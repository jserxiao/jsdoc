# 命名空间

## 概述

命名空间用于组织代码,避免全局污染。

## 一、基本语法

```typescript
namespace Utils {
  export function log(message: string): void {
    console.log(message)
  }

  export const PI = 3.14159
}

Utils.log('hello')
console.log(Utils.PI)
```

## 二、命名空间分割

```typescript
// 文件1
namespace Utils {
  export function log(msg: string) {
    console.log(msg)
  }
}

// 文件2
namespace Utils {
  export function error(msg: string) {
    console.error(msg)
  }
}

// 合并后的 Utils 包含 log 和 error
```

## 三、命名空间嵌套

```typescript
namespace App {
  export namespace Models {
    export interface User {
      id: number
      name: string
    }
  }

  export namespace Services {
    export class UserService {
      getUser(): Models.User {
        return { id: 1, name: 'Alice' }
      }
    }
  }
}

const user: App.Models.User = { id: 1, name: 'Alice' }
```

## 四、别名

```typescript
namespace Company {
  export namespace Department {
    export class Employee {
      constructor(public name: string) {}
    }
  }
}

import Employee = Company.Department.Employee

const emp = new Employee('Alice')
```

## 五、模块中的命名空间

```typescript
// utils.ts
export namespace StringUtils {
  export function upper(str: string): string {
    return str.toUpperCase()
  }
}

// 使用
import { StringUtils } from './utils'
StringUtils.upper('hello')
```

## 六、总结

- 命名空间用 `namespace` 定义
- 使用 `export` 暴露成员
- 支持分割和嵌套
- 现代 TypeScript 推荐使用 ES 模块
