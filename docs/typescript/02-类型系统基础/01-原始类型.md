# 原始类型

## 概述

TypeScript 支持与 JavaScript 相同的原始类型,并提供了类型注解功能。

## 一、string (字符串)

```typescript
// 基本用法
let name: string = 'Alice'
let greeting: string = `Hello, ${name}`

// 模板字符串
let message: string = `
  Hello, ${name}!
  Welcome to TypeScript.
`

// 字符串方法
let upper: string = name.toUpperCase()
let lower: string = name.toLowerCase()
let parts: string[] = name.split('')
```

## 二、number (数字)

```typescript
// 整数和浮点数
let integer: number = 42
let float: number = 3.14

// 二进制、八进制、十六进制
let binary: number = 0b1010     // 10
let octal: number = 0o744       // 484
let hex: number = 0xf00d        // 61453

// 特殊值
let infinity: number = Infinity
let negInfinity: number = -Infinity
let notANumber: number = NaN

// 数字运算
let sum: number = 10 + 20
let product: number = 5 * 6
```

## 三、boolean (布尔值)

```typescript
// 基本用法
let isTrue: boolean = true
let isFalse: boolean = false

// 布尔表达式
let isAdult: boolean = age >= 18
let hasPermission: boolean = user.role === 'admin'

// 布尔运算
let and: boolean = true && false   // false
let or: boolean = true || false    // true
let not: boolean = !true           // false
```

## 四、null 和 undefined

```typescript
// null - 表示"无值"或"空"
let empty: null = null

// undefined - 表示"未定义"
let notAssigned: undefined = undefined

// 严格空值检查 (strictNullChecks: true)
let name: string = null           // ❌ 编译错误
let age: number = undefined       // ❌ 编译错误

// 联合类型允许 null 和 undefined
let maybeString: string | null = null
let maybeNumber: number | undefined = undefined

// 可选参数和属性
interface User {
  name: string
  age?: number  // age?: number | undefined
}

function greet(name?: string) {
  // name: string | undefined
  if (name) {
    console.log(`Hello, ${name}`)
  } else {
    console.log('Hello, stranger')
  }
}
```

## 五、symbol (符号)

```typescript
// 创建 symbol
let sym1: symbol = Symbol()
let sym2: symbol = Symbol('description')

// symbol 是唯一的
console.log(Symbol('foo') === Symbol('foo')) // false

// 用作对象属性键
const obj = {
  [Symbol('key')]: 'value'
}

// 获取 symbol 属性
const symbols = Object.getOwnPropertySymbols(obj)

// 全局 symbol 注册表
const globalSym = Symbol.for('app.key')
const sameSym = Symbol.for('app.key')
console.log(globalSym === sameSym) // true

// 获取 symbol 键
const key = Symbol.keyFor(globalSym) // 'app.key'
```

## 六、bigint (大整数)

```typescript
// 创建 bigint
let big1: bigint = 100n
let big2: bigint = BigInt(100)

// 大整数运算
let sum: bigint = big1 + big2
let product: bigint = big1 * big2

// bigint 和 number 不兼容
let num: number = 100
// let big: bigint = num  // ❌ 编译错误

// 需要显式转换
let big: bigint = BigInt(num)
let numberFromBig: number = Number(big1)

// 使用场景: 大数计算
const largeNumber: bigint = 9007199254740993n
console.log(largeNumber + 1n) // 9007199254740994n
```

## 七、类型推断与类型注解

```typescript
// 类型推断
let inferred = 'hello'    // string
let num = 42              // number
let bool = true           // boolean

// 类型注解
let explicit: string = 'hello'
let explicitNum: number = 42

// 类型推断 vs 类型注解
// 推荐使用类型推断,除非需要更精确的类型
let name = 'Alice'        // ✅ 推荐
let age: number = 25      // ✅ 也可以

// 需要类型注解的场景
let nullable: string | null = null
let union: string | number = 'hello'
union = 42
```

## 八、类型兼容性

```typescript
// 原始类型之间不兼容
let str: string = 'hello'
let num: number = 42

// str = num  // ❌ 编译错误

// null 和 undefined 的兼容性
// 在 strictNullChecks: false 时
let str2: string = null      // ✅ 允许
let num2: number = undefined // ✅ 允许

// 在 strictNullChecks: true 时
let str3: string = null      // ❌ 编译错误
let num3: number = undefined // ❌ 编译错误

// 使用联合类型
let strOrNull: string | null = null      // ✅
let numOrUndefined: number | undefined   // ✅
```

## 九、最佳实践

### 1. 优先使用类型推断

```typescript
// ✅ 好的做法
let name = 'Alice'
let age = 25
let isActive = true

// ❌ 不必要的类型注解
let name: string = 'Alice'
let age: number = 25
let isActive: boolean = true
```

### 2. 明确处理 null 和 undefined

```typescript
// ✅ 明确的空值处理
interface User {
  name: string
  email: string | null
}

function getEmail(user: User): string {
  return user.email ?? 'no-email@example.com'
}
```

### 3. 使用 const 断言

```typescript
// 普通类型推断
let x = 'hello'  // string

// const 断言
let y = 'hello' as const  // 'hello' (字面量类型)

const obj = {
  name: 'Alice',
  age: 25
} as const
// obj: { readonly name: 'Alice'; readonly age: 25 }
```

## 十、总结

### 原始类型对照表

| TypeScript 类型 | JavaScript typeof | 示例值 |
|----------------|-------------------|--------|
| string | 'string' | 'hello', "world" |
| number | 'number' | 42, 3.14, NaN |
| boolean | 'boolean' | true, false |
| null | 'object' | null |
| undefined | 'undefined' | undefined |
| symbol | 'symbol' | Symbol() |
| bigint | 'bigint' | 100n |

### 关键要点

1. TypeScript 的原始类型与 JavaScript 一致
2. 严格模式下 null 和 undefined 需要显式处理
3. 优先使用类型推断,必要时使用类型注解
4. symbol 和 bigint 提供了额外的功能
