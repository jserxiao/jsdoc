# 类型断言

## 概述

类型断言告诉编译器"我知道这个类型"。

## 一、as 语法

```typescript
let value: unknown = 'hello'
let length: number = (value as string).length
```

## 二、尖括号语法

```typescript
let value: unknown = 'hello'
let length: number = (<string>value).length
```

## 三、非空断言

```typescript
interface User {
  name: string | null
}

const user: User = { name: 'Alice' }
const name = user.name!  // string (断言不为 null)
```

## 四、const 断言

```typescript
let x = 'hello' as const  // 'hello' (字面量类型)

const obj = {
  name: 'Alice',
  age: 25
} as const
// { readonly name: 'Alice'; readonly age: 25 }
```

## 五、双重断言

```typescript
// 不安全,避免使用
let value: string = 'hello'
let num = value as any as number
```

## 六、实战示例

```typescript
// DOM 元素断言
const input = document.querySelector('#input') as HTMLInputElement
console.log(input.value)

// API 响应断言
const response = await fetch('/api/users')
const users = await response.json() as User[]
```

## 七、最佳实践

```typescript
// ✅ 好: 明确类型
const value = data as User

// ❌ 不好: 滥用 any
const value = data as any

// ✅ 好: 使用类型守卫代替断言
if (isUser(data)) {
  console.log(data.name)
}
```

## 八、总结

- `as Type`: 类型断言
- `!`: 非空断言
- `as const`: const 断言
- 避免滥用断言
