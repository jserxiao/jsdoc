# 类型兼容性

## 概述

TypeScript 使用结构化类型系统,基于成员兼容性。

## 一、基本兼容性

```typescript
interface Point {
  x: number
  y: number
}

interface Point3D {
  x: number
  y: number
  z: number
}

let point: Point = { x: 0, y: 0 }
let point3D: Point3D = { x: 0, y: 0, z: 0 }

// ✅ 多余属性可以赋值
point = point3D

// ❌ 缺少属性不能赋值
// point3D = point
```

## 二、函数兼容性

### 参数数量

```typescript
type Handler1 = (a: number) => void
type Handler2 = (a: number, b: number) => void

let h1: Handler1 = a => {}
let h2: Handler2 = (a, b) => {}

// ✅ 参数少可以赋值
h2 = h1

// ❌ 参数多不能赋值
// h1 = h2
```

### 参数类型(逆变)

```typescript
type HandlerA = (a: Animal) => void
type HandlerD = (d: Dog) => void

// 在严格模式下
// HandlerD 不能赋值给 HandlerA
```

### 返回值类型(协变)

```typescript
type Creator1 = () => Animal
type Creator2 = () => Dog

let c1: Creator1 = () => new Animal()
let c2: Creator2 = () => new Dog()

// ✅ 子类型返回值可以赋值
c1 = c2
```

## 三、类兼容性

```typescript
class Animal {
  name: string = ''
}

class Dog extends Animal {
  breed: string = ''
}

let animal: Animal = new Animal()
let dog: Dog = new Dog()

// ✅ 子类可以赋值给父类
animal = dog

// ❌ 父类不能赋值给子类
// dog = animal
```

## 四、泛型兼容性

```typescript
interface Empty<T> {}

let e1: Empty<number> = {}
let e2: Empty<string> = {}

// ✅ 没有使用类型参数时兼容
e1 = e2

interface NotEmpty<T> {
  value: T
}

let n1: NotEmpty<number> = { value: 42 }
let n2: NotEmpty<string> = { value: 'hello' }

// ❌ 使用了类型参数时不兼容
// n1 = n2
```

## 五、枚举兼容性

```typescript
enum Color {
  Red,
  Green,
  Blue
}

let c: Color = Color.Red

// 枚举与数字不兼容
let n: number = c  // ✅
// let c2: Color = 0  // ❌ 严格模式
```

## 六、总结

- 结构化类型:基于成员兼容
- 对象: 多余属性可以赋值
- 函数: 参数逆变,返回值协变
- 类: 继承关系决定兼容性
- 泛型: 类型参数使用影响兼容性
