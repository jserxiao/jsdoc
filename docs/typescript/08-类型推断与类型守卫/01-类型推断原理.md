# 类型推断原理

## 概述

TypeScript 编译器能够自动推断类型,无需显式注解。

## 一、变量类型推断

```typescript
let x = 10          // number
let y = 'hello'     // string
let z = true        // boolean
let arr = [1, 2, 3] // number[]

// 对象推断
const obj = {
  name: 'Alice',
  age: 25
}  // { name: string; age: number }
```

## 二、函数类型推断

### 参数推断

```typescript
// 根据上下文推断
const numbers = [1, 2, 3, 4, 5]
numbers.forEach(n => {
  console.log(n.toFixed(2))  // n: number
})

// 类型别名提供上下文
type Handler = (event: MouseEvent) => void
const handler: Handler = e => {
  console.log(e.clientX)  // e: MouseEvent
}
```

### 返回值推断

```typescript
function add(a: number, b: number) {
  return a + b  // 返回 number
}

function getUser() {
  return {
    id: 1,
    name: 'Alice'
  }
}
// 返回 { id: number; name: string }
```

## 三、最佳通用类型

```typescript
// 混合类型推断
let mixed = [1, 'two', 3]  // (string | number)[]

// 对象推断
const animals = [
  { name: 'Dog', age: 2 },
  { name: 'Cat', age: 3 }
]
// { name: string; age: number }[]
```

## 四、上下文类型

```typescript
interface Event {
  type: string
  target: Element
}

function onEvent(handler: (e: Event) => void) {}

onEvent(e => {
  console.log(e.type)  // e: Event
})
```

## 五、控制流分析

```typescript
function process(value: string | number) {
  // value: string | number

  if (typeof value === 'string') {
    // value: string (缩小)
    return value.toUpperCase()
  } else {
    // value: number (缩小)
    return value * 2
  }
}
```

## 六、实战示例

```typescript
// API 响应推断
async function fetchData() {
  const response = await fetch('/api/users')
  const data = await response.json()
  // data: any (需要类型注解)

  return data
}

// 更好的方式
interface User {
  id: number
  name: string
}

async function fetchUsers(): Promise<User[]> {
  const response = await fetch('/api/users')
  return response.json()
}

const users = await fetchUsers()  // User[]
```

## 七、总结

- TypeScript 自动推断变量、参数和返回值类型
- 上下文类型提供更好的推断
- 控制流分析实现类型缩小
- 复杂场景建议显式类型注解
