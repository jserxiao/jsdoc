# 实战项目：@jserxiao/xcli

本章将以 [`@jserxiao/xcli`](https://github.com/jserxiao/xcli) 为实战案例，深入分析一个完整 CLI 脚手架工具的设计与实现。

---

## 项目简介

**@jserxiao/xcli** 是一个可插拔的 TypeScript 项目脚手架 CLI 工具，支持快速创建 Library、React、Vue 等类型的项目，并提供丰富的插件生态。

### 核心特性

| 特性 | 说明 |
|------|------|
| 🎯 **多项目类型** | 支持 Library、React、Vue 三种项目模板 |
| 🔌 **插件系统** | 内置 15+ 插件，支持灵活组合 |
| 🎨 **样式支持** | 支持 CSS、Less、Sass 三种样式方案 |
| 📦 **打包工具** | 支持 Vite、Webpack、Rollup |
| 🔧 **状态管理** | 支持 Redux、MobX、Pinia |
| 🚀 **一键升级** | 内置 `upgrade` 命令自动更新 |
| 📝 **类型安全** | 完整的 TypeScript 类型定义 |

### 快速开始

```bash
# 使用 npx 创建项目
npx @jserxiao/xcli init my-project

# 或全局安装后使用
npm install -g @jserxiao/xcli
xcli init my-project
```

│   │   ├── plugin.ts         # plugin 命令

│   │   ├── react.ts          # React 模板
│   │   ├── vue.ts            # Vue 模板
│   │   └── files/            # 模板文件生成器
│   ├── types/                # 类型定义
│   │   └── index.ts          # 核心类型
│   ├── utils/                # 工具函数
│   │   ├── logger.ts         # 日志工具
│   │   ├── fileGenerator.ts  # 文件生成器
│   │   └── dependency.ts     # 依赖管理
│   └── constants/            # 常量定义

├── package.json
├── tsconfig.json
├── rollup.config.mjs         # Rollup 配置
└── publish.sh                # 发布脚本

## 架构设计

### 分层架构

```
┌─────────────────────────────────────────────────────────────────┐
│                      用户交互层 (CLI Layer)                      │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │  Commander.js 命令定义 → Inquirer.js 交互问答            │   │
│  └──────────────────────────────────────────────────────────┘   │
├─────────────────────────────────────────────────────────────────┤
│                      业务逻辑层 (Service Layer)                  │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐              │
│  │ init 命令   │  │ plugin 命令 │  │ upgrade 命令│              │
│  └─────────────┘  └─────────────┘  └─────────────┘              │
├─────────────────────────────────────────────────────────────────┤
│                      核心模块层 (Core Layer)                     │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐              │
│  │ 插件系统    │  │ 模板系统    │  │ 类型系统    │              │
│  └─────────────┘  └─────────────┘  └─────────────┘              │
├─────────────────────────────────────────────────────────────────┤
│                      基础设施层 (Infrastructure Layer)           │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐              │
│  │ 日志工具    │  │ 文件生成    │  │ 依赖管理    │              │
│  └─────────────┘  └─────────────┘  └─────────────┘              │
└─────────────────────────────────────────────────────────────────┘
```

### 核心设计理念

1. **插件化**：通过统一的插件接口，实现功能模块的灵活组合
2. **模板化**：通过模板生成器，支持多种项目类型的快速创建
3. **类型安全**：完整的 TypeScript 类型定义，提供优秀的开发体验
4. **配置驱动**：用户选择转换为配置对象，驱动后续生成流程
