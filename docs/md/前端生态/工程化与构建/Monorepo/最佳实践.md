# Monorepo 最佳实践

## 项目结构设计

### 标准结构

```
my-monorepo/
├── apps/                    # 应用
│   ├── web/                 # Web 应用
│   ├── admin/               # 管理后台
│   └── mobile/              # 移动端
├── packages/                # 共享包
│   ├── ui/                  # UI 组件库
│   ├── utils/               # 工具函数
│   ├── config/              # 共享配置
│   ├── tsconfig/            # TypeScript 配置
│   └── eslint-config/       # ESLint 配置
├── tools/                   # 工具脚本
├── docs/                    # 文档
├── package.json             # 根配置
├── pnpm-workspace.yaml      # pnpm 工作空间
├── turbo.json               # Turborepo 配置
└── tsconfig.json            # 基础 TypeScript 配置
```

### 命名规范

```
@scope/类型-名称

示例：
@my-org/ui-button
@my-org/util-date
@my-org/app-web
```

## 依赖管理

### 统一版本

```json
// package.json (根)
{
  "devDependencies": {
    "typescript": "5.3.3",
    "vite": "5.0.10",
    "vitest": "1.0.4"
  },
  "pnpm": {
    "overrides": {
      "lodash": "4.17.21"
    }
  }
}
```

### 共享开发依赖

```json
// packages/tsconfig/package.json
{
  "name": "@my-org/tsconfig",
  "files": ["base.json", "react.json", "vue.json"]
}

// apps/web/tsconfig.json
{
  "extends": "@my-org/tsconfig/react"
}
```

### 包间依赖

```json
// packages/ui/package.json
{
  "dependencies": {
    "@my-org/utils": "workspace:*",
    "@my-org/theme": "workspace:*"
  }
}
```

## 构建优化

### 任务编排

```json
// turbo.json
{
  "pipeline": {
    "build": {
      "dependsOn": ["^build"],
      "outputs": ["dist/**"]
    },
    "test": {
      "dependsOn": ["build"],
      "outputs": ["coverage/**"]
    },
    "lint": {
      "outputs": []
    }
  }
}
```

### 缓存策略

```bash
# 本地缓存
turbo run build --cache-dir=.turbo

# 远程缓存
turbo run build --token=xxx
```

## 发布流程

### Changesets 配置

```json
// .changeset/config.json
{
  "access": "public",
  "baseBranch": "main",
  "updateInternalDependencies": "patch",
  "ignore": ["@my-org/web", "@my-org/admin"]
}
```

### 发布脚本

```json
// package.json
{
  "scripts": {
    "version": "changeset version",
    "release": "changeset publish",
    "version:ci": "changeset version && pnpm install --lockfile-only"
  }
}
```

### CI/CD 配置

```yaml
# .github/workflows/release.yml
name: Release

on:
  push:
    branches: [main]

jobs:
  release:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v2
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          registry-url: 'https://registry.npmjs.org'

      - run: pnpm install
      - run: pnpm build

      - name: Create Release
        run: |
          npx changeset version
          pnpm install --lockfile-only
          npx changeset publish
        env:
          NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}

      - name: Push Changes
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git push --follow-tags
```

## 常见问题

### 依赖版本冲突

```json
// package.json
{
  "pnpm": {
    "overrides": {
      "react": "18.2.0",
      "react-dom": "18.2.0"
    }
  }
}
```

### 循环依赖

```bash
# 检测循环依赖
npx madge --circular packages/
```

### 构建顺序

```json
// turbo.json
{
  "pipeline": {
    "build": {
      "dependsOn": ["^build"]
    }
  }
}
```

## 最佳实践总结

1. **统一依赖版本**：避免版本冲突
2. **清晰的包职责**：单一职责原则
3. **合理的依赖方向**：避免循环依赖
4. **完善的文档**：每个包有 README
5. **自动化发布**：Changesets 自动化
6. **增量构建**：Turborepo 缓存
7. **代码共享**：抽离公共模块
8. **规范命名**：统一命名风格
