# 核心特性

## 增量编译

Turbopack 只编译变化的部分，实现极快的更新速度。

### 工作原理

```
源代码
    ↓
Turbo Engine（增量计算引擎）
    ↓
只重新计算变化的部分
    ↓
缓存结果，下次直接使用
```

### 编译粒度

```
项目
├── 函数级别缓存
├── 文件级别缓存
├── 模块级别缓存
└── Chunk 级别缓存
```

## 持久化缓存

Turbopack 的缓存存储在内存中，重启后依然可用。

### 缓存策略

```javascript
// 缓存存储位置
.turbo/
├── cache/          # 缓存数据
└── daemon/         # 后台进程
```

### 缓存命中

```javascript
// 文件 A 修改
// 只有依赖 A 的模块需要重新编译

// 文件 B 未修改
// 直接使用缓存结果
```

| Webpack | ~10s |

## 内存缓存架构

```
请求
    ↓
检查缓存
    ├── 命中 → 返回缓存结果
    └── 未命中 → 编译 → 存入缓存 → 返回结果
```

### 缓存结构

```rust
struct Cache {
    // 函数结果缓存
    function_cache: HashMap<FunctionId, Result>,
    
    // 文件内容缓存
    file_cache: HashMap<Path, Content>,
    
    // 模块依赖图
    dependency_graph: Graph,
}
```

## 并行处理

Turbopack 利用 Rust 的并发能力，自动并行处理任务。

```rust
// 并行编译多个模块
modules.par_iter().for_each(|module| {
    compile(module)
})
```

## Lazy Compilation

Turbopack 只编译当前页面需要的代码。

```
用户访问 /page-a
    ↓
只编译 page-a 及其依赖
    ↓
其他页面延迟编译
```
