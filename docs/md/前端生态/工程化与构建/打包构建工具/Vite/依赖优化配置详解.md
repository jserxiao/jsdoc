# Vite 依赖优化配置详解

## 概述

Vite 的依赖优化是提升开发体验的关键特性，通过预构建依赖来加速开发服务器的启动和热更新。

## 依赖预构建

### 工作原理

Vite 在首次启动时会扫描项目依赖，并使用 esbuild 将它们预构建为 ESM 格式：

```
node_modules/.vite/
├── deps/
│   ├── vue.js
│   ├── vue-router.js
│   ├── axios.js
│   └── _metadata.json
└── deps_temp/
```

### optimizeDeps.include

强制预构建依赖。

```typescript
import { defineConfig } from 'vite'

export default defineConfig({
  optimizeDeps: {
    include: [
      'vue',
      'vue-router',
      'pinia',
      'axios',
      'lodash',
      'dayjs',
      // 指定具体文件
      'element-plus/es/components/button/style/css',
      // 使用 glob 模式
      'my-lib/components/**/*.vue'
    ]
  }
})
```

### optimizeDeps.exclude

排除预构建依赖。

```typescript
export default defineConfig({
  optimizeDeps: {
    exclude: [
      'my-linked-package',  // 链接的本地包
      'my-unbundled-package',  // 未打包的依赖
    ]
  }
})
```

### optimizeDeps.esbuildOptions

自定义 esbuild 配置。

```typescript
export default defineConfig({
  optimizeDeps: {
    esbuildOptions: {
      // 目标环境
      target: 'es2020',

      // 定义环境变量
      define: {
        'process.env.NODE_ENV': '"development"'
      },

      // 插件
      plugins: [
        {
          name: 'my-plugin',
          setup(build) {
            build.onLoad({ filter: /\.js$/ }, async (args) => {
              // 自定义处理
            })
          }
        }
      ],

      // 外部依赖
      external: ['fs', 'path'],

      // JSX 支持
      jsx: 'automatic',
      jsxImportSource: 'react',
    }
  }
})
```

## 强制预构建

### optimizeDeps.force

强制重新预构建。

```typescript
export default defineConfig({
  optimizeDeps: {
    force: true,  // 强制重新预构建
  }
})
```

### 命令行强制

```bash
vite --force
vite --force --clearScreen false
```

## 发现依赖

### 自动发现

Vite 会自动扫描以下文件发现依赖：

```typescript
// 默认扫描位置
[
  'index.html',
  'src/main.ts',
  'src/main.js',
  'src/App.vue',
  'src/App.tsx',
]

// 或在配置中指定
export default defineConfig({
  optimizeDeps: {
    entries: [
      'index.html',
      'src/**/*.{ts,tsx,vue}',
      '!src/**/*.spec.ts'
    ]
  }
})
```

### optimizeDeps.entries

自定义扫描入口。

```typescript
export default defineConfig({
  optimizeDeps: {
    entries: [
      'src/main.ts',
      'src/pages/**/*.vue',
      'src/components/**/*.tsx'
    ]
  }
})
```

## 缓存配置

### 缓存位置

```typescript
export default defineConfig({
  cacheDir: 'node_modules/.vite',  // 默认
  cacheDir: '.vite-cache',         // 自定义
})
```

### 清除缓存

```bash
# 清除缓存
rm -rf node_modules/.vite

# 或使用命令
vite --force
```

### 缓存失效条件

Vite 会在以下情况下重新预构建：

1. `package.json` 中的 `dependencies` 改变
2. `package-lock.json` 或 `yarn.lock` 改变
3. `vite.config.js` 中的相关配置改变
4. `optimizeDeps.force: true`

## 处理特殊依赖

### CJS 依赖转 ESM

```typescript
// 自动转换
import { someFunction } from 'cjs-package'  // Vite 自动处理

// 手动指定
export default defineConfig({
  optimizeDeps: {
    include: ['cjs-package']
  }
})
```

### 链接的本地包

```typescript
export default defineConfig({
  optimizeDeps: {
    exclude: [
      'my-local-package',  // 链接到本地开发的包
    ]
  }
})
```

### 未打包的依赖

```typescript
export default defineConfig({
  optimizeDeps: {
    include: [
      // 包含许多导出的包
      'lodash-es',
      // 使用特定导出路径
      'my-lib/components/Button',
      'my-lib/utils/format',
    ]
  }
})
```

### Worker 中的依赖

```typescript
export default defineConfig({
  worker: {
    rollupOptions: {
      external: ['fs', 'path'],
    }
  },
  optimizeDeps: {
    include: [
      'my-worker-dependency'
    ]
  }
})
```

## 性能优化

### 预构建大型依赖

```typescript
export default defineConfig({
  optimizeDeps: {
    include: [
      // 大型 UI 库
      'element-plus',
      'ant-design-vue',
      '@mui/material',

      // 工具库
      'lodash',
      'moment',
      'dayjs',

      // 图表库
      'echarts',
      'd3',

      // 富文本编辑器
      'monaco-editor',
      'quill',
    ]
  }
})
```

### 避免不必要的预构建

```typescript
export default defineConfig({
  optimizeDeps: {
    exclude: [
      // 已经是 ESM 格式的包
      'modern-esm-package',

      // 不使用的包（按需加载）
      'heavy-package',

      // 服务端专用包
      'server-only-package',
    ]
  }
})
```

### 批量预构建

```typescript
export default defineConfig({
  optimizeDeps: {
    include: [
      // 将相关依赖批量预构建
      'vue',
      'vue-router',
      'pinia',
      '@vueuse/core',
    ],
    esbuildOptions: {
      // 并行处理
      concurrency: 10,
    }
  }
})
```

## 常见问题处理

### 1. 依赖预构建失败

```typescript
export default defineConfig({
  optimizeDeps: {
    include: ['problematic-package'],
    esbuildOptions: {
      // 处理特殊语法
      loader: {
        '.js': 'jsx',
      },
      // 忽略警告
      logLevel: 'warning',
    }
  }
})
```

### 2. CJS 默认导出问题

```typescript
// 错误
import someDefault from 'cjs-package'

// 正确
import { default as someDefault } from 'cjs-package'
// 或
import * as somePackage from 'cjs-package'
const someDefault = somePackage.default
```

```typescript
// vite.config.ts
export default defineConfig({
  optimizeDeps: {
    esbuildOptions: {
      mainFields: ['module', 'main'],
      conditions: ['import', 'require', 'default'],
    }
  }
})
```

### 3. 动态导入问题

```typescript
// 错误 - Vite 无法静态分析
const pkg = await import(someVariable)

// 正确 - 静态字符串
const pkg = await import('my-package')

// 如果必须使用变量
export default defineConfig({
  optimizeDeps: {
    include: [
      'package-a',
      'package-b',
      'package-c',
    ]
  }
})
```

### 4. Monorepo 中链接包

```typescript
export default defineConfig({
  optimizeDeps: {
    exclude: [
      // 排除所有工作空间包
      '@myorg/shared',
      '@myorg/utils',
      '@myorg/components',
    ]
  }
})
```

### 5. 开发环境警告

```typescript
export default defineConfig({
  optimizeDeps: {
    // 禁用预构建警告
    disabled: false,
  }
})
```

## 完整配置示例

### 企业级项目配置

```typescript
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import path from 'path'

export default defineConfig({
  plugins: [vue()],

  // 缓存目录
  cacheDir: 'node_modules/.vite',

  // 依赖优化
  optimizeDeps: {
    // 强制预构建
    include: [
      'vue',
      'vue-router',
      'pinia',
      '@vueuse/core',
      'axios',
      'lodash-es',
      'dayjs',
      'element-plus/es',
      'element-plus/es/components/message/style/css',
      'element-plus/es/components/notification/style/css',
    ],

    // 排除预构建
    exclude: [
      '@myorg/shared',
      '@myorg/utils',
    ],

    // esbuild 配置
    esbuildOptions: {
      target: 'es2020',
      define: {
        'process.env.NODE_ENV': '"development"'
      },
      plugins: [],
    },

    // 强制重新构建
    force: false,
  },

  resolve: {
    alias: {
      '@': path.resolve(__dirname, 'src')
    }
  }
})
```

### 开发环境优化

```typescript
import { defineConfig } from 'vite'

export default defineConfig(({ mode }) => {
  const isDev = mode === 'development'

  return {
    optimizeDeps: {
      include: isDev ? [
        // 开发环境只包含常用依赖
        'vue',
        'vue-router',
        'pinia',
      ] : [],

      esbuildOptions: {
        target: 'esnext',
        logLevel: 'warning',
      },

      force: isDev,
    }
  }
})
```

### Monorepo 配置

```typescript
import { defineConfig } from 'vite'
import { readdirSync } from 'fs'
import { join } from 'path'

// 自动发现工作空间包
const workspacePkgs = readdirSync(join(__dirname, 'packages'))

export default defineConfig({
  optimizeDeps: {
    exclude: workspacePkgs.map(pkg => `@myorg/${pkg}`),

    include: [
      // 共享依赖
      'vue',
      'vue-router',
      'pinia',
      'lodash-es',
    ],
  }
})
```
