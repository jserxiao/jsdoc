# 打包构建工具

打包构建工具是前端工程化的核心基础设施，负责将源代码转换为可在浏览器中运行的产物，处理模块依赖、代码转换、资源优化等工作。

---

## 主流工具对比

| 工具 | 定位 | 核心优势 | 适用场景 |
|------|------|----------|----------|
| **Vite** | 下一代构建工具 | 极速冷启动、ESM 原生 | 新项目首选 |
| **Webpack** | 模块打包器 | 生态完善、高度可配置 | 企业级复杂项目 |
| **Rollup** | 库打包工具 | Tree Shaking、输出干净 | 库开发、组件库 |
| **esbuild** | 极速打包器 | Go 编写、速度极快 | 追求速度场景 |
| **Turbopack** | 增量打包器 | Rust 编写、增量编译 | Next.js 大型应用 |

## 技术选型建议

| 场景 | 推荐方案 | 理由 |
|------|---------|------|
| 新项目 | Vite | 开发体验好、速度快 |
| 企业级项目 | Webpack | 灵活可控、生态完善 |
| 库开发 | Rollup / Vite Library Mode | 输出干净、多格式支持 |
| Next.js 项目 | Turbopack | 原生集成、增量编译 |
| 快速原型 | Vite | 极速启动、零配置 |

## 常见问题

### 构建速度慢

1. 开启持久化缓存
2. 使用 esbuild/swc 替代 Babel
3. 并行构建
4. 减少不必要的编译范围

### 包体积过大

1. 分析依赖树，定位问题依赖
2. 按需引入，避免全量导入
3. 代码分割，动态加载
4. 使用更轻量的替代方案

### 开发服务器启动慢

1. 切换到 Vite
2. 配置忽略监听目录
3. 减少监听文件数量
