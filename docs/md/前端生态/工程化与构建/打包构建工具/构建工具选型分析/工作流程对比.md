# 工作流程对比

本文对比 Webpack 和 Vite 的构建流程，帮助理解两种工具的核心差异。

---

## Webpack 构建流程

### 完整构建流程

```
┌─────────────────────────────────────────────────────────────────┐
│                      Webpack 构建流程                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 初始化参数                                                    │
│     └── 读取 webpack.config.js，合并命令行参数                      │
│                                                                 │
│  2. 创建 Compiler 对象                                           │
│     └── 初始化插件系统，加载所有插件                                  │
│                                                                 │
│  3. run() / watch()                                              │
│     └── 触发 run 钩子，开始编译                                     │
│                                                                 │
│  4. 编译模块                                                      │
│     ├── 从入口文件开始分析                                          │
│     ├── 调用 Loader 转换文件                                       │
│     ├── 分析依赖，构建依赖图                                        │
│     └── 递归处理所有依赖                                           │
│                                                                 │
│  5. 生成输出                                                      │
│     ├── 根据依赖图生成 Chunk                                       │
│     ├── 代码分割、Tree Shaking                                     │
│     └── 写入文件系统                                               │
│                                                                 │
│  6. 完成                                                          │
│     └── 触发 done 钩子                                            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### Webpack 开发模式流程

```
启动 Webpack Dev Server
    │
    ▼
┌─────────────────────┐
│  遍历入口文件         │
│  分析所有依赖         │  ← 需要遍历整个项目
│  编译所有模块         │
│  打包成 Bundle        │
└─────────────────────┘
    │
    ▼
启动开发服务器 (数秒 ~ 数十秒)
    │
    ▼
浏览器加载 Bundle
    │
    ▼
文件修改触发重新构建
    │
    ▼
┌─────────────────────┐
│  增量编译            │  ← 只编译变化的部分
│  HMR 更新            │
└─────────────────────┘
```

### Webpack 流程详解

```javascript
// 1. 初始化阶段
// 读取配置文件，合并命令行参数
const options = require('./webpack.config.js')
const finalOptions = merge(options, cliOptions)

// 2. 创建 Compiler
// Compiler 是整个构建过程的核心控制器
const compiler = webpack(finalOptions)

// 3. 注册插件
finalOptions.plugins.forEach(plugin => {
  plugin.apply(compiler)
})

// 4. 开始编译
compiler.run((err, stats) => {
  // 编译完成回调
})

// 5. 编译过程
// - make 阶段：从 entry 开始，递归分析依赖
// - seal 阶段：根据依赖图生成 chunks
// - emit 阶段：输出文件到 output 目录
```

## Vite 构建流程

### 开发模式流程

```
┌─────────────────────────────────────────────────────────────────┐
│                      Vite 开发模式流程                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 启动服务器                                                    │
│     ├── 扫描依赖，预构建 node_modules                             │
│     └── 启动 HTTP 服务器（毫秒级）                                  │
│                                                                 │
│  2. 浏览器请求入口文件                                             │
│     └── 返回 index.html                                          │
│                                                                 │
│  3. 浏览器加载 ES Module                                          │
│     └── <script type="module" src="/src/main.js">               │
│                                                                 │
│  4. 按需编译                                                      │
│     ├── 浏览器请求哪个文件，就编译哪个文件                           │
│     ├── 使用 esbuild 快速编译 TS/JSX                              │
│     └── 返回编译后的 ESM 模块                                      │
│                                                                 │
│  5. 文件修改                                                      │
│     ├── 触发 HMR                                                 │
│     └── 只更新变化的模块                                          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### Vite 生产模式流程

```
Vite Build
    │
    ▼
┌─────────────────────┐
│  使用 Rollup 打包    │
│  代码分割            │
│  Tree Shaking       │
│  压缩优化            │
└─────────────────────┘
    │
    ▼
生成生产产物
```

### Vite 流程详解

```javascript
// 1. 启动开发服务器
const server = await createServer({
  // 配置选项
})

// 2. 预构建依赖
// 扫描源码中的 import 语句
// 使用 esbuild 预构建 node_modules
await server.optimizeDeps()

// 3. 启动 HTTP 服务器
await server.listen(5173)

// 4. 请求处理
// 浏览器请求 -> 中间件处理 -> 返回编译结果
server.middlewares.use((req, res, next) => {
  // 处理各种请求
  // - JS/TS: esbuild 编译
  // - Vue: @vitejs/plugin-vue 编译
  // - CSS: 原生支持
})

// 5. HMR
// 文件变化 -> 触发 update -> WebSocket 通知浏览器
```

## 核心差异对比

### 启动方式

| 特性 | Webpack | Vite |
|------|---------|------|
| 启动时机 | 打包完成后启动 | 立即启动服务器 |
| 编译时机 | 启动时编译所有文件 | 请求时按需编译 |
| 依赖处理 | 打包进 Bundle | 预构建后单独加载 |

### 模块处理

| 特性 | Webpack | Vite |
|------|---------|------|
| 模块格式 | 打包成函数 | 保持 ESM 格式 |
| 依赖图 | 启动时完整构建 | 请求时动态构建 |
| 缓存策略 | 文件系统缓存 | 内存缓存 + 预构建缓存 |

### HMR 实现

| 特性 | Webpack | Vite |
|------|---------|------|
| 更新触发 | 重新编译变化模块 | 直接重新加载模块 |
| 更新传播 | 沿依赖图向上传播 | 基于边界更新 |
| 更新速度 | 依赖项目大小 | 几乎恒定 |

---

## 总结

- **Webpack**：先打包再启动，适合复杂项目和成熟团队
- **Vite**：按需编译，适合新项目和追求开发体验
