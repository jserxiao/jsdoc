# 核心概念

Webpack 是现代 JavaScript 应用程序的静态模块打包工具。当 webpack 处理应用程序时，它会在内部从一个或多个入口点构建一个依赖图(dependency graph)，然后将你项目中所需的每一个模块组合成一个或多个 bundles，它们均为静态资源，用于展示你的内容。

## 核心概念总览

Webpack 有五个核心概念需要理解：

| 概念 | 说明 | 作用 |
|------|------|------|
| **Entry** | 入口 | 指示 webpack 从哪个文件开始构建依赖图 |
| **Output** | 输出 | 告诉 webpack 在哪里输出打包后的文件 |
| **Loader** | 加载器 | 让 webpack 能够处理非 JavaScript 文件 |
| **Plugin** | 插件 | 用于执行更广泛的任务，如优化、资源管理等 |
| **Mode** | 模式 | 启用相应模式下的内置优化 |

## Entry（入口）

### 概念

入口起点(entry point)指示 webpack 应该使用哪个模块，来作为构建其内部依赖图的开始。进入入口起点后，webpack 会找出有哪些模块和库是入口起点（直接和间接）依赖的。

### 基本用法

```javascript
// webpack.config.js
module.exports = {
  entry: './path/to/my/entry/file.js'
}
```

### 单入口配置

```javascript
// 简写语法
module.exports = {
  entry: './src/index.js'
}

// 完整语法（对象）
module.exports = {
  entry: {
    main: './src/index.js'
  }
}
```

### 多入口配置

```javascript
// 多页面应用
module.exports = {
  entry: {
    home: './src/home.js',
    about: './src/about.js',
    contact: './src/contact.js'
  }
}

// 入口数组（创建多个主入口）
module.exports = {
  entry: {
    app: ['./src/index.js', './src/other.js']
  }
}
```

### 动态入口

```javascript
// 使用函数
module.exports = {
  entry: () => './src/index.js'
}

// 使用 Promise
module.exports = {
  entry: () => new Promise((resolve) => resolve('./src/index.js'))
}

// 动态读取多入口
const glob = require('glob')

module.exports = {
  entry: glob.sync('./src/pages/*/index.js').reduce((entries, path) => {
    const name = path.match(/\/pages\/(.+)\/index\.js/)[1]
    entries[name] = path
    return entries
  }, {})
}
```

### 最佳实践

```javascript
// 单页应用 (SPA)
module.exports = {
  entry: './src/index.js'
}

// 多页应用 (MPA)
module.exports = {
  entry: {
    pageOne: './src/pageOne/index.js',
    pageTwo: './src/pageTwo/index.js',
    pageThree: './src/pageThree/index.js'
  }
}

// 分离 vendor
module.exports = {
  entry: {
    main: './src/index.js',
    vendor: ['react', 'react-dom', 'react-router']
  }
}
```

## Output（输出）

### 概念

output 属性告诉 webpack 在哪里输出它所创建的 bundle，以及如何命名这些文件。

### 基本配置

```javascript
const path = require('path')

module.exports = {
  entry: './src/index.js',
  output: {
    path: path.resolve(__dirname, 'dist'),
    filename: 'bundle.js'
  }
}
```

### 多入口输出

```javascript
module.exports = {
  entry: {
    app: './src/app.js',
    admin: './src/admin.js'
  },
  output: {
    path: path.resolve(__dirname, 'dist'),
    filename: '[name].bundle.js'  // 使用占位符
  }
}
// 输出: app.bundle.js, admin.bundle.js
```

### 文件名占位符

| 占位符 | 说明 | 示例 |
|--------|------|------|
| `[name]` | 入口名称 | `app.bundle.js` |
| `[id]` | chunk id | `1.bundle.js` |
| `[hash]` | 模块标识符的 hash | `bundle.abc123.js` |
| `[chunkhash]` | chunk 内容的 hash | `bundle.def456.js` |
| `[contenthash]` | 提取内容的 hash | `bundle.ghi789.js` |

```javascript
module.exports = {
  output: {
    filename: '[name].[contenthash:8].js',
    chunkFilename: '[name].[contenthash:8].chunk.js'
  }
}
```

### 高级输出配置

```javascript
module.exports = {
  output: {
    // 输出目录（必须是绝对路径）
    path: path.resolve(__dirname, 'dist'),

    // 文件名
    filename: '[name].[contenthash:8].js',

    // chunk 文件名
    chunkFilename: '[name].[contenthash:8].chunk.js',

    // 静态资源文件名
    assetModuleFilename: 'assets/[hash][ext][query]',

    // 公共路径
    publicPath: '/assets/',

    // 库配置
    library: {
      name: 'MyLibrary',
      type: 'umd'
    },

    // 清空输出目录
    clean: true,

    // source map 文件名
    sourceMapFilename: '[name].map',

    // 开发工具名
    devtoolModuleFilenameTemplate: info => {
      return `file:///${path.resolve(info.absoluteResourcePath)}`
    }
  }
}
```

### CDN 配置

```javascript
module.exports = {
  output: {
    path: path.resolve(__dirname, 'dist'),
    publicPath: 'https://cdn.example.com/assets/',
    filename: '[name].[contenthash:8].js'
  }
}
```

## Loader（加载器）

### 概念

Webpack 只能理解 JavaScript 和 JSON 文件，这是 webpack 开箱可用的自带能力。loader 让 webpack 能够去处理其他类型的文件，并将它们转换为有效模块，以供应用程序使用，以及被添加到依赖图中。

### Loader 特性

- **链式调用**：loader 从右到左、从下到上执行
- **可同步可异步**：loader 可以同步或异步运行
- **模块化**：loader 在 Node.js 中运行，具有所有功能
- **可配置**：loader 可以通过 options 配置

### 基本用法

```javascript
module.exports = {
  module: {
    rules: [
      {
        test: /\.css$/,
        use: ['style-loader', 'css-loader']
      }
    ]
  }
}
```

### Loader 执行顺序

```javascript
// 从右到左、从下到上执行
module.exports = {
  module: {
    rules: [
      {
        test: /\.scss$/,
        use: [
          'style-loader',   // 3. 将 CSS 注入 DOM
          'css-loader',     // 2. 解析 CSS 文件
          'sass-loader'     // 1. 编译 Sass 为 CSS
        ]
      }
    ]
  }
}
```

### Loader 配置选项

```javascript
module.exports = {
  module: {
    rules: [
      {
        test: /\.js$/,
        exclude: /node_modules/,  // 排除
        include: path.resolve(__dirname, 'src'),  // 包含
        use: [
          {
            loader: 'babel-loader',
            options: {
              presets: ['@babel/preset-env'],
              cacheDirectory: true
            }
          }
        ]
      }
    ]
  }
}
```

### 条件匹配

```javascript
module.exports = {
  module: {
    rules: [
      {
        // 匹配文件
        test: /\.js$/,

        // 包含目录
        include: path.resolve(__dirname, 'src'),

        // 排除目录
        exclude: /node_modules/,

        // 或使用 resource 条件
        resource: {
          test: /\.js$/,
          include: path.resolve(__dirname, 'src')
        },

        // issuer：模块来源
        issuer: {
          test: /\.js$/
        },

        // 一对一映射
        oneOf: [
          {
            test: /\.css$/,
            use: ['style-loader', 'css-loader']
          },
          {
            test: /\.scss$/,
            use: ['style-loader', 'css-loader', 'sass-loader']
          }
        ]
      }
    ]
  }
}
```

### 常用 Loader 示例

```javascript
module.exports = {
  module: {
    rules: [
      // JavaScript 转换
      {
        test: /\.js$/,
        exclude: /node_modules/,
        use: 'babel-loader'
      },

      // TypeScript
      {
        test: /\.tsx?$/,
        use: 'ts-loader',
        exclude: /node_modules/
      },

      // CSS
      {
        test: /\.css$/,
        use: ['style-loader', 'css-loader']
      },

      // Sass
      {
        test: /\.scss$/,
        use: ['style-loader', 'css-loader', 'sass-loader']
      },

      // Less
      {
        test: /\.less$/,
        use: ['style-loader', 'css-loader', 'less-loader']
      },

      // 图片
      {
        test: /\.(png|jpg|gif|svg)$/,
        type: 'asset',
        parser: {
          dataUrlCondition: {
            maxSize: 8 * 1024 // 8KB
          }
        }
      },

      // 字体
      {
        test: /\.(woff|woff2|eot|ttf|otf)$/,
        type: 'asset/resource'
      },

      // CSV/TSV
      {
        test: /\.(csv|tsv)$/,
        use: ['csv-loader']
      },

      // XML
      {
        test: /\.xml$/,
        use: ['xml-loader']
      },

      // Markdown
      {
        test: /\.md$/,
        use: ['html-loader', 'markdown-loader']
      }
    ]
  }
}
```

## Plugin（插件）

### 概念

loader 用于转换某些类型的模块，而插件则可以用于执行范围更广的任务。包括：打包优化、资源管理、注入环境变量等。

### 基本用法

```javascript
const HtmlWebpackPlugin = require('html-webpack-plugin')
const { CleanWebpackPlugin } = require('clean-webpack-plugin')

module.exports = {
  plugins: [
    new CleanWebpackPlugin(),
    new HtmlWebpackPlugin({
      template: './src/index.html'
    })
  ]
}
```

### 插件配置示例

```javascript
const HtmlWebpackPlugin = require('html-webpack-plugin')
const MiniCssExtractPlugin = require('mini-css-extract-plugin')
const { BundleAnalyzerPlugin } = require('webpack-bundle-analyzer')
const webpack = require('webpack')

module.exports = {
  plugins: [
    // HTML 生成
    new HtmlWebpackPlugin({
      template: './src/index.html',
      filename: 'index.html',
      minify: {
        removeComments: true,
        collapseWhitespace: true
      }
    }),

    // CSS 提取
    new MiniCssExtractPlugin({
      filename: '[name].[contenthash:8].css',
      chunkFilename: '[name].[contenthash:8].chunk.css'
    }),

    // 环境变量
    new webpack.DefinePlugin({
      'process.env.NODE_ENV': JSON.stringify(process.env.NODE_ENV),
      __DEV__: JSON.stringify(true)
    }),

    // 模块热替换
    new webpack.HotModuleReplacementPlugin(),

    // 进度显示
    new webpack.ProgressPlugin(),

    // 包分析
    new BundleAnalyzerPlugin({
      analyzerMode: 'static',
      openAnalyzer: false
    }),

    // 复制文件
    new CopyWebpackPlugin({
      patterns: [
        { from: 'public', to: 'dist' }
      ]
    })
  ]
}
```

### 常用插件列表

| 插件 | 作用 |
|------|------|
| `html-webpack-plugin` | 生成 HTML 文件 |
| `mini-css-extract-plugin` | 提取 CSS 到单独文件 |
| `css-minimizer-webpack-plugin` | 压缩 CSS |
| `terser-webpack-plugin` | 压缩 JavaScript |
| `clean-webpack-plugin` | 清空构建目录 |
| `copy-webpack-plugin` | 复制文件 |
| `webpack-bundle-analyzer` | 分析构建产物 |
| `define-plugin` | 定义环境变量 |
| `provide-plugin` | 自动加载模块 |
| `hot-module-replacement-plugin` | 模块热替换 |

## Mode（模式）

### 概念

通过选择 `development`, `production` 或 `none` 之中的一个，来设置 `mode` 参数，你可以启用 webpack 内置在相应环境下的优化。

### 基本用法

```javascript
module.exports = {
  mode: 'production' // 或 'development' 或 'none'
}
```

### 模式差异

| 特性 | development | production |
|------|-------------|------------|
| `process.env.NODE_ENV` | `'development'` | `'production'` |
| 代码压缩 | ❌ | ✅ |
| Source Map | ✅ | 可配置 |
| Tree Shaking | ❌ | ✅ |
| Scope Hoisting | ❌ | ✅ |
| 自动优化 | ❌ | ✅ |

### 开发模式

```javascript
module.exports = {
  mode: 'development',
  devtool: 'eval-cheap-module-source-map',
  optimization: {
    usedExports: true
  }
}

// 内置优化：
// - 开发工具
// - 缓存
// - 详细错误信息
```

### 生产模式

```javascript
module.exports = {
  mode: 'production',
  devtool: 'source-map',
  optimization: {
    minimize: true,
    splitChunks: {
      chunks: 'all'
    }
  }
}

// 内置优化：
// - 代码压缩
// - Tree Shaking
// - Scope Hoisting
// - 代码分割
```

## Module（模块）

### 概念

在模块化编程中，开发者将程序分解为功能离散的 chunk，并称之为模块。Webpack 支持以下模块类型：

- **ES Modules** (ESM)：ECMAScript 标准模块
- **CommonJS**：Node.js 模块
- **AMD**：异步模块定义
- **Assets**：资源模块

### ES Modules

```javascript
// 导出
export const name = 'value'
export default function() {}

// 导入
import name from './module'
import { name } from './module'
import * as module from './module'

// 动态导入
import('./module').then(module => {
  console.log(module.default)
})
```

### CommonJS

```javascript
// 导出
module.exports = {}
exports.name = 'value'

// 导入
const module = require('./module')
const { name } = require('./module')
```

### 模块解析

```javascript
module.exports = {
  resolve: {
    // 文件扩展名
    extensions: ['.js', '.json', '.ts'],

    // 路径别名
    alias: {
      '@': path.resolve(__dirname, 'src'),
      '@components': path.resolve(__dirname, 'src/components')
    },

    // 模块搜索路径
    modules: ['node_modules', 'src'],

    // 主字段
    mainFields: ['browser', 'module', 'main'],

    // 主文件
    mainFiles: ['index']
  }
}
```

## Chunk（代码块）

### 概念

Chunk 是 webpack 打包过程中的代码块，分为三种类型：

1. **Entry Chunk**：入口 chunk
2. **Runtime Chunk**：运行时代码
3. **Normal Chunk**：普通 chunk

### 代码分割

```javascript
module.exports = {
  optimization: {
    splitChunks: {
      chunks: 'all',  // 'initial' | 'async' | 'all'
      minSize: 20000,
      maxSize: 244000,
      minChunks: 1,
      maxAsyncRequests: 30,
      maxInitialRequests: 30,
      automaticNameDelimiter: '~',
      cacheGroups: {
        vendors: {
          test: /[\\/]node_modules[\\/]/,
          priority: -10,
          name: 'vendors'
        },
        default: {
          minChunks: 2,
          priority: -20,
          reuseExistingChunk: true
        }
      }
    },

    // 运行时代码分离
    runtimeChunk: 'single'
  }
}
```

### 动态导入

```javascript
// 使用 import() 语法
const module = await import('./module')

// 路由懒加载
const routes = [
  {
    path: '/about',
    component: () => import('./views/About.vue')
  }
]

// 预取/预加载
import(
  /* webpackPrefetch: true */
  /* webpackChunkName: "module" */
  './module'
)
```

## Manifest（清单）

### 概念

Webpack 生成的 Manifest 文件包含了所有模块的映射关系，用于追踪模块输出结果。

```javascript
const { WebpackManifestPlugin } = require('webpack-manifest-plugin')

module.exports = {
  plugins: [
    new WebpackManifestPlugin({
      fileName: 'manifest.json',
      basePath: '/assets/',
      seed: {
        name: 'My App',
        version: '1.0.0'
      }
    })
  ]
}
```

## 完整配置示例

```javascript
const path = require('path')
const HtmlWebpackPlugin = require('html-webpack-plugin')
const MiniCssExtractPlugin = require('mini-css-extract-plugin')
const { CleanWebpackPlugin } = require('clean-webpack-plugin')

module.exports = {
  // 入口
  entry: './src/index.js',

  // 输出
  output: {
    path: path.resolve(__dirname, 'dist'),
    filename: '[name].[contenthash:8].js',
    chunkFilename: '[name].[contenthash:8].chunk.js',
    publicPath: '/',
    clean: true
  },

  // 模式
  mode: 'production',

  // 模块规则
  module: {
    rules: [
      {
        test: /\.js$/,
        exclude: /node_modules/,
        use: 'babel-loader'
      },
      {
        test: /\.css$/,
        use: [MiniCssExtractPlugin.loader, 'css-loader']
      },
      {
        test: /\.(png|jpg|gif|svg)$/,
        type: 'asset',
        parser: {
          dataUrlCondition: {
            maxSize: 8 * 1024
          }
        }
      }
    ]
  },

  // 插件
  plugins: [
    new CleanWebpackPlugin(),
    new HtmlWebpackPlugin({
      template: './src/index.html'
    }),
    new MiniCssExtractPlugin({
      filename: '[name].[contenthash:8].css'
    })
  ],

  // 解析
  resolve: {
    extensions: ['.js', '.json'],
    alias: {
      '@': path.resolve(__dirname, 'src')
    }
  },

  // 优化
  optimization: {
    splitChunks: {
      chunks: 'all'
    },
    runtimeChunk: 'single'
  }
}
```

## Browser Compatibility（浏览器兼容性）

Webpack 支持所有符合 ES5 标准的浏览器（IE8 及以下版本不支持）。如果需要支持旧版本浏览器，可以使用以下配置：

```javascript
// 使用 Babel
module.exports = {
  module: {
    rules: [
      {
        test: /\.js$/,
        exclude: /node_modules/,
        use: {
          loader: 'babel-loader',
          options: {
            presets: [
              ['@babel/preset-env', {
                targets: {
                  ie: '11'
                },
                useBuiltIns: 'usage',
                corejs: 3
              }]
            ]
          }
        }
      }
    ]
  }
}

// 使用 @vitejs/plugin-legacy
const legacy = require('@vitejs/plugin-legacy')

module.exports = {
  plugins: [
    legacy({
      targets: ['ie >= 11']
    })
  ]
}
```
