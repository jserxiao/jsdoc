# 组件通信方式

组件通信是前端组件化开发的核心问题。根据组件之间的关系（父子、兄弟、跨层级），需要选择不同的通信方式。本章涵盖 Vue 3 和 React 两种框架的组件通信方案。

---

## 通信方式总览

| 方式 | Vue 3 | React | 方向 | 适用场景 |
|------|-------|-------|------|----------|
| Props | ✅ | ✅ | 父→子 | 简单数据传递 |
| Events/Emit | ✅ | ✅ (回调) | 子→父 | 子组件通知父组件 |
| v-model / 双向绑定 | ✅ | ✅ (受控组件) | 双向 | 表单组件 |
| Provide/Inject | ✅ | ❌ (Context) | 跨层级 | 深层嵌套组件 |
| Context | ❌ | ✅ | 跨层级 | 全局主题、用户信息 |
| 状态管理 (Pinia/Redux) | ✅ | ✅ | 全局 | 复杂状态管理 |
| Event Bus | ✅ (mitt) | ✅ (自定义) | 任意 | 简单跨组件通信 |
| Refs | ✅ | ✅ (useRef) | 父→子 | 直接访问子组件 |
| Slots | ✅ | ✅ (children/render props) | 父→子 | 内容分发 |

## 一、Props / Events（父子通信）

### Vue 3

#### 父传子（Props）

```vue
<!-- 父组件 -->
<template>
  <ChildComponent 
    :message="greeting" 
    :count="10"
    :user="currentUser"
  />
</template>

<script setup>
import { ref } from 'vue'
import ChildComponent from './ChildComponent.vue'

const greeting = ref('Hello')
const currentUser = ref({ name: 'John', email: 'john@example.com' })
</script>

<!-- 子组件 -->
<template>
  <div>
    <p>{{ message }}</p>
    <span>Count: {{ count }}</span>
  </div>
</template>

<script setup>
const props = defineProps({
  message: {
    type: String,
    required: true
  },
  count: {
    type: Number,
    default: 0
  },
  user: {
    type: Object,
    default: () => ({})
  }
})

// 使用 props
console.log(props.message)
</script>
```

#### 子传父（Events）

```vue
<!-- 子组件 -->
<template>
  <button @click="handleClick">更新</button>
</template>

<script setup>
const emit = defineEmits(['update', 'delete'])

function handleClick() {
  emit('update', { id: 1, name: 'updated' })
}

function handleDelete() {
  emit('delete', { id: 1 })
}
</script>

<!-- 父组件 -->
<template>
  <ChildComponent 
    @update="handleUpdate" 
    @delete="handleDelete" 
  />
</template>

<script setup>
function handleUpdate(data) {
  console.log('更新:', data)
}

function handleDelete(data) {
  console.log('删除:', data)
}
</script>
```

#### v-model 双向绑定

```vue
<!-- 自定义输入组件 -->
<template>
  <input 
    :value="modelValue" 
    @input="emit('update:modelValue', $event.target.value)"
  />
</template>

<script setup>
const props = defineProps(['modelValue'])
const emit = defineEmits(['update:modelValue'])
</script>

<!-- 多个 v-model -->
<template>
  <UserForm 
    v-model:username="form.username"
    v-model:email="form.email"
  />
</template>

<!-- UserForm.vue -->
<script setup>
const props = defineProps(['username', 'email'])
const emit = defineEmits(['update:username', 'update:email'])
</script>
```

### React

#### 父传子（Props）

```jsx
// 父组件
function ParentComponent() {
  const [greeting, setGreeting] = useState('Hello')
  const [currentUser, setCurrentUser] = useState({ name: 'John' })
  
  return (
    <ChildComponent 
      message={greeting}
      count={10}
      user={currentUser}
    />
  )
}

// 子组件
function ChildComponent({ message, count, user }) {
  return (
    <div>
      <p>{message}</p>
      <span>Count: {count}</span>
      <span>User: {user.name}</span>
    </div>
  )
}

// 带默认值
function ChildComponent({ 
  message, 
  count = 0, 
  user = { name: 'Anonymous' } 
}) {
  // ...
}
```

#### 子传父（回调函数）

```jsx
// 子组件
function ChildComponent({ onUpdate, onDelete }) {
  const handleClick = () => {
    onUpdate({ id: 1, name: 'updated' })
  }
  
  return (
    <button onClick={handleClick}>更新</button>
  )
}

// 父组件
function ParentComponent() {
  const handleUpdate = (data) => {
    console.log('更新:', data)
  }
  
  return (
    <ChildComponent 
      onUpdate={handleUpdate}
      onDelete={(data) => console.log('删除:', data)}
    />
  )
}
```

#### 受控组件（双向绑定）

```jsx
// 受控输入组件
function CustomInput({ value, onChange, placeholder }) {
  return (
    <input
      value={value}
      onChange={(e) => onChange(e.target.value)}
      placeholder={placeholder}
    />
  )
}

// 使用
function Form() {
  const [text, setText] = useState('')
  
  return (
    <CustomInput 
      value={text} 
      onChange={setText}
      placeholder="请输入"
    />
  )
}
```

## 二、跨层级通信

### Vue 3: Provide / Inject

```vue
<!-- 祖先组件 -->
<script setup>
import { provide, ref, readonly } from 'vue'

const theme = ref('dark')
const user = ref({ name: 'John', role: 'admin' })

// 提供只读状态
provide('theme', readonly(theme))

// 提供修改方法
provide('themeActions', {
  setTheme: (newTheme) => { theme.value = newTheme },
  toggleTheme: () => { 
    theme.value = theme.value === 'dark' ? 'light' : 'dark' 
  }
})

// 提供响应式对象
provide('user', user)
</script>

<!-- 后代组件（任意层级） -->
<script setup>
import { inject } from 'vue'

// 注入并设置默认值
const theme = inject('theme', 'light')
const { setTheme, toggleTheme } = inject('themeActions', {
  setTheme: () => {},
  toggleTheme: () => {}
})
const user = inject('user')
</script>
```

### React: Context

```jsx
import { createContext, useContext, useState } from 'react'

// 1. 创建 Context
const ThemeContext = createContext(undefined)
const UserContext = createContext(undefined)

// 2. Provider 组件
function ThemeProvider({ children }) {
  const [theme, setTheme] = useState('dark')
  
  const value = {
    theme,
    setTheme,
    toggleTheme: () => setTheme(t => t === 'dark' ? 'light' : 'dark')
  }
  
  return (
    <ThemeContext.Provider value={value}>
      {children}
    </ThemeContext.Provider>
  )
}

// 3. 自定义 Hook（推荐）
function useTheme() {
  const context = useContext(ThemeContext)
  if (context === undefined) {
    throw new Error('useTheme must be used within a ThemeProvider')
  }
  return context
}

// 4. 使用
function App() {
  return (
    <ThemeProvider>
      <ThemeProvider>
        <Header />
        <Main />
      </ThemeProvider>
    </ThemeProvider>
  )
}

function DeepChild() {
  const { theme, toggleTheme } = useTheme()
  
  return (
    <div className={`theme-${theme}`}>
      <button onClick={toggleTheme}>切换主题</button>
    </div>
  )
}
```

#### 多 Context 组合

```jsx
// 多个 Context
function AppProviders({ children }) {
  return (
    <ThemeProvider>
      <UserProvider>
        <NotificationProvider>
          {children}
        </NotificationProvider>
      </UserProvider>
    </ThemeProvider>
  )
}

// 组合 Hook
function useAppContext() {
  const theme = useTheme()
  const user = useUser()
  const notification = useNotification()
  
  return { theme, user, notification }
}
```

## 三、状态管理

### Vue 3: Pinia

```javascript
// stores/user.js
import { defineStore } from 'pinia'
import { ref, computed } from 'vue'

// Composition API 风格
export const useUserStore = defineStore('user', () => {
  // State
  const user = ref(null)
  const token = ref(null)
  
  // Getters
  const isLoggedIn = computed(() => !!token.value)
  const userName = computed(() => user.value?.name ?? 'Guest')
  
  // Actions
  async function login(credentials) {
    const response = await api.login(credentials)
    user.value = response.user
    token.value = response.token
  }
  
  function logout() {
    user.value = null
    token.value = null
  }
  
  return { user, token, isLoggedIn, userName, login, logout }
})

// Options API 风格
export const useCounterStore = defineStore('counter', {
  state: () => ({
    count: 0
  }),
  
  getters: {
    doubleCount: (state) => state.count * 2
  },
  
  actions: {
    increment() {
      this.count++
    },
    async fetchCount() {
      const result = await api.getCount()
      this.count = result
    }
  }
})
```

```vue
<!-- 组件中使用 -->
<script setup>
import { storeToRefs } from 'pinia'
import { useUserStore, useCounterStore } from '@/stores'

const userStore = useUserStore()
const counterStore = useCounterStore()

// 解构保持响应式
const { user, isLoggedIn, userName } = storeToRefs(userStore)
const { count, doubleCount } = storeToRefs(counterStore)

// Actions 直接解构
const { login, logout } = userStore
const { increment, fetchCount } = counterStore

// 直接修改
userStore.user = { name: 'John' }

// $patch 批量修改
userStore.$patch({
  user: { name: 'John' },
  token: 'xxx'
})

// $patch 函数式
counterStore.$patch((state) => {
  state.count++
  state.lastUpdated = Date.now()
})
</script>
```

### React: Redux Toolkit

```javascript
// store/userSlice.js
import { createSlice, createAsyncThunk } from '@reduxjs/toolkit'

export const loginUser = createAsyncThunk(
  'user/login',
  async (credentials) => {
    const response = await api.login(credentials)
    return response
  }
)

const userSlice = createSlice({
  name: 'user',
  initialState: {
    user: null,
    token: null,
    loading: false,
    error: null
  },
  reducers: {
    logout: (state) => {
      state.user = null
      state.token = null
    },
    updateUser: (state, action) => {
      state.user = { ...state.user, ...action.payload }
    }
  },
  extraReducers: (builder) => {
    builder
      .addCase(loginUser.pending, (state) => {
        state.loading = true
      })
      .addCase(loginUser.fulfilled, (state, action) => {
        state.loading = false
        state.user = action.payload.user
        state.token = action.payload.token
      })
      .addCase(loginUser.rejected, (state, action) => {
        state.loading = false
        state.error = action.error.message
      })
  }
})

export const { logout, updateUser } = userSlice.actions
export default userSlice.reducer
```

```jsx
// 组件中使用
import { useSelector, useDispatch } from 'react-redux'
import { loginUser, logout } from '@/store/userSlice'

function UserProfile() {
  const dispatch = useDispatch()
  const { user, loading, error } = useSelector((state) => state.user)
  const isLoggedIn = useSelector((state) => !!state.user.token)
  
  const handleLogin = async (credentials) => {
    dispatch(loginUser(credentials))
  }
  
  if (loading) return <Spinner />
  if (error) return <ErrorMessage error={error} />
  
  return (
    <div>
      {isLoggedIn ? (
        <>
          <span>{user.name}</span>
          <button onClick={() => dispatch(logout())}>登出</button>
        </>
      ) : (
        <button onClick={handleLogin}>登录</button>
      )}
    </div>
  )
}
```

### React: Zustand（轻量级方案）

```javascript
// stores/userStore.js
import { create } from 'zustand'
import { devtools, persist } from 'zustand/middleware'

const useUserStore = create(
  devtools(
    persist(
      (set, get) => ({
        user: null,
        token: null,
        
        // 派生状态
        isLoggedIn: () => !!get().token,
        
        // Actions
        login: async (credentials) => {
          const response = await api.login(credentials)
          set({ user: response.user, token: response.token })
        },
        
        logout: () => set({ user: null, token: null }),
        
        updateUser: (updates) => set((state) => ({
          user: { ...state.user, ...updates }
        }))
      }),
      { name: 'user-storage' }
    )
  )
)

export default useUserStore
```

```jsx
// 组件中使用
function UserProfile() {
  // 选择器避免不必要渲染
  const user = useUserStore((state) => state.user)
  const login = useUserStore((state) => state.login)
  const logout = useUserStore((state) => state.logout)
  
  // 多个状态
  const { user, token } = useUserStore((state) => ({
    user: state.user,
    token: state.token
  }))
  
  return (
    <div>
      {user ? (
        <>
          <span>{user.name}</span>
          <button onClick={logout}>登出</button>
        </>
      ) : (
        <button onClick={() => login({ email, password })}>登录</button>
      )}
    </div>
  )
}
```

## 四、事件总线

### Vue 3: mitt

```javascript
// utils/eventBus.js
import mitt from 'mitt'

const emitter = mitt()

// 类型定义
type Events = {
  'user:login': { id: number; name: string }
  'user:logout': void
  'notification:show': { type: 'success' | 'error'; message: string }
  'cart:update': { items: CartItem[]; total: number }
}

export const eventBus = mitt<Events>()

// 使用
eventBus.emit('user:login', { id: 1, name: 'John' })
eventBus.on('user:login', (user) => console.log('User logged in:', user))
eventBus.off('user:login', handler)
```

```vue
<script setup>
import { onMounted, onUnmounted } from 'vue'
import { eventBus } from '@/utils/eventBus'

function handleUserLogin(user) {
  console.log('用户登录:', user)
}

onMounted(() => {
  eventBus.on('user:login', handleUserLogin)
})

onUnmounted(() => {
  // 清理监听器
  eventBus.off('user:login', handleUserLogin)
})
</script>
```

### React: 自定义事件

```jsx
// utils/eventBus.js
class EventBus {
  constructor() {
    this.events = new Map()
  }
  
  on(event, callback) {
    if (!this.events.has(event)) {
      this.events.set(event, new Set())
    }
    this.events.get(event).add(callback)
    return () => this.off(event, callback)
  }
  
  emit(event, data) {
    const callbacks = this.events.get(event)
    if (callbacks) {
      callbacks.forEach(cb => cb(data))
    }
  }
  
  off(event, callback) {
    const callbacks = this.events.get(event)
    if (callbacks) {
      callbacks.delete(callback)
    }
  }
}

export const eventBus = new EventBus()

// Hook 封装
function useEventBus(event, callback) {
  useEffect(() => {
    const unsubscribe = eventBus.on(event, callback)
    return unsubscribe
  }, [event, callback])
}

// 使用
function NotificationListener() {
  useEventBus('notification:show', ({ type, message }) => {
    toast[type](message)
  })
  
  return null
}
```

## 五、Refs / 直接访问

### Vue 3: Template Refs

```vue
<script setup>
import { ref, onMounted } from 'vue'

const childRef = ref(null)

onMounted(() => {
  // 访问子组件方法
  childRef.value?.validate()
  childRef.value?.focus()
})
</script>

<template>
  <ChildComponent ref="childRef" />
</template>

<!-- 子组件 -->
<script setup>
import { defineExpose } from 'vue'

function validate() {
  return true
}

function focus() {
  inputRef.value?.focus()
}

// 显式暴露方法
defineExpose({
  validate,
  focus
})
</script>
```

### React: useRef + useImperativeHandle

```jsx
import { useRef, useImperativeHandle, forwardRef } from 'react'

// 子组件
const ChildComponent = forwardRef(function ChildComponent(props, ref) {
  const inputRef = useRef(null)
  
  // 自定义暴露给父组件的方法
  useImperativeHandle(ref, () => ({
    validate: () => {
      return inputRef.current?.value.length > 0
    },
    focus: () => {
      inputRef.current?.focus()
    },
    clear: () => {
      inputRef.current.value = ''
    }
  }))
  
  return <input ref={inputRef} />
})

// 父组件
function ParentComponent() {
  const childRef = useRef(null)
  
  const handleSubmit = () => {
    if (childRef.current?.validate()) {
      // 提交逻辑
    }
  }
  
  return (
    <>
      <ChildComponent ref={childRef} />
      <button onClick={handleSubmit}>提交</button>
    </>
  )
}
```

## 六、兄弟组件通信

### 方案一：状态提升

```jsx
// 将共享状态提升到共同父组件
function ParentComponent() {
  const [sharedData, setSharedData] = useState(null)
  
  return (
    <>
      <SiblingA data={sharedData} onDataChange={setSharedData} />
      <SiblingB data={sharedData} onDataChange={setSharedData} />
    </>
  )
}
```

### 方案二：Context/状态管理

```jsx
// 使用 Context 或状态管理库
function SiblingA() {
  const { data, setData } = useSharedState()
  return <div onClick={() => setData('new data')}>Sibling A</div>
}

function SiblingB() {
  const { data } = useSharedState()
  return <div>{data}</div>
}
```

### 方案三：事件总线

```javascript
// SiblingA 发送事件
eventBus.emit('data:update', newData)

// SiblingB 监听事件
useEventBus('data:update', (data) => {
  // 处理数据
})
```

## 七、通信方式选择指南

### 根据组件关系选择

```
父子通信
├── 简单数据 → Props
├── 子通知父 → Events / 回调
└── 表单双向绑定 → v-model / 受控组件

跨层级通信
├── 1-2 层 → Props 传递（可接受）
├── 3+ 层 → Provide/Inject 或 Context
└── 全局状态 → Pinia / Redux / Zustand

兄弟组件
├── 共同父组件状态提升
├── 状态管理库
└── 事件总线（简单场景）

任意组件
├── 状态管理库（推荐）
└── 事件总线（轻量场景）
```

### 根据数据复杂度选择

| 数据类型 | 推荐方案 |
|----------|----------|
| 临时 UI 状态 | Props / useState |
| 表单数据 | v-model / 受控组件 |
| 主题/用户信息 | Context / Provide |
| 复杂业务状态 | Redux / Pinia |
| 跨组件事件 | 事件总线 / 状态管理 |
