# Web 应用开发

## 概述

解决传统 Web 应用开发场景的需求，包括框架选型、状态管理、数据请求等核心问题。这是前端开发最常见、最基础的场景。

## 框架选型

### React
- **定位**：用于构建用户界面的 JavaScript 库
- **特点**：组件化、声明式、虚拟 DOM、强大的生态
- **适用场景**：中大型应用、团队协作、需要灵活性

### Vue
- **定位**：渐进式 JavaScript 框架
- **特点**：易上手、模板语法、响应式系统、完善的文档
- **适用场景**：快速开发、中小型项目、渐进式增强

### Angular
- **定位**：企业级 Web 应用框架
- **特点**：完整解决方案、TypeScript 原生支持、依赖注入
- **适用场景**：大型企业应用、团队规范统一

### Svelte
- **定位**：编译时框架
- **特点**：无虚拟 DOM、编译时优化、体积小
- **适用场景**：性能敏感项目、轻量级应用

### Solid
- **定位**：高性能响应式框架
- **特点**：细粒度响应式、类 React 语法、极致性能
- **适用场景**：高性能要求的应用

## 全栈框架

### Next.js
- **定位**：React 全栈框架
- **特点**：SSR/SSG/ISR、文件路由、API Routes
- **适用场景**：SEO 要求高、内容站点、全栈应用

### Nuxt.js
- **定位**：Vue 全栈框架
- **特点**：文件路由、自动导入、服务端渲染
- **适用场景**：Vue 技术栈的全栈应用

### Remix
- **定位**：全栈 Web 框架
- **特点**：拥抱 Web 标准、嵌套路由、数据加载策略
- **适用场景**：现代全栈 Web 应用

### Astro
- **定位**：静态站点生成器
- **特点**：多框架支持、岛屿架构、默认零 JS
- **适用场景**：内容网站、博客、文档站

## 状态管理

### Redux Toolkit
- **定位**：可预测的状态容器
- **特点**：单一数据源、时间旅行调试、中间件生态
- **适用场景**：大型应用、需要严格状态管理

### MobX
- **定位**：简单、可扩展的状态管理
- **特点**：响应式编程、自动追踪依赖、学习成本低
- **适用场景**：需要简洁代码的状态管理

### Zustand
- **定位**：简洁的 React 状态管理
- **特点**：API 简单、无样板代码、支持中间件
- **适用场景**：中小型 React 应用

### Pinia
- **定位**：Vue 官方状态管理
- **特点**：TypeScript 友好、模块化、DevTools 支持
- **适用场景**：Vue 3 应用首选

### Jotai
- **定位**：原子化状态管理
- **特点**：原子化设计、极简 API、无需 Provider
- **适用场景**：细粒度状态管理

## 数据请求

### Axios
- **定位**：HTTP 客户端
- **特点**：拦截器、请求取消、自动转换
- **适用场景**：通用 HTTP 请求场景

### SWR
- **定位**：数据请求 Hooks
- **特点**：自动缓存、重新验证、焦点自动刷新
- **适用场景**：React 数据获取场景

### React Query / TanStack Query
- **定位**：服务端状态管理
- **特点**：缓存管理、后台更新、乐观更新
- **适用场景**：复杂的数据请求与缓存场景

### GraphQL + Apollo Client
- **定位**：API 查询语言与客户端
- **特点**：按需查询、类型安全、缓存策略
- **适用场景**：复杂 API 场景、多数据源

## 技术选型建议

| 项目类型 | 框架推荐 | 状态管理 | 数据请求 |
|---------|---------|---------|---------|
| 企业后台 | React | Redux Toolkit | React Query |
| 中小型应用 | Vue | Pinia | Axios |
| 内容站点 | Next.js/Nuxt.js | 按需选择 | SWR |
| 快速原型 | Vue/React | Zustand/Pinia | Axios |
