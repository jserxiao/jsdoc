# 移动端与跨端开发

## 概述

解决移动端 H5、原生 App、小程序等多端开发场景的需求。随着移动互联网发展，多端开发已成为前端工程师必备技能。

## 跨平台移动开发

### React Native
- **定位**：基于 React 的跨平台移动开发
- **特点**：JavaScript 开发、原生组件、热更新
- **适用场景**：已有 React 技术栈团队、需要接近原生性能

### Flutter
- **定位**：Google 跨平台框架
- **特点**：Dart 语言、自绘渲染引擎、高性能
- **适用场景**：新项目、需要高度一致的跨平台 UI

### uni-app
- **定位**：Vue 语法开发多端应用
- **特点**：Vue 语法、多端编译、丰富的插件市场
- **适用场景**：国内开发、多端需求、Vue 技术栈

### Taro
- **定位**：多端开发框架
- **特点**：支持 React/Vue、多端编译、京东团队维护
- **适用场景**：多端小程序、H5 + 小程序

### Capacitor
- **定位**：Web 应用转原生 App
- **特点**：渐进式迁移、原生插件支持、与框架无关
- **适用场景**：已有 Web 应用转原生

## 小程序开发

### 微信小程序
- **定位**：原生小程序开发
- **特点**：WXML/WXSS/WXS、微信生态、性能最优
- **适用场景**：仅微信平台、追求极致性能

### Taro 小程序
- **定位**：多端小程序框架
- **特点**：React/Vue 语法、多端编译
- **适用场景**：多平台小程序统一开发

### uni-app 小程序
- **定位**：一套代码发布多端小程序
- **特点**：Vue 语法、丰富的组件库
- **适用场景**：快速开发、多端需求

### Remax
- **定位**：React 语法开发小程序
- **特点**：真正的 React 运行时、完整的 React 特性
- **适用场景**：React 技术栈开发小程序

## 移动端 UI 组件库

### Vant
- **定位**：轻量可靠的移动端组件库
- **特点**：Vue/React 双版本、60+ 高质量组件
- **适用场景**：移动端 H5、小程序

### Ant Design Mobile
- **定位**：移动端 UI 组件
- **特点**：React 技术栈、Ant Design 设计语言
- **适用场景**：React 移动端项目

### NutUI
- **定位**：京东移动端组件库
- **特点**：Vue 3 支持、多端适配
- **适用场景**：电商场景、Vue 3 项目

## 移动端适配方案

### rem 方案
- **原理**：根据屏幕宽度动态设置 html 字体大小
- **实现**：lib-flexible + postcss-pxtorem
- **优点**：兼容性好、开发便捷
- **缺点**：需要计算 rem 值

### vw/vh 方案
- **原理**：使用视口单位，元素大小随视口变化
- **实现**：postcss-px-to-viewport
- **优点**：无需 JS、性能好
- **缺点**：部分兼容性问题

### 媒体查询
- **原理**：根据设备宽度应用不同样式
- **适用场景**：响应式设计、PC + 移动端适配
- **特点**：精细控制、代码量较大

### 安全区域适配
- **问题**：刘海屏、底部指示器遮挡
- **方案**：env(safe-area-inset-*)、viewport-fit=cover
- **适用场景**：全屏应用、沉浸式体验

## 移动端性能优化

### 首屏优化
- 骨架屏 - 提升感知速度
- 预加载关键资源
- 路由懒加载

### 交互优化
- 防抖节流 - 处理高频事件
- 虚拟列表 - 长列表性能
- 惰性加载 - 图片、组件延迟加载

### 包体积优化
- 代码分割
- Tree Shaking
- 压缩资源

## 技术选型建议

| 需求场景 | 推荐方案 | 理由 |
|---------|---------|------|
| 原生 App | React Native / Flutter | 跨平台、接近原生体验 |
| 小程序多端 | uni-app / Taro | 一套代码多端发布 |
| 仅微信小程序 | 原生开发 / Taro | 性能最优或统一技术栈 |
| 移动端 H5 | Vue + Vant | 开发效率高、组件丰富 |
| Web 转 App | Capacitor | 复用现有代码 |
