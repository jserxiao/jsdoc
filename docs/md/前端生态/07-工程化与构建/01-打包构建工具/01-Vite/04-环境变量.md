# 环境变量

Vite 内置了对环境变量的支持，可以通过 `.env` 文件管理不同环境的配置。

## 环境变量文件

### 文件命名

```bash
.env                # 所有环境
.env.local          # 所有环境，不提交到 Git
.env.development    # 开发环境
.env.development.local
.env.production     # 生产环境
.env.production.local
.env.staging        # 自定义环境
```

### 文件优先级

```
.env.[mode].local > .env.[mode] > .env.local > .env
```

## 定义环境变量

```bash
# .env
VITE_APP_TITLE=My App
VITE_API_BASE_URL=/api

# .env.development
VITE_APP_TITLE=My App (Dev)
VITE_API_BASE_URL=http://localhost:3000/api
NODE_ENV=development

# .env.production
VITE_APP_TITLE=My App
VITE_API_BASE_URL=https://api.example.com
NODE_ENV=production

# .env.staging
VITE_APP_TITLE=My App (Staging)
VITE_API_BASE_URL=https://staging-api.example.com
```

> **注意**：只有 `VITE_` 前缀的变量才会暴露给客户端代码。

## 使用环境变量

### 在代码中使用

```javascript
// 访问环境变量
const title = import.meta.env.VITE_APP_TITLE
const apiUrl = import.meta.env.VITE_API_BASE_URL

console.log(import.meta.env.MODE)        // 当前模式
console.log(import.meta.env.DEV)         // 是否开发环境
console.log(import.meta.env.PROD)        // 是否生产环境
console.log(import.meta.env.SSR)         // 是否 SSR
console.log(import.meta.env.BASE_URL)    // 基础路径
```

### 在 HTML 中使用

```html
<!-- index.html -->
<!DOCTYPE html>
<html>
<head>
  <title>%VITE_APP_TITLE%</title>
</head>
<body>
  <div id="app"></div>
</body>
</html>
```

### 在配置文件中使用

```javascript
// vite.config.js
import { defineConfig, loadEnv } from 'vite'

export default defineConfig(({ mode }) => {
  // 加载环境变量
  const env = loadEnv(mode, process.cwd())
  
  return {
    define: {
      __APP_VERSION__: JSON.stringify(env.VITE_APP_VERSION)
    },
    server: {
      proxy: {
        '/api': {
          target: env.VITE_API_BASE_URL
        }
      }
    }
  }
})
```

## TypeScript 类型支持

```typescript
// src/env.d.ts
/// <reference types="vite/client" />

interface ImportMetaEnv {
  readonly VITE_APP_TITLE: string
  readonly VITE_API_BASE_URL: string
  readonly VITE_APP_VERSION: string
}

interface ImportMeta {
  readonly env: ImportMetaEnv
}
```

## 自定义模式

### 定义模式

```bash
# .env.staging
VITE_APP_TITLE=My App (Staging)
NODE_ENV=production
```

### 启动自定义模式

```json
// package.json
{
  "scripts": {
    "dev": "vite",
    "staging": "vite --mode staging",
    "build": "vite build",
    "build:staging": "vite build --mode staging"
  }
}
```

## Node.js 内置环境变量

```javascript
// 访问 NODE_ENV
console.log(process.env.NODE_ENV)

// 在配置中使用
export default defineConfig({
  define: {
    'process.env.NODE_ENV': JSON.stringify(process.env.NODE_ENV)
  }
})
```

## 环境变量验证

```javascript
// vite.config.js
import { defineConfig, loadEnv } from 'vite'
import z from 'zod'

const envSchema = z.object({
  VITE_APP_TITLE: z.string(),
  VITE_API_BASE_URL: z.string().url()
})

export default defineConfig(({ mode }) => {
  const env = loadEnv(mode, process.cwd())
  
  // 验证环境变量
  const result = envSchema.safeParse(env)
  if (!result.success) {
    console.error('环境变量验证失败:', result.error)
    process.exit(1)
  }
  
  return {
    // 配置...
  }
})
```

## 安全注意事项

### 不要暴露敏感信息

```bash
# ❌ 错误：敏感信息会被暴露到客户端
VITE_DB_PASSWORD=secret123
VITE_API_SECRET=abc123

# ✅ 正确：使用服务端环境变量
# DB_PASSWORD=secret123（仅在服务端使用）
```

### 本地环境变量

```bash
# .env.local - 不会被提交到 Git
VITE_API_KEY=my-secret-key
```

```gitignore
# .gitignore
.env.local
.env.*.local
```

## 动态环境变量

```javascript
// vite.config.js
export default defineConfig({
  define: {
    __BUILD_TIME__: JSON.stringify(new Date().toISOString()),
    __GIT_COMMIT__: JSON.stringify(
      require('child_process')
        .execSync('git rev-parse HEAD')
        .toString()
        .trim()
    )
  }
})
```

## 多环境配置示例

```javascript
// config/index.js
const configs = {
  development: {
    api: 'http://localhost:3000/api',
    debug: true
  },
  staging: {
    api: 'https://staging-api.example.com',
    debug: false
  },
  production: {
    api: 'https://api.example.com',
    debug: false
  }
}

export const config = configs[import.meta.env.MODE]
```
