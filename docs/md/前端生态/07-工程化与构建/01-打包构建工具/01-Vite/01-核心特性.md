# 核心特性

## 极速冷启动

Vite 利用浏览器原生 ESM 能力，无需打包即可启动开发服务器。

```
传统打包工具：
启动 → 分析依赖 → 编译打包 → 启动服务器 → 浏览器加载
（数十秒）

Vite：
启动 → 启动服务器 → 浏览器按需加载模块
（毫秒级）
```

### 工作原理

```
浏览器请求模块
    ↓
Vite 拦截请求
    ↓
按需编译单个文件
    ↓
返回 ESM 模块
```

## 即时 HMR

模块级别的热更新，无论项目大小，HMR 速度保持一致。

### HMR 原理

```javascript
// Vite 通过 HMR 边界实现精确更新
// 只有修改的模块及其依赖者会被更新

// 热更新 API
if (import.meta.hot) {
  import.meta.hot.accept((newModule) => {
    // 处理更新
  })
  
  import.meta.hot.dispose(() => {
    // 清理副作用
  })
}
```

### HMR 边界

```
App.vue (HMR 边界)
    ├── Header.vue
    │       └── Logo.vue
    └── Content.vue
            └── Button.vue

修改 Button.vue → 只更新 Button.vue 和 Content.vue
```

## ESM 原生

### 开发环境

```javascript
// 浏览器直接加载 ES Module
<script type="module">
  import { createApp } from '/node_modules/.vite/deps/vue.js'
  import App from '/src/App.vue'
  
  createApp(App).mount('#app')
</script>
```

### 依赖预构建

```javascript
// Vite 会预构建依赖
// 1. 将 CommonJS/UMD 转换为 ESM
// 2. 合并小模块，减少请求数

// 预构建缓存位置
node_modules/.vite/deps/
```

```javascript
// vite.config.js - 配置预构建
export default defineConfig({
  optimizeDeps: {
    include: ['lodash-es'],
    exclude: ['your-linked-package']
  }
})
```

## 生产构建

Vite 生产环境使用 Rollup 打包，确保最佳性能。

```javascript
// vite.config.js
export default defineConfig({
  build: {
    // 输出目录
    outDir: 'dist',
    // 生成 sourcemap
    sourcemap: true,
    // 代码分割策略
    rollupOptions: {
      output: {
        manualChunks: {
          vendor: ['vue', 'vue-router'],
          utils: ['lodash', 'dayjs']
        }
      }
    }
  }
})
```

```javascript

import svgUrl from './icon.svg'
import json from './data.json'

// URL 转换
// 小于 4KB：base64 内联
// 大于 4KB：单独文件

// 显式加载为 URL
import logoUrl from './logo.png?url'

// 显式加载为字符串
import svgString from './icon.svg?raw'
```
