# 选型决策指南

帮助开发者根据项目特点选择合适的构建工具。

---

## 选型决策树

```
项目需求
    │
    ├── 是否是 Next.js 项目？
    │   └── 是 → Turbopack（开发）+ Webpack（生产）
    │
    ├── 是否是库开发？
    │   └── 是 → Vite Library Mode 或 Rollup
    │
    ├── 是否有复杂 Webpack 配置需要迁移？
    │   ├── 是，且时间紧张 → Webpack
    │   └── 是，有时间重构 → Vite
    │
    ├── 是否需要特定 Webpack 插件？
    │   ├── 是，无替代品 → Webpack
    │   └── 否 → Vite
    │
    ├── 团队经验？
    │   ├── 熟悉 Webpack → 可选 Webpack
    │   └── 无偏好 → Vite（学习成本低）
    │
    └── 新项目？
        └── 是 → 推荐 Vite
```

---

## 场景选择

### 选择 Vite 的场景

| 场景 | 说明 |
|------|------|
| 新项目 | 开箱即用，配置简单 |
| Vue/React/Svelte SPA | 官方模板支持完善 |
| 库开发 | 库模式输出干净 |
| 中小型项目 | 开发体验极佳 |
| 团队快速上手 | 学习成本低 |
| 追求开发效率 | 启动和更新快 |

### 选择 Webpack 的场景

| 场景 | 说明 |
|------|------|
| 大型企业级项目 | 需要精细控制 |
| 依赖特定插件 | 无替代品可用 |
| 团队有 Webpack 经验 | 不需要重新学习 |
| 需要支持旧浏览器 | 配合 polyfill |
| 复杂构建流程 | 高度定制需求 |
| 微前端架构 | qiankun 等方案成熟 |

## 迁移指南

### Webpack → Vite

#### 1. 配置迁移

```javascript
// webpack.config.js
module.exports = {
  resolve: {
    alias: { '@': path.resolve(__dirname, 'src') }
  },
  module: {
    rules: [
      { test: /\.css$/, use: ['style-loader', 'css-loader'] }
    ]
  }
}

// ↓ 迁移到

// vite.config.js
export default defineConfig({
  resolve: {
    alias: { '@': '/src' }
  },
  // CSS 不需要配置
})
```

#### 2. 环境变量

```javascript
// Webpack
process.env.VUE_APP_TITLE

// ↓ 迁移到

// Vite
import.meta.env.VITE_APP_TITLE
```

#### 3. 动态导入

```javascript
// Webpack
require.context('./dir', true, /\.vue$/)

// ↓ 迁移到

// Vite
const modules = import.meta.glob('./dir/**/*.vue')
```

#### 4. 常见问题

```javascript
// 问题 1：require 语法不兼容
const config = require('./config')
// 解决：改为 ESM
import config from './config'

// 问题 2：全局变量
new webpack.DefinePlugin({ __DEV__: true })
// 解决：使用 define 配置或 import.meta.env

// 问题 3：特定的 Webpack Loader
// 解决：寻找 Vite 插件替代或使用 vite-plugin-static-copy
```

#### 5. 迁移检查清单

```markdown
- [ ] 更新 package.json scripts
- [ ] 创建 vite.config.js
- [ ] 迁移环境变量（.env 文件）
- [ ] 更新 index.html 位置
- [ ] 替换 require 为 import
- [ ] 替换 require.context 为 import.meta.glob
- [ ] 更新环境变量访问方式
- [ ] 迁移自定义 Loader/Plugin
- [ ] 测试所有功能
```

## 最佳实践

### Vite 最佳实践

```javascript
// vite.config.js
import { defineConfig } from 'vite'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [react()],
  
  // 开发服务器
  server: {
    port: 3000,
    proxy: {
      '/api': {
        target: 'http://localhost:8080',
        changeOrigin: true
      }
    }
  },
  
  // 构建配置
  build: {
    outDir: 'dist',
    sourcemap: true,
    rollupOptions: {
      output: {
        manualChunks: {
          vendor: ['react', 'react-dom']
        }
      }
    }
  },
  
  // 依赖优化
  optimizeDeps: {
    include: ['react', 'react-dom']
  }
})
```

### Webpack 最佳实践

```javascript
// webpack.config.js
const path = require('path')

module.exports = {
  mode: process.env.NODE_ENV || 'development',
  
  entry: './src/index.js',
  
  output: {
    path: path.resolve(__dirname, 'dist'),
    filename: '[name].[contenthash].js',
    clean: true
  },
  
  // 缓存
  cache: {
    type: 'filesystem'
  },
  
  // 模块规则
  module: {
    rules: [
      {
        test: /\.js$/,
        exclude: /node_modules/,
        use: 'swc-loader'
      },
      {
        test: /\.css$/,
        use: ['style-loader', 'css-loader']
      }
    ]
  },
  
  // 优化
  optimization: {
    splitChunks: {
      chunks: 'all'
    }
  },
  
  // 开发服务器
  devServer: {
    port: 3000,
    hot: true,
    proxy: {
      '/api': 'http://localhost:8080'
    }
  }
}
```

## 混合方案

开发环境用 Vite，生产环境用 Webpack：

```javascript
// package.json
{
  "scripts": {
    "dev": "vite",
    "build": "webpack --config webpack.prod.js"
  }
}
```

**适用场景：**
- 已有 Webpack 生产构建配置
- 想要更好的开发体验
- 需要保证生产构建稳定

## 工具对比总结

| 维度 | Vite | Webpack |
|------|------|---------|
| 学习曲线 | 低 | 高 |
| 开发体验 | 极好 | 好 |
| 构建速度 | 快 | 较慢 |
| 配置复杂度 | 简单 | 复杂 |
| 生态成熟度 | 成长中 | 成熟 |
| 定制能力 | 中等 | 极高 |
| 调试体验 | 好 | 一般 |
