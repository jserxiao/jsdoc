# 组件设计原则

组件化是现代前端开发的核心思想。良好的组件设计能够提高代码复用性、可维护性和可测试性。本章结合《Clean Code》、《设计模式》、React/Vue 官方最佳实践，深入探讨组件设计的核心原则。

---

## 核心设计原则概述

### 软件设计原则在组件中的应用

| 原则 | 来源 | 在组件中的应用 |
|------|------|----------------|
| 单一职责原则 (SRP) | SOLID | 每个组件只做一件事 |
| 开闭原则 (OCP) | SOLID | 对扩展开放，对修改关闭 |
| 里氏替换原则 (LSP) | SOLID | 子组件可替换父组件 |
| 接口隔离原则 (ISP) | SOLID | 不应强制依赖不使用的 Props |
| 依赖倒置原则 (DIP) | SOLID | 依赖抽象而非具体实现 |
| 迪米特法则 (LoD) | 面向对象 | 最小知识原则，减少耦合 |
| 组合优于继承 | GoF 设计模式 | 优先使用组合而非继承 |

---

## 一、单一职责原则 (Single Responsibility Principle)

> "一个类应该只有一个引起它变化的原因。" —— Robert C. Martin

### 核心思想

每个组件应该只有一个职责，只有一个理由去改变。如果一个组件承担多个职责，这些职责就会耦合在一起，导致：

- 修改一个职责可能影响其他职责
- 代码难以理解和维护
- 复用变得困难

### 反模式示例

```vue
<!-- ❌ 违反 SRP：组件职责过多 -->
<template>
  <div class="user-dashboard">
    <!-- 用户信息展示 -->
    <div class="user-info">

    <div v-else-if="error">{{ error }}</div>
    
    <!-- 图表渲染 -->
    <div class="chart-container">
      <!-- ECharts 初始化和渲染 -->
    </div>
    
    <!-- 表格展示 -->

      <!-- 复杂的表格逻辑 -->
    </table>
    
    <!-- 表单编辑 -->
    <form @submit="handleSubmit">
      <!-- 表单字段 -->
    </form>
  </div>

    initChart()

    error.value = e.message
  } finally {
    loading.value = false
  }

  chartInstance.value = echarts.init(/* ... */)

  // 表单提交逻辑...
```

## 二、开闭原则 (Open/Closed Principle)

> "软件实体应该对扩展开放，对修改关闭。" —— Bertrand Meyer

### 核心思想

组件应该在不修改现有代码的情况下，能够扩展其行为。这通常通过以下方式实现：

- 使用插槽（Slots）或渲染属性（Render Props）
- 使用组合模式
- 使用高阶组件（HOC）或 Hooks

### 示例：可扩展的列表组件

```vue
<!-- ✅ 对扩展开放：通过插槽支持自定义渲染 -->
<template>
  <ul class="list">
    <li v-for="item in items" :key="getKey(item)">
      <slot name="item" :item="item" :index="index">
        <!-- 默认渲染 -->
        {{ item }}
      </slot>
    </li>
  </ul>
</template>

<script setup>
import { withDefaults } from 'vue'

const props = withDefaults(defineProps({
  items: Array,
  itemKey: {
    type: [String, Function],
    default: 'id'
  }
}))

function getKey(item) {
  return typeof props.itemKey === 'string' 
    ? item[props.itemKey] 
    : props.itemKey(item)
}
</script>

<!-- 使用：扩展为用户列表 -->
<UserList :items="users">
  <template #item="{ item }">
    <div class="user-item">
      <img :src="item.avatar" />
      <span>{{ item.name }}</span>
    </div>
  </template>
</UserList>

<!-- 使用：扩展为商品列表 -->
<ProductList :items="products">
  <template #item="{ item }">
    <div class="product-item">
      <img :src="item.image" />
      <span>{{ item.name }}</span>
      <span class="price">{{ item.price }}</span>
    </div>
  </template>
</ProductList>
```

### React Render Props 模式

```jsx
// ✅ 使用 Render Props 实现开闭原则
function List({ items, renderItem, keyExtractor }) {
  return (
    <ul className="list">
      {items.map((item, index) => (
        <li key={keyExtractor(item, index)}>
          {renderItem(item, index)}
        </li>
      ))}
    </ul>
  )
}

// 使用：扩展为用户列表
<List
  items={users}
  keyExtractor={(user) => user.id}
  renderItem={(user) => (
    <div className="user-item">
      <img src={user.avatar} alt={user.name} />
      <span>{user.name}</span>
    </div>
  )}
/>
```

## 三、组合优于继承 (Composition over Inheritance)

> "优先使用对象组合而不是类继承。" —— GoF 设计模式

### 为什么组合优于继承

| 组合 | 继承 |
|------|------|
| 灵活，运行时可变 | 静态，编译时确定 |
| 松耦合 | 紧耦合 |
| 易于测试 | 测试困难 |
| 避免继承层次过深 | 可能导致脆弱基类问题 |

### React 组合模式

```jsx
// ❌ 继承方式：脆弱且不灵活
class Button extends BaseComponent {
  render() {
    return <button className="btn">{this.props.children}</button>
  }
}

class IconButton extends Button {
  render() {
    return (
      <button className="btn">
        <Icon name={this.props.icon} />
        {this.props.children}
      </button>
    )
  }
}

// ✅ 组合方式：灵活且可扩展
function Button({ children, variant = 'default', size = 'medium', ...props }) {
  return (
    <button 
      className={cn('btn', `btn-${variant}`, `btn-${size}`)} 
      {...props}
    >
      {children}
    </button>
  )
}

function IconButton({ icon, children, ...props }) {
  return (
    <Button {...props}>
      <Icon name={icon} className="btn-icon" />
      {children}
    </Button>
  )
}

// 使用
<Button variant="primary">Click me</Button>
<IconButton icon="plus" variant="success">Add Item</IconButton>
```

### Vue 组合模式

```vue
<!-- ✅ 组合模式：通过插槽和组合 -->
```

## 四、Props 设计原则

### 4.1 Props 验证与类型安全

```vue
<script setup>
// ✅ 完整的 Props 验证
const props = defineProps({
  // 基础类型
  title: {
    type: String,
    required: true
  },
  
  // 多类型
  value: {
    type: [String, Number, Boolean],
    default: ''
  },
  
  // 枚举验证
  variant: {
    type: String,
    default: 'default',
    validator: (v) => ['default', 'primary', 'success', 'warning', 'danger'].includes(v)
  },
  
  // 对象默认值（必须使用工厂函数）
  config: {
    type: Object,
    default: () => ({
      theme: 'light',
      size: 'medium'
    })
  },
  
  // 数组默认值
  items: {
    type: Array,
    default: () => []
  },
  
  // 函数类型
  formatter: {
    type: Function,
    default: (value) => value
  },
  
  // 自定义类实例
  user: {
    type: Object,
    // 自定义验证器
    validator: (user) => {
      return user && typeof user.id === 'number' && typeof user.name === 'string'
    }
  }
})
</script>
```

### 4.2 TypeScript 增强

```typescript

interface ButtonProps {
  variant?: 'default' | 'primary' | 'success' | 'warning' | 'danger'
  size?: 'small' | 'medium' | 'large'
  disabled?: boolean
  loading?: boolean
  onClick?: (event: React.MouseEvent<HTMLButtonElement>) => void
  children: React.ReactNode
}

function Button({ 
  variant = 'default', 
  size = 'medium',
  disabled = false,
  loading = false,
  onClick,
  children 
}: ButtonProps) {
  return (
    <button 
      className={cn('btn', `btn-${variant}`, `btn-${size}`)}
      disabled={disabled || loading}
      onClick={onClick}
    >
      {loading ? <Spinner /> : children}
    </button>
  )
}
```

### 5.1 事件命名规范

### 5.2 事件数据设计

```vue
<script setup>
// ✅ 事件携带结构化数据
const emit = defineEmits({
  // 无验证
  'click': null,
  
  // 带验证的事件
  'submit': (payload) => {
    // 验证数据结构
    return (
      typeof payload === 'object' &&
      typeof payload.email === 'string' &&
      typeof payload.password === 'string'
    )
  },
  
  'item-select': (payload) => {
    return (
      typeof payload.id === 'string' &&
      typeof payload.selected === 'boolean'
    )
  }
})

// 发送事件时携带完整上下文
function handleSubmit(formData) {
  emit('submit', {
    email: formData.email,
    password: formData.password,
    rememberMe: formData.rememberMe,
    timestamp: Date.now()
  })
}
</script>
```

### React 回调设计

```jsx
// ✅ 清晰的回调接口
interface ListProps {
  items: Item[]
  onItemSelect?: (item: Item, index: number) => void
  onItemDelete?: (itemId: string) => void
  onSelectionChange?: (selectedItems: Item[]) => void
}

function List({ items, onItemSelect, onItemDelete, onSelectionChange }: ListProps) {
  const handleItemClick = (item: Item, index: number) => {
    onItemSelect?.(item, index)
  }
  
  return (
    <ul>
      {items.map((item, index) => (
        <li key={item.id} onClick={() => handleItemClick(item, index)}>
          {item.name}
          <button onClick={(e) => {
            e.stopPropagation()
            onItemDelete?.(item.id)
          }}>
            Delete
          </button>
        </li>
      ))}
    </ul>
  )
}
```

## 七、组件分层架构

### 容器组件 vs 展示组件

> 模式来源：Dan Abramov（Redux 作者）

```jsx
// 展示组件：只关心 UI
function UserList({ users, onSelect, selectedId }) {
  return (
    <ul className="user-list">
      {users.map(user => (
        <li 
          key={user.id}
          className={cn({ selected: user.id === selectedId })}
          onClick={() => onSelect(user)}
        >
          <img src={user.avatar} alt={user.name} />
          <span>{user.name}</span>
        </li>
      ))}
    </ul>
  )
}

// 容器组件：负责数据和逻辑
function UserListContainer() {
  const dispatch = useDispatch()
  const { users, selectedId, loading, error } = useSelector(state => state.users)
  
  useEffect(() => {
    dispatch(fetchUsers())
  }, [dispatch])
  
  const handleSelect = useCallback((user) => {
    dispatch(selectUser(user.id))
  }, [dispatch])
  
  if (loading) return <Spinner />
  if (error) return <ErrorMessage error={error} />
  
  return <UserList users={users} selectedId={selectedId} onSelect={handleSelect} />
}
```

### 组件分类

| 类型 | 职责 | 示例 |
|------|------|------|
| 基础组件 (Base) | UI 基础元素 | Button、Input、Icon |
| 组合组件 (Composite) | 组合基础组件 | Form、Modal、Table |
| 业务组件 (Business) | 特定业务逻辑 | UserCard、OrderItem |
| 布局组件 (Layout) | 页面布局 | Header、Sidebar、Grid |
| 容器组件 (Container) | 数据获取和状态管理 | UserListContainer |
| 高阶组件 (HOC) | 逻辑复用 | withAuth、withLoading |

## 九、最佳实践总结

### 组件设计检查清单

- [ ] **单一职责**：组件是否只做一件事？
- [ ] **可复用性**：组件是否能在其他地方使用？
- [ ] **Props 设计**：Props 是否精简、类型安全？
- [ ] **事件设计**：事件命名是否清晰、数据是否结构化？
- [ ] **插槽设计**：是否预留扩展点？
- [ ] **可测试性**：组件是否易于单元测试？
- [ ] **可访问性**：是否支持键盘导航和屏幕阅读器？
- [ ] **性能优化**：是否避免不必要的渲染？

### 反模式警示

```vue
<!-- ❌ 反模式：Props Drilling -->
<GrandParent :data="data" />
  <Parent :data="data" />  <!-- 只是为了传递 -->
    <Child :data="data" />  <!-- 只是为了传递 -->
      <GrandChild :data="data" />  <!-- 真正使用者 -->
```
