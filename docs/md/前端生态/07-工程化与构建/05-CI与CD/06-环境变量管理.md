# 环境变量管理

## GitHub Secrets 配置

在项目 Settings → Secrets and variables → Actions 中配置：

| Secret 名称 | 说明 |
|-------------|------|
| `SERVER_HOST` | 服务器地址 |
| `SERVER_USER` | 服务器用户名 |
| `SSH_PRIVATE_KEY` | SSH 私钥 |
| `DOCKERHUB_USERNAME` | Docker Hub 用户名 |
| `DOCKERHUB_TOKEN` | Docker Hub Token |
| `SLACK_WEBHOOK` | Slack 通知 Webhook |
| `DATABASE_URL` | 数据库连接字符串 |
| `API_KEY` | API 密钥 |

## 环境变量文件

```yaml
# .github/workflows/deploy.yml
env:
  NODE_ENV: production

jobs:
  deploy:
    environment: production
    env:
      DATABASE_URL: ${{ secrets.DATABASE_URL }}
      API_KEY: ${{ secrets.API_KEY }}
```

## 多环境配置

```yaml
# .github/workflows/deploy.yml
jobs:
  deploy_staging:
    runs-on: ubuntu-latest
    environment: staging
    env:
      API_URL: ${{ secrets.STAGING_API_URL }}
    steps:
      - run: echo "Deploying to staging"

  deploy_production:
    runs-on: ubuntu-latest
    environment: production
    env:
      API_URL: ${{ secrets.PRODUCTION_API_URL }}
    steps:
      - run: echo "Deploying to production"
```

## 本地环境变量

### .env 文件

```bash
# .env.development
VITE_API_URL=http://localhost:3000

# .env.production
VITE_API_URL=https://api.example.com
```

### 使用方式

```javascript
// Vite 项目
const apiUrl = import.meta.env.VITE_API_URL

// Create React App
const apiUrl = process.env.REACT_APP_API_URL
```

## 安全最佳实践

### 1. 不要提交敏感信息

```gitignore
# .gitignore
.env
.env.local
.env.*.local
```

### 2. 使用 Secrets 而非明文

```yaml
# ❌ 不安全
env:
  API_KEY: "sk-xxxxx"

# ✅ 安全
env:
  API_KEY: ${{ secrets.API_KEY }}
```

### 3. 限制 Secrets 访问

```yaml
jobs:
  deploy:
    environment: production  # 需要审批
    steps:
      - run: deploy
```

### 4. 使用环境保护规则

在 GitHub 项目 Settings → Environments 中配置：
- Required reviewers（需要审批）
- Wait timer（延迟部署）
- Deployment branches（限制分支）
