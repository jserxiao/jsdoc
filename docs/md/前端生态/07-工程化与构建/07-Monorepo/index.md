# Monorepo

## 概述

Monorepo（单一代码仓库）是一种将多个项目代码存储在同一个仓库中的开发策略。在前端工程化中，Monorepo 可以有效管理多个包、共享代码、统一配置，提高大型项目的开发效率。

## Monorepo vs Multirepo

| 特性 | Monorepo | Multirepo |
|------|----------|-----------|
| 代码共享 | ✅ 容易 | ❌ 需要发布包 |
| 统一配置 | ✅ 容易 | ❌ 每个仓库独立 |
| 依赖管理 | ✅ 统一版本 | ❌ 可能版本冲突 |
| CI/CD | ✅ 统一流程 | ❌ 各自配置 |
| 历史追踪 | ✅ 完整历史 | ❌ 分散在各仓库 |
| 权限管理 | ❌ 需要细化 | ✅ 独立权限 |
| 仓库大小 | ❌ 可能很大 | ✅ 相对较小 |

## 主流工具对比

| 工具 | 核心特点 | 适用场景 |
|------|----------|----------|
| **pnpm workspace** | 原生支持、硬链接存储 | 新项目首选 |
| **Turborepo** | 增量构建、远程缓存 | 大型项目 |
| **Nx** | 代码生成、依赖图可视化 | 企业级项目 |
| **Lerna** | 版本管理、发布流程 | npm 包开发 |
| **Rush** | 微软出品、企业级方案 | 大型企业 |

## 技术选型建议

| 场景 | 推荐方案 | 理由 |
|------|---------|------|
| 新项目 | pnpm workspace + Turborepo | 简单高效 |
| 组件库开发 | pnpm workspace + Changesets | 发布流程完善 |
| 企业级项目 | Nx | 功能全面 |
| npm 包开发 | Lerna / Changesets | 发布管理成熟 |
| 大型团队 | Turborepo + 远程缓存 | 构建效率高 |
