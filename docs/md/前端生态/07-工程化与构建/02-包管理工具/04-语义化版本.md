# 语义化版本

## 版本号规则

格式：`major.minor.patch`

- **major（主版本）**：不兼容的 API 修改
- **minor（次版本）**：向下兼容的功能新增
- **patch（补丁版本）**：向下兼容的 bug 修复

```
MAJOR.MINOR.PATCH
  │     │     │
  │     │     └── 修复 bug，向后兼容
  │     └──────── 新功能，向后兼容
  └────────────── 重大变更，不兼容
```

## 版本范围语法

| 符号 | 含义 | 示例 |
|------|------|------|
| `^` | 兼容次版本 | `^1.2.3` → `>=1.2.3 <2.0.0` |
| `~` | 兼容补丁版本 | `~1.2.3` → `>=1.2.3 <1.3.0` |
| `>` | 大于 | `>1.2.3` |
| `>=` | 大于等于 | `>=1.2.3` |
| `<` | 小于 | `<2.0.0` |
| `<=` | 小于等于 | `<=2.0.0` |
| `x` | 通配符 | `1.2.x` → `>=1.2.0 <1.3.0` |
| `||` | 或 | `^1.0.0 || ^2.0.0` |
| `-` | 范围 | `1.0.0 - 2.0.0` |

## 预发布版本

```bash
# 预发布标签
1.0.0-alpha.1
1.0.0-beta.2
1.0.0-rc.1

# 安装预发布版本
npm install <package>@beta
npm install <package>@next
```

## 锁定文件

| 工具 | 锁定文件 | 作用 |
|------|----------|------|
| npm | package-lock.json | 锁定依赖版本树 |
| pnpm | pnpm-lock.yaml | 锁定依赖版本 |
| Yarn | yarn.lock | 锁定依赖版本 |

## 最佳实践

### 始终提交锁文件

```bash
git add package-lock.json  # 或 yarn.lock / pnpm-lock.yaml
```

### 明确依赖类型

```bash
# 生产环境需要的
npm install <package> --save

# 开发环境需要的（构建工具、测试框架等）
npm install <package> --save-dev
```

### 定期检查过期依赖

```bash
npm outdated
pnpm outdated
yarn outdated
```
