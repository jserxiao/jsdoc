# 最佳实践

## 选择包管理器

- **npm**：Node.js 默认自带，适合小型项目和快速原型
- **yarn**：适合中大型项目，需要更好的稳定性和性能
- **pnpm**：适合 Monorepo 项目，磁盘空间敏感场景

## 使用建议

### 1. 始终提交锁文件

```bash
git add package-lock.json  # 或 yarn.lock / pnpm-lock.yaml
```

### 2. 明确依赖类型

```bash
# 生产环境需要的
npm install <package> --save

# 开发环境需要的（构建工具、测试框架等）
npm install <package> --save-dev
```

### 3. 定期检查安全漏洞

```bash
npm audit
npm audit fix
```

### 4. 使用 npx 运行一次性命令

```bash
npx create-react-app my-app
npx cowsay "Hello"
```

### 5. 避免全局安装过多包

```bash
# ❌ 全局安装
npm install -g eslint

# ✅ 本地安装
npm install eslint --save-dev
npx eslint --init
```

## 常见问题

### 依赖安装失败

```bash
# 清理缓存
npm cache clean --force
rm -rf node_modules package-lock.json
npm install

# 切换镜像源
npm config set registry https://registry.npmmirror.com
```

### 版本冲突

```bash
# 查看依赖树
npm list <package>
pnpm why lodash

# 强制安装特定版本
npm install <package>@1.2.3 --force
```

### 权限问题（macOS/Linux）

```bash
# 修复 npm 权限
sudo chown -R $(whoami) ~/.npm
sudo chown -R $(whoami) /usr/local/lib/node_modules
```

### 依赖冲突

```bash
# 解决 peer 依赖冲突
# npm: 手动安装缺失的 peer 依赖
# pnpm: 配置 .npmrc
strict-peer-dependencies=false
```

### 安装慢

1. 切换镜像源
2. 使用 pnpm 或 Bun
3. 配置代理

```bash
# 切换淘宝镜像
npm config set registry https://registry.npmmirror.com
pnpm config set registry https://registry.npmmirror.com
```

### 幽灵依赖

使用 pnpm 的严格模式避免访问未声明的依赖。
