# 前端监控

> 前端监控是一个系统工程，涵盖错误监控、性能监控、用户行为追踪、请求监控等多个维度，为应用稳定性和用户体验保驾护航。

## 学习要点

- 🐛 掌握各类错误捕获与上报
- ⚡ 理解 Web Vitals 性能指标采集
- 👤 学会用户行为追踪方案
- 🌐 实现请求监控与劫持
- 🔒 掌握数据脱敏与采样控制

## 快速开始

### SDK 初始化

```javascript
import MonitorSDK from './SDK整合';

const monitor = MonitorSDK.getInstance({
    // 基础配置
    appId: 'my-app',
    env: 'production',
    version: '1.0.0',
    endpoint: 'https://monitor.example.com/api/collect',
    
    // 模块开关
    enableError: true,
    enablePerformance: true,
    enableBehavior: true,
    enableRequest: true,
    
    // 采样配置
    sampling: {
        global: 1,
        byType: {
            error: 1,
            performance: 0.1,
            behavior: 0.05,
            request: 0.1
        }
    }
});
```

### 手动上报

```javascript
// 捕获错误
monitor.captureError(new Error('Custom error'), { 
    context: 'additional info' 
});

// 捕获行为
monitor.captureBehavior('button_click', { 
    buttonId: 'submit' 
});

// 手动上报
monitor.capture({
    type: 'custom',
    event: 'purchase',
    data: { amount: 100 }
});
```

## 核心指标体系

### Web Vitals

| 指标 | 全称 | 优秀值 | 需改进 | 差 |
|------|------|--------|--------|-----|
| LCP | Largest Contentful Paint | ≤ 2.5s | 2.5s-4s | > 4s |
| INP | Interaction to Next Paint | ≤ 200ms | 200-500ms | > 500ms |
| CLS | Cumulative Layout Shift | ≤ 0.1 | 0.1-0.25 | > 0.25 |
| FCP | First Contentful Paint | ≤ 1.8s | 1.8s-3s | > 3s |
| TTFB | Time to First Byte | ≤ 800ms | 800ms-1.8s | > 1.8s |

### 错误类型

| 类型 | 触发场景 | 捕获方式 |
|------|---------|---------|
| JS 错误 | 运行时错误 | window.onerror |
| Promise 错误 | 未捕获的拒绝 | unhandledrejection |
| 资源错误 | 加载失败 | error 事件捕获 |
| Vue 错误 | 组件错误 | errorHandler |
| React 错误 | 渲染错误 | ErrorBoundary |
