# 性能优化

> 前端性能优化是提升用户体验的核心能力，涵盖加载性能、渲染性能、资源优化和网络优化等多个维度。

## 概述

性能优化是一个系统工程，需要从多个维度进行考虑：

- **加载性能**：减少首屏加载时间，让用户更快看到内容
- **渲染性能**：确保流畅的交互和动画，避免卡顿
- **资源优化**：减少资源体积，优化资源加载
- **网络优化**：减少网络延迟，优化请求策略

## 核心性能指标

### Web Vitals

Google 定义的核心性能指标：

| 指标 | 名称 | 优秀值 | 说明 |
|------|------|--------|------|
| LCP | 最大内容绘制 | ≤ 2.5s | 页面主要内容加载完成的时间 |
| INP | 交互到下一次绘制 | ≤ 200ms | 页面交互响应速度 |
| CLS | 累积布局偏移 | ≤ 0.1 | 页面视觉稳定性 |
| FCP | 首次内容绘制 | ≤ 1.8s | 首次渲染内容的时间 |
| TTFB | 首字节时间 | ≤ 800ms | 服务器响应速度 |

### 性能预算

建立性能预算，持续监控：

```javascript
// 性能预算示例
const performanceBudget = {
    // 资源大小预算
    javascript: 300,    // KB
    css: 100,           // KB
    images: 500,        // KB
    fonts: 100,         // KB
    total: 1000,        // KB
    
    // 时间预算
    fcp: 1800,          // ms
    lcp: 2500,          // ms
    tti: 3800,          // ms
    tbt: 300,           // ms (Total Blocking Time)
    cls: 0.1            // score
};
```

## 性能优化检查清单

### 加载优化
- [ ] 关键 CSS 内联
- [ ] JavaScript 使用 defer/async
- [ ] 路由懒加载
- [ ] 图片懒加载
- [ ] 关键资源预加载
- [ ] 预连接 CDN 域名

### 渲染优化
- [ ] 减少 DOM 操作
- [ ] 使用 transform/opacity 动画
- [ ] 虚拟滚动大列表
- [ ] 高频事件防抖节流
- [ ] Web Worker 处理重计算
- [ ] 避免 long tasks

### 资源优化
- [ ] 使用 WebP/AVIF 图片
- [ ] 图片压缩
- [ ] Tree Shaking
- [ ] 代码压缩
- [ ] Gzip/Brotli 压缩
- [ ] 字体子集化

### 网络优化
- [ ] CDN 加速
- [ ] HTTP/2 或 HTTP/3
- [ ] 合理的缓存策略
- [ ] 减少重定向
- [ ] 合并/拆分请求
- [ ] Cookie-free 域名
