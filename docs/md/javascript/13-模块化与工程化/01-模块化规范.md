# 模块化规范

## 概述

JavaScript 模块化发展经历了多个阶段，从无模块化到现代的 ES Modules。模块化解决了以下问题：
- **命名冲突**：全局变量污染
- **依赖管理**：脚本加载顺序
- **代码复用**：难以共享代码
- **维护性**：代码难以维护和测试

---

## 模块化演进历史

```
1995-2009: 无模块化 / 全局函数
    ↓
2009: CommonJS (Node.js)
    ↓
2010: AMD (RequireJS) / CMD (SeaJS)
    ↓
2011: UMD (统一模块定义)
    ↓
2015: ES Modules (ES6)
    ↓
2020+: ESM 成为标准
```

## 早期解决方案

### 全局函数

最早的代码组织方式，所有函数都是全局的：

```javascript
// ❌ 问题：全局命名空间污染
function foo() {}
function bar() {}
```

### 命名空间模式

使用对象封装：

```javascript
// 简单的命名空间
var MyModule = {
  name: 'John',
  greet: function() {
    console.log('Hello ' + this.name);
  }
};

MyModule.greet();
```

**问题**：所有成员都可以被外部修改。

### IIFE（立即执行函数）

最早期模块化的基础模式：

```javascript
// 基本形式
(function() {
  var privateVar = 'private';

  function privateFunc() {
    console.log(privateVar);
  }

  // 暴露公共接口
  window.MyModule = {
    publicMethod: privateFunc
  };
})();

// 带参数的 IIFE
(function(global, $) {
  var name = 'Module';

  global.ModuleA = {
    getName: function() {
      return name;
    }
  };
})(window, jQuery);
```

**优点**：
- 创建独立作用域，避免全局污染
- 可以封装私有变量和函数

**缺点**：
- 依赖关系不明确
- 手动管理加载顺序

## CommonJS

### 简介

CommonJS 是 Node.js 采用的模块规范，2009 年提出。

### 基本语法

#### 导出模块

```javascript
// 方式1：导出整个对象
module.exports = {
  name: 'John',
  greet: function() {
    console.log('Hello ' + this.name);
  }
};

// 方式2：导出函数/类
module.exports = function(name) {
  this.name = name;
  this.greet = function() {
    console.log('Hello ' + this.name);
  };
};

// 方式3：逐个添加属性
exports.name = 'John';
exports.greet = function() {
  console.log('Hello');
};
```

#### 导入模块

```javascript
// 导入整个模块
const module = require('./module');

// 解构导入
const { name, greet } = require('./module');

// 动态导入（运行时）
const moduleName = process.env.DEBUG ? './debug' : './prod';
const config = require(moduleName);

// 条件导入
if (condition) {
  const module = require('./optional-module');
}
```

### 模块查找规则

```javascript
// 1. 核心模块
require('fs');           // Node.js 内置模块
require('path');

// 2. 文件模块
require('./module');     // 相对路径
require('/abs/path');    // 绝对路径
require('./module.js');  // 扩展名可选

// 3. 目录模块
require('./dir');        // 查找 dir/index.js 或 dir/main.js

// 4. node_modules 查找
require('lodash');       // 查找 node_modules/lodash
require('@scope/pkg');   // 作用域包

// 查找顺序：
// 当前目录/node_modules -> 父目录/node_modules -> ... -> 全局 node_modules
```

### module 对象

```javascript
// module 对象属性
console.log(module.id);           // 模块标识符
console.log(module.filename);     // 模块绝对路径
console.log(module.loaded);       // 是否已加载
console.log(module.parent);       // 父模块
console.log(module.children);     // 子模块数组
console.log(module.paths);        // node_modules 查找路径
console.log(module.exports);      // 导出对象

// __dirname 和 __filename
console.log(__dirname);   // 当前模块所在目录
console.log(__filename);  // 当前模块文件路径
```

### 模块加载机制

```javascript
// 模块缓存
// 第一次 require 时执行并缓存
// 后续 require 返回缓存结果
const a1 = require('./a');  // 执行 a.js
const a2 = require('./a');  // 返回缓存，不重复执行
console.log(a1 === a2);     // true

// 清除缓存（不推荐）
delete require.cache[require.resolve('./a')];
```

### 循环依赖

CommonJS 处理循环依赖的机制：

```javascript
// a.js
console.log('a starting');
exports.done = false;
const b = require('./b');
console.log('in a, b.done =', b.done);
exports.done = true;
console.log('a done');

// b.js
console.log('b starting');
exports.done = false;
const a = require('./a');
console.log('in b, a.done =', a.done);
exports.done = true;
console.log('b done');

// main.js
console.log('main starting');
const a = require('./a');
const b = require('./b');
console.log('in main, a.done =', a.done, 'b.done =', b.done);

// 输出：
// a starting
// b starting
// in b, a.done = false  (a 还没执行完)
// b done
// in a, b.done = true
// a done
// main starting
// in main, a.done = true, b.done = true
```

**原理**：CommonJS 返回的是值的拷贝，循环依赖时返回已执行部分的值。

## AMD（Asynchronous Module Definition）

### 简介

AMD 是为浏览器设计的异步模块加载规范，RequireJS 是其代表实现。

### 基本语法

#### 定义模块

```javascript
// 简单模块
define({
  name: 'John',
  greet: function() {
    return 'Hello ' + this.name;
  }
});

// 函数式定义
define(function() {
  const name = 'John';

  return {
    name: name,
    greet: function() {
      return 'Hello ' + name;
    }
  };
});

// 带依赖的模块
define(['jquery', './utils'], function($, utils) {
  return {
    getElement: function(selector) {
      return $(selector);
    },
    format: utils.format
  };
});

// 命名模块（不推荐）
define('myModule', ['jquery'], function($) {
  return { /* ... */ };
});
```

#### 加载模块

```javascript
// 基本用法
require(['moduleA', 'moduleB'], function(moduleA, moduleB) {
  // 模块加载完成后执行
  moduleA.init();
  moduleB.doSomething();
});

// 配置 RequireJS
require.config({
  baseUrl: '/js',
  paths: {
    jquery: 'lib/jquery.min',
    underscore: 'lib/underscore-min'
  },
  shim: {
    'backbone': {
      deps: ['underscore', 'jquery'],
      exports: 'Backbone'
    }
  }
});
```

### CommonJS 风格的 AMD

```javascript
define(function(require, exports, module) {
  const a = require('./a');
  const b = require('./b');

  exports.name = 'Module';
  exports.doSomething = function() {
    return a.method() + b.method();
  };
});
```

### 优点

- 异步加载，不阻塞页面
- 依赖前置，提前执行
- 浏览器原生支持（需加载 RequireJS）

### 缺点

- 语法复杂
- 依赖前置可能导致不必要的加载

## CMD（Common Module Definition）

### 简介

CMD 是 SeaJS 推广的规范，与 AMD 类似但支持依赖就近。

### 基本语法

```javascript
// 定义模块
define(function(require, exports, module) {
  // 同步加载
  const a = require('./a');
  a.doSomething();

  // 异步加载
  require.async('./b', function(b) {
    b.doSomething();
  });

  // 导出
  exports.name = 'Module';
  module.exports = {
    name: 'Module'
  };
});
```

### AMD vs CMD

```javascript
// AMD - 依赖前置，提前执行
define(['a', 'b', 'c'], function(a, b, c) {
  // a, b, c 都已加载执行
  if (false) {
    b.doSomething(); // 即使不用，b 也已加载
  }
});

// CMD - 依赖就近，延迟执行
define(function(require, exports, module) {
  const a = require('./a'); // 用到时才加载

  if (condition) {
    const b = require('./b'); // 条件加载
  }
});
```

## UMD（Universal Module Definition）

### 简介

UMD 是通用模块定义，兼容 AMD、CommonJS 和全局变量三种模式。

### 基本模式

```javascript
(function(root, factory) {
  if (typeof define === 'function' && define.amd) {
    // AMD
    define(['jquery'], factory);
  } else if (typeof exports === 'object') {
    // CommonJS
    module.exports = factory(require('jquery'));
  } else {
    // 全局变量（浏览器）
    root.MyModule = factory(root.jQuery);
  }
})(typeof self !== 'undefined' ? self : this, function($) {
  // 模块实际代码
  return {
    name: 'MyModule',
    version: '1.0.0',
    init: function() {
      console.log('Module initialized');
    }
  };
});
```

### 更完整的 UMD 模板

```javascript
(function(root, factory) {
  // 环境检测
  if (typeof define === 'function' && define.amd) {
    // AMD 环境
    define(['dependency'], factory);
  } else if (typeof module === 'object' && module.exports) {
    // Node.js / CommonJS
    module.exports = factory(require('dependency'));
  } else if (typeof exports === 'object') {
    // CommonJS 简单环境
    exports = factory(require('dependency'));
  } else {
    // 浏览器全局变量
    root.MyLibrary = factory(root.dependency);
  }
})(typeof globalThis !== 'undefined' ? globalThis :
   typeof self !== 'undefined' ? self :
   typeof window !== 'undefined' ? window :
   typeof global !== 'undefined' ? global : this,
   function(dependency) {
     'use strict';

     // 私有变量
     const privateVar = 'private';

     // 私有方法
     function privateMethod() {
       return privateVar;
     }

     // 公共接口
     const MyLibrary = {
       version: '1.0.0',

       method: function() {
         return privateMethod();
       }
     };

     return MyLibrary;
   });
```

### Webpack UMD 输出

```javascript
// webpack.config.js
module.exports = {
  output: {
    library: {
      name: 'MyLibrary',
      type: 'umd',
      export: 'default'
    },
    globalObject: 'this'
  }
};
```

## ES Modules

### 简介

ES Modules 是 ES6 引入的官方模块规范，是现代 JavaScript 的标准模块系统。

### 导出语法

```javascript
// ========== 导出方式 ==========

// 命名导出
export const name = 'John';
export const age = 30;

export function greet() {
  return 'Hello';
}

export class Person {
  constructor(name) {
    this.name = name;
  }
}

// 导出列表
const a = 1;
const b = 2;
export { a, b };

// 重命名导出
export { a as alpha, b as beta };

// 默认导出
export default function() {
  console.log('Default export');
}

export default class {
  constructor() {}
}

// 导出其他模块的成员（聚合导出）
export { name, greet } from './module.js';
export * from './utils.js';
export * as utils from './utils.js';
export { default } from './module.js';
```

### 导入语法

```javascript
// ========== 导入方式 ==========

// 命名导入
import { name, age } from './module.js';

// 重命名导入
import { name as userName, age as userAge } from './module.js';

// 默认导入
import MyModule from './module.js';

// 混合导入
import MyModule, { name, age } from './module.js';

// 导入整个模块
import * as module from './module.js';
console.log(module.name);
console.log(module.default);

// 仅执行模块（副作用导入）
import './setup.js';

// 动态导入
const module = await import('./module.js');
const { name, age } = await import('./module.js');

// 条件导入
if (condition) {
  const module = await import('./module.js');
}
```

### import.meta

```javascript
// 获取当前模块 URL
console.log(import.meta.url);

// 在 Node.js 中
console.log(import.meta.url);        // file:///path/to/module.mjs
console.log(import.meta.resolve('./utils.js')); // 解析模块路径

// 配合 URL 使用
const data = await fetch(new URL('./data.json', import.meta.url));
```

### 模块特性

```javascript
// 1. 严格模式（自动启用）
// 'use strict';

// 2. 顶层 this 为 undefined
console.log(this); // undefined

// 3. 模块作用域
// 每个模块有自己的作用域，不会污染全局

// 4. 静态结构
// import/export 必须在顶层
// if (condition) {
//   import a from './a.js'; // SyntaxError
// }

// 5. export 导出的是引用（Live Binding）
// counter.js
export let count = 0;
export function increment() {
  count++;
}

// main.js
import { count, increment } from './counter.js';
console.log(count);  // 0
increment();
console.log(count);  // 1 (实时绑定)
```

### 循环依赖

ESM 的循环依赖处理：

```javascript
// a.js
import { b } from './b.js';
export const a = 'a';
console.log('a.js:', b);

// b.js
import { a } from './a.js';
export const b = 'b';
console.log('b.js:', a);

// main.js
import './a.js';

// 执行顺序：
// 1. 解析 a.js，发现依赖 b.js
// 2. 解析 b.js，发现依赖 a.js（已解析但未执行）
// 3. b.js 获取到 a 的引用（此时 a 为 undefined）
// 4. b.js 执行完毕
// 5. a.js 继续执行，获取到 b 的值
```

**原理**：ESM 使用"活绑定"（Live Binding），导入的是引用而非值的拷贝。

## ESM 与 CommonJS 的区别

### 语法差异

```javascript
// CommonJS
const module = require('./module');
module.exports = { /* ... */ };

// ESM
import module from './module.js';
export default { /* ... */ };
```

### 加载机制

| 特性 | CommonJS | ES Modules |
|------|----------|------------|
| 加载时机 | 运行时加载 | 编译时静态分析 |
| 导出值 | 值的拷贝 | 值的引用（Live Binding） |
| this | `{}` | `undefined` |
| `__dirname` | 支持 | 不支持（需用 `import.meta`） |
| `__filename` | 支持 | 不支持 |
| 动态导入 | 天然支持 | `import()` 函数 |
| 循环依赖 | 返回已执行部分 | Live Binding |
| Tree Shaking | 不支持 | 支持 |

### 值的拷贝 vs 引用

```javascript
// ========== CommonJS（值的拷贝）==========

// counter.js
let count = 0;
function increment() {
  count++;
}
module.exports = { count, increment };

// main.js
const counter = require('./counter');
console.log(counter.count);  // 0
counter.increment();
console.log(counter.count);  // 0（仍是 0，因为是拷贝）

// ========== ES Modules（值的引用）==========

// counter.mjs
export let count = 0;
export function increment() {
  count++;
}

// main.mjs
import { count, increment } from './counter.mjs';
console.log(count);  // 0
increment();
console.log(count);  // 1（实时更新）
```

### 模块解析

```javascript
// CommonJS
require('./module');     // 可以省略 .js
require('module');       // 包名

// ESM
import './module.js';    // 必须写完整扩展名（浏览器）
import './module';       // Node.js 可配置
```

### 互操作

```javascript
// ========== 在 ESM 中导入 CommonJS ==========

// 导入 CommonJS 模块（作为默认导出）
import module from './commonjs-module.cjs';
import { name, greet } from './commonjs-module.cjs';

// 动态导入
const module = await import('./commonjs-module.cjs');

// ========== 在 CommonJS 中导入 ESM ==========

// 只能使用动态 import
const module = await import('./esm-module.mjs');
```

## Node.js 中的模块

### 模块类型判断

```javascript
// .mjs 文件 -> ES Modules
// .cjs 文件 -> CommonJS
// .js 文件 -> 取决于 package.json 的 type 字段

// package.json
```

```javascript
// 在 ESM 中使用 __dirname
import { fileURLToPath } from 'url';

const __filename = fileURLToPath(import.meta.url);

// 在 CommonJS 中使用 ESM
```

## 浏览器中的 ESM

### 原生使用

```html
<!-- 外部模块 -->
<script type="module" src="./app.js"></script>

<!-- 内联模块 -->
<script type="module">
  import { greet } from './utils.js';
  greet('World');
</script>

<!-- 模块默认延迟加载（类似 defer）-->
<script type="module" src="./app.js"></script>
<!-- 等同于 -->
<script type="module" src="./app.js" defer></script>

<!-- Nomodule 降级方案 -->
<script type="module" src="./app.js"></script>
<script nomodule src="./fallback.js"></script>
```

### 模块特性

```html
<!-- 1. 模块只执行一次 -->
<script type="module" src="./module.js"></script>
<script type="module" src="./module.js"></script>
<!-- 只执行一次 -->

<!-- 2. 严格模式 -->
<script type="module">
  this; // undefined
</script>

<!-- 3. 作用域隔离 -->
<script type="module">
  const secret = 'secret';
</script>
<script>
  console.log(typeof secret); // undefined
</script>

<!-- 4. CORS 限制 -->
<script type="module" src="https://other.com/module.js"></script>
<!-- 需要 CORS 头 -->
```

### 动态导入

```javascript
// 按需加载
button.addEventListener('click', async () => {
  const module = await import('./heavy-module.js');
  module.init();
});

// 条件加载
if (featureFlags.newUI) {
  const { renderUI } = await import('./new-ui.js');
  renderUI();
}

// 路由懒加载（SPA）
const routes = {
  '/home': () => import('./pages/home.js'),
  '/about': () => import('./pages/about.js')
};
```

## SystemJS

### 简介

SystemJS 是一个通用的模块加载器，支持所有模块格式。

### 基本使用

```html
<script src="system.js"></script>
<script>
  // 配置
  System.config({
    baseURL: '/js',
    map: {
      'jquery': 'https://cdn.example.com/jquery.min.js'
    }
  });

  // 加载模块
  System.import('./app.js').then(function(module) {
    module.init();
  });
</script>
```

### 支持多种格式

```javascript
// 加载 ESM
System.import('./module.mjs');

// 加载 CommonJS
System.import('./module.cjs');

// 加载 AMD
System.import('./module.js');

// 加载 UMD
System.import('./umd-module.js');
```

## 最佳实践

### 选择模块格式

```javascript
// 浏览器项目
// ✅ 优先使用 ESM

// Node.js 项目
// ✅ 新项目使用 ESM (.mjs 或 type: "module")
// ✅ 库项目提供双格式支持

// 库开发
// ✅ 提供 ESM 和 CommonJS 两种格式
// ✅ 使用 package.json exports 字段
```

### 模块设计原则

```javascript
// 1. 单一职责
// ✅ 每个模块只做一件事
export function formatDate(date) { /* ... */ }
export function parseDate(str) { /* ... */ }

// 2. 避免循环依赖
// ❌ 不好的设计
// a.js 依赖 b.js，b.js 又依赖 a.js

// ✅ 重构：提取公共模块
// common.js -> a.js, b.js

// 3. 清晰的导出
// ✅ 命名导出更明确
export { formatDate, parseDate };

// ✅ 默认导出用于单一功能
```

## 总结对比

| 规范 | 环境 | 加载方式 | 使用场景 |
|------|------|----------|----------|
| IIFE | 浏览器 | 同步 script | 早期简单封装 |
| AMD | 浏览器 | 异步 | 老项目、RequireJS |
| CMD | 浏览器 | 异步/就近 | 老项目、SeaJS |
| UMD | 通用 | 兼容多种 | 库开发、兼容包 |
| CommonJS | Node.js | 同步 | Node.js 项目 |
| ES Modules | 通用 | 静态/异步 | 现代项目首选 |
