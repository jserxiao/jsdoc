## 单线程模型

### 为什么是单线程？

JavaScript 从诞生之初就是单线程语言，主要原因：

1. **DOM 操作**：避免多线程同时操作 DOM 导致冲突
2. **简单性**：无需考虑线程同步、锁等复杂问题
3. **历史原因**：设计用于浏览器脚本，单线程足够

```javascript
// 单线程：一次只能做一件事
function slowOperation() {
  // 这个操作会阻塞整个页面
  const start = Date.now();
  while (Date.now() - start < 1000) {
    // 同步等待 1 秒
  }
  console.log('Done');
}

slowOperation();  // 这 1 秒内页面无响应
console.log('After slow operation');
```

### 异步解决方案

单线程不意味着不能处理并发任务，JavaScript 通过异步机制实现：

```javascript
// 异步：不阻塞主线程
console.log('Start');

setTimeout(() => {
  console.log('Timeout callback');
}, 1000);

console.log('End');

// 输出：Start, End, (1秒后) Timeout callback
```

## 调用栈

### 什么是调用栈？

调用栈（Call Stack）是 JavaScript 解释器追踪函数执行的一种机制，采用后进先出（LIFO）原则。

```javascript
function first() {
  console.log('first start');
  second();
  console.log('first end');
}

function second() {
  console.log('second');
}

first();
console.log('global');

// 调用栈变化：
// 1. first() 入栈
// 2. second() 入栈
// 3. second() 执行完毕，出栈
// 4. first() 继续执行，出栈
// 5. console.log('global') 入栈出栈

// 输出：
// first start
// second
// first end
// global
```

### 栈溢出

```javascript
// 无限递归导致栈溢出
function infiniteRecursion() {
  return infiniteRecursion();
}

try {
  infiniteRecursion();
} catch (error) {
  console.log(error.name);  // RangeError
  console.log(error.message);  // Maximum call stack size exceeded
}

// 递归改迭代避免栈溢出
function factorial(n) {
  let result = 1;
  for (let i = 2; i <= n; i++) {
    result *= i;
  }
  return result;
}

// 尾递归优化（ES6，需在严格模式下）
'use strict';
function factorialTail(n, acc = 1) {
  if (n <= 1) return acc;
  return factorialTail(n - 1, n * acc);  // 尾调用
}
```

## 事件循环机制

### 核心概念

事件循环（Event Loop）是 JavaScript 实现异步的核心机制：

```
┌───────────────────────────┐
│         Call Stack        │  执行同步代码
│   (调用栈 - LIFO)          │
└─────────────┬─────────────┘
              │
              ▼
┌───────────────────────────┐
│       Web APIs            │  浏览器提供的 API
│  (setTimeout, DOM, HTTP)  │
└─────────────┬─────────────┘
              │
              ▼
┌───────────────────────────┐
│       Callback Queue      │  等待执行的任务
│   (任务队列 - FIFO)        │
└─────────────┬─────────────┘
              │
              ▼
┌───────────────────────────┐
│        Event Loop         │  持续检查
│  "Stack empty? → Push"    │
└───────────────────────────┘
```

### 事件循环工作流程

```javascript
console.log('1. Script start');

setTimeout(() => {
  console.log('2. setTimeout');
}, 0);

Promise.resolve()
  .then(() => {
    console.log('3. Promise then');
  });

console.log('4. Script end');

// 执行流程：
// 1. 同步代码执行：1, 4
// 2. 调用栈清空
// 3. 执行微任务：3
// 4. 执行宏任务：2

// 输出：1, 4, 3, 2
```

### 事件循环循环

```javascript
// 模拟事件循环
while (true) {
  // 1. 执行调用栈中的所有同步任务
  while (callStack.length > 0) {
    const task = callStack.pop();
    task();
  }

  // 2. 执行所有微任务
  while (microtaskQueue.length > 0) {
    const task = microtaskQueue.shift();
    callStack.push(task);
    task();
  }

  // 3. UI 渲染（如果需要）

  // 4. 执行一个宏任务
  if (macrotaskQueue.length > 0) {
    const task = macrotaskQueue.shift();
    callStack.push(task);
    task();
  }
}
```

## 宏任务与微任务

### 宏任务（Macrotask）

| 来源 | 示例 |
|------|------|
| script | 主程序代码 |
| setTimeout | `setTimeout(callback, 0)` |
| setInterval | `setInterval(callback, 1000)` |
| setImmediate | Node.js 环境 |
| I/O | 文件读写、网络请求 |
| UI Rendering | 浏览器渲染 |

### 微任务（Microtask）

| 来源 | 示例 |
|------|------|
| Promise.then | `Promise.resolve().then(callback)` |
| Promise.catch | `Promise.reject().catch(callback)` |
| Promise.finally | `promise.finally(callback)` |
| queueMicrotask | `queueMicrotask(callback)` |
| MutationObserver | DOM 变化监听 |
| process.nextTick | Node.js 环境（优先级最高） |

### 执行优先级

```javascript
console.log('1. Script start');

setTimeout(() => {
  console.log('2. setTimeout 1');

  Promise.resolve().then(() => {
    console.log('3. Promise inside setTimeout');
  });
}, 0);

Promise.resolve()
  .then(() => {
    console.log('4. Promise 1');
  })
  .then(() => {
    console.log('5. Promise 2');
  });

queueMicrotask(() => {
  console.log('6. queueMicrotask');
});

console.log('7. Script end');

// 输出顺序：
// 1. Script start      (同步)
// 7. Script end        (同步)
// 4. Promise 1         (微任务)
// 6. queueMicrotask    (微任务)
// 5. Promise 2         (微任务)
// 2. setTimeout 1      (宏任务)
// 3. Promise inside setTimeout  (微任务)
```

### 执行规则总结

1. **同步代码**：立即执行
2. **微任务**：每个宏任务后，清空所有微任务
3. **宏任务**：每次只执行一个
4. **循环往复**：执行一个宏任务 → 清空微任务 → 渲染 → 下一个宏任务

## 回调函数

### 什么是回调函数？

回调函数是作为参数传递给另一个函数的函数，在特定事件或条件发生时被调用。

```javascript
// 同步回调
const numbers = [1, 2, 3, 4, 5];
numbers.forEach(function(num) {
  console.log(num);  // 立即执行
});

// 异步回调
setTimeout(function() {
  console.log('Delayed execution');  // 延迟执行
}, 1000);

// 事件回调
document.getElementById('btn').addEventListener('click', function(event) {
  console.log('Button clicked');  // 事件触发时执行
});
```

### 回调函数模式

#### 简单回调

```javascript
function greet(name, callback) {
  console.log(`Hello, ${name}!`);
  callback();
}

greet('World', function() {
  console.log('Greeting complete');
});
```

#### 错误优先回调（Node.js 风格）

```javascript
// Node.js 回调约定：第一个参数是错误对象
const fs = require('fs');

fs.readFile('file.txt', 'utf8', function(err, data) {
  if (err) {
    console.error('Error:', err);
    return;
  }
  console.log('Data:', data);
});

// 自定义错误优先回调函数
function asyncOperation(options, callback) {
  setTimeout(() => {
    if (!options.valid) {
      callback(new Error('Invalid options'), null);
      return;
    }
    callback(null, { result: 'success' });
  }, 1000);
}

asyncOperation({ valid: true }, (err, result) => {
  if (err) {
    console.error(err);
    return;
  }
  console.log(result);
});
```

#### 回调模式对比

```javascript
// 模式 1：分离成功和失败回调
function fetch(url, onSuccess, onError) {
  // ...
}

fetch('/api/data',
  data => console.log('Success:', data),
  error => console.error('Error:', error)
);

// 模式 2：单一回调处理结果
function fetch(url, callback) {
  // callback(null, data) 表示成功
  // callback(error, null) 表示失败
}

// 模式 3：返回 Promise（推荐）
function fetch(url) {
  return new Promise((resolve, reject) => {
    // resolve(data) 表示成功
    // reject(error) 表示失败
  });
}
```

## 回调地狱

### 问题演示

```javascript
// 回调地狱（Callback Hell）/ 金字塔厄运（Pyramid of Doom）
getData(function(a) {
  getMoreData(a, function(b) {
    getMoreData(b, function(c) {
      getMoreData(c, function(d) {
        getMoreData(d, function(e) {
          // 嵌套过深，难以维护
          // 代码可读性差
          // 错误处理复杂
        });
      });
    });
  });
});

// 实际例子：多个 AJAX 请求
$.ajax({
  url: '/api/user',
  success: function(user) {
    $.ajax({
      url: '/api/posts/' + user.id,
      success: function(posts) {
        $.ajax({
          url: '/api/comments/' + posts[0].id,
          success: function(comments) {
            // 嵌套越来越深
            renderComments(comments);
          },
          error: function(err) {
            console.error(err);
          }
        });
      },
      error: function(err) {
        console.error(err);
      }
    });
  },
  error: function(err) {
    console.error(err);
  }
});
```

### 回调地狱的问题

1. **可读性差**：代码横向增长，难以阅读
2. **错误处理复杂**：每层都需要错误处理
3. **难以维护**：修改逻辑需要大量嵌套调整
4. **难以调试**：堆栈信息不清晰

### 解决方案

#### 1. 函数提取

```javascript
// 提取函数，减少嵌套
function getUser(callback) {
  $.ajax({ url: '/api/user', success: callback });
}

function getPosts(userId, callback) {
  $.ajax({
    url: '/api/posts/' + userId,
    success: callback
  });
}

function getComments(postId, callback) {
  $.ajax({
    url: '/api/comments/' + postId,
    success: callback
  });
}

// 链式调用
getUser(function(user) {
  getPosts(user.id, function(posts) {
    getComments(posts[0].id, function(comments) {
      renderComments(comments);
    });
  });
});
```

#### 2. Promise 链式调用

```javascript
// 使用 Promise 解决回调地狱
getData()
  .then(a => getMoreData(a))
  .then(b => getMoreData(b))
  .then(c => getMoreData(c))
  .then(d => console.log(d))
  .catch(error => console.error('Error:', error));

// AJAX 示例
fetch('/api/user')
  .then(response => response.json())
  .then(user => fetch('/api/posts/' + user.id))
  .then(response => response.json())
  .then(posts => fetch('/api/comments/' + posts[0].id))
  .then(response => response.json())
  .then(comments => renderComments(comments))
  .catch(error => console.error('Error:', error));
```

#### 3. async/await

```javascript
// 使用 async/await（最佳方案）
async function fetchComments() {
  try {
    const userResponse = await fetch('/api/user');
    const user = await userResponse.json();

    const postsResponse = await fetch('/api/posts/' + user.id);
    const posts = await postsResponse.json();

    const commentsResponse = await fetch('/api/comments/' + posts[0].id);
    const comments = await commentsResponse.json();

    renderComments(comments);
  } catch (error) {
    console.error('Error:', error);
  }
}

// 并行请求
async function fetchAll() {
  const [users, products, orders] = await Promise.all([
    fetch('/api/users').then(r => r.json()),
    fetch('/api/products').then(r => r.json()),
    fetch('/api/orders').then(r => r.json())
  ]);

  return { users, products, orders };
}
```

## 异步编程演进

### 第一阶段：回调函数

```javascript
// 简单场景可用
setTimeout(() => {
  console.log('Delayed');
}, 1000);

// 复杂场景难以管理
asyncOp1(function(result1) {
  asyncOp2(result1, function(result2) {
    asyncOp3(result2, function(result3) {
      // ...
    });
  });
});
```

### 第二阶段：Promise

```javascript
// 更好的错误处理和链式调用
asyncOp1()
  .then(result1 => asyncOp2(result1))
  .then(result2 => asyncOp3(result2))
  .then(result3 => {
    // ...
  })
  .catch(error => {
    // 统一错误处理
  });
```

### 第三阶段：Generator

```javascript
// Generator + Promise
function* fetchGenerator() {
  try {
    const result1 = yield asyncOp1();
    const result2 = yield asyncOp2(result1);
    const result3 = yield asyncOp3(result2);
    return result3;
  } catch (error) {
    console.error(error);
  }
}

// 执行器
function runGenerator(generator) {
  const gen = generator();

  function handle(result) {
    if (result.done) return Promise.resolve(result.value);

    return Promise.resolve(result.value).then(
      data => handle(gen.next(data)),
      error => handle(gen.throw(error))
    );
  }

  return handle(gen.next());
}

runGenerator(fetchGenerator);
```

### 第四阶段：async/await

```javascript
// 语法糖，基于 Promise
async function fetchAll() {
  try {
    const result1 = await asyncOp1();
    const result2 = await asyncOp2(result1);
    const result3 = await asyncOp3(result2);
    return result3;
  } catch (error) {
    console.error(error);
  }
}
```

## 事件循环面试题

### 题目 1：基础执行顺序

```javascript
console.log('1');

setTimeout(() => {
  console.log('2');
}, 0);

Promise.resolve().then(() => {
  console.log('3');
});

console.log('4');

// 输出：1, 4, 3, 2
```

### 题目 2：async/await

```javascript
async function async1() {
  console.log('async1 start');
  await async2();
  console.log('async1 end');
}

async function async2() {
  console.log('async2');
}

console.log('script start');

setTimeout(() => {
  console.log('setTimeout');
}, 0);

async1();

new Promise(resolve => {
  console.log('promise1');
  resolve();
}).then(() => {
  console.log('promise2');
});

console.log('script end');

// 输出顺序：
// script start
// async1 start
// async2
// promise1
// script end
// async1 end
// promise2
// setTimeout
```

### 题目 3：嵌套微任务

```javascript
console.log('start');

Promise.resolve()
  .then(() => {
    console.log('then1');
    Promise.resolve()
      .then(() => {
        console.log('then1-1');
        return Promise.resolve();
      })
      .then(() => {
        console.log('then1-2');
      });
  })
  .then(() => {
    console.log('then2');
  });

Promise.resolve()
  .then(() => {
    console.log('then3');
  });

console.log('end');

// 输出顺序：
// start
// end
// then1
// then3
// then1-1
// then2
// then1-2
```

### 题目 4：复杂场景

```javascript
console.log('1');

setTimeout(() => {
  console.log('2');
  Promise.resolve().then(() => {
    console.log('3');
  });
});

new Promise((resolve) => {
  console.log('4');
  resolve();
}).then(() => {
  console.log('5');
}).then(() => {
  console.log('6');
});

setTimeout(() => {
  console.log('7');
  Promise.resolve().then(() => {
    console.log('8');
  });
});

console.log('9');

// 输出顺序：
// 1, 4, 9, 5, 6, 2, 3, 7, 8
```

### 题目 5：Node.js process.nextTick

```javascript
// Node.js 环境
console.log('1');

process.nextTick(() => {
  console.log('2');
});

Promise.resolve().then(() => {
  console.log('3');
});

setImmediate(() => {
  console.log('4');
});

setTimeout(() => {
  console.log('5');
}, 0);

console.log('6');

// 输出顺序：
// 1, 6, 2, 3, 5, 4
// process.nextTick 优先级高于 Promise.then
// setTimeout 优先级高于 setImmediate（在 IO 回调中相反）
```
