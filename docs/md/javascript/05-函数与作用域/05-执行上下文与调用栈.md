# 执行上下文与调用栈

> 执行上下文是 JavaScript 代码执行的环境，理解它是深入理解 JavaScript 运行机制的关键。《你不知道的JavaScript》将此作为核心概念详细讲解。

## 学习要点

- 🔍 理解执行上下文的概念和类型
- 📦 掌握变量对象和活动对象
- 🔗 理解作用域链的构建过程
- 📚 掌握调用栈的工作原理

---

## 1. 什么是执行上下文？

执行上下文（Execution Context）是 JavaScript 代码执行时的环境抽象，它决定了代码可以访问哪些变量、函数和对象。

### 执行上下文的类型

```javascript
// 1. 全局执行上下文
// - 任何不在函数内部的代码都在全局上下文中
// - 浏览器中，全局上下文的 this 指向 window
// - 一个程序只有一个全局执行上下文

var globalVar = '全局变量';

// 2. 函数执行上下文
// - 每个函数调用时都会创建一个新的执行上下文
// - 函数被调用多少次，就创建多少个执行上下文

function foo() {
    var localVar = '局部变量';
    // 函数执行上下文
}

foo(); // 创建一个函数执行上下文
foo(); // 创建另一个函数执行上下文

// 3. Eval 执行上下文（不推荐使用）
eval('var x = 1'); // 创建 eval 执行上下文
```

## 2. 执行上下文的创建

### 执行上下文的两个阶段

```javascript
function foo(a, b) {
    var c = a + b;
    function bar() {
        return c * 2;
    }
    return bar();
}

foo(1, 2);

/*
执行上下文生命周期：

1. 创建阶段（进入执行上下文）
   - 创建变量对象（Variable Object）
   - 建立作用域链（Scope Chain）
   - 确定 this 的值

2. 执行阶段
   - 变量赋值
   - 函数引用
   - 执行其他代码
*/
```

### 创建阶段详解

```javascript
// 模拟执行上下文对象
ExecutionContext = {
    VariableObject: { ... },  // 变量对象
    ScopeChain: [ ... ],      // 作用域链
    this: { ... }             // this 绑定
};

// 创建阶段：函数 foo(1, 2) 的执行上下文
ExecutionContext = {
    // 变量对象（参数、函数声明、变量声明）
    VariableObject: {
        arguments: {
            0: 1,
            1: 2,
            length: 2,
            callee: foo
        },
        a: 1,              // 参数 a
        b: 2,              // 参数 b
        c: undefined,      // 变量声明（未赋值）
        bar: function() {} // 函数声明（完整提升）
    },
    ScopeChain: [ ... ],
    this: window
};
```

## 3. 变量对象（Variable Object）

变量对象是执行上下文中的一个特殊对象，存储了该上下文中定义的变量和函数声明。

### 变量对象的创建过程

```javascript
// 变量对象创建顺序：
// 1. 函数的形参（arguments 对象）
// 2. 函数声明（如果同名，后者覆盖前者）
// 3. 变量声明（不覆盖同名的函数声明）

// 示例
function test(a, b) {
    console.log(a);      // function a() {}（函数声明覆盖参数）
    console.log(b);      // 2
    console.log(c);      // undefined
    console.log(d);      // function d() {}
    
    var c = 10;
    function a() {}
    var b = 20;
    function d() {}
    var a = 30;          // 变量声明不会覆盖函数声明
}

test(1, 2);

/*
变量对象创建过程：
1. 处理参数：a = 1, b = 2
2. 处理函数声明：
   - a: function a() {}（覆盖参数 a = 1）
   - d: function d() {}
3. 处理变量声明：
   - c: undefined
   - b: 已经存在，忽略
   - a: 已经是函数，忽略变量声明

最终变量对象：
VO = {
    arguments: { 0: 1, 1: 2, length: 2 },
    a: function a() {},
    b: 2,
    c: undefined,
    d: function d() {}
}
*/
```

### 活动对象（Activation Object）

```javascript
// 活动对象是函数上下文中的变量对象
// 只有进入函数执行上下文时，变量对象才被激活成为活动对象

function foo(x) {
    var y = 2;
    
    // 此时变量对象已激活
    // AO = {
    //     arguments: { 0: 1, length: 1 },
    //     x: 1,
    //     y: 2
    // }
}

foo(1);
```

### 变量对象 vs 活动对象

```
全局执行上下文：
┌─────────────────────────────┐
│ 全局变量对象 = 全局对象      │
│ (Global Object = window)    │
│                             │
│ var a = 1 → window.a = 1    │
│ function fn() → window.fn   │
└─────────────────────────────┘

函数执行上下文：
┌─────────────────────────────┐
│ 变量对象（VO）               │
│ 创建阶段：只有声明           │
│                             │
│ 激活后 → 活动对象（AO）      │
│ 执行阶段：包含实际值         │
└─────────────────────────────┘
```

## 4. 作用域链（Scope Chain）

作用域链是由当前执行上下文的变量对象和所有父级执行上下文的变量对象组成的链表。

### 作用域链的构建

```javascript
// 作用域链在函数定义时就确定了（词法作用域）
// 它存储在函数的内部属性 [[Scopes]] 中

var globalVar = 'global';

function outer() {
    var outerVar = 'outer';
    
    function inner() {
        var innerVar = 'inner';
        console.log(innerVar);  // 'inner'
        console.log(outerVar);  // 'outer'
        console.log(globalVar); // 'global'
    }
    
    inner();
}

outer();

/*
inner 函数的作用域链：
[0] innerContext.VO (innerVar)
[1] outerContext.VO (outerVar)
[2] globalContext.VO (globalVar, outer, global)

查找变量时，从作用域链前端开始查找，找到即停止
*/
```

### 作用域链图解

```
┌─────────────────────────────────────────────────────────────┐
│                    作用域链示意图                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  inner 函数上下文                                           │
│  ┌─────────────────────┐                                   │
│  │ innerVar = 'inner'  │ ──→ 找到 innerVar ✓               │
│  └─────────────────────┘                                   │
│            ↓                                               │
│  outer 函数上下文                                           │
│  ┌─────────────────────┐                                   │
│  │ outerVar = 'outer'  │ ──→ 找到 outerVar ✓               │
│  └─────────────────────┘                                   │
│            ↓                                               │
│  全局上下文                                                 │
│  ┌─────────────────────┐                                   │
│  │ globalVar = 'global'│ ──→ 找到 globalVar ✓              │
│  │ outer = function    │                                   │
│  └─────────────────────┘                                   │
│            ↓                                               │
│         null（链的终点）                                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 闭包的作用域链

```javascript
function createCounter() {
    let count = 0;
    
    return {
        increment() {
            count++;
            return count;
        },
        decrement() {
            count--;
            return count;
        },
        getCount() {
            return count;
        }
    };
}

const counter = createCounter();
counter.increment(); // 1
counter.increment(); // 2
counter.getCount();  // 2

/*
createCounter 执行完毕后，其执行上下文已出栈
但 counter 对象的方法仍然持有对 count 的引用
因为它们的作用域链包含 createCounter 的变量对象

这就是闭包的本质：函数保持对其词法作用域的引用
*/
```

## 5. 调用栈（Call Stack）

调用栈是一个后进先出（LIFO）的数据结构，用于存储代码执行期间创建的所有执行上下文。

### 调用栈的工作原理

```javascript
function first() {
    console.log('first start');
    second();
    console.log('first end');
}

function second() {
    console.log('second start');
    third();
    console.log('second end');
}

function third() {
    console.log('third start');
    console.log('third end');
}

first();

/*
执行过程：

1. 全局代码入栈
   栈：[global]

2. first() 调用
   栈：[global, first]
   输出：first start

3. second() 调用
   栈：[global, first, second]
   输出：second start

4. third() 调用
   栈：[global, first, second, third]
   输出：third start
   输出：third end

5. third() 执行完毕，出栈
   栈：[global, first, second]
   输出：second end

6. second() 执行完毕，出栈
   栈：[global, first]
   输出：first end

7. first() 执行完毕，出栈
   栈：[global]

8. 全局代码执行完毕
*/
```

### 调用栈可视化

```
调用栈变化过程：

步骤1              步骤2              步骤3              步骤4
┌─────────┐       ┌─────────┐       ┌─────────┐       ┌─────────┐
│         │       │         │       │         │       │  third  │
├─────────┤       ├─────────┤       ├─────────┤       ├─────────┤
│         │       │         │       │ second  │       │ second  │
├─────────┤       ├─────────┤       ├─────────┤       ├─────────┤
│         │       │  first  │       │  first  │       │  first  │
├─────────┤       ├─────────┤       ├─────────┤       ├─────────┤
│ global  │       │ global  │       │ global  │       │ global  │
└─────────┘       └─────────┘       └─────────┘       └─────────┘

步骤5              步骤6              步骤7              步骤8
┌─────────┐       ┌─────────┐       ┌─────────┐       ┌─────────┐
│         │       │         │       │         │       │         │
├─────────┤       ├─────────┤       ├─────────┤       ├─────────┤
│         │       │         │       │         │       │         │

```

### 栈溢出

```javascript
// 无限递归导致栈溢出
function recursive() {
    recursive(); // 无限调用自己
}

recursive(); // RangeError: Maximum call stack size exceeded

// 尾递归优化（ES6 严格模式）
function factorial(n, acc = 1) {
    if (n <= 1) return acc;
    return factorial(n - 1, n * acc); // 尾调用，不会增加栈深度
}

factorial(100000); // 在支持尾递归优化的引擎中不会溢出
```

### 调试调用栈

```javascript
// 使用 console.trace() 打印调用栈
function funcA() {
    funcB();
}

function funcB() {
    funcC();
}

function funcC() {
    console.trace();
}

funcA();

/*
输出：
Trace
    at funcC (repl:3:9)
    at funcB (repl:2:5)
    at funcA (repl:1:5)
    at repl:1:1
    ...
*/

// 使用 debugger 语句
function debug() {
    debugger; // 在此处暂停，可在开发者工具中查看调用栈
}
```

## 6. this 的确定

在执行上下文的创建阶段，this 的值会被确定。

### this 绑定规则

```javascript
// 1. 默认绑定
function showThis() {
    console.log(this);
}

showThis(); // window（非严格模式）/ undefined（严格模式）

// 2. 隐式绑定
const obj = {
    name: 'obj',
    showThis: showThis
};

obj.showThis(); // obj

// 3. 显式绑定
showThis.call({ name: 'call' });   // { name: 'call' }
showThis.apply({ name: 'apply' }); // { name: 'apply' }

const bound = showThis.bind({ name: 'bind' });
bound(); // { name: 'bind' }

// 4. new 绑定
function Person(name) {
    this.name = name;
}

const person = new Person('John'); // this 指向新创建的对象
console.log(person.name); // 'John'
```

### this 绑定优先级

```javascript
// 优先级：new 绑定 > 显式绑定 > 隐式绑定 > 默认绑定

function foo(something) {
    this.a = something;
}

// 显式绑定
const obj1 = {};
const bar = foo.bind(obj1);
bar(2);
console.log(obj1.a); // 2

// new 绑定覆盖显式绑定
const baz = new bar(3);
console.log(obj1.a); // 2（仍然是 2）
console.log(baz.a);  // 3（new 创建的新对象）
```

## 7. 完整执行流程示例

```javascript
var globalVar = 'global';

function outer(outerParam) {
    var outerVar = 'outer';
    
    function inner(innerParam) {
        var innerVar = 'inner';
        console.log(innerVar);      // 'inner'
        console.log(innerParam);    // 参数值
        console.log(outerVar);      // 'outer'
        console.log(outerParam);    // 参数值
        console.log(globalVar);     // 'global'
    }
    
    inner('innerParam');
}

outer('outerParam');

/*
完整执行流程：

1. 全局上下文创建
   GlobalContext = {
       VO: {
           globalVar: undefined → 'global',
           outer: function
       },
       ScopeChain: [GlobalContext.VO],
       this: window
   }

2. 全局代码执行
   globalVar = 'global'

3. outer('outerParam') 调用
   OuterContext = {
       VO: {
           arguments: { 0: 'outerParam', length: 1 },
           outerParam: 'outerParam',
           outerVar: undefined → 'outer',
           inner: function
       },
       ScopeChain: [OuterContext.VO, GlobalContext.VO],
       this: window
   }

4. outer 执行
   outerVar = 'outer'

5. inner('innerParam') 调用
   InnerContext = {
       VO: {
           arguments: { 0: 'innerParam', length: 1 },
           innerParam: 'innerParam',
           innerVar: undefined → 'inner'
       },
       ScopeChain: [InnerContext.VO, OuterContext.VO, GlobalContext.VO],
       this: window
   }

6. inner 执行
   innerVar = 'inner'
   console.log 输出各种变量
*/
```

## 小结

| 概念 | 说明 |
|------|------|
| **执行上下文** | 代码执行的环境抽象 |
| **变量对象** | 存储变量、函数声明、参数 |
| **活动对象** | 函数执行时激活的变量对象 |
| **作用域链** | 变量查找的路径 |
| **调用栈** | 管理执行上下文的 LIFO 栈 |
| **this** | 执行上下文的 this 绑定 |
