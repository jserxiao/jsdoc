# 创建型模式

创建型模式（Creational Patterns）关注对象的创建过程，将对象的创建和使用分离，降低系统耦合度。

---

## 概述

创建型模式主要包括：
- **单例模式**（Singleton）：确保只有一个实例
- **工厂模式**（Factory）：创建对象的接口
- **抽象工厂模式**（Abstract Factory）：创建相关对象族
- **建造者模式**（Builder）：复杂对象的分步构建
- **原型模式**（Prototype）：通过复制创建对象

---

## 单例模式

### 概念

确保一个类只有一个实例，并提供全局访问点。

### 应用场景

- 全局配置管理
- 数据库连接池
- 日志记录器
- 缓存管理器
- 状态管理（如 Redux Store）

### 实现方式

#### 1. 类静态属性方式

```javascript
class Singleton {
  static instance = null;

  constructor() {
    if (Singleton.instance) {
      return Singleton.instance;
    }
    Singleton.instance = this;
    this.data = {};
  }

  static getInstance() {
    if (!Singleton.instance) {
      Singleton.instance = new Singleton();
    }
    return Singleton.instance;
  }
}

// 使用
const instance1 = new Singleton();
const instance2 = new Singleton();
console.log(instance1 === instance2);  // true

const instance3 = Singleton.getInstance();
console.log(instance1 === instance3);  // true
```

#### 2. 闭包方式

```javascript
const Singleton = (function() {
  let instance;

  function init() {
    return {
      name: 'Singleton',
      method() {
        console.log('Method called');
      }
    };
  }

  return {
    getInstance() {
      if (!instance) {
        instance = init();
      }
      return instance;
    }
  };
})();

// 使用
const inst1 = Singleton.getInstance();
const inst2 = Singleton.getInstance();
console.log(inst1 === inst2);  // true
```

#### 3. 模块方式（ES6）

```javascript
// singleton.js
class Singleton {
  constructor() {
    this.data = {};
  }

  set(key, value) {
    this.data[key] = value;
  }

  get(key) {
    return this.data[key];
  }
}

// 模块单例
export default new Singleton();

// 使用
import singleton from './singleton.js';
singleton.set('name', 'John');
```

#### 4. Symbol 防止外部实例化

```javascript
const INSTANCE = Symbol('instance');

class Singleton {
  constructor() {
    if (Singleton[INSTANCE]) {
      return Singleton[INSTANCE];
    }
    Singleton[INSTANCE] = this;
  }

  static getInstance() {
    if (!Singleton[INSTANCE]) {
      Singleton[INSTANCE] = new Singleton();
    }
    return Singleton[INSTANCE];
  }
}

// 外部无法访问 INSTANCE Symbol
const inst1 = new Singleton();
const inst2 = new Singleton();
console.log(inst1 === inst2);  // true
```

class DatabaseConnection {
  static instance = null;

  constructor(connectionString) {
    if (DatabaseConnection.instance) {
      return DatabaseConnection.instance;
    }

    this.connectionString = connectionString;
    this.isConnected = false;
    DatabaseConnection.instance = this;

    this.isConnected = false;
    console.log('Disconnected');
  }
}

#### 状态管理

```javascript
class Store {
  static instance = null;

  constructor(initialState = {}) {
    if (Store.instance) {
      return Store.instance;
    }

    this.state = initialState;
    this.listeners = [];
    Store.instance = this;
  }

  getState() {
    return this.state;
  }

  setState(newState) {
    this.state = { ...this.state, ...newState };
    this.listeners.forEach(listener => listener(this.state));
  }

  subscribe(listener) {
    this.listeners.push(listener);
    return () => {
      this.listeners = this.listeners.filter(l => l !== listener);
    };
  }
}

// 使用
const store = new Store({ count: 0 });
store.subscribe(state => console.log('State changed:', state));

store.setState({ count: 1 });  // State changed: { count: 1 }
```

### 优缺点

**优点**：
- 保证唯一实例
- 提供全局访问点
- 延迟初始化

**缺点**：
- 违反单一职责原则
- 可能隐藏依赖关系
- 难以单元测试
- 全局状态可能导致问题

## 工厂模式

### 概念

定义创建对象的接口，让子类决定实例化哪个类。

### 简单工厂

```javascript
// 产品类
class Product {
  constructor(name) {
    this.name = name;
  }

  operation() {
    console.log(`${this.name} operation`);
  }
}

// 具体产品
class ConcreteProductA extends Product {
  constructor() {
    super('Product A');
  }

  operation() {
    console.log('Product A specific operation');
  }
}

class ConcreteProductB extends Product {
  constructor() {
    super('Product B');
  }

  operation() {
    console.log('Product B specific operation');
  }
}

// 简单工厂
class SimpleFactory {
  static create(type) {
    switch (type) {
      case 'A':
        return new ConcreteProductA();
      case 'B':
        return new ConcreteProductB();
      default:
        throw new Error(`Unknown product type: ${type}`);
    }
  }
}

// 使用
const productA = SimpleFactory.create('A');
productA.operation();  // Product A specific operation

const productB = SimpleFactory.create('B');
productB.operation();  // Product B specific operation
```

#### 表单验证器工厂

```javascript
// 验证器接口
class Validator {
  validate(value) {
    throw new Error('Must implement validate method');
  }
}

// 必填验证器
class RequiredValidator extends Validator {
  validate(value) {
    return {
      valid: value !== null && value !== undefined && value !== '',
      message: 'This field is required'
    };
  }
}

// 邮箱验证器
class EmailValidator extends Validator {
  validate(value) {
    const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
    return {
      valid: emailRegex.test(value),
      message: 'Invalid email format'
    };
  }
}

// 长度验证器
class LengthValidator extends Validator {
  constructor(min, max) {
    super();
    this.min = min;
    this.max = max;
  }

  validate(value) {
    const length = value.length;
    return {
      valid: length >= this.min && length <= this.max,
      message: `Length must be between ${this.min} and ${this.max}`
    };
  }
}

// 验证器工厂
class ValidatorFactory {
  static validators = {
    required: RequiredValidator,
    email: EmailValidator,
    length: LengthValidator
  };

  static create(type, ...args) {
    const Validator = this.validators[type];
    if (!Validator) {
      throw new Error(`Unknown validator type: ${type}`);
    }
    return new Validator(...args);
  }
}

// 使用
const validators = [
  ValidatorFactory.create('required'),
  ValidatorFactory.create('email'),
  ValidatorFactory.create('length', 6, 50)
];

const value = 'test@example.com';
validators.forEach(validator => {
  const result = validator.validate(value);
  console.log(result);
});
```

- 增加系统复杂度

## 建造者模式

### 概念

将复杂对象的构建与表示分离，使得同样的构建过程可以创建不同的表示。

### 基本实现

```javascript
// 产品
class Computer {
  constructor() {
    this.parts = {};
  }

  show() {
    console.log('Computer Configuration:');
    for (const [key, value] of Object.entries(this.parts)) {
      console.log(`  ${key}: ${value}`);
    }
  }
}

// 建造者
class ComputerBuilder {
  constructor() {
    this.computer = new Computer();
  }

  setCPU(cpu) {
    this.computer.parts.cpu = cpu;
    return this;
  }

  setRAM(ram) {
    this.computer.parts.ram = ram;
    return this;
  }

  setStorage(storage) {
    this.computer.parts.storage = storage;
    return this;
  }

  setGPU(gpu) {
    this.computer.parts.gpu = gpu;
    return this;
  }

  build() {
    return this.computer;
  }
}

// 使用
const computer = new ComputerBuilder()
  .setCPU('Intel i9')
  .setRAM('32GB DDR5')
  .setStorage('1TB NVMe SSD')
  .setGPU('NVIDIA RTX 4090')
  .build();

computer.show();
// Computer Configuration:
//   cpu: Intel i9
//   ram: 32GB DDR5
//   storage: 1TB NVMe SSD
//   gpu: NVIDIA RTX 4090
```

### 实际应用

#### SQL 查询构建器

```javascript
class QueryBuilder {
  constructor() {
    this.query = {
      select: [],
      from: '',
      where: [],
      orderBy: [],
      limit: null,
      offset: null
    };
  }

  select(...fields) {
    this.query.select = fields;
    return this;
  }

  from(table) {
    this.query.from = table;
    return this;
  }

  where(condition) {
    this.query.where.push(condition);
    return this;
  }

  orderBy(field, direction = 'ASC') {
    this.query.orderBy.push(`${field} ${direction}`);
    return this;
  }

  limit(count) {
    this.query.limit = count;
    return this;
  }

  offset(count) {
    this.query.offset = count;
    return this;
  }

  build() {
    let sql = `SELECT ${this.query.select.join(', ') || '*'}`;
    sql += ` FROM ${this.query.from}`;

    if (this.query.where.length > 0) {
      sql += ` WHERE ${this.query.where.join(' AND ')}`;
    }

    if (this.query.orderBy.length > 0) {
      sql += ` ORDER BY ${this.query.orderBy.join(', ')}`;
    }

    if (this.query.limit !== null) {
      sql += ` LIMIT ${this.query.limit}`;
    }

    if (this.query.offset !== null) {
      sql += ` OFFSET ${this.query.offset}`;
    }

    return sql;
  }
}

// 使用
const query = new QueryBuilder()
  .select('id', 'name', 'email')
  .from('users')
  .where('active = 1')
  .where('created_at > "2024-01-01"')
  .orderBy('name', 'ASC')
  .limit(10)
  .offset(20)
  .build();

console.log(query);
// SELECT id, name, email FROM users WHERE active = 1 AND created_at > "2024-01-01" ORDER BY name ASC LIMIT 10 OFFSET 20
```

  }

  params(params) {
    Object.assign(this.queryParams, params);
    return this;
  }

  setHash(hash) {
    this.hash = hash;
    return this;
  }

  build() {
    let url = this.baseUrl;

    if (this.pathSegments.length > 0) {
      url += '/' + this.pathSegments.join('/');
    }

    const queryString = Object.entries(this.queryParams)
      .map(([key, value]) => `${encodeURIComponent(key)}=${encodeURIComponent(value)}`)
      .join('&');

    if (queryString) {
      url += '?' + queryString;
    }

    if (this.hash) {
      url += '#' + this.hash;
    }

    return url;
  }
}

// 使用
const url = new URLBuilder('https://api.example.com')
  .path('v1', 'users', 'profile')
  .param('userId', '123')
  .params({ format: 'json', lang: 'en' })
  .setHash('settings')
  .build();

console.log(url);
// https://api.example.com/v1/users/profile?userId=123&format=json&lang=en#settings
```

class FormBuilder {
  constructor() {
    this.fields = [];
    this.styles = {};
  }

  addField(name, type, options = {}) {
    this.fields.push({ name, type, ...options });
    return this;
  }

  addTextField(name, label, options = {}) {
    return this.addField(name, 'text', { label, ...options });
  }

  addEmailField(name, label, options = {}) {
    return this.addField(name, 'email', { label, ...options });
  }

  addPasswordField(name, label, options = {}) {
    return this.addField(name, 'password', { label, ...options });
  }

  addSubmitButton(text = 'Submit') {
    return this.addField('submit', 'submit', { text });
  }

  setStyle(key, value) {
    this.styles[key] = value;
    return this;
  }

  build() {
    const styleStr = Object.entries(this.styles)
      .map(([key, value]) => `${key}: ${value}`)
      .join('; ');

    const fieldsHtml = this.fields.map(field => {
      if (field.type === 'submit') {
        return `<button type="submit">${field.text}</button>`;
      }

      return `
        <div class="form-field">
          <label for="${field.name}">${field.label}</label>
          <input type="${field.type}" id="${field.name}" name="${field.name}"
                 ${field.required ? 'required' : ''}
                 ${field.placeholder ? `placeholder="${field.placeholder}"` : ''}>
        </div>
      `;
    }).join('');

    return `<form style="${styleStr}">${fieldsHtml}</form>`;
  }
}

// 使用
const form = new FormBuilder()
  .addTextField('username', 'Username', { required: true, placeholder: 'Enter username' })
  .addEmailField('email', 'Email', { required: true })
  .addPasswordField('password', 'Password', { required: true })
  .addSubmitButton('Sign Up')
  .setStyle('max-width', '400px')
  .setStyle('margin', '0 auto')
  .build();

console.log(form);
```

- 支持链式调用

**缺点**：

- 产品差异大时不适用

## 原型模式

### 概念

通过复制现有对象来创建新对象，而不是从头创建。

### 实现

#### Object.create()

```javascript
// 原型对象
const vehiclePrototype = {
  init(make, model) {
    this.make = make;
    this.model = model;
    return this;
  },

  getInfo() {
    return `${this.make} ${this.model}`;
  },

  clone() {
    return Object.create(this);
  }
};

// 使用
const car = Object.create(vehiclePrototype).init('Toyota', 'Camry');
console.log(car.getInfo());  // Toyota Camry

const anotherCar = car.clone();
anotherCar.init('Honda', 'Accord');
console.log(anotherCar.getInfo());  // Honda Accord
```

#### 类实现

```javascript
class Prototype {
  clone() {
    // 浅拷贝
    return Object.assign(Object.create(Object.getPrototypeOf(this)), this);
  }

  deepClone() {
    // 深拷贝
    return JSON.parse(JSON.stringify(this));
  }
}

class Document extends Prototype {
  constructor(title, content) {
    super();
    this.title = title;
    this.content = content;
    this.metadata = {
      createdAt: new Date(),
      author: 'Unknown'
    };
  }

  clone() {
    const cloned = super.clone();
    cloned.metadata = { ...this.metadata };
    return cloned;
  }
}

// 使用
const original = new Document('Original', 'Content');
const cloned = original.clone();

cloned.title = 'Cloned';
console.log(original.title);  // Original
console.log(cloned.title);    // Cloned
console.log(cloned.metadata !== original.metadata);  // true
```

### 实际应用

```javascript
// 模板对象
class EmailTemplate {
  constructor() {
    this.subject = '';
    this.body = '';
    this.recipients = [];
    this.cc = [];
    this.attachments = [];
  }

  setSubject(subject) {
    this.subject = subject;
    return this;
  }

  setBody(body) {
    this.body = body;
    return this;
  }

  addRecipient(email) {
    this.recipients.push(email);
    return this;
  }

  clone() {
    const cloned = new EmailTemplate();
    cloned.subject = this.subject;
    cloned.body = this.body;
    cloned.recipients = [...this.recipients];
    cloned.cc = [...this.cc];
    cloned.attachments = [...this.attachments];
    return cloned;
  }
}

// 创建原型模板
const welcomeEmail = new EmailTemplate()
  .setSubject('Welcome to Our Service')
  .setBody('Dear {name}, welcome!');

// 克隆并定制
const user1Email = welcomeEmail.clone()
  .addRecipient('user1@example.com');
user1Email.body = user1Email.body.replace('{name}', 'John');

const user2Email = welcomeEmail.clone()
  .addRecipient('user2@example.com');
user2Email.body = user2Email.body.replace('{name}', 'Jane');
```

### 优缺点

**优点**：
- 简化对象创建
- 动态添加/删除原型
- 减少子类数量

**缺点**：
- 深拷贝实现复杂
- 循环引用问题

## 模式对比

| 模式 | 用途 | 创建方式 |
|------|------|----------|
| 单例 | 唯一实例 | 静态方法或闭包 |
| 工厂 | 创建对象 | 工厂类/方法 |
| 抽象工厂 | 创建对象族 | 工厂组合 |
| 建造者 | 复杂对象 | 分步构建 |
| 原型 | 复制对象 | 克隆 |

---

## 最佳实践

1. **选择合适的模式**
   - 简单对象：直接 new 或对象字面量
   - 需要唯一实例：单例模式
   - 创建逻辑复杂：工厂模式
   - 多步骤构建：建造者模式
   - 避免重复初始化：原型模式

2. **避免过度设计**
   - 不要为了使用模式而使用
   - 优先考虑简单方案

3. **注意测试性**
   - 单例可能影响测试
   - 依赖注入可提高可测试性
