# 结构型模式

结构型模式（Structural Patterns）关注类和对象的组合，通过组合实现新的功能。

---

## 概述

结构型模式主要包括：
- **代理模式**（Proxy）：控制对象访问
- **装饰器模式**（Decorator）：动态添加功能
- **适配器模式**（Adapter）：接口转换
- **外观模式**（Facade）：简化接口
- **桥接模式**（Bridge）：分离抽象与实现
- **组合模式**（Composite）：树形结构
- **享元模式**（Flyweight）：共享细粒度对象

---

## 代理模式

### 概念

为其他对象提供代理，控制对原对象的访问。

### 应用场景

- 虚拟代理：延迟加载
- 保护代理：权限控制
- 缓存代理：缓存结果
- 智能引用：访问统计

### 实现方式

#### ES6 Proxy

```javascript
const target = {
  name: 'John',
  age: 30
};

const handler = {
  // 拦截属性读取
  get(target, prop, receiver) {
    console.log(`Getting ${prop}`);
    return target[prop];
  },

  // 拦截属性设置
  set(target, prop, value, receiver) {
    console.log(`Setting ${prop} = ${value}`);
    target[prop] = value;
    return true;
  },

  // 拦截 in 操作符
  has(target, prop) {
    console.log(`Checking ${prop} in target`);
    return prop in target;
  },

  // 拦截 delete 操作
  deleteProperty(target, prop) {
    console.log(`Deleting ${prop}`);
    delete target[prop];
    return true;
  },

  // 拦截 Object.keys 等
  ownKeys(target) {
    console.log('Getting own keys');
    return Object.keys(target);
  }
};

const proxy = new Proxy(target, handler);

proxy.name;         // Getting name
proxy.age = 31;     // Setting age = 31
'name' in proxy;    // Checking name in target
```

#### 缓存代理

```javascript
function createCacheProxy(fn) {
  const cache = new Map();

  return new Proxy(fn, {
    apply(target, thisArg, args) {
      const key = JSON.stringify(args);

      if (cache.has(key)) {
        console.log('From cache');
        return cache.get(key);
      }

      console.log('Computing');
      const result = target.apply(thisArg, args);
      cache.set(key, result);
      return result;
    }
  });
}

// 使用
function expensiveCalculation(n) {
  return n * n;
}

const proxiedCalc = createCacheProxy(expensiveCalculation);

proxiedCalc(5);  // Computing 25
proxiedCalc(5);  // From cache 25
proxiedCalc(10); // Computing 100
proxiedCalc(10); // From cache 100
```

#### 验证代理

```javascript
function createValidationProxy(obj, schema) {
  return new Proxy(obj, {
    set(target, prop, value) {
      if (prop in schema) {
        const validator = schema[prop];

        if (!validator(value)) {
          throw new Error(`Invalid value for ${prop}: ${value}`);
        }
      }

      target[prop] = value;
      return true;
    }
  });
}

// 使用
const schema = {
  age: value => Number.isInteger(value) && value >= 0,
  name: value => typeof value === 'string' && value.length > 0,
  email: value => /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(value)
};

const user = createValidationProxy({}, schema);

user.name = 'John';     // OK
user.age = 30;          // OK
user.email = 'john@example.com';  // OK

try {
  user.age = -5;        // Error: Invalid value for age: -5
} catch (e) {
  console.error(e.message);
}
```

#### 保护代理

```javascript
function createProtectedProxy(obj, rules) {
  return new Proxy(obj, {
    get(target, prop) {
      if (rules.readonly && rules.readonly.includes(prop)) {
        console.log(`Read-only access to ${prop}`);
      }
      return target[prop];
    },

    set(target, prop, value) {
      if (rules.readonly && rules.readonly.includes(prop)) {
        throw new Error(`${prop} is read-only`);
      }

      if (rules.private && rules.private.includes(prop)) {
        throw new Error(`${prop} is private`);
      }

      target[prop] = value;
      return true;
    }
  });
}

// 使用
const sensitiveData = {
  name: 'John',
  email: 'john@example.com',
  password: 'secret'
};

const protectedData = createProtectedProxy(sensitiveData, {
  readonly: ['name'],
  private: ['password']
});

console.log(protectedData.name);  // John

try {
  protectedData.name = 'Jane';    // Error: name is read-only
} catch (e) {
  console.error(e.message);
}

try {
  console.log(protectedData.password);  // Error: password is private
} catch (e) {
  console.error(e.message);
}
```

### 实际应用 - API 客户端

```javascript
class APIClient {
  constructor(baseUrl) {
    this.baseUrl = baseUrl;
  }

  async get(endpoint) {
    const response = await fetch(`${this.baseUrl}${endpoint}`);
    return response.json();
  }

  async post(endpoint, data) {
    const response = await fetch(`${this.baseUrl}${endpoint}`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(data)
    });
    return response.json();
  }
}

function createAPIProxy(client) {
  const cache = new Map();
  const pending = new Map();

  return new Proxy(client, {
    get(target, method) {
      if (method === 'get') {
        return async function(endpoint) {
          const key = `get:${endpoint}`;

          // 检查缓存
          if (cache.has(key)) {
            console.log('Cache hit:', endpoint);
            return cache.get(key);
          }

          // 避免重复请求
          if (pending.has(key)) {
            console.log('Pending request:', endpoint);
            return pending.get(key);
          }

          // 发起请求
          const promise = target.get(endpoint);
          pending.set(key, promise);

          try {
            const result = await promise;
            cache.set(key, result);
            return result;
          } finally {
            pending.delete(key);
          }
        };
      }

      return target[method].bind(target);
    }
  });
}

// 使用
const client = new APIClient('https://api.example.com');
const proxiedClient = createAPIProxy(client);

// 同时发起多个相同请求，只会发送一次
proxiedClient.get('/users');
proxiedClient.get('/users');
```

### 优缺点

**优点**：
- 控制对象访问
- 增强功能
- 符合开闭原则

**缺点**：
- 增加复杂度
- 可能影响性能

## 装饰器模式

### 概念

动态地给对象添加额外功能，比生成子类更灵活。

### 函数装饰器

```javascript
// 日志装饰器
function log(fn) {
  return function(...args) {
    console.log(`Calling ${fn.name} with:`, args);
    const result = fn.apply(this, args);
    console.log(`Result:`, result);
    return result;
  };
}

// 使用
function add(a, b) {
  return a + b;
}

const loggedAdd = log(add);
loggedAdd(2, 3);
// Calling add with: [2, 3]
// Result: 5
```

#### 计时装饰器

```javascript
function timing(fn) {
  return function(...args) {
    const start = performance.now();
    const result = fn.apply(this, args);
    const end = performance.now();
    console.log(`${fn.name} took ${(end - start).toFixed(2)}ms`);
    return result;
  };
}

// 使用
function slowFunction() {
  for (let i = 0; i < 1000000; i++) {}
  return 'Done';
}

const timedFunction = timing(slowFunction);
timedFunction();  // slowFunction took 5.23ms
```

#### 重试装饰器

```javascript
function retry(maxRetries = 3, delay = 1000) {
  return function(fn) {
    return async function(...args) {
      let lastError;

      for (let i = 0; i < maxRetries; i++) {
        try {
          return await fn.apply(this, args);
        } catch (error) {
          lastError = error;
          console.log(`Attempt ${i + 1} failed, retrying...`);
          await new Promise(resolve => setTimeout(resolve, delay));
        }
      }

      throw lastError;
    };
  };
}

// 使用
const fetchWithRetry = retry(3)(fetch);
await fetchWithRetry('https://api.example.com/data');
```

#### 记忆化装饰器

```javascript
function memoize(fn) {
  const cache = new Map();

  return function(...args) {
    const key = JSON.stringify(args);

    if (cache.has(key)) {
      return cache.get(key);
    }

    const result = fn.apply(this, args);
    cache.set(key, result);
    return result;
  };
}

// 使用
function fibonacci(n) {
  if (n <= 1) return n;
  return fibonacci(n - 1) + fibonacci(n - 2);
}

const memoizedFib = memoize(function fibonacci(n) {
  if (n <= 1) return n;
  return memoizedFib(n - 1) + memoizedFib(n - 2);
});

console.log(memoizedFib(50));  // 快速计算
```

### 类装饰器（TypeScript/ES Next）

```javascript
// 方法装饰器
function readonly(target, key, descriptor) {
  descriptor.writable = false;
  return descriptor;
}

function log(target, key, descriptor) {
  const original = descriptor.value;

  descriptor.value = function(...args) {
    console.log(`Calling ${key} with:`, args);
    return original.apply(this, args);
  };

  return descriptor;
}

class Person {
  constructor(name) {
    this.name = name;
  }

  @readonly
  getName() {
    return this.name;
  }

  @log
  greet(greeting) {
    return `${greeting}, I'm ${this.name}`;
  }
}

// 注意：装饰器语法需要编译器支持
```

### 组合装饰器

```javascript
// 多个装饰器组合
function compose(...decorators) {
  return function(fn) {
    return decorators.reduceRight((acc, decorator) => decorator(acc), fn);
  };
}

// 使用
const enhanced = compose(
  log,
  timing,
  memoize
)(expensiveFunction);
```

#### 防抖装饰器

```javascript
function debounce(delay) {
  return function(target, key, descriptor) {
    const original = descriptor.value;
    let timeoutId;

    descriptor.value = function(...args) {
      clearTimeout(timeoutId);
      timeoutId = setTimeout(() => original.apply(this, args), delay);
    };

    return descriptor;
  };
}

class SearchComponent {
  @debounce(300)
  handleSearch(query) {
    console.log('Searching:', query);
    // 发起搜索请求
  }
}
```

- 增加复杂度
- 可能产生过多小对象

## 适配器模式

### 概念

将一个类的接口转换成客户端期望的另一个接口，使不兼容的类可以一起工作。

### 应用场景

- 接口不兼容
- 第三方库集成
- 旧系统改造

### 实现

#### 对象适配器

```javascript
// 旧接口
class OldCalculator {
  operations(a, b, operation) {
    switch (operation) {
      case 'add': return a + b;
      case 'sub': return a - b;
      default: return NaN;
    }
  }
}

// 新接口期望
class NewCalculator {
  add(a, b) {
    return a + b;
  }

  subtract(a, b) {
    return a - b;
  }

  multiply(a, b) {
    return a * b;
  }
}

// 适配器
class CalculatorAdapter {
  constructor() {
    this.calculator = new OldCalculator();
  }

  add(a, b) {
    return this.calculator.operations(a, b, 'add');
  }

  subtract(a, b) {
    return this.calculator.operations(a, b, 'sub');
  }

  multiply(a, b) {
    // 旧系统不支持，需要扩展
    console.log('Operation not supported in old system');
    return a * b;
  }
}

// 使用
const adapter = new CalculatorAdapter();
adapter.add(5, 3);      // 8
adapter.subtract(5, 3); // 2
```

## 外观模式

### 概念

为子系统中的一组接口提供一个统一的高层接口，使子系统更容易使用。

### 实现

```javascript
// 复杂子系统
class CPU {
  freeze() { console.log('CPU freeze'); }
  jump(position) { console.log(`CPU jump to ${position}`); }
  execute() { console.log('CPU execute'); }
}

class Memory {
  load(position, data) { console.log(`Memory load ${data} at ${position}`); }
}

class HardDrive {
  read(lba, size) {
    console.log(`HardDrive read ${size} bytes from ${lba}`);
    return 'data';
  }
}

// 外观
class ComputerFacade {
  constructor() {
    this.cpu = new CPU();
    this.memory = new Memory();
    this.hardDrive = new HardDrive();
  }

  start() {
    this.cpu.freeze();
    this.memory.load(0, this.hardDrive.read(0, 1024));
    this.cpu.jump(0);
    this.cpu.execute();
    console.log('Computer started');
  }
}

// 使用
const computer = new ComputerFacade();
computer.start();
// CPU freeze
// HardDrive read 1024 bytes from 0
// Memory load data at 0
// CPU jump to 0
// CPU execute
// Computer started
```

// 浏览器兼容性外观
const EventUtils = {
  addEvent(element, type, handler) {
    if (element.addEventListener) {
      element.addEventListener(type, handler, false);
    } else if (element.attachEvent) {
      element.attachEvent(`on${type}`, handler);
    } else {
      element[`on${type}`] = handler;
    }
  },

  removeEvent(element, type, handler) {
    if (element.removeEventListener) {
      element.removeEventListener(type, handler, false);
    } else if (element.detachEvent) {
      element.detachEvent(`on${type}`, handler);
    } else {
      element[`on${type}`] = null;
    }
  },

  getEvent(event) {
    return event || window.event;
  },

  getTarget(event) {
    return event.target || event.srcElement;
  },

  preventDefault(event) {
    if (event.preventDefault) {
      event.preventDefault();
    } else {
      event.returnValue = false;
    }
  }
};

## 桥接模式

### 概念

将抽象部分与实现部分分离，使它们都可以独立变化。

### 实现

```javascript
// 实现接口
class Renderer {
  renderCircle(radius) {
    throw new Error('Must implement');
  }
}

// 具体实现
class VectorRenderer extends Renderer {
  renderCircle(radius) {
    console.log(`Drawing circle with radius ${radius} (Vector)`);
  }
}

class RasterRenderer extends Renderer {
  renderCircle(radius) {
    console.log(`Drawing circle with radius ${radius} (Raster)`);
  }
}

// 抽象
class Shape {
  constructor(renderer) {
    this.renderer = renderer;
  }

  draw() {
    throw new Error('Must implement');
  }
}

// 扩展抽象
class Circle extends Shape {
  constructor(renderer, radius) {
    super(renderer);
    this.radius = radius;
  }

  draw() {
    this.renderer.renderCircle(this.radius);
  }

  resize(factor) {
    this.radius *= factor;
  }
}

// 使用
const vectorRenderer = new VectorRenderer();
const rasterRenderer = new RasterRenderer();

const vectorCircle = new Circle(vectorRenderer, 5);
const rasterCircle = new Circle(rasterRenderer, 5);

vectorCircle.draw();  // Drawing circle with radius 5 (Vector)
rasterCircle.draw();  // Drawing circle with radius 5 (Raster)

vectorCircle.resize(2);
vectorCircle.draw();  // Drawing circle with radius 10 (Vector)
```

### 优缺点

**优点**：
- 分离抽象和实现
- 扩展性好
- 符合开闭原则

**缺点**：
- 增加复杂度

## 组合模式

### 概念

将对象组合成树形结构以表示"部分-整体"的层次结构。

### 实现

```javascript
// 组件接口
class Component {
  add(component) {
    throw new Error('Must implement');
  }

  remove(component) {
    throw new Error('Must implement');
  }

  display(depth = 0) {
    throw new Error('Must implement');
  }
}

// 叶子节点
class Leaf extends Component {
  constructor(name) {
    super();
    this.name = name;
  }

  add(component) {
    throw new Error('Cannot add to leaf');
  }

  remove(component) {
    throw new Error('Cannot remove from leaf');
  }

  display(depth = 0) {
    console.log('  '.repeat(depth) + this.name);
  }
}

// 组合节点
class Composite extends Component {
  constructor(name) {
    super();
    this.name = name;
    this.children = [];
  }

  add(component) {
    this.children.push(component);
  }

  remove(component) {
    const index = this.children.indexOf(component);
    if (index > -1) {
      this.children.splice(index, 1);
    }
  }

  display(depth = 0) {
    console.log('  '.repeat(depth) + this.name);
    this.children.forEach(child => child.display(depth + 1));
  }
}

// 使用
const root = new Composite('root');
const branch1 = new Composite('branch1');
const branch2 = new Composite('branch2');
const leaf1 = new Leaf('leaf1');
const leaf2 = new Leaf('leaf2');
const leaf3 = new Leaf('leaf3');

branch1.add(leaf1);
branch1.add(leaf2);
branch2.add(leaf3);
root.add(branch1);
root.add(branch2);

root.display();
// root
//   branch1
//     leaf1
//     leaf2
//   branch2
//     leaf3
```

### 优缺点

**优点**：
- 统一处理简单和复杂对象
- 方便构建层次结构
- 符合开闭原则

**缺点**：
- 难以限制类型
- 设计变得复杂

## 享元模式

### 概念

运用共享技术有效地支持大量细粒度对象。

### 实现

```javascript
// 享元工厂
class FlyweightFactory {
  constructor() {
    this.flyweights = {};
  }

  getFlyweight(key) {
    if (!this.flyweights[key]) {
      this.flyweights[key] = this.createFlyweight(key);
    }
    return this.flyweights[key];
  }

  createFlyweight(key) {
    // 创建共享对象
    return { sharedState: key };
  }
}

// 使用示例：文本编辑器中的字符
class CharacterFactory {
  constructor() {
    this.characters = {};
  }

  getCharacter(char, font) {
    const key = `${char}-${font}`;
    if (!this.characters[key]) {
      this.characters[key] = {
        char,
        font,
        render(position) {
          console.log(`Rendering '${this.char}' at ${position}`);
        }
      };
    }
    return this.characters[key];
  }
}

// 使用
const factory = new CharacterFactory();
const charA1 = factory.getCharacter('A', 'Arial');
const charA2 = factory.getCharacter('A', 'Arial');

console.log(charA1 === charA2);  // true（共享同一个对象）

charA1.render(0);  // Rendering 'A' at 0
charA2.render(1);  // Rendering 'A' at 1
```

### 优缺点

**优点**：
- 减少内存占用
- 减少对象数量

**缺点**：
- 增加复杂度
- 需要区分内部状态和外部状态

## 模式对比

| 模式 | 用途 | 关键点 |
|------|------|--------|
| 代理 | 控制访问 | 控制对象访问 |
| 装饰器 | 添加功能 | 动态扩展 |
| 适配器 | 接口转换 | 接口兼容 |
| 外观 | 简化接口 | 统一入口 |
| 桥接 | 分离抽象 | 独立变化 |
| 组合 | 树形结构 | 统一处理 |
| 享元 | 共享对象 | 减少内存 |

---

## 最佳实践

1. **选择合适的模式**
   - 需要控制访问：代理模式
   - 需要动态添加功能：装饰器模式
   - 需要接口兼容：适配器模式
   - 需要简化接口：外观模式

2. **避免过度设计**
   - 简单问题用简单方案
   - 不要为了模式而模式

3. **组合使用**
   - 多种模式可以组合
   - 提高灵活性
