# 行为型模式

行为型模式（Behavioral Patterns）关注对象之间的通信、职责划分和算法封装。

---

## 概述

行为型模式主要包括：
- **观察者模式**（Observer）：对象间的一对多依赖
- **发布订阅模式**（Pub/Sub）：事件驱动的消息传递
- **策略模式**（Strategy）：算法族的封装和切换
- **命令模式**（Command）：请求的封装
- **迭代器模式**（Iterator）：遍历聚合对象
- **责任链模式**（Chain of Responsibility）：请求的链式处理
- **状态模式**（State）：状态驱动的行为变化
- **模板方法模式**（Template Method）：算法骨架

---

## 观察者模式

### 概念

定义对象间一对多的依赖关系，当一个对象状态改变时，所有依赖它的对象都会收到通知。

### 应用场景

- 事件处理系统
- 数据绑定
- 消息通知
- 状态监控

### 基本实现

```javascript
class Subject {
  constructor() {
    this.observers = [];
  }

  // 添加观察者
  subscribe(observer) {
    this.observers.push(observer);
    return () => this.unsubscribe(observer);
  }

  // 移除观察者
  unsubscribe(observer) {
    this.observers = this.observers.filter(o => o !== observer);
  }

  // 通知所有观察者
  notify(data) {
    this.observers.forEach(observer => observer.update(data));
  }
}

// 观察者接口
class Observer {
  update(data) {
    throw new Error('Must implement update method');
  }
}

// 具体观察者
class ConsoleObserver extends Observer {
  constructor(name) {
    super();
    this.name = name;
  }

  update(data) {
    console.log(`${this.name} received:`, data);
  }
}

// 使用
const subject = new Subject();
const observer1 = new ConsoleObserver('Observer 1');
const observer2 = new ConsoleObserver('Observer 2');

subject.subscribe(observer1);
subject.subscribe(observer2);

subject.notify('Hello World!');
// Observer 1 received: Hello World!
// Observer 2 received: Hello World!
```

```javascript
class Store {
  constructor(initialState = {}) {
    this.state = initialState;
    this.listeners = [];
```

#### 表单验证

```javascript
class FormValidator {
  constructor() {
    this.validators = [];
    this.errors = [];
    this.subscribers = [];
  }

  addValidator(validator) {
    this.validators.push(validator);
  }

  validate(value) {
    this.errors = this.validators
      .map(validator => validator(value))
      .filter(result => !result.valid)
      .map(result => result.message);

    this.notify({ value, errors: this.errors, valid: this.errors.length === 0 });
    return this.errors.length === 0;
  }

  subscribe(callback) {
    this.subscribers.push(callback);
    return () => {
      this.subscribers = this.subscribers.filter(cb => cb !== callback);
    };
  }

  notify(data) {
    this.subscribers.forEach(callback => callback(data));
  }
}

// 使用
const validator = new FormValidator();

validator.addValidator(value => ({
  valid: value.length >= 8,
  message: 'Minimum 8 characters'
}));

validator.addValidator(value => ({
  valid: /\d/.test(value),
  message: 'Must contain a number'
}));

validator.subscribe(({ valid, errors }) => {
  if (valid) {
    console.log('✅ Valid');
  } else {
    console.log('❌ Errors:', errors);
  }
});

validator.validate('password');
// ❌ Errors: ['Must contain a number']

validator.validate('password123');
// ✅ Valid
```

## 发布订阅模式

### 概念

发布订阅模式是观察者模式的变体，引入了事件通道（Event Channel）作为中介。

### 与观察者模式的区别

```
观察者模式：Subject <-> Observer（直接通信）
发布订阅模式：Publisher -> Event Channel -> Subscriber（通过中介）
```

### 实现

```javascript
class EventEmitter {
  constructor() {
    this.events = {};
  }

  // 订阅事件
  on(event, callback) {
    if (!this.events[event]) {
      this.events[event] = [];
    }
    this.events[event].push(callback);
    return () => this.off(event, callback);
  }

  // 订阅一次性事件
  once(event, callback) {
    const wrapper = (...args) => {
      callback(...args);
      this.off(event, wrapper);
    };
    this.on(event, wrapper);
  }

  // 发布事件
  emit(event, ...args) {
    if (this.events[event]) {
      this.events[event].forEach(callback => callback(...args));
    }
  }

  // 取消订阅
  off(event, callback) {
    if (this.events[event]) {
      this.events[event] = this.events[event].filter(cb => cb !== callback);
    }
  }

  // 清除所有订阅
  removeAllListeners(event) {
    if (event) {
      delete this.events[event];
    } else {
      this.events = {};
    }
  }
}

// 使用
const emitter = new EventEmitter();

emitter.on('userCreated', (user) => {
  console.log('User created:', user);
});

emitter.on('userCreated', (user) => {
  console.log('Send welcome email to:', user.email);
});

emitter.emit('userCreated', { id: 1, name: 'John', email: 'john@example.com' });
// User created: { id: 1, name: 'John', email: 'john@example.com' }
// Send welcome email to: john@example.com
```

```javascript
class MessageQueue {
  constructor() {
    this.emitter = new EventEmitter();
    this.queues = {};
  }

  // 发布消息到队列
  publish(queueName, message) {
    if (!this.queues[queueName]) {
      this.queues[queueName] = [];
    }
    this.queues[queueName].push(message);
    this.emitter.emit(queueName, message);
  }

  // 订阅队列
  subscribe(queueName, handler) {
    return this.emitter.on(queueName, handler);
  }

  // 获取队列消息
  getMessages(queueName) {
    return this.queues[queueName] || [];
  }
}

// 使用
const messageQueue = new MessageQueue();

  console.log('Notification:', message);
});

messageQueue.publish('notifications', { type: 'alert', text: 'System update' });
```

## 策略模式

### 概念

定义一系列算法，把它们封装起来，并使它们可以相互替换。

### 应用场景

- 表单验证
- 排序算法选择
- 支付方式选择
- 价格计算策略

### 基本实现

```javascript
// 策略对象
const strategies = {
  add: (a, b) => a + b,
  subtract: (a, b) => a - b,
  multiply: (a, b) => a * b,
  divide: (a, b) => a / b
};

// 上下文
function calculate(strategy, a, b) {
  if (!strategies[strategy]) {
    throw new Error(`Unknown strategy: ${strategy}`);
  }
  return strategies[strategy](a, b);
}

// 使用
console.log(calculate('add', 5, 3));      // 8
console.log(calculate('multiply', 5, 3)); // 15
```

### 实际应用

#### 表单验证策略

```javascript
const validationStrategies = {
  required: (value) => ({
    valid: value !== null && value !== undefined && value !== '',
    message: 'This field is required'
  }),

  minLength: (value, min) => ({
    valid: value.length >= min,
    message: `Minimum ${min} characters required`
  }),

  maxLength: (value, max) => ({
    valid: value.length <= max,
    message: `Maximum ${max} characters allowed`
  }),

  email: (value) => ({
    valid: /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(value),
    message: 'Invalid email format'
  }),

  phone: (value) => ({
    valid: /^1[3-9]\d{9}$/.test(value),
    message: 'Invalid phone number'
  }),

  pattern: (value, regex) => ({
    valid: regex.test(value),
    message: 'Invalid format'
  })
};

// 验证器
class Validator {
  constructor() {
    this.rules = [];
  }

  addRule(strategy, ...args) {
    this.rules.push({ strategy, args });
    return this;
  }

  validate(value) {
    for (const rule of this.rules) {
      const validator = validationStrategies[rule.strategy];
      const result = validator(value, ...rule.args);

      if (!result.valid) {
        return result;
      }
    }

    return { valid: true };
  }
}

// 使用
const emailValidator = new Validator()
  .addRule('required')
  .addRule('email');

console.log(emailValidator.validate('test@example.com'));  // { valid: true }
console.log(emailValidator.validate(''));  // { valid: false, message: '...' }

const passwordValidator = new Validator()
  .addRule('required')
  .addRule('minLength', 8)
  .addRule('pattern', /\d/);

console.log(passwordValidator.validate('password123'));  // { valid: true }
```

```javascript
// 价格策略
const priceStrategies = {
  normal: (price) => price,

  discount: (price, discount) => price * (1 - discount),

  member: (price, memberLevel) => {
    const discounts = {
      bronze: 0.95,
      silver: 0.9,
      gold: 0.85,
      platinum: 0.8
    };
    return price * (discounts[memberLevel] || 1);
  },

  bulk: (price, quantity) => {
    if (quantity >= 100) return price * 0.8;
    if (quantity >= 50) return price * 0.9;
    if (quantity >= 10) return price * 0.95;
    return price;
  }
};

class PriceCalculator {
  constructor(strategy = 'normal') {
    this.strategy = strategy;
    this.params = [];

  setStrategy(strategy) {
    this.strategy = strategy;
    return this;
  }

  setParams(...params) {
    this.params = params;
    return this;
  }

  calculate(price) {
    const calculator = priceStrategies[this.strategy];
    return calculator(price, ...this.params);
  }
}

calculator.setStrategy('discount').setParams(0.2);
```

```javascript
const sortStrategies = {
  ascending: (arr, key) => [...arr].sort((a, b) => a[key] - b[key]),

  descending: (arr, key) => [...arr].sort((a, b) => b[key] - a[key]),

  alphabetical: (arr, key) => [...arr].sort((a, b) =>
    a[key].localeCompare(b[key])
  ),

  byDate: (arr, key, order = 'asc') => [...arr].sort((a, b) => {
    const dateA = new Date(a[key]);
    const dateB = new Date(b[key]);
    return order === 'asc' ? dateA - dateB : dateB - dateA;
  })
};

];

console.log(sortStrategies.ascending(products, 'price'));
// [{ name: 'Cherry', price: 1.0, ... }, ...]

console.log(sortStrategies.alphabetical(products, 'name'));
// [{ name: 'Apple', ... }, { name: 'Banana', ... }, ...]
```

## 命令模式

### 概念

将请求封装为对象，从而使你可用不同的请求对客户进行参数化。

### 实现

```javascript
// 命令接口
class Command {
  execute() {
    throw new Error('Must implement execute method');
  }

  undo() {
    throw new Error('Must implement undo method');
  }
}

// 接收者
class Light {
  turnOn() {
    console.log('Light is ON');
    this.isOn = true;
  }

  turnOff() {
    console.log('Light is OFF');
    this.isOn = false;
  }
}

// 具体命令
class TurnOnCommand extends Command {
  constructor(light) {
    super();
    this.light = light;
  }

  execute() {
    this.light.turnOn();
  }

  undo() {
    this.light.turnOff();
  }
}

class TurnOffCommand extends Command {
  constructor(light) {
    super();
    this.light = light;
  }

  execute() {
    this.light.turnOff();
  }

  undo() {
    this.light.turnOn();
  }
}

// 调用者
class RemoteControl {
  constructor() {
    this.command = null;
    this.history = [];
  }

  setCommand(command) {
    this.command = command;
  }

  pressButton() {
    if (this.command) {
      this.command.execute();
      this.history.push(this.command);
    }
  }

  pressUndo() {
    const lastCommand = this.history.pop();
    if (lastCommand) {
      lastCommand.undo();
    }
  }
}

// 使用
const light = new Light();
const remote = new RemoteControl();

remote.setCommand(new TurnOnCommand(light));
remote.pressButton();  // Light is ON

remote.setCommand(new TurnOffCommand(light));
remote.pressButton();  // Light is OFF

remote.pressUndo();    // Light is ON
```

### 实际应用 - 文本编辑器

```javascript
class TextEditor {
  constructor() {
    this.content = '';
    this.history = [];
    this.redoStack = [];
  }

  execute(command) {
    this.content = command.execute(this.content);
    this.history.push(command);
    this.redoStack = [];
  }

  undo() {
    const command = this.history.pop();
    if (command) {
      this.content = command.undo(this.content);
      this.redoStack.push(command);
    }
  }

  redo() {
    const command = this.redoStack.pop();
    if (command) {
      this.content = command.execute(this.content);
      this.history.push(command);
    }
  }

  show() {
    console.log(this.content);
  }
}

// 命令类
class InsertCommand {
  constructor(text, position) {
    this.text = text;
    this.position = position;
  }

  execute(content) {
    return content.slice(0, this.position) + this.text + content.slice(this.position);
  }

  undo(content) {
    return content.slice(0, this.position) + content.slice(this.position + this.text.length);
  }
}

class DeleteCommand {
  constructor(start, end) {
    this.start = start;
    this.end = end;
    this.deletedText = '';
  }

  execute(content) {
    this.deletedText = content.slice(this.start, this.end);
    return content.slice(0, this.start) + content.slice(this.end);
  }

  undo(content) {
    return content.slice(0, this.start) + this.deletedText + content.slice(this.start);
  }
}

// 使用
const editor = new TextEditor();

editor.execute(new InsertCommand('Hello', 0));
editor.show();  // Hello

editor.execute(new InsertCommand(' World', 5));
editor.show();  // Hello World

editor.undo();
editor.show();  // Hello

editor.redo();
editor.show();  // Hello World
```

## 迭代器模式

### 概念

提供一种方法顺序访问聚合对象中的元素，而不暴露其内部表示。

### 实现

```javascript
// 迭代器接口
class Iterator {
  hasNext() {
    throw new Error('Must implement hasNext method');
  }

  next() {
    throw new Error('Must implement next method');
  }
}

// 数组迭代器
class ArrayIterator extends Iterator {
  constructor(array) {
    super();
    this.array = array;
    this.index = 0;
  }

  hasNext() {
    return this.index < this.array.length;
  }

  next() {
    if (this.hasNext()) {
      return {
        value: this.array[this.index++],
        done: false
      };
    }
    return { done: true };
  }
}

// 聚合对象
class ArrayCollection {
  constructor(items) {
    this.items = items;
  }

  createIterator() {
    return new ArrayIterator(this.items);
  }

  // ES6 迭代器协议
  [Symbol.iterator]() {
    let index = 0;
    const items = this.items;

    return {
      next() {
        if (index < items.length) {
          return { value: items[index++], done: false };
        }
        return { done: true };
      }
    };
  }
}

// 使用
const collection = new ArrayCollection([1, 2, 3, 4, 5]);

// 自定义迭代器
const iterator = collection.createIterator();
while (iterator.hasNext()) {
  console.log(iterator.next().value);
}

// ES6 for...of
for (const item of collection) {
  console.log(item);
}
```

### 实际应用

#### 树形结构迭代

```javascript
class TreeNode {
  constructor(value) {
    this.value = value;
    this.children = [];
  }

  add(child) {
    this.children.push(child);
  }

  // 深度优先迭代器
  [Symbol.iterator]() {
    const stack = [this];

    return {
      next() {
        if (stack.length === 0) {
          return { done: true };
        }

        const node = stack.pop();

        // 子节点入栈（反向以保证顺序）
        for (let i = node.children.length - 1; i >= 0; i--) {
          stack.push(node.children[i]);
        }

        return { value: node.value, done: false };
      }
    };
  }
}

// 使用
const root = new TreeNode('A');
const b = new TreeNode('B');
const c = new TreeNode('C');
const d = new TreeNode('D');
const e = new TreeNode('E');

root.add(b);
root.add(c);
b.add(d);
b.add(e);

for (const value of root) {
  console.log(value);
}
// A, B, D, E, C（深度优先遍历）
```

## 责任链模式

### 概念

使多个对象都有机会处理请求，从而避免请求的发送者和接收者之间的耦合关系。

### 实现

```javascript
// 处理者接口
class Handler {
  constructor() {
    this.next = null;
  }

  setNext(handler) {
    this.next = handler;
    return handler;
  }

  handle(request) {
    if (this.next) {
      return this.next.handle(request);
    }
    return null;
  }
}

// 具体处理者
class AuthHandler extends Handler {
  handle(request) {
    if (!request.user) {
      return { error: 'Unauthorized' };
    }
    console.log('Auth check passed');
    return super.handle(request);
  }
}

class RoleHandler extends Handler {
  handle(request) {
    if (!request.user.roles.includes('admin')) {
      return { error: 'Forbidden' };
    }
    console.log('Role check passed');
    return super.handle(request);
  }
}

class RateLimitHandler extends Handler {
  handle(request) {
    if (request.rateLimitExceeded) {
      return { error: 'Rate limit exceeded' };
    }
    console.log('Rate limit check passed');
    return super.handle(request);
  }
}

// 使用
const auth = new AuthHandler();
const role = new RoleHandler();
const rateLimit = new RateLimitHandler();

auth.setNext(role).setNext(rateLimit);

const result1 = auth.handle({
  user: { roles: ['admin'] }
});
// Auth check passed
// Role check passed
// Rate limit check passed

const result2 = auth.handle({
  user: { roles: ['user'] }
});
// Auth check passed
// { error: 'Forbidden' }
```

### 实际应用 - 中间件系统

```javascript
class Middleware {
  constructor() {
    this.middlewares = [];
  }

  use(middleware) {
    this.middlewares.push(middleware);
  }

  async execute(context) {
    let index = 0;

    const next = async () => {
      if (index < this.middlewares.length) {
        const middleware = this.middlewares[index++];
        await middleware(context, next);
      }
    };

    await next();
    return context;
  }
}

// 使用
const app = new Middleware();

// 日志中间件
app.use(async (ctx, next) => {
  console.log(`[${new Date().toISOString()}] ${ctx.method} ${ctx.url}`);
  await next();
});

// 认证中间件
app.use(async (ctx, next) => {
  if (!ctx.token) {
    ctx.status = 401;
    ctx.body = 'Unauthorized';
    return;
  }
  await next();
});

// 响应中间件
app.use(async (ctx, next) => {
  ctx.status = 200;
  ctx.body = 'Success';
  await next();
});

// 执行
app.execute({
  method: 'GET',
  url: '/api/users',
  token: 'valid-token'
});
```

## 状态模式

### 概念

允许对象在其内部状态改变时改变其行为。

### 实现

```javascript
// 状态接口
class State {
  handle(context) {
    throw new Error('Must implement handle method');
  }
}

// 具体状态
class RedState extends State {
  handle(context) {
    console.log('Red light - STOP');
    context.setState(new GreenState());
  }
}

class GreenState extends State {
  handle(context) {

    console.log('Yellow light - CAUTION');
    context.setState(new RedState());
  }
}

// 上下文
class TrafficLight {

    this.state = new RedState();
```

## 模板方法模式

### 概念

定义算法的骨架，将某些步骤延迟到子类中实现。

### 实现

```javascript
// 抽象类
class DataParser {
  // 模板方法
  parse(data) {
    const rawData = this.readFile(data);
    const parsedData = this.processData(rawData);
    const result = this.formatOutput(parsedData);
    this.saveResult(result);
    return result;
  }

  readFile(data) {
    console.log('Reading file...');
    return data;
  }

  // 抽象方法，由子类实现
  processData(data) {
    throw new Error('Must implement processData method');
  }

  formatOutput(data) {
    return JSON.stringify(data, null, 2);
  }

  saveResult(result) {
    console.log('Result saved');
  }
}

// 具体实现
class CSVParser extends DataParser {
  processData(data) {
    console.log('Parsing CSV...');
    const lines = data.split('\n');
    const headers = lines[0].split(',');
    return lines.slice(1).map(line => {
      const values = line.split(',');
      return headers.reduce((obj, header, index) => {
        obj[header] = values[index];
        return obj;
      }, {});
    });
  }
}

class XMLParser extends DataParser {
  processData(data) {
    console.log('Parsing XML...');
    // XML 解析逻辑
    return { parsed: 'xml' };
  }
}

// 使用
const csvParser = new CSVParser();
const result = csvParser.parse('name,age\nJohn,30\nJane,25');
console.log(result);
```

## 模式对比

| 模式 | 用途 | 关键点 |
|------|------|--------|
| 观察者 | 一对多依赖 | 状态通知 |
| 发布订阅 | 事件驱动 | 消息中介 |
| 策略 | 算法切换 | 封装策略 |
| 命令 | 请求封装 | 命令对象 |
| 迭代器 | 遍历聚合 | 统一遍历 |
| 责任链 | 链式处理 | 传递请求 |
| 状态 | 状态驱动 | 状态对象 |
| 模板方法 | 算法骨架 | 固定流程 |

---

## 最佳实践

1. **选择合适的模式**
   - 需要通知机制：观察者/发布订阅
   - 需要切换算法：策略模式
   - 需要撤销操作：命令模式
   - 需要状态管理：状态模式

2. **避免过度设计**
   - 简单逻辑不需要复杂模式
   - 保持代码可读性

3. **组合使用**
   - 多种模式可以组合
   - 提高系统灵活性
