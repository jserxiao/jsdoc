# 脚本加载与执行

> 理解 JavaScript 脚本的加载和执行机制对于优化页面性能至关重要。不同的加载方式会显著影响页面的渲染速度和用户体验。

## 学习要点

- 📝 理解 `<script>` 标签的工作原理
- 🔄 掌握 async 和 defer 的区别
- ⚡ 学会优化脚本加载
- 🎯 了解动态脚本加载技术

---

## 1. script 标签基础

### 默认行为

```html
<!DOCTYPE html>
<html>
<head>
    <title>脚本加载示例</title>
    <!-- 默认行为：阻塞渲染 -->
    <script src="script.js"></script>
    <!-- 
        浏览器遇到 <script> 时：
        1. 暂停 HTML 解析
        2. 下载脚本
        3. 执行脚本
        4. 继续解析 HTML
    -->
</head>
<body>
    <div>内容</div>
    <script src="another.js"></script>
    <!-- body 底部的脚本：减少阻塞时间 -->
</body>
</html>
```

### 内联脚本 vs 外部脚本

```html
<!-- 内联脚本：立即执行 -->
<script>
    console.log('内联脚本');
    // 优点：无需额外请求
    // 缺点：无法缓存，HTML 文件变大
</script>

<!-- 外部脚本：需要下载 -->
<script src="external.js"></script>
<!-- 优点：可缓存、可复用、代码分离 -->
<!-- 缺点：额外 HTTP 请求 -->
```

## 2. defer 属性

### 工作原理

```html
<script defer src="script.js"></script>
<!-- 
    defer 的工作流程：
    1. 异步下载脚本（不阻塞 HTML 解析）
    2. 等待 HTML 解析完成
    3. DOMContentLoaded 之前按顺序执行
-->
```

### defer 的特点

```html
<!DOCTYPE html>
<html>
<head>
    <script defer src="a.js"></script>
    <script defer src="b.js"></script>
    <script defer src="c.js"></script>
    <!-- 
        即使 a.js 最先开始下载，
        它们也会按照声明顺序执行：a.js → b.js → c.js
        执行时机：HTML 解析完成后，DOMContentLoaded 之前
    -->
</head>
<body>
    <div>内容</div>
</body>
</html>

<script>
document.addEventListener('DOMContentLoaded', function() {
    console.log('DOM 加载完成');
    // defer 脚本在此之前已执行
});
</script>
```

### defer 的使用场景

```html
<!-- 
    defer 适用于：
    1. 脚本需要访问完整的 DOM
    2. 多个脚本有依赖关系，需要按顺序执行
    3. 想要尽早开始下载，但不阻塞渲染
-->

<!-- 示例：jQuery 和依赖 jQuery 的插件 -->
<script defer src="jquery.js"></script>
<script defer src="jquery-plugin.js"></script>
<script defer src="main.js"></script>
<!-- 保证 jquery.js 先执行 -->
```

## 3. async 属性

### 工作原理

```html
<script async src="script.js"></script>
<!-- 
    async 的工作流程：
    1. 异步下载脚本（不阻塞 HTML 解析）
    2. 下载完成后立即执行（会暂停 HTML 解析）
    3. 执行顺序不保证
-->
```

### async 的特点

```html
<!DOCTYPE html>
<html>
<head>
    <script async src="a.js"></script>
    <script async src="b.js"></script>
    <script async src="c.js"></script>
    <!-- 
        哪个先下载完，哪个先执行
        执行顺序：不保证！可能是 b → a → c
        执行时机：下载完成后立即执行（可能在 DOMContentLoaded 之前或之后）
    -->
</head>
<body>
    <div>内容</div>
</body>
</html>
```

### async 的使用场景

```html
<!-- 
    async 适用于：
    1. 独立的第三方脚本（不依赖其他脚本）
    2. 不需要访问完整 DOM 的脚本
    3. 希望尽快执行，不在乎执行顺序
-->

<!-- 示例：第三方统计脚本 -->
<script async src="https://www.googletagmanager.com/gtag/js"></script>
<script async src="analytics.js"></script>

<!-- 示例：广告脚本 -->
<script async src="ads.js"></script>
```

## 4. defer vs async 对比

### 图解对比

```
┌──────────────────────────────────────────────────────────────────┐
│                        HTML 解析时间线                            │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│ 无属性（阻塞）：                                                  │
│ ──解析──[暂停]──下载──────────执行──[继续]──解析───────────────►  │
│                    └─ 阻塞 ─┘                                    │
│                                                                  │
│ defer：                                                          │
│ ──解析───────────────────────[完成]──执行──DOMContentLoaded───►  │
│         └─ 异步下载 ─┘              └─ 按顺序执行 ─┘             │
│                                                                  │
│ async：                                                          │
│ ──解析──────[暂停]──执行──解析──────[暂停]──执行──解析───────►   │
│       └─ 下载 ─┘     └─ 下载 ─┘                                   │
│       └─ 立即执行 ─┘   └─ 立即执行 ─┘（顺序不保证）               │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

### 详细对比表

| 特性 | 无属性 | defer | async |
|------|--------|-------|-------|
| 下载方式 | 同步阻塞 | 异步 | 异步 |
| 执行时机 | 立即 | DOMContentLoaded 前 | 下载完成后立即 |
| 执行顺序 | 声明顺序 | 声明顺序 | 不保证 |
| 阻塞渲染 | 是 | 否 | 短暂（执行时） |
| 访问 DOM | 可能不完整 | 完整 | 不保证 |
| 使用场景 | 简单脚本 | 依赖 DOM/顺序 | 独立脚本 |

### 代码示例

```html
<!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <title>脚本加载对比</title>
    
    <!-- 场景1：需要按顺序执行的依赖脚本 -->
    <script defer src="library.js"></script>
    <script defer src="plugin.js"></script>
    <script defer src="app.js"></script>
    
    <!-- 场景2：独立的第三方脚本 -->
    <script async src="analytics.js"></script>
    <script async src="ads.js"></script>
    
    <!-- 场景3：内联脚本配合 defer -->
    <script defer src="jquery.js"></script>
    <script>
        // 等待 defer 脚本执行完毕
        document.addEventListener('DOMContentLoaded', function() {
            // 这里可以使用 jQuery
            $('body').addClass('loaded');
        });
    </script>
</head>
<body>
    <div id="app">应用内容</div>
</body>
</html>
```

## 5. 动态脚本加载

### 创建 script 元素

```javascript
// 动态创建 script 标签
function loadScript(url, callback) {
    const script = document.createElement('script');
    script.src = url;
    
    // 监听加载完成
    script.onload = function() {
        console.log(url + ' 加载完成');
        if (callback) callback();
    };
    
    script.onerror = function() {
        console.error(url + ' 加载失败');
    };
    
    document.head.appendChild(script);
}

// 使用
loadScript('https://cdn.example.com/library.js', function() {
    console.log('脚本已加载，可以使用');
});
```

### 动态加载实现按需加载

```javascript
// 按需加载模块
const moduleCache = {};

function loadModule(name) {
    if (moduleCache[name]) {
        return Promise.resolve(moduleCache[name]);
    }
    
    return new Promise((resolve, reject) => {
        const script = document.createElement('script');
        script.src = `/modules/${name}.js`;
        
        script.onload = () => {
            moduleCache[name] = true;
            resolve();
        };
        
        script.onerror = reject;
        document.head.appendChild(script);
    });
}

// 使用
loadModule('chart').then(() => {
    // 模块加载完成
    createChart();
});
```

### 动态加载与 async

```javascript
// 动态添加的 script 默认是 async 的
const script = document.createElement('script');
script.src = 'script.js';
document.head.appendChild(script);
// 等同于 <script async src="script.js">

// 设置为同步加载（不推荐）
const syncScript = document.createElement('script');
syncScript.src = 'script.js';
syncScript.async = false; // 设置为 false 变成 defer 行为
document.head.appendChild(syncScript);
```

## 6. 预加载和预获取

### preload（预加载）

```html
<!-- preload：提前加载资源，但不执行 -->
<link rel="preload" href="important.js" as="script">

<!-- 
    preload 的作用：
    1. 提前开始下载，优先级高
    2. 不阻塞页面渲染
    3. 脚本不会立即执行
-->

<!-- 配合 preload 使用 -->
<link rel="preload" href="critical.js" as="script">
<script src="critical.js"></script>
<!-- 脚本会从缓存中加载 -->
```

### prefetch（预获取）

```html
<!-- prefetch：预获取未来可能需要的资源 -->
<link rel="prefetch" href="next-page.js" as="script">

<!-- 
    prefetch 的作用：
    1. 浏览器空闲时下载
    2. 优先级低
    3. 用于预加载下一页资源
-->

<!-- 示例：预加载下一页数据 -->
<link rel="prefetch" href="page2.js">
<link rel="prefetch" href="page2.css">
```

### modulepreload（ES6 模块预加载）

```html
<!-- 预加载 ES6 模块 -->
<link rel="modulepreload" href="module.js">

<script type="module" src="app.js"></script>
<!-- app.js 中 import './module.js' 会更快 -->
```

## 7. type="module"

### ES6 模块的加载

```html
<!-- ES6 模块默认是 defer 的 -->
<script type="module" src="module.js"></script>
<!-- 等同于 <script defer src="module.js"> -->

<!-- 内联模块 -->
<script type="module">
    import { greeting } from './utils.js';
    console.log(greeting);
</script>

<!-- 模块脚本的特点：
     1. 自动启用严格模式
     2. 自动 defer
     3. 支持 async（下载后立即执行）
     4. 只执行一次
     5. 跨域需要 CORS
-->

<!-- async 模块 -->
<script type="module" async src="module.js"></script>
<!-- 下载完成后立即执行 -->
```

### nomodule 属性

```html
<!-- 为不支持 ES6 模块的浏览器提供后备 -->
<script type="module" src="modern.js"></script>
<script nomodule src="legacy.js"></script>

<!-- 
    支持模块的浏览器：执行 modern.js，忽略 legacy.js
    不支持模块的浏览器：忽略 modern.js，执行 legacy.js
-->
```

## 8. 脚本加载最佳实践

### 关键渲染路径优化

```html
<!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <title>优化的页面</title>
    
    <!-- 1. 预加载关键资源 -->
    <link rel="preload" href="critical.js" as="script">
    
    <!-- 2. 内联关键 CSS -->
    <style>
        /* 首屏关键 CSS */
        .header { /* ... */ }
        .hero { /* ... */ }
    </style>
    
    <!-- 3. 异步加载非关键 CSS -->
    <link rel="preload" href="styles.css" as="style" onload="this.rel='stylesheet'">
    
    <!-- 4. defer 非关键 JS -->
    <script defer src="analytics.js"></script>
</head>
<body>
    <!-- 内容 -->
    
    <!-- 5. 关键脚本放在 body 底部或使用 defer -->
    <script defer src="app.js"></script>
</body>
</html>
```

### 性能建议

```html
<!-- ✅ 推荐 -->
<script defer src="app.js"></script>
<script type="module" src="app.js"></script>

<!-- ⚠️ 谨慎使用 -->
<script async src="third-party.js"></script>

<!-- ❌ 避免 -->
<script src="large-bundle.js"></script> <!-- 阻塞渲染 -->

<!-- ✅ 代码分割 -->
<script defer src="vendor.js"></script>
<script defer src="app.js"></script>
```

### 第三方脚本加载策略

```javascript
// 延迟加载第三方脚本
function loadThirdParty() {
    // 用户交互后加载
    document.addEventListener('click', function handler() {
        loadScript('third-party.js');
        document.removeEventListener('click', handler);
    }, { once: true });
}

// 页面加载完成后加载
window.addEventListener('load', function() {
    setTimeout(() => {
        loadScript('analytics.js');
    }, 1000);
});

// 使用 requestIdleCallback
if ('requestIdleCallback' in window) {
    requestIdleCallback(() => {
        loadScript('non-critical.js');
    });
} else {
    setTimeout(loadScript, 1, 'non-critical.js');
}
```

## 小结

| 属性 | 下载 | 执行时机 | 执行顺序 | 适用场景 |
|------|------|---------|---------|---------|
| **无属性** | 同步阻塞 | 立即 | 声明顺序 | 简单页面 |
| **defer** | 异步 | DOMContentLoaded 前 | 声明顺序 | 依赖 DOM 的脚本 |
| **async** | 异步 | 下载后立即 | 不保证 | 独立第三方脚本 |
| **type="module"** | 异步 | defer 行为 | 声明顺序 | ES6 模块 |
