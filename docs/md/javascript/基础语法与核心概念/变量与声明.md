# 变量与声明

> 变量是存储数据的容器。理解 JavaScript 中的变量声明机制是编程的第一步，也是避免很多 bug 的基础。

## 学习要点

- 🔑 理解三种声明方式的核心区别
- 📦 掌握作用域的概念和影响
- ⚠️ 了解变量提升机制，避免常见陷阱
- ✅ 形成良好的变量声明习惯

---

## 1. var、let、const 的区别与最佳实践

### 三种声明方式对比

JavaScript 提供了三种声明变量的方式：`var`、`let` 和 `const`。它们在**作用域**、**提升机制**、**重复声明**等方面有本质区别。

### var 关键字（传统方式，现已不推荐）

```javascript
// var 声明的变量具有函数作用域（function scope）
// 这意味着变量在声明它的函数内部任何地方都可访问
function example() {
    if (true) {
        var x = 1;  // x 在整个 example 函数内都可访问
    }
    console.log(x); // 1 - 可以访问！
}

// var 的问题：变量泄漏到预期的作用域外
for (var i = 0; i < 3; i++) {
    // ...
}
console.log(i); // 3 - i 泄漏到全局！
```

**var 的主要问题：**
1. **函数作用域**而非块级作用域，容易造成变量泄漏
2. **变量提升**可能导致意外行为
3. **允许重复声明**，可能覆盖已有变量
4. **全局变量污染**，var 声明的全局变量会成为 window 属性

### let 关键字（ES6 引入，推荐用于可变变量）

```javascript
// let 声明的变量具有块级作用域（block scope）
// 块级作用域由 {} 界定，如 if、for、while 等
function example() {
    if (true) {
        let x = 1;  // x 只在这个 if 块内可访问
    }
    console.log(x); // ReferenceError: x is not defined
}

// let 解决了循环变量问题
for (let i = 0; i < 3; i++) {
    setTimeout(() => console.log(i), 100);
}
// 输出：0, 1, 2（而不是 3, 3, 3）
```

**let 的优势：**
- 块级作用域，变量不会泄漏
- 不会提升，减少意外行为
- 不允许重复声明
- 全局声明不会污染 window

### const 关键字（ES6 引入，推荐用于不可变绑定）

```javascript
// const 声明的是"常量绑定"，而非"不可变值"
const PI = 3.14159;
PI = 3; // TypeError: Assignment to constant variable

// ⚠️ 重要：const 不等于"不可变对象"
// const 只保证绑定不变，不保证值不变
const obj = { name: 'John' };
obj.name = 'Jane'; // ✅ 允许！修改属性
obj.age = 30;      // ✅ 允许！添加属性
obj = {};          // ❌ TypeError！不能重新赋值

const arr = [1, 2, 3];
arr.push(4);       // ✅ 允许！修改数组内容
arr = [];          // ❌ TypeError！不能重新赋值

// 如果需要真正的不可变对象
const frozenObj = Object.freeze({ name: 'John' });
frozenObj.name = 'Jane'; // 静默失败（严格模式报错）
```

### 最佳实践总结

| 关键字 | 作用域 | 提升 | 重复声明 | 全局污染 | 使用场景 |
|--------|--------|------|----------|----------|----------|
| `var` | 函数作用域 | ✅ 是 | ✅ 允许 | ✅ 会 | ⛔ 避免使用 |
| `let` | 块级作用域 | ❌ 否 | ❌ 不允许 | ❌ 不会 | 🔄 需要重新赋值时 |
| `const` | 块级作用域 | ❌ 否 | ❌ 不允许 | ❌ 不会 | ⭐ 默认首选 |

**推荐规则：**
1. **默认使用 `const`** — 大多数变量不需要重新赋值
2. **需要重新赋值时用 `let`** — 如循环计数器、累加器
3. **避免使用 `var`** — 它的设计问题太多

## 2. 作用域（Scope）

> 作用域决定了变量的可见性和生命周期。理解作用域是理解闭包、避免变量冲突的基础。

### 作用域类型

```
全局作用域
    └── 函数作用域
            └── 块级作用域
                    └── 更内层的块级作用域
```

### 全局作用域

```javascript
// 在任何函数或块外部声明的变量具有全局作用域
var globalVar = 'global';  // 全局变量
let globalLet = 'global';  // 全局变量（但不会成为 window 属性）

// 浏览器环境中
console.log(window.globalVar);  // 'global'（var 声明的会成为 window 属性）
console.log(window.globalLet);  // undefined（let/const 不会）

// ⚠️ 避免使用全局变量！
// 原因：命名冲突、难以维护、内存泄漏风险
```

### 函数作用域

```javascript
// 在函数内部声明的变量（使用 var）具有函数作用域
function foo() {
    var functionVar = 'function scoped';
    console.log(functionVar); // 'function scoped'
}
console.log(functionVar); // ReferenceError - 函数外部无法访问

// 函数作用域是 JavaScript 传统的作用域单元
// 所有 var 变量在整个函数内都有效
function example() {
    console.log(x); // undefined（提升，但未赋值）
    if (true) {
        var x = 1;
    }
    console.log(x); // 1（函数作用域，不受 if 块限制）
}
```

### 块级作用域（ES6）

```javascript
// {} 创建一个块级作用域
// let 和 const 声明的变量只在该块内有效
{
    let blockLet = 'block scoped';
    const blockConst = 'block scoped';
    var blockVar = 'function scoped'; // var 不受块级限制！
}
console.log(blockVar);    // 'function scoped'（var 泄漏）
console.log(blockLet);    // ReferenceError
console.log(blockConst);  // ReferenceError

// 常见的块级作用域场景
// 1. if 语句
if (true) {
    let x = 1;
}
console.log(x); // ReferenceError

// 2. for 循环
for (let i = 0; i < 3; i++) {
    // i 只在循环体内有效
}
console.log(i); // ReferenceError

// 3. try-catch
try {
    throw new Error();
} catch (error) {
    // error 只在 catch 块内有效（ES2019 之前）
}
```

### 词法作用域（静态作用域）

```javascript
// JavaScript 使用词法作用域（在编写代码时确定，而非运行时）
const x = 10;

function outer() {
    const x = 20;
    
    function inner() {
        console.log(x); // 20（不是 10）
        // inner 函数在定义时就确定了它访问的是 outer 中的 x
    }
    
    inner();
}

outer(); // 输出 20
```

## 3. 变量提升（Hoisting）

> 提升是 JavaScript 引擎将变量和函数声明"移动"到作用域顶部的行为。理解提升有助于避免意外错误。

### var 的提升

```javascript
// 看起来是这样
console.log(hoistedVar); // undefined（不是 ReferenceError！）
var hoistedVar = 'value';

// 实际上 JavaScript 引擎这样处理：
// var hoistedVar;           ← 声明被提升到顶部
// console.log(hoistedVar);  // undefined
// hoistedVar = 'value';     ← 赋值留在原地

// ⚠️ 这就是著名的"变量提升"问题
// 变量存在但值为 undefined，而不是 ReferenceError
```

### 函数声明的提升

```javascript
// 函数声明会被完整提升（声明 + 定义）
hoistedFunction(); // 'Hello!' - 可以正常调用
function hoistedFunction() {
    console.log('Hello!');
}

// 函数表达式只会提升声明，不会提升定义
notHoisted(); // TypeError: notHoisted is not a function
var notHoisted = function() {
    console.log('Hello');
};

// 箭头函数也不会提升
arrowFunction(); // TypeError
var arrowFunction = () => console.log('Hello');
```

### let/const 的"不提升"

```javascript
// let/const 也有提升，但存在"暂时性死区"
console.log(x); // ReferenceError: Cannot access 'x' before initialization
let x = 1;

// 这比 var 的 undefined 更安全
// 因为明确告诉你"在初始化前不能访问"
```

## 4. 暂时性死区（Temporal Dead Zone, TDZ）

> TDZ 是从作用域开始到变量声明语句之间的区域，在此区域内访问变量会报错。

```javascript
// TDZ 示意图
{
    // ──────────────────────
    // ↑ TDZ 开始              ↑ TDZ 结束
    // 变量 x 已存在（提升），   变量 x 初始化
    // 但不能访问              可以正常访问
    // ──────────────────────
    
    console.log(x); // ❌ ReferenceError（在 TDZ 中）
    let x = 1;      // TDZ 结束
    console.log(x); // ✅ 1
}

// typeof 在 TDZ 中也会报错
console.log(typeof y); // ✅ 'undefined'（y 未声明）
{
    console.log(typeof z); // ❌ ReferenceError（z 在 TDZ 中）
    let z = 1;
}
```

### TDZ 的意义

1. **防止在声明前使用变量** — 避免难以追踪的 bug
2. **让代码更可预测** — 变量只在声明后才可用
3. **符合直觉** — "先声明，后使用"更符合人类思维

## 5. 常见陷阱与练习

### 陷阱 1：循环中的 var

```javascript
// ❌ 错误写法
for (var i = 0; i < 3; i++) {
    setTimeout(() => console.log(i), 100);
}
// 输出：3, 3, 3（不是 0, 1, 2）

// 原因：var 是函数作用域，三个回调函数共享同一个 i
// 当回调执行时，i 已经是 3 了

// ✅ 正确写法 1：使用 let
for (let j = 0; j < 3; j++) {
    setTimeout(() => console.log(j), 100);
}
// 输出：0, 1, 2

// ✅ 正确写法 2：使用 IIFE（立即执行函数）
for (var k = 0; k < 3; k++) {
    ((m) => {
        setTimeout(() => console.log(m), 100);
    })(k);
}
```

### 陷阱 2：全局变量污染

```javascript
// ❌ 在函数中忘记使用 var/let/const
function setName() {
    name = 'John'; // 意外创建了全局变量！
}
setName();
console.log(window.name); // 'John'（污染了全局）

// ✅ 始终使用 let 或 const
function setNameCorrect() {
    const name = 'John'; // 局部变量
}
```

### 练习题

```javascript
// 题目 1：预测输出
console.log(a);
var a = 1;
// 答案：undefined（变量提升）

// 题目 2：预测输出
let b = 2;
{
    console.log(b);
    let b = 3;
}
// 答案：ReferenceError（TDZ）

// 题目 3：以下代码有什么问题？
const person = { name: 'John' };
person = { name: 'Jane' };
// 答案：TypeError，const 不能重新赋值

// 题目 4：如何让下面的代码输出 0, 1, 2？
for (var i = 0; i < 3; i++) {
    setTimeout(() => console.log(i), 100);
}
// 答案：将 var 改为 let
```

## 小结

| 概念 | 核心要点 |
|------|----------|
| **var** | 函数作用域、会提升、可重复声明、避免使用 |
| **let** | 块级作用域、TDZ、不可重复声明、用于可变变量 |
| **const** | 块级作用域、TDZ、不可重复声明、默认首选 |
| **作用域** | 全局 > 函数 > 块级，词法作用域 |
| **提升** | 声明提升，赋值不提升 |
| **TDZ** | let/const 声明前的区域，访问会报错 |
