## 发布选项

### 三种方式

1. **内置声明**：在 npm 包中包含 `.d.ts` 文件
2. **@types 包**：发布到 DefinitelyTyped
3. **独立类型包**：发布单独的类型声明包

## 打包到 npm 包

### package.json 配置

```json
{
  "name": "my-library",
  "version": "1.0.0",
  "main": "dist/index.js",
  "types": "dist/index.d.ts",
  "files": [
    "dist"
  ],
  "scripts": {
    "build": "tsc",
    "prepublishOnly": "npm run build"
  }
}
```

### 目录结构

```
my-library/
├── package.json
├── tsconfig.json
├── src/
│   ├── index.ts
│   └── utils.ts
└── dist/
    ├── index.js
    ├── index.d.ts
    ├── utils.js
    └── utils.d.ts
```

### tsconfig.json

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "ESNext",
    "declaration": true,           // 生成 .d.ts
    "declarationMap": true,        // 生成 .d.ts.map
    "outDir": "./dist",
    "rootDir": "./src",
    "strict": true
  },
  "include": ["src"]
}
```

### 使用 exports 字段

```json
{
  "name": "my-library",
  "exports": {
    ".": {
      "import": "./dist/index.mjs",
      "require": "./dist/index.js",
      "types": "./dist/index.d.ts"
    },
    "./utils": {
      "import": "./dist/utils.mjs",
      "require": "./dist/utils.js",
      "types": "./dist/utils.d.ts"
    }
  }
}
```

## 发布到 @types

### 场景

- 库没有 TypeScript 类型
- 库作者不维护类型
- 需要社区维护类型

### 流程

1. Fork DefinitelyTyped 仓库
2. 创建类型包目录
3. 添加声明文件和测试
4. 提交 PR

### 目录结构

```
DefinitelyTyped/
└── types/
    └── my-library/
        ├── index.d.ts
        ├── my-library-tests.ts
        ├── tsconfig.json
        └── README.md
```

## 版本管理

### 语义化版本

```json
{
  "version": "1.2.3"
}
```

- **主版本号**：不兼容的 API 变更
- **次版本号**：向后兼容的功能新增
- **修订号**：向后兼容的 bug 修复

### 类型版本匹配

```json
// 库 package.json
{
  "name": "my-library",
  "version": "2.1.0"
}

// 类型包 package.json
{
  "name": "@types/my-library",
  "version": "2.1.0"  // 主版本号应匹配
}
```

### TypeScript 版本声明

```json
// package.json
{
  "peerDependencies": {
    "typescript": ">=4.0.0"
  }
}
```

## 最佳实践

### 1. 导出所有类型

```typescript
// index.ts
export { User, Client, Config } from './types';
export { UserService } from './service';
export type { Result, Options } from './types';
```

### 2. 保持同步

```typescript
// 确保 .ts 和 .d.ts 同步
// 使用 declaration: true 自动生成
```

### 3. 提供文档

```typescript
/**
 * 用户服务
 * @example
 * ```ts
 * const service = new UserService();
 * const user = await service.getUser('123');
 * ```
 */
export class UserService {
  // ...
}
```
