# 类型断言

类型断言（Type Assertion）是一种告诉 TypeScript 编译器"相信我，我知道这个值的确切类型"的方式。它不会在运行时做任何检查，只在编译时起作用。

## 概述

类型断言有两种语法形式，功能完全相同：

```typescript
// 1. as 语法（推荐）
let value: unknown = 'hello';
let length: number = (value as string).length;

// 2. 尖括号语法
let value2: unknown = 'hello';
let length2: number = (<string>value2).length;
```

### 为什么需要类型断言

```typescript
// 1. 类型推断不够精确
const element = document.querySelector('#myInput');
// element: Element | null

const input = document.querySelector('#myInput') as HTMLInputElement;
// input: HTMLInputElement | null

// 2. 从联合类型中缩小范围
let value: string | number = 'hello';
// value.length; // Error: number 没有 length
console.log((value as string).length); // OK

// 3. 处理 unknown 类型
function process(value: unknown) {
  // value.toFixed(2); // Error
  (value as number).toFixed(2); // OK
}
```

## as 语法

### 基本用法

```typescript
// 基本类型断言
let value: unknown = 'hello world';
let str: string = value as string;
console.log(str.toUpperCase()); // HELLO WORLD

// 对象类型断言
interface User {
  name: string;
  age: number;
}

const data: unknown = { name: 'Alice', age: 30 };
const user = data as User;
console.log(user.name); // Alice

// 联合类型断言
let id: string | number = '123';
let strId = id as string;
let numId = id as number;
```

### DOM 元素断言

```typescript
// HTML 元素断言
const input = document.querySelector('#username') as HTMLInputElement;
console.log(input.value);

const button = document.querySelector('button') as HTMLButtonElement;
button.disabled = true;

const canvas = document.querySelector('canvas') as HTMLCanvasElement;
const ctx = canvas.getContext('2d');

// 更安全的写法
const element = document.querySelector('#myElement');
if (element instanceof HTMLInputElement) {
  console.log(element.value); // 类型自动收窄
}

// 使用可选链
const maybeInput = document.querySelector('#input') as HTMLInputElement | null;
console.log(maybeInput?.value);
```

### API 响应断言

```typescript
interface User {
  id: number;
  name: string;
  email: string;
}

interface ApiResponse<T> {
  data: T;
  status: number;
  message: string;
}

// API 响应断言
async function fetchUser(id: number): Promise<User> {
  const response = await fetch(`/api/users/${id}`);
  const json = await response.json();
  return json as User;
}

// 更安全的做法
async function fetchUserSafe(id: number): Promise<User> {
  const response = await fetch(`/api/users/${id}`);
  const json = await response.json();

  if (
    typeof json === 'object' &&
    json !== null &&
    'id' in json &&
    'name' in json
  ) {
    return json as User;
  }

  throw new Error('Invalid response');
}
```

## 尖括号语法

### 基本用法

```typescript
// 尖括号语法（与 as 功能相同）
let value: unknown = 'hello';
let length: number = (<string>value).length;

// 在 JSX 中的限制
// ❌ 在 JSX 中不能使用尖括号语法
// const element = <string>value; // 与 JSX 语法冲突
```

### 适用场景

```typescript
// 在非 JSX 文件中可以使用
function getFirstItem<T>(items: T[]): T | undefined {
  return items.length > 0 ? items[0] : undefined;
}

// 使用尖括号指定泛型类型
const result = <string>getFirstItem(['a', 'b', 'c']);

// 或者使用 as 语法
const result2 = getFirstItem(['a', 'b', 'c']) as string;
```

### 与 JSX 的冲突

```typescript
// ❌ 在 .tsx 文件中，尖括号语法会与 JSX 冲突
// const element = <string>value; // Error

// ✅ 在 .tsx 文件中使用 as 语法
const element = value as string;

// ✅ 如果确实需要在 .tsx 中使用尖括号，可以这样写
const element = <string>(value);
```

## 非空断言

### 基本语法

非空断言 `!` 用于断言某个值不是 `null` 或 `undefined`。

```typescript
// 非空断言
interface User {
  name: string | null;
  email: string | undefined;
}

const user: User = {
  name: 'Alice',
  email: 'alice@example.com'
};

// 使用非空断言
const name: string = user.name!;      // 断言不为 null
const email: string = user.email!;    // 断言不为 undefined

// 函数返回值
function getElement(id: string): HTMLElement | null {
  return document.getElementById(id);
}

const element = getElement('myId')!; // 断言不为 null
element.textContent = 'Hello';
```

### 常见场景

```typescript
// 1. DOM 元素
const input = document.querySelector<HTMLInputElement>('#input')!;
input.value = 'Hello';

// 2. 可选属性
interface Config {
  timeout?: number;
}

const config: Config = { timeout: 1000 };
const timeout: number = config.timeout!; // 断言 timeout 存在

// 3. 数组查找
const users = [
  { id: 1, name: 'Alice' },
  { id: 2, name: 'Bob' }
];

const user = users.find(u => u.id === 1)!;
console.log(user.name);

// 4. Map 查找
const map = new Map<string, number>();
map.set('key', 42);
const value = map.get('key')!; // number
```

### 非空断言 vs 可选链

```typescript
interface User {
  profile?: {
    name?: string;
  };
}

const user: User = {};

// 非空断言：断言值存在，运行时可能出错
const name1 = user.profile!.name!; // 运行时可能崩溃

// 可选链：安全处理，返回 undefined
const name2 = user.profile?.name; // string | undefined

// 使用默认值
const name3 = user.profile?.name ?? 'Unknown'; // string
```

## const 断言

### 基本用法

`as const` 将类型推断为最具体的字面量类型。

```typescript
// 没有使用 as const
let x = 'hello';
// x: string

// 使用 as const
let y = 'hello' as const;
// y: 'hello'（字面量类型）

// 数字
let num = 42 as const;
// num: 42

// 布尔
let bool = true as const;
// bool: true
```

### 对象的 const 断言

```typescript
// 没有使用 as const
const config = {
  endpoint: 'https://api.example.com',
  timeout: 5000
};
// config: { endpoint: string; timeout: number }

// 使用 as const
const configConst = {
  endpoint: 'https://api.example.com',
  timeout: 5000
} as const;
// configConst: {
//   readonly endpoint: 'https://api.example.com';
//   readonly timeout: 5000;
// }

// 访问属性
config.endpoint = 'new url';  // OK
// configConst.endpoint = 'new url'; // Error: readonly
```

### 数组的 const 断言

```typescript
// 没有使用 as const
const arr = [1, 2, 3];
// arr: number[]

// 使用 as const
const arrConst = [1, 2, 3] as const;
// arrConst: readonly [1, 2, 3]（只读元组）

// 元组用途
function getPair(): readonly [string, number] {
  return ['hello', 42] as const;
}
```

### 实际应用

```typescript
// 1. 定义常量
const ROUTES = {
  HOME: '/',
  ABOUT: '/about',
  CONTACT: '/contact'
} as const;
// 类型: { readonly HOME: '/'; readonly ABOUT: '/about'; readonly CONTACT: '/contact' }

type Route = typeof ROUTES[keyof typeof ROUTES];
// '/main' | '/about' | '/contact'

// 2. 定义动作类型
const ActionTypes = {
  ADD_TODO: 'ADD_TODO',
  REMOVE_TODO: 'REMOVE_TODO',
  TOGGLE_TODO: 'TOGGLE_TODO'
} as const;

type ActionType = typeof ActionTypes[keyof typeof ActionTypes];
// 'ADD_TODO' | 'REMOVE_TODO' | 'TOGGLE_TODO'

// 3. 函数参数
function setMode(mode: 'dark' | 'light') {
  // ...
}

const mode = 'dark' as const;
setMode(mode); // OK

// 4. 排除类型扩展
function createConfig() {
  return {
    debug: false,
    logLevel: 'info'
  } as const;
}
```

## satisfies 运算符

### 基本用法（TypeScript 4.9+）

`satisfies` 用于检查值是否满足类型，同时保留值的精确类型。

```typescript
type Colors = 'red' | 'green' | 'blue';

// 使用 as：类型被扩展
const colors1 = { primary: 'red' } as Record<string, Colors>;
colors1.primary.toUpperCase(); // OK
colors1.secondary; // OK (类型为 Colors)

// 使用 satisfies：保留精确类型
const colors2 = { primary: 'red' } satisfies Record<string, Colors>;
colors2.primary.toUpperCase(); // OK
// colors2.secondary; // Error: 属性不存在
```

### 对比 as 和 satisfies

```typescript
interface Config {
  host: string;
  port: number;
}

// as: 类型被改变
const config1 = {
  host: 'localhost',
  port: 3000
} as Config;
// config1.host: string
// config1.port: number

// satisfies: 检查类型但保留字面量
const config2 = {
  host: 'localhost',
  port: 3000
} satisfies Config;
// config2.host: 'localhost'
// config2.port: 3000
```

### 实际应用

```typescript
// 1. 配置对象
type RouteConfig = {
  path: string;
  methods: ('GET' | 'POST' | 'PUT' | 'DELETE')[];
};

const routes = {
  users: {
    path: '/users',
    methods: ['GET', 'POST']
  },
  posts: {
    path: '/posts',
    methods: ['GET']
  }
} satisfies Record<string, RouteConfig>;

// 保留精确类型
routes.users.path;    // string
routes.users.methods; // ('GET' | 'POST')[]

// 2. 环境变量
type EnvVars = Record<string, string | undefined>;

const env = {
  API_URL: 'https://api.example.com',
  NODE_ENV: 'production'
} satisfies EnvVars;

env.API_URL;    // string
env.NODE_ENV;   // string
```

## 双重断言

### 基本概念

双重断言（Double Assertion）是通过 `any` 或 `unknown` 中间类型进行断言。

```typescript
// 不安全的双重断言
let value: string = 'hello';
let num = value as any as number; // 不安全，避免使用

// 正常断言会失败
// let num = value as number; // Error: string 不能断言为 number

// 通过 unknown 中转
let num2 = value as unknown as number; // 可以编译通过，但不安全
```

### 危险性

```typescript
// 运行时错误示例
function dangerous(): number {
  const str = 'hello';
  return str as unknown as number; // 编译通过
}

const result = dangerous();
// result.toFixed(); // 运行时错误：str.toFixed is not a function
```

### 替代方案

```typescript
// ❌ 避免：双重断言
const value = data as any as User;

// ✅ 更好：使用类型守卫
function isUser(data: unknown): data is User {
  return (
    typeof data === 'object' &&
    data !== null &&
    'id' in data &&
    'name' in data
  );
}

if (isUser(data)) {
  console.log(data.name);
}

// ✅ 或者使用验证库（如 zod）
import { z } from 'zod';

const UserSchema = z.object({
  id: z.number(),
  name: z.string()
});

const user = UserSchema.parse(data);
```

## 类型断言的限制

### 类型兼容性

```typescript
// ✅ 允许：子类型断言为父类型
class Animal {}
class Dog extends Animal {}

const dog = new Dog();
const animal = dog as Animal; // OK

// ✅ 允许：父类型断言为子类型
const animal2 = new Animal();
const dog2 = animal2 as Dog; // 编译通过（可能不安全）

// ❌ 不允许：不相关的类型
let str: string = 'hello';
// let num: number = str as number; // Error

// 需要通过 unknown 中转
let num: number = str as unknown as number; // 编译通过
```

### 类型断言 vs 类型转换

```typescript
// 类型断言：只影响编译时类型
const value = '123' as number; // 编译通过
// value.toFixed(); // 运行时错误

// 类型转换：实际转换值
const num = Number('123'); // 123
console.log(typeof num); // 'number'
```

## 实际应用场景

### 1. DOM 操作

```typescript
// 表单元素
const form = document.querySelector('form') as HTMLFormElement;
const formData = new FormData(form);

// 输入框
const emailInput = document.querySelector<HTMLInputElement>('input[type="email"]');
if (emailInput) {
  console.log(emailInput.value);
}

// Canvas
const canvas = document.getElementById('canvas') as HTMLCanvasElement;
const ctx = canvas.getContext('2d') as CanvasRenderingContext2D;
```

### 2. 事件处理

```typescript
// 键盘事件
document.addEventListener('keydown', (e: Event) => {
  const keyEvent = e as KeyboardEvent;
  console.log(keyEvent.key);
});

// 更好的方式
document.addEventListener('keydown', (e: KeyboardEvent) => {
  console.log(e.key);
});
```

### 3. JSON 解析

```typescript
interface User {
  id: number;
  name: string;
  email: string;
}

// API 响应
const response = await fetch('/api/user/1');
const user = await response.json() as User;

// localStorage
const stored = localStorage.getItem('user');
const savedUser = stored ? JSON.parse(stored) as User : null;
```

### 4. 泛型函数

```typescript
function getProperty<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}

const user = { name: 'Alice', age: 30 };
const name = getProperty(user, 'name');

// 处理 unknown 类型
function parseJSON<T>(json: string): T {
  return JSON.parse(json) as T;
}

const data = parseJSON<User>('{"id":1,"name":"Alice","email":"a@b.com"}');
```

## 最佳实践

### 优先使用类型守卫

```typescript
// ❌ 避免：过度使用断言
const value = data as User;
console.log(value.name); // 可能在运行时崩溃

// ✅ 更好：使用类型守卫
function isUser(data: unknown): data is User {
  return (
    typeof data === 'object' &&
    data !== null &&
    'id' in data &&
    typeof (data as User).id === 'number' &&
    'name' in data &&
    typeof (data as User).name === 'string'
  );
}

if (isUser(data)) {
  console.log(data.name); // 安全
}
```

### 避免滥用 any

```typescript
// ❌ 不好：滥用 any
const value = data as any;
value.whatever(); // 失去类型检查

// ✅ 更好：使用更精确的类型
const value = data as unknown as User;

// ✅ 最好：使用 unknown 和类型守卫
function process(data: unknown) {
  if (isUser(data)) {
    console.log(data.name);
  }
}
```

### 合理使用 const 断言

```typescript
// ✅ 定义常量时使用
const CONFIG = {
  API_URL: 'https://api.example.com',
  TIMEOUT: 5000
} as const;

// ✅ 创建元组时使用
const pair = ['hello', 42] as const;
// pair: readonly ['hello', 42]

// ❌ 不必要时不要使用
let dynamicValue = getValue() as const; // 可能过度限制
```

## 小结

| 语法 | 用途 | 安全性 |
|------|------|--------|
| `as Type` | 类型断言 | 需谨慎 |
| `<Type>` | 类型断言（旧语法） | 需谨慎 |
| `!` | 非空断言 | 需谨慎 |
| `as const` | 字面量类型 | 安全 |
| `satisfies Type` | 类型检查+保留精确类型 | 安全 |
