# 接口合并

## 概述

TypeScript 支持声明合并,相同名称的接口会自动合并为一个。

## 一、基本合并

```typescript
interface User {
  name: string
}

interface User {
  age: number
}

// 合并后
const user: User = {
  name: 'Alice',
  age: 25
}
```

## 二、非函数成员合并

```typescript
interface Document {
  title: string
}

interface Document {
  // title: number  // ❌ 编译错误: 类型必须一致
  title: string     // ✅ 相同类型
  author: string
}

const doc: Document = {
  title: 'My Document',
  author: 'Alice'
}
```

## 三、函数成员合并

```typescript
interface Album {
  listen(): void
}

interface Album {
  listen(): void  // ✅ 可以重复
  play(): void
}

const album: Album = {
  listen() {},
  play() {}
}
```

## 四、合并为函数重载

```typescript
interface Document {
  createElement(tag: 'div'): HTMLDivElement
  createElement(tag: 'span'): HTMLSpanElement
}

interface Document {
  createElement(tag: string): HTMLElement
}

// 合并后的顺序: 后声明的接口在前
// createElement(tag: string): HTMLElement
// createElement(tag: 'div'): HTMLDivElement
// createElement(tag: 'span'): HTMLSpanElement
```

## 五、扩展第三方类型

```typescript
// 扩展 Express Request
declare module 'express' {
  interface Request {
    user?: {
      id: string
      name: string
    }
  }
}

// 使用
import { Request } from 'express'

function handler(req: Request) {
  console.log(req.user?.name)
}
```

## 六、全局接口扩展

```typescript
// 扩展 Window
declare global {
  interface Window {
    myCustomAPI: {
      init(): void
      destroy(): void
    }
  }
}

// 使用
window.myCustomAPI.init()
```

## 七、最佳实践

### 1. 谨慎使用合并

```typescript
// ❌ 滥用合并可能导致混乱
interface User {
  name: string
}

// 在其他文件中
interface User {
  age: number
}

// ✅ 推荐: 在同一文件中组织
interface User {
  name: string
  age: number
}
```

### 2. 合理扩展第三方库

```typescript
// ✅ 好: 为第三方库添加类型
declare module 'lodash' {
  interface LoDashStatic {
    myCustomMethod(value: any): any
  }
}
```

## 八、总结

- 相同名称的接口自动合并
- 非函数成员类型必须一致
- 函数成员形成重载
- 用于扩展第三方类型
- 谨慎使用避免混乱
