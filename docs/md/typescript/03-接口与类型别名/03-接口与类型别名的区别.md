# 接口与类型别名的区别

## 概述

interface 和 type 都可以定义对象类型,但它们有一些重要区别。

## 一、核心区别对比

| 特性 | Interface | Type |
|------|-----------|------|
| 定义对象类型 | ✅ | ✅ |
| 定义联合类型 | ❌ | ✅ |
| 定义元组 | ❌ | ✅ |
| 定义基本类型别名 | ❌ | ✅ |
| 声明合并 | ✅ | ❌ |
| extends | ✅ | ❌ (使用 &) |
| implements | ✅ | ✅ |
| 类型运算 | ❌ | ✅ |
| 映射类型 | ❌ | ✅ |

## 二、详细对比

### 2.1 类型范围

```typescript
// ✅ type 可以表示任何类型
type ID = string | number
type Point = [number, number]
type Name = string
type Handler = (a: number) => void

// ❌ interface 只能表示对象类型
// interface ID = string | number  // 编译错误
```

### 2.2 扩展方式

```typescript
// interface 使用 extends
interface Animal {
  name: string
}

interface Dog extends Animal {
  breed: string
}

// type 使用交叉类型
type Animal2 = {
  name: string
}

type Dog2 = Animal2 & {
  breed: string
}
```

### 2.3 声明合并

```typescript
// ✅ interface 可以合并
interface User {
  name: string
}

interface User {
  age: number
}

// 合并后
const user: User = {
  name: 'Alice',
  age: 25
}

// ❌ type 不能合并
type Person = { name: string }
// type Person = { age: number }  // 编译错误
```

### 2.4 类实现

```typescript
interface UserInterface {
  name: string
  age: number
}

type UserType = {
  name: string
  age: number
}

// ✅ 两者都可以被实现
class Person1 implements UserInterface {
  name = 'Alice'
  age = 25
}

class Person2 implements UserType {
  name = 'Bob'
  age = 30
}
```

## 三、性能差异

### 3.1 类型检查

```typescript
// interface - 类型检查性能更好
interface User {
  name: string
  age: number
}

// type - 交叉类型可能稍慢
type User2 = {
  name: string
} & {
  age: number
}
```

### 3.2 编译速度

对于大型项目,interface 通常编译更快:
- Interface 是声明式的,易于缓存
- Type 可能需要递归解析

## 四、使用场景选择

### 4.1 使用 Interface 的场景

```typescript
// 1. 对象类型定义
interface User {
  id: number
  name: string
}

// 2. 需要声明合并
interface Window {
  customProperty: string
}

// 3. 类的契约
interface Repository<T> {
  findById(id: string): Promise<T>
  save(entity: T): Promise<void>
}

// 4. 库的类型定义
// 第三方库通常使用 interface,方便用户扩展
```

### 4.2 使用 Type 的场景

```typescript
// 1. 联合类型
type ID = string | number
type Status = 'pending' | 'completed'

// 2. 元组类型
type Point = [number, number]

// 3. 映射类型
type Readonly<T> = {
  readonly [P in keyof T]: T[P]
}

// 4. 条件类型
type NonNullable<T> = T extends null | undefined ? never : T

// 5. 复杂类型表达式
type DeepPartial<T> = {
  [P in keyof T]?: T[P] extends object ? DeepPartial<T[P]> : T[P]
}
```

## 五、最佳实践总结

### 5.1 官方建议

TypeScript 官方建议:
- **对象类型优先使用 interface**
- **其他情况使用 type**

### 5.2 实践原则

```typescript
// ✅ 推荐: 对象类型用 interface
interface User {
  name: string
  age: number
}

// ✅ 推荐: 联合类型用 type
type ID = string | number

// ✅ 推荐: 工具类型用 type
type ReadonlyUser = Readonly<User>

// ✅ 推荐: 元组用 type
type Point = [number, number]

// ✅ 推荐: 函数类型用 type
type Handler = (event: Event) => void
```

## 六、总结

### 选择 Interface 当:
- 定义对象类型
- 需要声明合并
- 需要扩展其他类型
- 定义类的契约
- 创建库的类型定义

### 选择 Type 当:
- 定义联合类型
- 定义元组
- 创建工具类型
- 需要类型运算
- 定义函数签名
- 创建复杂的类型表达式

### 核心区别记忆
- **Interface**: 对象类型,声明式,可合并
- **Type**: 通用类型,表达式,不可合并
