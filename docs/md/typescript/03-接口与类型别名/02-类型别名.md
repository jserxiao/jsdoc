# 类型别名

## 概述

类型别名(Type Alias)使用 `type` 关键字为类型创建新名称。

## 一、基本语法

### 1.1 基本类型别名

```typescript
type Name = string
type Age = number
type ID = string | number

let name: Name = 'Alice'
let age: Age = 25
let id: ID = '123'
```

### 1.2 对象类型别名

```typescript
type User = {
  id: number
  name: string
  email: string
}

const user: User = {
  id: 1,
  name: 'Alice',
  email: 'alice@example.com'
}
```

### 1.3 联合类型别名

```typescript
type ID = string | number
type Status = 'pending' | 'processing' | 'completed'
type Direction = 'up' | 'down' | 'left' | 'right'
```

### 1.4 交叉类型别名

```typescript
type Name = { name: string }
type Age = { age: number }
type Person = Name & Age

const person: Person = {
  name: 'Alice',
  age: 25
}
```

## 二、类型别名的用途

### 2.1 简化复杂类型

```typescript
// 复杂的函数类型
type RequestHandler = (
  req: {
    method: 'GET' | 'POST'
    path: string
    body?: any
  },
  res: {
    status: (code: number) => void
    json: (data: any) => void
  }
) => void

// 使用
const handler: RequestHandler = (req, res) => {
  res.status(200)
  res.json({ message: 'OK' })
}
```

### 2.2 泛型别名

```typescript
type Container<T> = {
  value: T
  metadata?: Record<string, any>
}

type Response<T> = {
  data: T
  status: number
  message: string
}

const userContainer: Container<User> = {
  value: { id: 1, name: 'Alice' }
}

const apiResponse: Response<User[]> = {
  data: [{ id: 1, name: 'Alice' }],
  status: 200,
  message: 'OK'
}
```

### 2.3 元组类型别名

```typescript
type Point = [number, number]
type Point3D = [number, number, number]
type KeyValuePair = [string, any]

const point: Point = [10, 20]
const point3D: Point3D = [10, 20, 30]
const pair: KeyValuePair = ['name', 'Alice']
```

### 2.4 映射类型别名

```typescript
type Readonly<T> = {
  readonly [P in keyof T]: T[P]
}

type Partial<T> = {
  [P in keyof T]?: T[P]
}

type Nullable<T> = {
  [P in keyof T]: T[P] | null
}

interface User {
  id: number
  name: string
}

type ReadonlyUser = Readonly<User>
type PartialUser = Partial<User>
type NullableUser = Nullable<User>
```

## 三、类型别名 vs 接口

### 3.1 相同点

```typescript
// 都可以定义对象类型
interface User1 {
  name: string
  age: number
}

type User2 = {
  name: string
  age: number
}

// 都可以被类实现
class Person implements User1 {
  name = 'Alice'
  age = 25
}

class Person2 implements User2 {
  name = 'Bob'
  age = 30
}
```

### 3.2 不同点

#### 1. 类型别名可以表示任何类型

```typescript
// ✅ type 可以
type ID = string | number
type Point = [number, number]
type Handler = (a: number) => void

// ❌ interface 不可以
// interface ID = string | number  // 编译错误
```

#### 2. 接口可以声明合并

```typescript
// ✅ interface 可以合并
interface User {
  name: string
}

interface User {
  age: number
}

const user: User = { name: 'Alice', age: 25 }

// ❌ type 不能合并
type Person = { name: string }
type Person = { age: number }  // 编译错误: 重复标识符
```

#### 3. 扩展方式不同

```typescript
// interface 使用 extends
interface Animal {
  name: string
}

interface Dog extends Animal {
  breed: string
}

// type 使用交叉类型
type Animal2 = {
  name: string
}

type Dog2 = Animal2 & {
  breed: string
}
```

## 四、类型运算

### 4.1 类型提取

```typescript
type User = {
  id: number
  name: string
  email: string
}

// 提取所有键
type UserKeys = keyof User  // 'id' | 'name' | 'email'

// 提取所有值类型
type UserValues = User[keyof User]  // number | string
```

### 4.2 条件类型

```typescript
type IsString<T> = T extends string ? true : false

type A = IsString<string>  // true
type B = IsString<number>  // false

// 实用类型
type NonNullable<T> = T extends null | undefined ? never : T
type ReturnType<T> = T extends (...args: any) => infer R ? R : never
```

### 4.3 类型推断

```typescript
const user = {
  id: 1,
  name: 'Alice',
  email: 'alice@example.com'
}

// 推断类型
type User = typeof user
// { id: number; name: string; email: string }

// 推断返回值类型
function getUser() {
  return { id: 1, name: 'Alice' }
}

type UserFromFunc = ReturnType<typeof getUser>
// { id: number; name: string }
```

## 五、实用类型别名

### 5.1 常用工具类型

```typescript
// 深度只读
type DeepReadonly<T> = {
  readonly [P in keyof T]: T[P] extends object
    ? DeepReadonly<T[P]>
    : T[P]
}

// 深度可选
type DeepPartial<T> = {
  [P in keyof T]?: T[P] extends object
    ? DeepPartial<T[P]>
    : T[P]
}

// 必需键
type RequiredKeys<T> = {
  [K in keyof T]-?: {} extends Pick<T, K> ? never : K
}[keyof T]

// 可选键
type OptionalKeys<T> = {
  [K in keyof T]-?: {} extends Pick<T, K> ? K : never
}[keyof T]
```

### 5.2 类型守卫

```typescript
type TypeName<T> = T extends string
  ? 'string'
  : T extends number
  ? 'number'
  : T extends boolean
  ? 'boolean'
  : T extends undefined
  ? 'undefined'
  : T extends Function
  ? 'function'
  : 'object'

type T0 = TypeName<string>     // 'string'
type T1 = TypeName<number>     // 'number'
type T2 = TypeName<boolean>    // 'boolean'
type T3 = TypeName<undefined>  // 'undefined'
type T4 = TypeName<() => void> // 'function'
type T5 = TypeName<object>     // 'object'
```

## 六、最佳实践

### 1. 对象类型优先使用 interface

```typescript
// ✅ 推荐
interface User {
  name: string
  age: number
}

// ❌ 不推荐
type UserType = {
  name: string
  age: number
}
```

### 2. 联合类型、元组使用 type

```typescript
// ✅ 好
type ID = string | number
type Point = [number, number]
```

### 3. 工具类型使用 type

```typescript
// ✅ 好
type ReadonlyUser = Readonly<User>
type PartialUser = Partial<User>
```

### 4. 复杂类型使用 type

```typescript
// ✅ 好
type ComplexFunction = (
  a: string,
  b: number,
  options: {
    flag?: boolean
    callback: () => void
  }
) => Promise<string | null>
```

## 七、总结

### 类型别名特点
- 可以表示任何类型
- 支持泛型
- 支持类型运算
- 不能声明合并

### 使用场景
- 联合类型
- 元组类型
- 复杂类型简化
- 工具类型
- 类型运算

### 选择建议
- **对象类型**: 优先使用 interface
- **联合/元组**: 使用 type
- **工具类型**: 使用 type
- **需要合并**: 使用 interface
