# 枚举类型

## 概述

枚举(Enum)是 TypeScript 中定义命名常量的方式,使代码更易读和维护。

## 一、数字枚举

### 1.1 基本用法

```typescript
enum Direction {
  Up,
  Down,
  Left,
  Right
}

let dir: Direction = Direction.Up
console.log(dir)  // 0

// 枚举值自动递增
console.log(Direction.Up)     // 0
console.log(Direction.Down)   // 1
console.log(Direction.Left)   // 2
console.log(Direction.Right)  // 3
```

### 1.2 自定义初始值

```typescript
enum Direction {
  Up = 1,
  Down,   // 2
  Left,   // 3
  Right   // 4
}

console.log(Direction.Up)    // 1
console.log(Direction.Down)  // 2
```

### 1.3 全部自定义

```typescript
enum Direction {
  Up = 1,
  Down = 2,
  Left = 3,
  Right = 4
}
```

### 1.4 反向映射

数字枚举支持反向映射:

```typescript
enum Direction {
  Up,
  Down
}

console.log(Direction.Up)        // 0
console.log(Direction[0])        // 'Up'
console.log(Direction['Up'])     // 0
```

编译后的 JavaScript:

```javascript
var Direction
(function (Direction) {
  Direction[(Direction['Up'] = 0)] = 'Up'
  Direction[(Direction['Down'] = 1)] = 'Down'
})(Direction || (Direction = {}))
```

## 二、字符串枚举

### 2.1 基本用法

```typescript
enum Direction {
  Up = 'UP',
  Down = 'DOWN',
  Left = 'LEFT',
  Right = 'RIGHT'
}

console.log(Direction.Up)  // 'UP'

// 字符串枚举不支持反向映射
// console.log(Direction['UP'])  // undefined
```

### 2.2 必须显式初始化

```typescript
enum Direction {
  Up = 'UP',
  // Down,  // ❌ 编译错误: 字符串枚举成员必须初始化
}
```

## 三、混合枚举

```typescript
enum BooleanLikeHeterogeneousEnum {

```

## 四、计算成员和常量成员

### 4.1 常量成员

```typescript
enum E {
  X,           // 常量
  Y = 1,       // 常量
  Z = 1 << 1,  // 常量 (常量表达式)
  A = X | Y,   // 常量
  B = 'hello'.length  // 计算成员
}
```

### 4.2 计算成员

```typescript
enum E {
  X = 'hello'.length,  // 计算成员
  Y = Math.random(),   // 计算成员
  Z = parseInt('10')   // 计算成员
}

console.log(E.X)  // 5
console.log(E.Y)  // 随机数
```

## 五、const 枚举

### 5.1 基本用法

```typescript
const enum Direction {
  Up,
  Down,
  Left,
  Right
}

let dir = Direction.Up  // 编译后: let dir = 0

// ❌ 不能访问计算成员
// console.log(Direction[0])  // 编译错误
```

### 5.2 编译结果对比

普通枚举:

```typescript
enum Direction {
  Up,
  Down
}

// 编译后:
var Direction
(function (Direction) {
  Direction[(Direction['Up'] = 0)] = 'Up'
  Direction[(Direction['Down'] = 1)] = 'Down'
})(Direction || (Direction = {}))
```

const 枚举:

```typescript
const enum Direction {
  Up,
  Down
}

let dir = Direction.Up

// 编译后:
let dir = 0 /* Direction.Up */
```

### 5.3 preserveConstEnums

```json
// tsconfig.json
{
  "compilerOptions": {
    "preserveConstEnums": true
  }
}
```

保留 const 枚举的编译输出。

## 六、枚举成员类型

### 6.1 作为类型使用

```typescript
enum Direction {
  Up,
  Down,
  Left,
  Right
}

function move(dir: Direction) {
  // ...
}

move(Direction.Up)  // ✅
// move(0)  // ❌ 编译错误 (严格模式)
```

### 6.2 联合枚举类型

```typescript
enum Direction {
  Up,
  Down,
  Left = 'LEFT',
  Right = 'RIGHT'
}

type AllDirections = Direction.Up | Direction.Down | Direction.Left | Direction.Right

function move(dir: AllDirections) {
  // ...
}
```

## 七、运行时枚举

### 7.1 枚举对象

```typescript
enum Direction {
  Up,
  Down,
  Left,
  Right
}

// 枚举是真实存在的对象
console.log(Object.keys(Direction))
// ['0', '1', '2', '3', 'Up', 'Down', 'Left', 'Right']

console.log(Object.values(Direction))
// ['Up', 'Down', 'Left', 'Right', 0, 1, 2, 3]
```

### 7.2 遍历枚举

```typescript
enum Direction {
  Up,
  Down,
  Left,
  Right
}

// 获取枚举值
for (const key in Direction) {
  if (isNaN(Number(key))) {
    console.log(key, Direction[key as keyof typeof Direction])
  }
}

// 或者
Object.keys(Direction)
  .filter(key => isNaN(Number(key)))
  .forEach(key => {
    console.log(key, Direction[key as keyof typeof Direction])
  })
```

## 八、实战示例

### 8.1 状态管理

```typescript
enum TaskStatus {
  Pending = 'PENDING',
  Processing = 'PROCESSING',
  Completed = 'COMPLETED',
  Failed = 'FAILED'
}

interface Task {
  id: string
  status: TaskStatus
}

function updateTask(task: Task, status: TaskStatus) {
  task.status = status
}

const task: Task = {
  id: 'task-1',
  status: TaskStatus.Pending
}

updateTask(task, TaskStatus.Processing)
```

### 8.2 权限系统

```typescript
enum Permission {
  Read = 1,
  Write = 2,
  Delete = 4,
  Admin = 7  // Read | Write | Delete
}

function hasPermission(userPerm: Permission, required: Permission): boolean {
  return (userPerm & required) === required
}

const userPermission = Permission.Read | Permission.Write

console.log(hasPermission(userPermission, Permission.Read))   // true
console.log(hasPermission(userPermission, Permission.Write))  // true
console.log(hasPermission(userPermission, Permission.Delete)) // false
```

### 8.3 HTTP 状态码

```typescript
enum HttpStatus {
  OK = 200,
  Created = 201,
  BadRequest = 400,
  Unauthorized = 401,
  Forbidden = 403,
  NotFound = 404,
  InternalServerError = 500
}

function handleResponse(status: HttpStatus) {
  switch (status) {
    case HttpStatus.OK:
      console.log('成功')
      break
    case HttpStatus.NotFound:
      console.log('资源未找到')
      break
    case HttpStatus.InternalServerError:
      console.log('服务器错误')
      break
  }
}
```

## 九、枚举 vs 字面量联合类型

### 9.1 对比

```typescript
// 枚举
enum Direction {
  Up = 'UP',
  Down = 'DOWN'
}

// 字面量联合类型
type DirectionType = 'UP' | 'DOWN'
```

| 特性 | 枚举 | 字面量联合类型 |
|------|------|--------------|
| 运行时存在 | ✅ | ❌ |
| 反向映射 | ✅ (数字枚举) | ❌ |
| 代码体积 | 较大 | 无 |
| 类型安全 | ✅ | ✅ |
| IDE 支持 | ✅ | ✅ |

### 9.2 选择建议

```typescript
// ✅ 使用字面量联合类型 - 推荐
type Status = 'pending' | 'processing' | 'completed'

// ✅ 使用枚举 - 需要运行时枚举或反向映射时
enum Direction {
  Up,
  Down,
  Left,
  Right
}

// 编译优化 - 使用 const 枚举
const enum Colors {
  Red = '#FF0000',
  Green = '#00FF00',
  Blue = '#0000FF'
}
```

## 十、最佳实践

### 1. 优先使用字符串枚举

```typescript
// ✅ 好: 字符串枚举更易调试
enum Status {
  Pending = 'PENDING',
  Completed = 'COMPLETED'
}

// ❌ 不推荐: 数字枚举不直观
enum Status {
  Pending,
  Completed
}
```

### 2. 使用 const 枚举优化性能

```typescript
// ✅ 好: 编译时内联,无运行时开销
const enum Direction {
  Up,
  Down
}

let dir = Direction.Up  // let dir = 0
```

### 3. 枚举命名使用单数

```typescript
// ✅ 好
enum Direction {
  Up,
  Down
}

// ❌ 不好
enum Directions {
  Up,
  Down
}
```

## 十一、总结

### 枚举类型要点
- **数字枚举**: 自动递增,支持反向映射
- **字符串枚举**: 更易读,需显式初始化
- **const 枚举**: 编译优化,无运行时开销
- **混合枚举**: 数字和字符串混合

### 使用场景
- 需要一组相关的命名常量
- 需要运行时枚举对象
- 需要反向映射(数字枚举)
- 位标志权限系统

### 选择建议
- 简单场景: 使用字面量联合类型
- 复杂场景: 使用枚举
- 性能优化: 使用 const 枚举
