# 数组与元组

## 概述

数组和元组是 TypeScript 中常用的集合类型,本章将详细介绍它们的定义、用法和类型安全特性。

## 一、数组类型

### 1.1 数组类型的两种定义方式

```typescript
// 方式一: 元素类型后面接 []
let numbers: number[] = [1, 2, 3, 4, 5]
let strings: string[] = ['a', 'b', 'c']
let booleans: boolean[] = [true, false]

// 方式二: 使用数组泛型 Array<元素类型>
let numbers2: Array<number> = [1, 2, 3]
let strings2: Array<string> = ['a', 'b', 'c']
```

### 1.2 数组元素的类型检查

```typescript
let numbers: number[] = [1, 2, 3]

// ✅ 添加正确类型的元素
numbers.push(4)
numbers[0] = 10

// ❌ 添加错误类型的元素
// numbers.push('string')  // 编译错误
// numbers[0] = 'string'   // 编译错误

// 数组方法也会进行类型检查
let doubled = numbers.map(n => n * 2)  // number[]
let strings = numbers.map(n => n.toString())  // string[]
```

### 1.3 多维数组

```typescript
// 二维数组
let matrix: number[][] = [
  [1, 2, 3],
  [4, 5, 6],
  [7, 8, 9]
]

// 三维数组
let cube: number[][][] = [
  [[1, 2], [3, 4]],
  [[5, 6], [7, 8]]
]

// 使用泛型语法
let matrix2: Array<Array<number>> = [
  [1, 2, 3],
  [4, 5, 6]
]
```

### 1.4 联合类型数组

```typescript
// 数组元素可以是多种类型
let mixed: (string | number)[] = [1, 'two', 3, 'four']

// 注意括号的位置
let arr1: string | number[]   // string 或 number[]
let arr2: (string | number)[] // (string | number)[]

// 示例
arr1 = 'hello'     // ✅
arr1 = [1, 2, 3]   // ✅
// arr1 = ['a', 'b']  // ❌

arr2 = [1, 'two', 3]  // ✅
```

### 1.5 对象数组

```typescript
interface User {
  id: number
  name: string
  email: string
}

let users: User[] = [
  { id: 1, name: 'Alice', email: 'alice@example.com' },
  { id: 2, name: 'Bob', email: 'bob@example.com' }
]

// 使用数组方法
const names = users.map(u => u.name)  // string[]
const user = users.find(u => u.id === 1)  // User | undefined
const sorted = users.sort((a, b) => a.name.localeCompare(b.name))
```

### 1.6 只读数组

```typescript
// 只读数组 - 不能修改
let readonlyArray: readonly number[] = [1, 2, 3]

// 或者使用 ReadonlyArray
let readonlyArray2: ReadonlyArray<number> = [1, 2, 3]

// ❌ 不能修改
// readonlyArray.push(4)      // 编译错误
// readonlyArray[0] = 10      // 编译错误
// readonlyArray.length = 0   // 编译错误

// ✅ 可以读取
let first = readonlyArray[0]
let length = readonlyArray.length

// 可以重新赋值整个数组
readonlyArray = [4, 5, 6]
```

## 二、元组类型

### 2.1 元组的定义

元组是一个**固定长度**和**固定类型顺序**的数组：

```typescript
// 定义元组
let tuple: [string, number] = ['hello', 42]

// ✅ 正确的类型和顺序
tuple = ['world', 100]

// ❌ 类型错误
// tuple = [42, 'hello']      // 顺序错误
// tuple = ['hello']          // 长度不足
// tuple = ['hello', 42, true] // 长度超出
```

### 2.2 访问元组元素

```typescript
let point: [number, number] = [10, 20]

// 访问元素 - 类型已知
let x: number = point[0]  // number
let y: number = point[1]  // number

// ❌ 越界访问
// let z = point[2]  // 编译错误

// 使用解构
let [xCoord, yCoord] = point
```

### 2.3 元组的操作

```typescript
let tuple: [string, number] = ['hello', 42]

// ✅ 支持数组方法
tuple.push('world')  // 允许,但不推荐
tuple.pop()

// ❌ 不推荐修改长度
// TypeScript 允许 push/pop,但会破坏类型安全

// 推荐: 重新赋值
tuple = ['new', 100]
```

### 2.4 可选元素

```typescript
// 可选的元组元素
let tuple: [string, number?] = ['hello']

// ✅ 可选元素可以省略
tuple = ['hello']
tuple = ['hello', 42]

// 访问可选元素
if (tuple[1] !== undefined) {
  let num: number = tuple[1]
}
```

### 2.5 剩余元素

```typescript
// 剩余元素
let tuple: [string, ...number[]] = ['hello', 1, 2, 3]

// 第一个元素是 string,其余都是 number
tuple = ['hello']
tuple = ['hello', 1]
tuple = ['hello', 1, 2, 3, 4, 5]

// 多种剩余类型
let tuple2: [number, ...string[], boolean]
tuple2 = [1, 'a', 'b', 'c', true]
```

### 2.6 只读元组

```typescript
// 只读元组
let readonlyTuple: readonly [string, number] = ['hello', 42]

// ❌ 不能修改
// readonlyTuple[0] = 'world'  // 编译错误
// readonlyTuple.push(100)     // 编译错误

// 类型推断中的只读元组
function getPoint() {
  return [10, 20] as const  // readonly [10, 20]
}

const point = getPoint()
// point[0] = 100  // 编译错误
```

## 三、数组 vs 元组

### 3.1 对比表格

| 特性 | 数组 | 元组 |
|------|------|------|
| 长度 | 可变 | 固定 |
| 元素类型 | 统一 | 可不同 |
| 类型顺序 | 不关心 | 严格 |
| 用途 | 同类数据集合 | 固定结构数据 |

### 3.2 使用场景

```typescript
// 使用数组: 同类数据集合
let scores: number[] = [90, 85, 92, 88]
let names: string[] = ['Alice', 'Bob', 'Charlie']

// 使用元组: 固定结构的数据
let coordinate: [number, number] = [10, 20]
let rgb: [number, number, number] = [255, 128, 0]
let httpResult: [number, string] = [200, 'OK']
let keyValue: [string, any] = ['name', 'Alice']
```

## 四、实战示例

### 4.1 函数返回多个值

```typescript
// 使用元组返回多个值
function getUserInfo(): [string, number, string] {
  return ['Alice', 25, 'alice@example.com']
}

const [name, age, email] = getUserInfo()

// 更好的方式: 使用对象
interface UserInfo {
  name: string
  age: number
  email: string
}

function getUserInfo2(): UserInfo {
  return {
    name: 'Alice',
    age: 25,
    email: 'alice@example.com'
  }
}
```

### 4.2 CSV 数据处理

```typescript
// CSV 数据行
type CSVRow = [string, string, number, boolean]

const data: CSVRow[] = [
  ['Alice', 'alice@example.com', 25, true],
  ['Bob', 'bob@example.com', 30, false]
]

// 处理 CSV 数据
data.forEach(([name, email, age, active]) => {
  console.log(`${name} (${age}): ${email} - ${active ? 'Active' : 'Inactive'}`)
})
```

### 4.3 React Hooks 返回值

```typescript
// useState 返回元组
const [count, setCount] = useState<number>(0)

// 自定义 Hook 返回元组
function useToggle(initial: boolean): [boolean, () => void] {
  const [value, setValue] = useState(initial)
  const toggle = () => setValue(v => !v)
  return [value, toggle]
}

// 使用
const [isOpen, toggleOpen] = useToggle(false)
```

### 4.4 函数参数元组

```typescript
// 参数类型
type Point = [number, number]

// 函数参数使用元组
function distance(a: Point, b: Point): number {
  return Math.sqrt(
    Math.pow(b[0] - a[0], 2) + Math.pow(b[1] - a[1], 2)
  )
}

const p1: Point = [0, 0]
const p2: Point = [3, 4]
console.log(distance(p1, p2))  // 5
```

### 4.5 异步迭代结果

```typescript
// 异步操作返回元组
async function fetchUser(id: string): Promise<[Error | null, User | null]> {
  try {
    const response = await fetch(`/api/users/${id}`)
    const user = await response.json()
    return [null, user]
  } catch (error) {
    return [error as Error, null]
  }
}

// 使用
const [error, user] = await fetchUser('123')
if (error) {
  console.error('Failed:', error.message)
} else {
  console.log('User:', user)
}
```

## 五、最佳实践

### 5.1 优先使用数组

```typescript
// ✅ 好: 同类数据使用数组
let scores: number[] = [90, 85, 92]

// ❌ 不好: 同类数据使用元组
let scoreTuple: [number, number, number] = [90, 85, 92]
```

### 5.2 元组用于固定结构

```typescript
// ✅ 好: 固定结构使用元组
type Coordinate = [number, number, number] // [x, y, z]
type Pair<T> = [T, T]

// ✅ 好: 明确的语义
type HTTPResponse = [number, string] // [statusCode, message]
```

### 5.3 避免元组越界

```typescript
let tuple: [string, number] = ['hello', 42]

// ❌ 避免: 越界操作
tuple.push('world')  // 虽然编译通过,但不推荐

// ✅ 推荐: 重新赋值
tuple = ['world', 100]
```

### 5.4 使用标签提升可读性

```typescript
// TypeScript 4.0+ 支持标签
type Point2D = [x: number, y: number]
type Point3D = [x: number, y: number, z: number]

const point: Point2D = [10, 20]
// 标签不影响类型,但提高可读性
```

## 六、常见错误

### 6.1 类型顺序错误

```typescript
let tuple: [string, number] = ['hello', 42]

// ❌ 类型顺序错误
// tuple = [42, 'hello']

// ✅ 正确顺序
tuple = ['world', 100]
```

### 6.2 数组与元组混淆

```typescript
// ❌ 错误: 数组和元组不兼容
let arr: number[] = [1, 2, 3]
let tuple: [number, number] = [1, 2]

// tuple = arr  // 编译错误
// arr = tuple  // 可以,但不推荐
```

### 6.3 忽略越界问题

```typescript
let tuple: [string, number] = ['hello', 42]

// TypeScript 允许 push/pop,但会破坏类型安全
tuple.push('extra')  // ⚠️ 长度变为 3,但类型仍然是 [string, number]
console.log(tuple[2])  // ❌ 编译错误: 没有 index 2
```

## 七、总结

### 数组要点
- 使用 `Type[]` 或 `Array<Type>` 定义
- 支持动态长度
- 所有元素类型相同
- 支持 readonly 修饰

### 元组要点
- 固定长度和类型顺序
- 每个位置可以有不同类型
- 支持可选元素和剩余元素
- TypeScript 4.0+ 支持标签

### 选择建议
- **数组**: 同类数据集合,长度可变
- **元组**: 固定结构,语义明确的组合数据
- **对象**: 需要命名属性时优先使用对象
