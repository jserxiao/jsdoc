# 对象类型

## 概述

对象类型是 TypeScript 中最常用的类型之一,用于描述对象的形状和结构。

## 一、对象类型基础

### 1.1 匿名对象类型

```typescript
// 内联对象类型
function greet(user: { name: string; age: number }): string {
  return `Hello, ${user.name}, you are ${user.age} years old`
}

greet({ name: 'Alice', age: 25 })
```

### 1.2 使用接口定义

```typescript
interface User {
  name: string
  age: number
  email: string
}

function greet(user: User): string {
  return `Hello, ${user.name}`
}
```

### 1.3 使用类型别名

```typescript
type User = {
  name: string
  age: number
  email: string
}

function greet(user: User): string {
  return `Hello, ${user.name}`
}
```

## 二、属性修饰符

### 2.1 可选属性

使用 `?` 标记可选属性:

```typescript
interface User {
  id: number
  name: string
  email?: string  // 可选
  age?: number    // 可选
}

// ✅ 可选属性可以省略
let user1: User = { id: 1, name: 'Alice' }
let user2: User = { id: 2, name: 'Bob', email: 'bob@example.com' }

// 访问可选属性
if (user1.email) {
  console.log(user1.email.toUpperCase())
}

// 使用可选链
console.log(user1.email?.toUpperCase())
```

### 2.2 只读属性

使用 `readonly` 标记只读属性:

```typescript
interface User {
  readonly id: number
  name: string
  readonly createdAt: Date
}

let user: User = {
  id: 1,
  name: 'Alice',
  createdAt: new Date()
}

// ✅ 可以修改 name
user.name = 'Bob'

// ❌ 不能修改 id 和 createdAt
// user.id = 2  // 编译错误
// user.createdAt = new Date()  // 编译错误
```

### 2.3 只读数组

```typescript
interface Config {
  readonly apiKeys: readonly string[]
}

let config: Config = {
  apiKeys: ['key1', 'key2']
}

// ❌ 不能修改数组
// config.apiKeys.push('key3')  // 编译错误
// config.apiKeys[0] = 'new'    // 编译错误
```

## 三、索引签名

### 3.1 字符串索引签名

```typescript
interface StringMap {
  [key: string]: string
}

let map: StringMap = {
  name: 'Alice',
  city: 'Beijing'
}

// 可以添加任意字符串键值对
map.country = 'China'
```

### 3.2 数字索引签名

```typescript
interface NumberArray {
  [index: number]: string
}

let arr: NumberArray = ['a', 'b', 'c']
console.log(arr[0])  // 'a'
```

### 3.3 混合索引签名

```typescript
interface MixedMap {
  [key: string]: string | number
  [index: number]: string  // 数字索引返回类型必须是字符串索引返回类型的子类型
  length: number  // 指定属性必须符合索引签名
}

let obj: MixedMap = {
  name: 'Alice',
  age: 25,
  length: 2
}
```

### 3.4 模板字面量索引

```typescript
// TypeScript 4.4+
interface Env {
  [key: `${string}_URL`]: string
  [key: `${string}_PORT`]: number
}

const env: Env = {
  API_URL: 'https://api.example.com',
  API_PORT: 3000,
  DB_URL: 'mongodb://localhost:27017'
}
```

## 四、对象类型推断

### 4.1 类型推断

```typescript
// 对象字面量推断
const user = {
  name: 'Alice',
  age: 25
}  // { name: string; age: number }

// 访问属性
console.log(user.name)  // string
console.log(user.age)   // number
```

### 4.2 as const 断言

```typescript
// 不使用 as const
let user1 = {
  name: 'Alice',
  age: 25
}  // { name: string; age: number }

// 使用 as const
const user2 = {
  name: 'Alice',
  age: 25
} as const  // { readonly name: 'Alice'; readonly age: 25 }

// user2.name = 'Bob'  // 编译错误
```

### 4.3 满足类型约束

```typescript
interface User {
  name: string
  age: number
}

// 使用 satisfies 确保类型正确,但保留具体类型
const user = {
  name: 'Alice',
  age: 25,
  email: 'alice@example.com'
} satisfies User  // 保留 email 属性

console.log(user.email)  // ✅ 可以访问
```

## 五、对象类型兼容性

### 5.1 结构化类型

```typescript
interface Point {
  x: number
  y: number
}

interface Point3D {
  x: number
  y: number
  z: number
}

let point2D: Point = { x: 0, y: 0 }
let point3D: Point3D = { x: 0, y: 0, z: 0 }

// ✅ Point3D 可以赋值给 Point (多余属性)
point2D = point3D

// ❌ Point 不能赋值给 Point3D (缺少 z 属性)
// point3D = point2D
```

### 5.2 多余属性检查

```typescript
interface User {
  name: string
  age: number
}

// ✅ 直接赋值对象变量
const user = { name: 'Alice', age: 25, email: 'alice@example.com' }
let u: User = user  // 允许多余属性

// ❌ 对象字面量会进行严格检查
// let u2: User = { name: 'Alice', age: 25, email: 'alice@example.com' }
// 编译错误: 对象字面量只能指定已知属性

// ✅ 使用类型断言绕过检查
let u3: User = { name: 'Alice', age: 25, email: 'alice@example.com' } as User
```

## 六、对象类型运算

### 6.1 联合类型

```typescript
interface User {
  type: 'user'
  name: string
}

interface Admin {
  type: 'admin'
  name: string
  permissions: string[]
}

type Person = User | Admin

function process(person: Person) {
  if (person.type === 'user') {
    console.log('User:', person.name)
  } else {
    console.log('Admin:', person.name, person.permissions)
  }
}
```

### 6.2 交叉类型

```typescript
interface Name {
  name: string
}

interface Age {
  age: number
}

type Person = Name & Age

const person: Person = {
  name: 'Alice',
  age: 25
}
```

### 6.3 类型运算

```typescript
interface User {
  id: number
  name: string
  email: string
}

// 部分属性
type PartialUser = Partial<User>

// 必需属性
type RequiredUser = Required<User>

// 只读属性
type ReadonlyUser = Readonly<User>

// 选取属性
type UserName = Pick<User, 'name'>

// 忽略属性
type UserWithoutEmail = Omit<User, 'email'>
```

## 七、对象类型工具

### 7.1 keyof 操作符

```typescript
interface User {
  id: number
  name: string
  email: string
}

type UserKeys = keyof User  // 'id' | 'name' | 'email'

function getProperty<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key]
}

const user: User = { id: 1, name: 'Alice', email: 'alice@example.com' }
const name = getProperty(user, 'name')  // string
```

### 7.2 typeof 操作符

```typescript
const user = {
  id: 1,
  name: 'Alice',
  email: 'alice@example.com'
}

type User = typeof user
// { id: number; name: string; email: string }
```

### 7.3 记录类型

```typescript
type Role = 'admin' | 'user' | 'guest'

const permissions: Record<Role, string[]> = {
  admin: ['read', 'write', 'delete'],
  user: ['read', 'write'],
  guest: ['read']
}
```

## 八、实战示例

### 8.1 API 响应类型

```typescript
interface APIResponse<T> {
  success: boolean
  data: T
  message?: string
  timestamp: number
}

interface User {
  id: number
  name: string
  email: string
}

async function fetchUser(id: number): Promise<APIResponse<User>> {
  const response = await fetch(`/api/users/${id}`)
  const data = await response.json()
  return {
    success: true,
    data,
    timestamp: Date.now()
  }
}
```

### 8.2 表单数据类型

```typescript
interface FormField<T> {
  value: T
  error?: string
  touched: boolean
}

type LoginForm = {
  username: FormField<string>
  password: FormField<string>
  rememberMe: FormField<boolean>
}

const loginForm: LoginForm = {
  username: { value: '', touched: false },
  password: { value: '', touched: false },
  rememberMe: { value: false, touched: false }
}
```

## 九、最佳实践

### 1. 优先使用 interface

```typescript
// ✅ 好: 对象类型使用 interface
interface User {
  name: string
  age: number
}

// ❌ 不推荐: 简单对象类型使用 type
type UserType = {
  name: string
  age: number
}
```

### 2. 明确可选属性

```typescript
// ✅ 好: 明确标记可选属性
interface User {
  name: string
  email?: string
}

// ❌ 不好: 使用联合类型 undefined
interface UserBad {
  name: string
  email: string | undefined
}
```

### 3. 使用 readonly 保护数据

```typescript
// ✅ 好: 不可变数据使用 readonly
interface Point {
  readonly x: number
  readonly y: number
}
```

### 4. 避免索引签名过于宽泛

```typescript
// ❌ 不好: 过于宽泛
interface BadMap {
  [key: string]: any
}

// ✅ 好: 限制值类型
interface GoodMap {
  [key: string]: string | number
}
```

## 十、总结

### 核心要点
1. 使用 `interface` 或 `type` 定义对象类型
2. 可选属性用 `?` 标记
3. 只读属性用 `readonly` 标记
4. 索引签名支持动态属性名
5. TypeScript 使用结构化类型系统

### 关键概念
- **可选属性**: 可以省略的属性
- **只读属性**: 不可修改的属性
- **索引签名**: 动态属性名
- **结构化类型**: 基于结构的类型兼容
