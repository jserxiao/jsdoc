# 可选参数与默认参数

## 概述

TypeScript 支持可选参数和默认参数,使函数调用更灵活。

## 一、可选参数

```typescript
function greet(name: string, greeting?: string): string {
  return `${greeting || 'Hello'}, ${name}`
}

greet('Alice')                // 'Hello, Alice'
greet('Alice', 'Hi')         // 'Hi, Alice'
```

### 可选参数必须在最后

```typescript
// ✅ 正确
function buildName(firstName: string, lastName?: string): string {
  return lastName ? `${firstName} ${lastName}` : firstName
}

// ❌ 错误
// function buildName(firstName?: string, lastName: string): string
```

## 二、默认参数

```typescript
function greet(name: string, greeting: string = 'Hello'): string {
  return `${greeting}, ${name}`
}

greet('Alice')                // 'Hello, Alice'
greet('Alice', 'Hi')         // 'Hi, Alice'
greet('Alice', undefined)    // 'Hello, Alice'
```

### 默认参数可以在任意位置

```typescript
function buildName(firstName: string, lastName: string = 'Smith'): string {
  return `${firstName} ${lastName}`
}

buildName('Alice')            // 'Alice Smith'
buildName('Alice', 'Johnson') // 'Alice Johnson'

function buildName2(firstName: string = 'John', lastName: string): string {
  return `${firstName} ${lastName}`
}

buildName2(undefined, 'Doe')  // 'John Doe'
```

## 三、可选 vs 默认参数

```typescript
// 可选参数
function foo1(a?: number) {
  return a  // number | undefined
}

// 默认参数
function foo2(a: number = 10) {
  return a  // number
}
```

## 四、解构参数

```typescript
interface Config {
  url: string
  method?: 'GET' | 'POST'
  timeout?: number
}

function request({ url, method = 'GET', timeout = 5000 }: Config) {
  console.log(`${method} ${url}, timeout: ${timeout}`)
}

request({ url: '/api/users' })
request({ url: '/api/users', method: 'POST' })
```

## 五、实战示例

```typescript
// API 客户端
class APIClient {
  constructor(private baseURL: string) {}

  async get<T>(path: string, options?: {
    params?: Record<string, any>
    headers?: Record<string, string>
  }): Promise<T> {
    const url = new URL(path, this.baseURL)

    if (options?.params) {
      Object.entries(options.params).forEach(([key, value]) => {
        url.searchParams.append(key, String(value))
      })
    }

    const response = await fetch(url, {
      method: 'GET',
      headers: options?.headers
    })

    return response.json()
  }
}

const client = new APIClient('https://api.example.com')
const users = await client.get<User[]>('/users')
const user = await client.get<User>('/users/1', {
  headers: { 'Authorization': 'Bearer token' }
})
```

## 六、最佳实践

### 1. 优先使用默认参数

```typescript
// ✅ 好: 默认参数提供明确的值
function config(options: { timeout?: number } = { timeout: 5000 }) {
  return options
}

// ❌ 不好: 可选参数需要处理 undefined
function config2(options?: { timeout?: number }) {
  return {
    timeout: options?.timeout ?? 5000
  }
}
```

### 2. 合理选择位置

```typescript
// ✅ 好: 可选参数在最后
function search(query: string, limit?: number, offset?: number) {}

// ✅ 好: 默认参数可以在前
function search2(query: string, limit: number = 10, offset: number = 0) {}
```

## 七、总结

- 可选参数用 `?` 标记
- 默认参数用 `= value` 设置
- 可选参数必须在最后
- 默认参数可以在任意位置
- 默认参数提供更好的类型推断
