# 命名空间

命名空间（Namespace）是 TypeScript 早期用于组织代码和避免全局污染的机制。虽然现代 TypeScript 开发更推荐使用 ES 模块，但命名空间在特定场景下仍然有用。

## 概述

### 什么是命名空间

命名空间是一种将相关代码组织在一起的方式，它可以避免变量名冲突，并提供逻辑分组。

```typescript
// 没有命名空间：全局污染
function log() {}
function error() {}
const PI = 3.14;

// 使用命名空间：组织代码
namespace Utils {
  export function log() {}
  export function error() {}
  export const PI = 3.14;
}
```

### 命名空间 vs 模块

| 特性 | 命名空间 | ES 模块 |
|------|----------|---------|
| 定义方式 | `namespace` | `export/import` |
| 文件关系 | 跨文件合并 | 每个文件一个模块 |
| 加载方式 | 编译后通过 IIFE | 需要 module loader |
| 推荐程度 | 传统/特定场景 | 现代 TypeScript 推荐 |

## 基本语法

### 定义命名空间

```typescript
namespace Utils {
  // 使用 export 暴露成员
  export function log(message: string): void {
    console.log(`[LOG]: ${message}`);
  }

  export function error(message: string): void {
    console.error(`[ERROR]: ${message}`);
  }

  export const PI = 3.14159;

  // 不使用 export 的成员是私有的
  function privateHelper(): string {
    return 'private';
  }
}

// 使用命名空间成员
Utils.log('Hello');
Utils.error('Something went wrong');
console.log(Utils.PI);
// Utils.privateHelper(); // Error: 私有成员无法访问
```

### 命名空间的编译结果

```typescript
// 源码
namespace Utils {
  export function log(message: string) {
    console.log(message);
  }
}

// 编译后（ES5）
var Utils;
(function (Utils) {
  function log(message) {
    console.log(message);
  }
  Utils.log = log;
})(Utils || (Utils = {}));
```

### 类型定义与值

```typescript
namespace Container {
  // 值
  export const value = 42;

  // 类型
  export interface User {
    id: number;
    name: string;
  }

  // 类（既是类型也是值）
  export class Person {
    constructor(public name: string) {}
  }
}

// 作为类型使用
const user: Container.User = { id: 1, name: 'Alice' };

// 作为值使用
console.log(Container.value);

// 类可以同时作为类型和值
const person: Container.Person = new Container.Person('Bob');
```

## 命名空间分割

命名空间可以在多个文件中定义，TypeScript 会自动合并它们。

### 跨文件定义

```typescript
// utils/logging.ts
namespace Utils {
  export function log(message: string): void {
    console.log(`[LOG]: ${message}`);
  }

  export function error(message: string): void {
    console.error(`[ERROR]: ${message}`);
  }
}

// utils/validation.ts
namespace Utils {
  export function isString(value: unknown): value is string {
    return typeof value === 'string';
  }

  export function isNumber(value: unknown): value is number {
    return typeof value === 'number';
  }
}

// 使用时需要引入所有文件
/// <reference path="utils/logging.ts" />
/// <reference path="utils/validation.ts" />

Utils.log('Hello');
Utils.isString('test'); // true
```

### 三斜线指令

```typescript
// 使用三斜线指令声明依赖关系
/// <reference path="types.ts" />
/// <reference path="utils.ts" />

namespace App {
  export function init() {
    Utils.log('App initialized');
  }
}
```

### 声明合并

```typescript
// 文件1: interfaces.d.ts
namespace Models {
  export interface User {
    id: number;
    name: string;
  }
}

// 文件2: services.d.ts
namespace Models {
  export interface UserService {
    getUser(id: number): User;
  }
}

// 合并后
const user: Models.User = { id: 1, name: 'Alice' };
const service: Models.UserService = {
  getUser: (id) => ({ id, name: 'Test' })
};
```

## 命名空间嵌套

### 基本嵌套

```typescript
namespace App {
  export namespace Models {
    export interface User {
      id: number;
      name: string;
    }

    export interface Post {
      id: number;
      title: string;
      authorId: number;
    }
  }

  export namespace Services {
    export class UserService {
      getUser(id: number): Models.User {
        return { id, name: 'Alice' };
      }
    }

    export class PostService {
      getPosts(userId: number): Models.Post[] {
        return [];
      }
    }
  }

  export namespace Utils {
    export function formatName(user: Models.User): string {
      return `${user.name} (#${user.id})`;
    }
  }
}

// 使用嵌套命名空间
const user: App.Models.User = { id: 1, name: 'Alice' };
const service = new App.Services.UserService();
const formatted = App.Utils.formatName(user);
```

### 嵌套的另一种写法

```typescript
namespace App {
  // 先声明
  export interface User {
    id: number;
    name: string;
  }
}

namespace App.Models {
  // 同一个命名空间的另一个位置
  export interface Post {
    id: number;
    title: string;
  }
}
```

## 别名

### 导入命名空间成员

```typescript
namespace Company {
  export namespace Department {
    export class Employee {
      constructor(public name: string, public role: string) {}
    }

    export class Manager extends Employee {
      constructor(name: string) {
        super(name, 'Manager');
      }
    }
  }
}

// 使用别名简化访问
import Employee = Company.Department.Employee;
import Manager = Company.Department.Manager;

const emp = new Employee('Alice', 'Developer');
const mgr = new Manager('Bob');
```

namespace App {

## 命名空间与模块

### 模块中使用命名空间

```typescript
// utils.ts
export namespace StringUtils {
  export function upper(str: string): string {
    return str.toUpperCase();
  }

  export function lower(str: string): string {
    return str.toLowerCase();
  }

  export function capitalize(str: string): string {
    return str.charAt(0).toUpperCase() + str.slice(1).toLowerCase();
  }
}

export namespace NumberUtils {
  export function clamp(value: number, min: number, max: number): number {
    return Math.min(Math.max(value, min), max);
  }

  export function random(min: number, max: number): number {
    return Math.floor(Math.random() * (max - min + 1)) + min;
  }
}
```

### 导入模块中的命名空间

```typescript
// main.ts
import { StringUtils, NumberUtils } from './utils';

StringUtils.upper('hello'); // 'HELLO'
StringUtils.capitalize('hello'); // 'Hello'
NumberUtils.clamp(5, 0, 10); // 5
NumberUtils.random(1, 100); // 随机数
```

### 命名空间导出

```typescript
// api.ts
export namespace API {
  export interface RequestConfig {
    baseURL: string;
    timeout: number;
    headers?: Record<string, string>;
  }

  export interface Response<T> {
    data: T;
    status: number;
    message: string;
  }

  export class Client {
    constructor(private config: RequestConfig) {}

    async get<T>(url: string): Promise<Response<T>> {
      // 实现
      return { data: null as T, status: 200, message: 'OK' };
    }
  }
}

// 使用
import { API } from './api';

const client = new API.Client({
  baseURL: 'https://api.example.com',
  timeout: 5000
});

const response = await client.get<{ id: number }>('/users/1');
```

### 3. 插件系统

```typescript
// plugin-system.ts
export namespace Plugins {
  export interface Plugin {
    name: string;
    version: string;
    install(app: Application): void;
  }

  export interface Application {
    use(plugin: Plugin): Application;
    plugins: Map<string, Plugin>;
  }

  export class PluginRegistry {
    private plugins = new Map<string, Plugin>();

    register(plugin: Plugin): void {
      this.plugins.set(plugin.name, plugin);
    }

    get(name: string): Plugin | undefined {
      return this.plugins.get(name);
    }

    getAll(): Plugin[] {
      return Array.from(this.plugins.values());
    }
  }
}
```

## 最佳实践

### 推荐使用 ES 模块

```typescript
// ❌ 不推荐：过度使用命名空间
namespace App {
  export namespace Utils {
    export function log(msg: string) {}
  }
}

// ✅ 推荐：使用 ES 模块
// utils.ts
export function log(msg: string) {}

// app.ts
import { log } from './utils';
log('Hello');
```

### 何时使用命名空间

```typescript
// ✅ 适合使用命名空间的场景

// 1. 全局类型声明
declare namespace NodeJS {
  interface ProcessEnv {
    NODE_ENV: 'development' | 'production';
  }
}

// 2. 扩展第三方库类型
declare module 'express' {
  interface Request {
    user?: User;
  }
}

// 3. 类型分组（与模块组合使用）
export namespace API {
  export interface Request {}
  export interface Response {}
}

// ❌ 不适合的场景

// 1. 普通业务代码（应该用模块）
// 2. 需要代码分割的场景
// 3. 需要动态导入的场景
```

### 命名空间命名规范

```typescript
// ✅ 好：使用 PascalCase
namespace UserService {}
namespace Models.User {}

// ❌ 差：使用 camelCase 或其他
namespace userService {}
namespace models_user {}

// ✅ 好：语义清晰的名称
namespace ApiEndpoints {}
namespace ValidationRules {}

// ❌ 差：过于抽象
namespace Helpers {}
namespace Utils {}
```

## 小结

| 特性 | 说明 |
|------|------|
| 定义 | `namespace Name { ... }` |
| 导出 | `export` 关键字 |
| 分割 | 多文件定义自动合并 |
| 嵌套 | `namespace A { export namespace B {} }` |
| 别名 | `import Alias = Namespace.Member` |
| 推荐 | 现代 TypeScript 优先使用 ES 模块 |
