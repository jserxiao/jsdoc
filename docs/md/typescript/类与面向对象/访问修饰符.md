# 访问修饰符

TypeScript 提供了三种访问修饰符来控制类成员的可见性：`public`、`private` 和 `protected`，以及 `readonly` 修饰符。

## public 公有修饰符

`public` 修饰的成员可以在任何地方访问，这是默认的可见性。

```typescript
class Animal {
  public name: string; // 显式声明 public

  constructor(name: string) {
    this.name = name;
  }

  public move(distance: number) {
    console.log(`${this.name} moved ${distance}m.`);
  }
}

const animal = new Animal('Dog');
console.log(animal.name); // 'Dog' - 可以在外部访问
animal.move(10);          // 可以在外部调用

// 默认就是 public
class Person {
  name: string; // 等同于 public name: string

  constructor(name: string) {
    this.name = name;
  }
}
```

## private 私有修饰符

`private` 修饰的成员只能在类内部访问，不能在类外部或子类中访问。

```typescript
class Animal {
  private name: string;

  constructor(name: string) {
    this.name = name;
  }

  public getName(): string {
    return this.name; // ✅ 类内部可以访问
  }

  public setName(name: string): void {
    this.name = name; // ✅ 类内部可以访问
  }
}

const animal = new Animal('Dog');
// console.log(animal.name);  // ❌ Error: 'name' 是私有的
console.log(animal.getName()); // ✅ 'Dog'

// 子类也无法访问
class Dog extends Animal {
  constructor(name: string) {
    super(name);
  }

  public printName() {
    // console.log(this.name); // ❌ Error: 'name' 是私有的
    console.log(this.getName()); // ✅ 通过公共方法访问
  }
}
```

### 私有字段 (#语法)

ES2022 引入了私有字段语法，使用 `#` 前缀：

```typescript
class Person {
  #name: string; // 真正的私有字段

  constructor(name: string) {
    this.#name = name;
  }

  getName() {
    return this.#name;
  }
}

const person = new Person('Alice');
// console.log(person.#name); // ❌ 语法错误
// console.log(person['#name']); // ❌ 也无法访问
console.log(person.getName()); // ✅ 'Alice'

// 区别：
// - private 是 TypeScript 编译时的私有性
// - # 是 JavaScript 运行时的真正私有
```

## protected 受保护修饰符

`protected` 修饰的成员可以在类内部和子类中访问，但不能在类外部访问。

```typescript
class Animal {
  protected name: string;

  constructor(name: string) {
    this.name = name;
  }

  protected move(distance: number) {
    console.log(`${this.name} moved ${distance}m.`);
  }
}

class Dog extends Animal {
  constructor(name: string, public breed: string) {
    super(name);
  }

  public bark() {
    console.log(`${this.name} barks!`); // ✅ 子类可以访问 protected 成员
  }

  public run(distance: number) {
    this.move(distance); // ✅ 子类可以调用 protected 方法
  }
}

const dog = new Dog('Rex', 'German Shepherd');
// console.log(dog.name);  // ❌ Error: 'name' 是受保护的
dog.bark(); // ✅ 'Rex barks!'
```

### protected 构造函数

`protected` 构造函数意味着类不能被实例化，只能被继承：

```typescript
abstract class Base {
  protected constructor() {}

  protected log(message: string) {
    console.log(message);
  }
}

// const base = new Base(); // ❌ Error: 构造函数是受保护的

class Derived extends Base {
  constructor() {
    super(); // ✅ 子类可以调用
  }

  public doSomething() {
    this.log('Doing something');
  }
}

const derived = new Derived();
derived.doSomething(); // ✅ 'Doing something'
```

## readonly 只读修饰符

`readonly` 修饰的属性只能在声明时或构造函数中初始化，之后不能修改。

```typescript
class Person {
  readonly name: string;
  readonly age: number = 0; // 声明时初始化

  constructor(name: string, age: number) {
    this.name = name;   // ✅ 构造函数中赋值
    this.age = age;     // ✅ 构造函数中赋值
  }

  changeName(newName: string) {
    // this.name = newName; // ❌ Error: 只读属性不能修改
  }
}

const person = new Person('Alice', 30);
console.log(person.name); // 'Alice'
// person.name = 'Bob'; // ❌ Error: 只读属性不能修改
// person.age = 31;     // ❌ Error: 只读属性不能修改
```

### readonly 与其他修饰符组合

```typescript
class Config {
  public readonly apiUrl: string;       // 公有只读
  private readonly apiKey: string;      // 私有只读
  protected readonly secret: string;    // 受保护只读

  constructor(apiUrl: string, apiKey: string, secret: string) {
    this.apiUrl = apiUrl;
    this.apiKey = apiKey;
    this.secret = secret;
  }
}
```

## 参数属性

在构造函数参数上使用修饰符，可以同时声明和初始化属性。

```typescript
// 传统写法
class Animal1 {
  public name: string;
  private age: number;
  protected species: string;

  constructor(name: string, age: number, species: string) {
    this.name = name;
    this.age = age;
    this.species = species;
  }
}

// 参数属性简写
class Animal2 {
  constructor(
    public name: string,
    private age: number,
    protected species: string,
    readonly id: number
  ) {
    // 自动创建并初始化属性
  }
}

const animal = new Animal2('Dog', 5, 'Canine', 1);
console.log(animal.name); // 'Dog'
console.log(animal.id);   // 1
```

## 存取器

TypeScript 支持 getter 和 setter 来控制属性的访问和修改。

```typescript
class Person {
  private _name: string = '';
  private _age: number = 0;

  // Getter
  get name(): string {
    return this._name;
  }

  // Setter
  set name(value: string) {
    if (value.length < 2) {
      throw new Error('Name is too short');
    }
    this._name = value;
  }

  get age(): number {
    return this._age;
  }

  set age(value: number) {
    if (value < 0 || value > 150) {
      throw new Error('Invalid age');
    }
    this._age = value;
  }
}

const person = new Person();
person.name = 'Alice';
console.log(person.name); // 'Alice'

// person.name = 'A'; // ❌ Error: Name is too short
// person.age = -1;   // ❌ Error: Invalid age
```

### 只读存取器

只提供 getter 不提供 setter 可以创建只读属性：

```typescript
class Circle {
  constructor(private _radius: number) {}

  get radius(): number {
    return this._radius;
  }

  get area(): number {
    return Math.PI * this._radius ** 2;
  }

  get circumference(): number {
    return 2 * Math.PI * this._radius;
  }
}

const circle = new Circle(5);
console.log(circle.radius);      // 5
console.log(circle.area);        // 78.54...
console.log(circle.circumference); // 31.42...
// circle.area = 100; // ❌ Error: 没有 setter
```

| `private` | ✅ | ❌ | ❌ |

## 最佳实践

### 1. 默认使用 private

```typescript
// ✅ 推荐：默认私有，需要暴露时再改为 public
class User {
  private id: string;
  private email: string;
  public name: string; // 只有需要暴露的才是 public

  constructor(id: string, name: string, email: string) {
    this.id = id;
    this.name = name;
    this.email = email;
  }

  public getEmail(): string {
    return this.email;
  }
}
```

### 2. 使用 readonly 保护不可变数据

```typescript
class Config {
  readonly apiUrl: string;
  readonly timeout: number;

  constructor() {
    this.apiUrl = 'https://api.example.com';
    this.timeout = 5000;
  }
}
```

### 3. 使用存取器进行验证

```typescript
class User {
  private _email: string = '';

  get email(): string {
    return this._email;
  }

  set email(value: string) {
    if (!this.validateEmail(value)) {
      throw new Error('Invalid email format');
    }
    this._email = value;
  }

  private validateEmail(email: string): boolean {
    return /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(email);
  }
}
```
