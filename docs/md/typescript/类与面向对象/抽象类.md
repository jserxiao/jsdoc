# 抽象类

抽象类是用于定义其他类的基类，不能被直接实例化。抽象类可以包含抽象方法和具体实现。

## 抽象类基础

使用 `abstract` 关键字定义抽象类，抽象类不能被实例化。

```typescript
abstract class Animal {
  name: string;

  constructor(name: string) {
    this.name = name;
  }

  move(distance: number = 0) {
    console.log(`${this.name} moved ${distance}m.`);
  }
}

// const animal = new Animal('Unknown'); // ❌ Error: 无法创建抽象类的实例

class Dog extends Animal {
  constructor(name: string) {
    super(name);
  }

  bark() {
    console.log('Woof! Woof!');
  }
}

const dog = new Dog('Rex');
dog.move(10); // 'Rex moved 10m.'
dog.bark();   // 'Woof! Woof!'
```

## 抽象方法

抽象方法只有声明没有实现，必须在派生类中实现。

```typescript
abstract class Animal {
  constructor(public name: string) {}

  // 抽象方法 - 没有实现
  abstract makeSound(): void;

  // 具体方法 - 有实现
  move(distance: number = 0): void {
    console.log(`${this.name} moved ${distance}m.`);
  }
}

class Dog extends Animal {
  constructor(name: string) {
    super(name);
  }

  // 必须实现抽象方法
  makeSound(): void {
    console.log('Woof! Woof!');
  }
}

class Cat extends Animal {
  constructor(name: string) {
    super(name);
  }

  makeSound(): void {
    console.log('Meow!');
  }
}

const dog = new Dog('Rex');
dog.makeSound(); // 'Woof! Woof!'
dog.move(5);     // 'Rex moved 5m.'

const cat = new Cat('Whiskers');
cat.makeSound(); // 'Meow!'
```

### 抽象方法标记

```typescript
abstract class Shape {
  // 抽象方法必须在派生类中实现
  abstract getArea(): number;
  abstract getPerimeter(): number;

  // 具体方法
  describe(): string {
    return `Area: ${this.getArea()}, Perimeter: ${this.getPerimeter()}`;
  }
}

class Rectangle extends Shape {
  constructor(
    public width: number,
    public height: number
  ) {
    super();
  }

  getArea(): number {
    return this.width * this.height;
  }

  getPerimeter(): number {
    return 2 * (this.width + this.height);
  }
}

class Circle extends Shape {
  constructor(public radius: number) {
    super();
  }

  getArea(): number {
    return Math.PI * this.radius ** 2;
  }

  getPerimeter(): number {
    return 2 * Math.PI * this.radius;
  }
}
```

## 抽象属性

抽象类也可以定义抽象属性，必须在派生类中实现。

```typescript
abstract class Entity {
  // 抽象属性
  abstract id: string;
  abstract createdAt: Date;

  // 抽象只读属性
  abstract readonly type: string;

  // 具体方法
  getInfo(): string {
    return `ID: ${this.id}, Type: ${this.type}, Created: ${this.createdAt.toISOString()}`;
  }
}

class User extends Entity {
  id: string;
  createdAt: Date;
  readonly type = 'user';

  constructor(
    id: string,
    public name: string
  ) {
    super();
    this.id = id;
    this.createdAt = new Date();
  }
}

class Product extends Entity {
  id: string;
  createdAt: Date;
  readonly type = 'product';

  constructor(
    id: string,
    public name: string,
    public price: number
  ) {
    super();
    this.id = id;
    this.createdAt = new Date();
  }
}
```

## 抽象类与接口

### 区别

| 特性 | 抽象类 | 接口 |
|------|--------|------|
| 实例化 | ❌ 不能 | ❌ 不能 |
| 具体实现 | ✅ 可以 | ❌ 不能 |
| 多继承 | ❌ 单继承 | ✅ 多实现 |
| 构造函数 | ✅ 可以有 | ❌ 不能 |
| 访问修饰符 | ✅ 支持 | ❌ 不支持 |
| 默认实现 | ✅ 可以 | ❌ 不能 |

### 使用场景

```typescript
// 抽象类 - 用于有共同实现的场景
abstract class Vehicle {
  constructor(
    public brand: string,
    public model: string
  ) {}

  // 具体实现
  getInfo(): string {
    return `${this.brand} ${this.model}`;
  }

  // 抽象方法
  abstract start(): void;
  abstract stop(): void;
}

class Car extends Vehicle {
  start(): void {
    console.log(`${this.getInfo()} engine started`);
  }

  stop(): void {
    console.log(`${this.getInfo()} engine stopped`);
  }
}

// 接口 - 用于定义契约
interface Driveable {
  drive(): void;
}

interface Flyable {
  fly(): void;
}

// 可以同时继承抽象类和实现多个接口
class FlyingCar extends Vehicle implements Driveable, Flyable {
  drive(): void {
    console.log('Driving on road');
  }

  fly(): void {
    console.log('Flying in sky');
  }

  start(): void {
    console.log('Flying car started');
  }

  stop(): void {
    console.log('Flying car stopped');
  }
}
```

## 实际应用

### 1. 模板方法模式

```typescript
abstract class DataProcessor {
  // 模板方法 - 定义算法骨架
  public process(data: unknown): void {
    const validated = this.validate(data);
    const transformed = this.transform(validated);
    this.save(transformed);
    this.notify();
  }

  // 具体方法
  private notify(): void {
    console.log('Processing completed');
  }

  // 抽象方法 - 子类实现
  protected abstract validate(data: unknown): any;
  protected abstract transform(data: any): any;
  protected abstract save(data: any): void;
}

class UserProcessor extends DataProcessor {
  protected validate(data: unknown): { name: string; email: string } {
    if (typeof data !== 'object' || data === null) {
      throw new Error('Invalid data');
    }
    return data as { name: string; email: string };
  }

  protected transform(data: { name: string; email: string }) {
    return {
      ...data,
      name: data.name.toUpperCase(),
      email: data.email.toLowerCase()
    };
  }

  protected save(data: any): void {
    console.log('Saving user:', data);
  }
}
```

### 2. 工厂方法模式

```typescript
abstract class Dialog {
  // 工厂方法
  abstract createButton(): Button;

  render(): void {
    const button = this.createButton();
    button.onClick();
    button.render();
  }
}

interface Button {
  render(): void;
  onClick(): void;
}

class WindowsDialog extends Dialog {
  createButton(): Button {
    return new WindowsButton();
  }
}

class WebDialog extends Dialog {
  createButton(): Button {
    return new HTMLButton();
  }
}

class WindowsButton implements Button {
  render(): void {
    console.log('Rendering Windows button');
  }

  onClick(): void {
    console.log('Windows button clicked');
  }
}

class HTMLButton implements Button {
  render(): void {
    console.log('Rendering HTML button');
  }

  onClick(): void {
    console.log('HTML button clicked');
  }
}

// 使用
function createDialog(os: 'windows' | 'web'): Dialog {
  if (os === 'windows') {
    return new WindowsDialog();
  }
  return new WebDialog();
}

const dialog = createDialog('web');
dialog.render();
```

### 3. 策略模式

```typescript
abstract class PaymentStrategy {
  abstract pay(amount: number): void;
}

class CreditCardPayment extends PaymentStrategy {
  constructor(
    private cardNumber: string,
    private cvv: string
  ) {
    super();
  }

  pay(amount: number): void {
    console.log(`Paid ${amount} via Credit Card ${this.cardNumber}`);
  }
}

class PayPalPayment extends PaymentStrategy {
  constructor(private email: string) {
    super();
  }

  pay(amount: number): void {
    console.log(`Paid ${amount} via PayPal ${this.email}`);
  }
}

class ShoppingCart {
  private paymentStrategy: PaymentStrategy;

  setPaymentStrategy(strategy: PaymentStrategy): void {
    this.paymentStrategy = strategy;
  }

  checkout(amount: number): void {
    if (!this.paymentStrategy) {
      throw new Error('Payment strategy not set');
    }
    this.paymentStrategy.pay(amount);
  }
}

// 使用
const cart = new ShoppingCart();
cart.setPaymentStrategy(new CreditCardPayment('1234-5678', '123'));
cart.checkout(100);
```

## 最佳实践

### 1. 抽象类用于共享代码

```typescript
// ✅ 好：抽象类提供共享实现
abstract class BaseService {
  protected baseUrl: string;

  constructor(baseUrl: string) {
    this.baseUrl = baseUrl;
  }

  protected async request<T>(endpoint: string, options?: RequestInit): Promise<T> {
    const response = await fetch(`${this.baseUrl}${endpoint}`, options);
    return response.json();
  }

  abstract get(id: string): Promise<any>;
  abstract list(): Promise<any[]>;
}

class UserService extends BaseService {
  async get(id: string) {
    return this.request(`/users/${id}`);
  }

  async list() {
    return this.request('/users');
  }
}
```

### 2. 接口用于定义契约

```typescript
// ✅ 好：接口定义能力
interface Serializable {
  serialize(): string;
}

interface Cloneable<T> {
  clone(): T;
}

class User implements Serializable, Cloneable<User> {
  constructor(
    public id: string,
    public name: string
  ) {}

  serialize(): string {
    return JSON.stringify({ id: this.id, name: this.name });
  }

  clone(): User {
    return new User(this.id, this.name);
  }
}
```
