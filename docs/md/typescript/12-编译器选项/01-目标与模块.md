# 目标与模块

TypeScript 编译器选项决定了代码如何被编译成 JavaScript。`target` 和 `module` 是两个最核心的配置项，它们分别控制输出 JavaScript 的语法版本和模块系统。

## 编译目标 target

### 基本配置

```json
{
  "compilerOptions": {
    "target": "ES2020"
  }
}
```

### 可选值说明

| 值 | 说明 | 适用场景 |
|---|------|---------|
| `ES3` | ECMAScript 3（默认） | 兼容最旧浏览器 |
| `ES5` | ECMAScript 5 | IE11+ 兼容 |
| `ES6` / `ES2015` | ES6 | 现代浏览器 |
| `ES2016` | ES7 | 包含 `**` 运算符等 |
| `ES2017` | ES8 | 包含 `async/await` |
| `ES2018` | ES9 | 包含异步迭代等 |
| `ES2019` | ES10 | 包含 `flatMap` 等 |
| `ES2020` | ES11 | 包含可选链、空值合并 |
| `ES2021` | ES12 | 包含 `replaceAll` 等 |
| `ES2022` | ES13 | 包含类私有字段等 |
| `ESNext` | 最新特性 | 使用最新提案 |

### target 对编译的影响

```typescript
// 源代码
class User {
  constructor(public name: string) {}

  greet() {
    console.log(`Hello, ${this.name}`);
  }
}

async function fetchUser() {
  const response = await fetch('/api/user');
  return response.json();
}
```

**编译为 ES5**：

```javascript
"use strict";
var User = /** @class */ (function () {
    function User(name) {
        this.name = name;
    }
    User.prototype.greet = function () {
        console.log("Hello, " + this.name);
    };
    return User;
}());

function fetchUser() {
    return __awaiter(this, void 0, void 0, function () {
        var response;
        return __generator(this, function (_a) {
            switch (_a.label) {
                case 0: return [4 /*yield*/, fetch('/api/user')];
                case 1:
                    response = _a.sent();
                    return [2 /*return*/, response.json()];
            }
        });
    });
}
```

**编译为 ES2020**：

```javascript
"use strict";
class User {
    constructor(name) {
        this.name = name;
    }
    greet() {
        console.log(`Hello, ${this.name}`);
    }
}
async function fetchUser() {
    const response = await fetch('/api/user');
    return response.json();
}
```

### target 选择建议

```json
// 现代浏览器项目
{
  "compilerOptions": {
    "target": "ES2020"
  }
}

// 需要兼容旧浏览器
{
  "compilerOptions": {
    "target": "ES5"
  }
}

// Node.js 项目（Node 16+）
{
  "compilerOptions": {
    "target": "ES2022"
  }
}
```

## 模块系统 module

### 基本配置

```json
{
  "compilerOptions": {
    "module": "ESNext"
  }
}
```

### 可选值说明

| 值 | 说明 | 适用场景 |
|---|------|---------|
| `CommonJS` | Node.js 模块 | Node.js 项目 |
| `AMD` | 异步模块定义 | RequireJS |
| `System` | SystemJS 模块 | SystemJS |
| `UMD` | 通用模块定义 | 同时支持浏览器和 Node |
| `ES6` / `ES2015` | ES 模块 | ES6 目标 |
| `ES2020` | ES 模块 + 动态导入 | 支持 `import()` |
| `ES2022` | ES 模块 + top-level await | 最新 ES 模块 |
| `ESNext` | 最新 ES 模块 | 前端打包项目 |
| `Node16` / `NodeNext` | Node.js ESM | Node.js ESM 项目 |
| `None` | 不使用模块 | 全局脚本 |

### 不同模块系统的输出

```typescript
// 源代码
export const name = 'Alice';
export function greet() {
  console.log('Hello');
}
import { name, greet } from './module';
```

**CommonJS 输出**：

```javascript
"use strict";
Object.defineProperty(exports, "__esModule", { value: true });
exports.greet = exports.name = void 0;
exports.name = 'Alice';
function greet() {
    console.log('Hello');
}
exports.greet = greet;

const module_1 = require("./module");
console.log(module_1.name);
(0, module_1.greet)();
```

**ESNext 输出**：

```javascript
export const name = 'Alice';
export function greet() {
    console.log('Hello');
}
import { name, greet } from './module';
console.log(name);
greet();
```

### module 选择建议

```json
// 前端项目（使用打包工具）
{
  "compilerOptions": {
    "module": "ESNext",
    "moduleResolution": "bundler"
  }
}

// Node.js CommonJS 项目
{
  "compilerOptions": {
    "module": "CommonJS",
    "moduleResolution": "node"
  }
}

// Node.js ESM 项目
{
  "compilerOptions": {
    "module": "NodeNext",
    "moduleResolution": "NodeNext"
  }
}
```

## 模块解析 moduleResolution

### 基本配置

```json
{
  "compilerOptions": {
    "moduleResolution": "node"
  }
}
```

### 可选值说明

| 值 | 说明 |
|---|------|
| `classic` | TypeScript 旧版解析策略 |
| `node` | Node.js 风格解析 |
| `node16` / `nodenext` | Node.js ESM 解析 |
| `bundler` | 打包工具解析（Vite、Webpack 5+） |

### 解析策略对比

```typescript
// 导入语句
import { foo } from './module';
```

**classic 策略**：

```
/root/src/module.ts
/root/module.ts
/module.ts
```

**node 策略**：

```
/root/src/module.ts
/root/src/module.tsx
/root/src/module.d.ts
/root/src/module/package.json (types 字段)
/root/src/module/index.ts
/root/src/module/index.tsx
/root/src/module/index.d.ts
```

### 路径映射

```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["src/*"],
      "@components/*": ["src/components/*"],
      "@utils/*": ["src/utils/*"]
    }
  }
}
```

```typescript
// 使用路径别名
import { Button } from '@components/Button';
import { formatDate } from '@utils/date';
```

## 库配置 lib

### 基本配置

```json
{
  "compilerOptions": {
    "lib": ["ES2020", "DOM", "DOM.Iterable"]
  }
}
```

### 常用库类型

| 库名 | 说明 |
|-----|------|
| `ES5` | ES5 核心 API |
| `ES6` / `ES2015` | ES6 核心 API |
| `ES2016` - `ES2023` | 各年份新增 API |
| `ESNext` | 最新 ECMAScript 特性 |
| `DOM` | DOM API |
| `DOM.Iterable` | DOM 可迭代对象 |
| `ScriptHost` | Windows Script Host |
| `WebWorker` | Web Worker API |
| `WebWorker.ImportScripts` | Web Worker importScripts |

### 不同环境的 lib 配置

```json
// 浏览器项目
{
  "compilerOptions": {
    "lib": ["ES2020", "DOM", "DOM.Iterable"]
  }
}

// Node.js 项目（配合 @types/node）
{
  "compilerOptions": {
    "lib": ["ES2022"]
  }
}

// 纯 ES 项目（无 DOM）
{
  "compilerOptions": {
    "lib": ["ES2022"]
  }
}

// Web Worker
{
  "compilerOptions": {
    "lib": ["ES2020", "WebWorker"]
  }
}
```

### lib 与 target 的关系

```json
// target 会自动包含默认的 lib
// 显式指定 lib 会覆盖默认值

// ES2020 target 默认包含:
// - ES2020 核心 API

// 如果需要 DOM，需要手动添加:
{
  "compilerOptions": {
    "target": "ES2020",
    "lib": ["ES2020", "DOM", "DOM.Iterable"]
  }
}
```

## 输出配置

### 基本输出配置

```json
{
  "compilerOptions": {
    "outDir": "./dist",           // 输出目录
    "rootDir": "./src",           // 源码根目录
    "outFile": "./bundle.js",     // 单文件输出（仅 AMD/System）
    "declaration": true,          // 生成 .d.ts 声明文件
    "declarationMap": true,       // 生成声明文件的 source map
    "sourceMap": true,            // 生成 .js.map 文件
    "declarationDir": "./types"   // 声明文件输出目录
  }
}
```

### 输入文件配置

```json
{
  "compilerOptions": {
    "rootDir": "./src",
    "rootDirs": ["./src", "./generated"],
    "include": ["src/**/*", "tests/**/*"],
    "exclude": ["node_modules", "dist"]
  }
}
```

### 声明文件配置

```json
{
  "compilerOptions": {
    "declaration": true,
    "declarationMap": true,
    "declarationDir": "./dist/types",
    "emitDeclarationOnly": true    // 只输出声明文件
  }
}
```

### Source Map 配置

```json
{
  "compilerOptions": {
    "sourceMap": true,
    "inlineSourceMap": true,      // 内联 source map
    "inlineSources": true,        // 内联源代码
    "mapRoot": "./maps"           // source map 根目录
  }
}
```

## 最佳实践

### 选择合适的 target

```typescript
// ✅ 推荐：根据实际运行环境选择
// - 现代浏览器: ES2020+
// - Node.js 16+: ES2022
// - 需要兼容旧浏览器: ES5（配合 core-js）

// ❌ 避免：不必要地使用过高版本
{
  "target": "ESNext" // 可能导致兼容性问题
}
```

### 模块配置一致性

```json
// ✅ 推荐：target 和 module 协调
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "ESNext"
  }
}

// ❌ 避免：不合理的组合
{
  "compilerOptions": {
    "target": "ES5",
    "module": "ES6" // ES5 没有 ES 模块语法
  }
}
```

### 使用 skipLibCheck

```json
{
  "compilerOptions": {
    "skipLibCheck": true  // 跳过声明文件检查，加快编译
  }
}
```

### 配合 Babel 使用

```json
// TypeScript 只做类型检查
{
  "compilerOptions": {
    "target": "ESNext",
    "module": "ESNext",
    "noEmit": true,           // 不输出文件
    "isolatedModules": true   // 确保可被 Babel 处理
  }
}
```

## 小结

| 选项 | 说明 |
|-----|------|
| `target` | 编译输出的 JavaScript 版本 |
| `module` | 模块系统类型 |
| `moduleResolution` | 模块解析策略 |
| `lib` | 运行时 API 类型声明 |
| `outDir` | 输出目录 |
| `rootDir` | 源码根目录 |
| `declaration` | 生成声明文件 |
| `sourceMap` | 生成 source map |
