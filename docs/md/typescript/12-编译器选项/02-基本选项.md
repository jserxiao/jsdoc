## 输出选项

### outDir

指定输出目录：

```json
{
  "compilerOptions": {
    "outDir": "./dist"
  }
}
```

### rootDir

指定源码根目录：

```json
{
  "compilerOptions": {
    "rootDir": "./src",
    "outDir": "./dist"
  }
}
```

### outFile

将所有文件合并输出到单个文件（仅用于 AMD 和 System 模块）：

```json
{
  "compilerOptions": {
    "module": "AMD",
    "outFile": "./dist/bundle.js"
  }
}
```

### declaration

生成 `.d.ts` 声明文件：

```json
{
  "compilerOptions": {
    "declaration": true,
    "declarationDir": "./types"
  }
}
```

### sourceMap

生成 source map：

```json
{
  "compilerOptions": {
    "sourceMap": true,
    "inlineSourceMap": false,
    "inlineSources": false
  }
}
```

### noEmit

不输出文件（仅做类型检查）：

```json
{
  "compilerOptions": {
    "noEmit": true
  }
}
```

## 模块选项

### module

指定模块系统：

```json
{
  "compilerOptions": {
    "module": "ESNext"
  }
}
```

| 值 | 说明 |
|---|------|
| `CommonJS` | Node.js 模块系统 |
| `AMD` | RequireJS 模块系统 |
| `System` | SystemJS 模块系统 |
| `UMD` | 通用模块定义 |
| `ES6` / `ES2015` | ES6 模块 |
| `ES2020` | ES2020 模块（支持动态导入） |
| `ES2022` | ES2022 模块（支持 top-level await） |
| `ESNext` | 最新 ES 模块 |
| `NodeNext` | Node.js ESM |

### moduleResolution

指定模块解析策略：

```json
{
  "compilerOptions": {
    "moduleResolution": "node"
  }
}
```

| 值 | 说明 |
|---|------|
| `classic` | TypeScript 旧版策略 |
| `node` | Node.js 风格解析 |
| `node16` / `nodenext` | Node.js ESM 解析 |
| `bundler` | 打包工具解析（Vite、Webpack 5+） |

### baseUrl 和 paths

配置路径别名：

```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["src/*"],
      "@components/*": ["src/components/*"],
      "@utils/*": ["src/utils/*"]
    }
  }
}
```

### rootDirs

多个根目录：

```json
{
  "compilerOptions": {
    "rootDirs": ["src", "generated"]
  }
}
```

## 互操作性选项

### esModuleInterop

启用 ES 模块互操作性：

```json
{
  "compilerOptions": {
    "esModuleInterop": true
  }
}
```

```typescript
// esModuleInterop: false
import * as fs from 'fs';
fs.default(); // 需要使用 .default

// esModuleInterop: true
import fs from 'fs';
fs.readFile(); // 直接使用
```

### allowSyntheticDefaultImports

允许从没有默认导出的模块中导入：

```json
{
  "compilerOptions": {
    "allowSyntheticDefaultImports": true
  }
}
```

### resolveJsonModule

允许导入 JSON 文件：

```json
{
  "compilerOptions": {
    "resolveJsonModule": true
  }
}
```

```typescript
import config from './config.json';
```

### allowJs

允许编译 JavaScript 文件：

```json
{
  "compilerOptions": {
    "allowJs": true,
    "checkJs": false
  }
}
```

## 类型检查选项

### strict

启用所有严格类型检查选项：

```json
{
  "compilerOptions": {
    "strict": true
  }
}
```

等同于：

```json
{
  "compilerOptions": {
    "noImplicitAny": true,
    "strictNullChecks": true,
    "strictFunctionTypes": true,
    "strictBindCallApply": true,
    "strictPropertyInitialization": true,
    "noImplicitThis": true,
    "alwaysStrict": true,
    "useUnknownInCatchVariables": true
  }
}
```

### noImplicitAny

禁止隐式 any 类型：

```json
{
  "compilerOptions": {
    "noImplicitAny": true
  }
}
```

```typescript
// noImplicitAny: false
function fn(x) { } // x 隐式 any

// noImplicitAny: true
function fn(x: number) { } // 必须显式声明类型
```

### strictNullChecks

严格的 null 检查：

```json
{
  "compilerOptions": {
    "strictNullChecks": true
  }
}
```

```typescript
let name: string = null; // Error: null 不能赋给 string
let name: string | null = null; // OK
```

### noUnusedLocals / noUnusedParameters

检查未使用的变量和参数：

```json
{
  "compilerOptions": {
    "noUnusedLocals": true,
    "noUnusedParameters": true
  }
}
```

### noImplicitReturns

检查函数所有路径都有返回值：

```json
{
  "compilerOptions": {
    "noImplicitReturns": true
  }
}
```

### noFallthroughCasesInSwitch

检查 switch 穿透：

```json
{
  "compilerOptions": {
    "noFallthroughCasesInSwitch": true
  }
}
```

## 其他常用选项

### skipLibCheck

跳过声明文件检查：

```json
{
  "compilerOptions": {
    "skipLibCheck": true
  }
}
```

### forceConsistentCasingInFileNames

强制文件名大小写一致：

```json
{
  "compilerOptions": {
    "forceConsistentCasingInFileNames": true
  }
}
```

### isolatedModules

确保文件可被单独编译：

```json
{
  "compilerOptions": {
    "isolatedModules": true
  }
}
```
