# 严格类型检查选项

TypeScript 的严格类型检查选项可以帮助发现更多潜在的类型错误，提高代码质量。

## strict 全局选项

### 启用所有严格选项

```json
{
  "compilerOptions": {
    "strict": true
  }
}
```

这会启用以下所有选项：

- `noImplicitAny`
- `strictNullChecks`
- `strictFunctionTypes`
- `strictBindCallApply`
- `strictPropertyInitialization`
- `noImplicitThis`
- `useUnknownInCatchVariables`
- `alwaysStrict`

### 渐进式启用

```json
{
  "compilerOptions": {
    "strict": true,
    // 可以单独关闭某些选项
    "strictPropertyInitialization": false
  }
}
```

## 空值检查

### strictNullChecks

严格检查 null 和 undefined：

```json
{
  "compilerOptions": {
    "strictNullChecks": true
  }
}
```

```typescript
// strictNullChecks: false
let name: string = null;     // OK
let age: number = undefined; // OK

// strictNullChecks: true
let name: string = null;     // Error!
let age: number = undefined; // Error!
let name: string | null = null;     // OK
let age: number | undefined;        // OK
```

**常见模式**：

```typescript
// 可选参数自动包含 undefined
function greet(name?: string) {
  // name 是 string | undefined
}

// 可选属性自动包含 undefined
interface User {
  name: string;
  email?: string;  // string | undefined
}

// 空值检查
function getLength(str: string | null) {
  if (str === null) {
    return 0;
  }
  return str.length;  // TypeScript 知道这里 str 是 string
}

// 使用可选链
const length = str?.length;  // number | undefined

// 使用空值合并
const value = input ?? 'default';
```

### noUncheckedIndexedAccess

检查数组/对象索引访问可能返回 undefined：

```json
{
  "compilerOptions": {
    "noUncheckedIndexedAccess": true
  }
}
```

```typescript
const arr = [1, 2, 3];
const first = arr[0];  // number | undefined

const obj: Record<string, number> = { a: 1 };
const value = obj['a'];  // number | undefined
```

## 类型推断检查

### noImplicitAny

禁止隐式 any 类型：

```json
{
  "compilerOptions": {
    "noImplicitAny": true
  }
}
```

```typescript
// noImplicitAny: false
function fn(x) {           // x 隐式 any
  return x.length;
}

// noImplicitAny: true
function fn(x: string) {   // 必须显式声明类型
  return x.length;
}

// 回调函数也需要类型
['a', 'b'].forEach(function(item) {  // item 隐式 any
  console.log(item);
});

['a', 'b'].forEach((item: string) => {
  console.log(item);
});
```

### noImplicitThis

禁止隐式 any 类型的 this：

```json
{
  "compilerOptions": {
    "noImplicitThis": true
  }
}
```

```typescript
const obj = {
  name: 'Alice',
  greet: function() {
    console.log(this.name);  // OK
  }
};

const bad = {
  name: 'Alice',
  greet: () => {
    console.log(this.name);  // Error: this 是 any
  }
};

// 使用 this 参数
function greet(this: { name: string }) {
  console.log(this.name);
}
```

### strictFunctionTypes

严格的函数类型检查：

```json
{
  "compilerOptions": {
    "strictFunctionTypes": true
  }
}
```

```typescript
// strictFunctionTypes: false
let fn: (x: string | number) => void;
fn = (x: string) => console.log(x.toUpperCase());  // OK (不安全)

// strictFunctionTypes: true
let fn: (x: string | number) => void;
fn = (x: string) => console.log(x.toUpperCase());  // Error!
```

## 其他严格选项

### strictBindCallApply

严格的 bind/call/apply 检查：

```json
{
  "compilerOptions": {
    "strictBindCallApply": true
  }
}
```

```typescript
function fn(x: string, y: number): boolean {
  return x.length > y;
}

// strictBindCallApply: false
fn.call(undefined, 1, 2);  // OK (但参数类型错误)

// strictBindCallApply: true
fn.call(undefined, 1, 2);  // Error: 参数类型不匹配
fn.call(undefined, 'hello', 2);  // OK
```

### strictPropertyInitialization

严格的类属性初始化检查：

```json
{
  "compilerOptions": {
    "strictPropertyInitialization": true
  }
}
```

```typescript
class User {
  name: string;      // Error: 属性未初始化
  email: string;     // Error: 属性未初始化
  
  // 解决方案1: 初始化
  age: number = 0;
  
  // 解决方案2: 构造函数赋值
  constructor(name: string, email: string) {
    this.name = name;
    this.email = email;
  }
  
  // 解决方案3: 可选属性
  nickname?: string;
  
  // 解决方案4: 断言
  id!: string;  // 确定会在其他地方初始化
}
```

### useUnknownInCatchVariables

catch 语句中的变量类型为 unknown：

```json
{
  "compilerOptions": {
    "useUnknownInCatchVariables": true
  }
}
```

```typescript
try {
  // ...
} catch (err) {
  // useUnknownInCatchVariables: false
  console.log(err.message);  // err 是 any
  
  // useUnknownInCatchVariables: true
  console.log(err.message);  // Error: err 是 unknown
  
  if (err instanceof Error) {
    console.log(err.message);  // OK
  }
}
```

### alwaysStrict

始终以严格模式解析：

```json
{
  "compilerOptions": {
    "alwaysStrict": true
  }
}
```

输出 JavaScript 文件会包含 `"use strict";`。

## 推荐配置

```json
{
  "compilerOptions": {
    "strict": true,
    "noUncheckedIndexedAccess": true,
    "noImplicitReturns": true,
    "noFallthroughCasesInSwitch": true,
    "noUnusedLocals": true,
    "noUnusedParameters": true
  }
}
```
