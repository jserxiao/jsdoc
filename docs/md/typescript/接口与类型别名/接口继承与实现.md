# 接口继承与实现

## 概述

接口可以通过继承扩展,也可以被类实现。本章详细介绍接口的继承和实现机制。

## 一、接口继承接口

### 1.1 单继承

```typescript
interface Animal {
  name: string
}

interface Dog extends Animal {
  breed: string
}

const dog: Dog = {
  name: 'Buddy',
  breed: 'Labrador'
}
```

### 1.2 多继承

```typescript
interface Name {
  name: string
}

interface Age {
  age: number
}

interface Person extends Name, Age {
  email: string
}

const person: Person = {
  name: 'Alice',
  age: 25,
  email: 'alice@example.com'
}
```

### 1.3 覆盖继承的属性

```typescript
interface Base {
  id: string | number
}

interface Derived extends Base {
  id: number  // 更具体的类型
}
```

## 二、类实现接口

### 2.1 实现单个接口

```typescript
interface ClockInterface {
  currentTime: Date
  setTime(d: Date): void
}

class Clock implements ClockInterface {
  currentTime: Date = new Date()

  setTime(d: Date) {
    this.currentTime = d
  }

  constructor(h: number, m: number) {}
}
```

### 2.2 实现多个接口

```typescript
interface Printable {
  print(): void
}

interface Loggable {
  log(): void
}

class Document implements Printable, Loggable {
  print() {
    console.log('Printing document')
  }

  log() {
    console.log('Logging document')
  }
}
```

### 2.3 接口继承类

```typescript
class Control {
  private state: any

  protected setState(state: any) {
    this.state = state
  }
}

interface SelectableControl extends Control {
  select(): void
}

class Button extends Control implements SelectableControl {
  select() {
    console.log('Button selected')
  }
}
```

## 三、接口继承 vs 类继承

### 3.1 对比

```typescript
// 接口继承
interface Animal {
  name: string
}

interface Dog extends Animal {
  breed: string
}

// 类继承
class AnimalClass {
  constructor(public name: string) {}
}

class DogClass extends AnimalClass {
  constructor(name: string, public breed: string) {
    super(name)
  }
}
```

### 3.2 混合使用

```typescript
interface Serializable {
  serialize(): string
}

abstract class Entity {
  constructor(public id: string) {}

  abstract toString(): string
}

class User extends Entity implements Serializable {
  constructor(id: string, public name: string) {
    super(id)
  }

  toString(): string {
    return `User(${this.id}, ${this.name})`
  }

  serialize(): string {
    return JSON.stringify({ id: this.id, name: this.name })
  }
}
```

```typescript
    // 实现查询所有

  }

    // 实现保存逻辑
    return user
  }

    // 实现删除逻辑
  }

```

### 4.2 插件系统

```typescript
interface Plugin {
  name: string
  version: string
  install(app: Application): void
}

interface Application {
  plugins: Plugin[]
  use(plugin: Plugin): void
}

class MyApp implements Application {
  plugins: Plugin[] = []

  use(plugin: Plugin) {
    this.plugins.push(plugin)
    plugin.install(this)
  }
}
```

## 五、最佳实践

### 1. 接口优先原则

```typescript
// ✅ 好: 先定义接口,再实现
interface UserService {
  getUser(id: string): Promise<User>
}

class UserServiceImpl implements UserService {
  async getUser(id: string): Promise<User> {
    // 实现
  }
}
```

### 2. 接口隔离

```typescript
// ❌ 不好: 臃肿的接口
interface User {
  id: string
  name: string
  email: string
  // 管理相关
  permissions: string[]
  // 支付相关
  paymentMethods: string[]
}

// ✅ 好: 分离接口
interface User {
  id: string
  name: string
  email: string
}

interface AdminUser extends User {
  permissions: string[]
}

interface CustomerUser extends User {
  paymentMethods: string[]
}
```

## 六、总结

- 接口可以单继承或多继承
- 类可以实现一个或多个接口
- 接口可以继承类(包含 private/protected 成员时有限制)
- 遵循接口隔离原则
