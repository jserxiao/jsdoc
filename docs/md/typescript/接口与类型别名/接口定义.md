# 接口定义

## 概述

接口(Interface)是 TypeScript 中定义对象类型的主要方式,用于描述对象的形状。

## 一、基本语法

### 1.1 定义对象类型

```typescript
interface User {
  id: number
  name: string
  email: string
}

const user: User = {
  id: 1,
  name: 'Alice',
  email: 'alice@example.com'
}
```

### 1.2 可选属性

```typescript
interface User {
  id: number
  name: string
  email?: string  // 可选
  age?: number    // 可选
}

const user1: User = { id: 1, name: 'Alice' }
const user2: User = { id: 2, name: 'Bob', email: 'bob@example.com' }
```

### 1.3 只读属性

```typescript
interface User {
  readonly id: number
  name: string
  readonly createdAt: Date
}

const user: User = {
  id: 1,
  name: 'Alice',
  createdAt: new Date()
}

user.name = 'Bob'        // ✅ 可以修改
// user.id = 2           // ❌ 不能修改
// user.createdAt = new Date()  // ❌ 不能修改
```

## 二、函数类型

### 2.1 函数接口

```typescript
interface SearchFunc {
  (source: string, subString: string): boolean
}

const search: SearchFunc = (source, subString) => {
  return source.search(subString) !== -1
}

console.log(search('hello world', 'world'))  // true
```

### 2.2 方法签名

```typescript
interface Calculator {
  add(a: number, b: number): number
  subtract(a: number, b: number): number
  multiply(a: number, b: number): number
  divide(a: number, b: number): number
}

const calc: Calculator = {
  add: (a, b) => a + b,
  subtract: (a, b) => a - b,
  multiply: (a, b) => a * b,
  divide: (a, b) => a / b
}
```

### 2.3 构造签名

```typescript
interface Constructor {
  new (name: string): User
}

class User {
  constructor(public name: string) {}
}

function createUser(ctor: Constructor, name: string): User {
  return new ctor(name)
}

const user = createUser(User, 'Alice')
```

## 三、可索引类型

### 3.1 数字索引

```typescript
interface StringArray {
  [index: number]: string
}

const arr: StringArray = ['a', 'b', 'c']
console.log(arr[0])  // 'a'
```

### 3.2 字符串索引

```typescript
interface StringMap {
  [key: string]: string
}

const map: StringMap = {
  name: 'Alice',
  city: 'Beijing'
}

map.country = 'China'  // ✅
// map.age = 25        // ❌ number 不能赋值给 string
```

### 3.3 混合索引

```typescript
interface MixedMap {
  [key: string]: string | number
  [index: number]: string  // 数字索引返回类型必须是字符串索引返回类型的子类型
  length: number           // 必须符合索引签名
}

const obj: MixedMap = {
  name: 'Alice',
  age: 25,
  length: 2
}
```

## 四、类类型

### 4.1 实现接口

```typescript
interface ClockInterface {
  currentTime: Date
  setTime(d: Date): void
}

class Clock implements ClockInterface {
  currentTime: Date = new Date()

  setTime(d: Date) {
    this.currentTime = d
  }

  constructor(h: number, m: number) {}
}
```

### 4.2 实现多个接口

```typescript
interface Printable {
  print(): void
}

interface Loggable {
  log(): void
}

class Document implements Printable, Loggable {
  print() {
    console.log('Printing document')
  }

  log() {
    console.log('Logging document')
  }
}
```

### 4.3 接口继承类

```typescript
class Control {
  private state: any
}

interface SelectableControl extends Control {
  select(): void
}

class Button extends Control implements SelectableControl {
  select() {}
}

class TextBox extends Control {
  select() {}
}
```

## 五、接口继承

### 5.1 单继承

```typescript
interface Animal {
  name: string
}

interface Dog extends Animal {
  breed: string
}

const dog: Dog = {
  name: 'Buddy',
  breed: 'Labrador'
}
```

### 5.2 多继承

```typescript
interface Name {
  name: string
}

interface Age {
  age: number
}

interface Person extends Name, Age {
  email: string
}

const person: Person = {
  name: 'Alice',
  age: 25,
  email: 'alice@example.com'
}
```

### 5.3 覆盖继承的属性

```typescript
interface Base {
  id: string | number
}

interface Derived extends Base {
  id: number  // 更具体的类型
}
```

## 六、混合类型

```typescript
interface Counter {
  (start: number): string
  interval: number
  reset(): void
}

function getCounter(): Counter {
  const counter = ((start: number) => {
    return `Started at ${start}`
  }) as Counter

  counter.interval = 123
  counter.reset = () => {}

  return counter
}

const c = getCounter()
c(10)
c.reset()
console.log(c.interval)
```

## 七、接口合并

### 7.1 自动合并

```typescript
interface User {
  name: string
}

interface User {
  age: number
}

// 合并后
const user: User = {
  name: 'Alice',
  age: 25
}
```

### 7.2 非函数成员冲突

```typescript
interface Document {
  title: string
}

interface Document {
  title: string  // ✅ 相同类型
  // title: number  // ❌ 编译错误: 后续声明必须具有相同的类型
  author: string
}
```

### 7.3 函数成员合并

```typescript
interface Album {
  listen(): void
}

interface Album {
  listen(): void  // ✅ 可以重复
  play(): void
}

const album: Album = {
  listen() {},
  play() {}
}
```

## 八、实战示例

### 8.1 API 响应

```typescript
interface APIResponse<T> {
  success: boolean
  data: T
  message?: string
  timestamp: number
}

interface User {
  id: number
  name: string
  email: string
}

async function fetchUser(id: number): Promise<APIResponse<User>> {
  const response = await fetch(`/api/users/${id}`)
  const data = await response.json()
  return {
    success: true,
    data,
    timestamp: Date.now()
  }
}
```

### 8.2 事件处理

```typescript
interface Event {
  type: string
  target: Element
  timestamp: number
}

interface ClickEvent extends Event {
  type: 'click'
  x: number
  y: number
}

interface KeyboardEvent extends Event {
  type: 'keydown' | 'keyup'
  key: string
  code: string
}

function handleEvent(event: ClickEvent | KeyboardEvent) {
  switch (event.type) {
    case 'click':
      console.log(`Clicked at (${event.x}, ${event.y})`)
      break
    case 'keydown':
    case 'keyup':
      console.log(`Key: ${event.key}`)
      break
  }
}
```

### 8.3 插件系统

```typescript
interface Plugin {
  name: string
  version: string
  install(app: Application): void
}

interface Application {
  plugins: Plugin[]
  use(plugin: Plugin): void
}

class MyApp implements Application {
  plugins: Plugin[] = []

  use(plugin: Plugin) {
    this.plugins.push(plugin)
    plugin.install(this)
  }
}
```

## 九、最佳实践

### 1. 接口命名规范

```typescript
// ✅ 好: 不加 I 前缀
interface User {}
interface Product {}

// ❌ 不好: C# 风格的前缀
interface IUser {}
interface IProduct {}
```

### 2. 优先使用接口

```typescript
// ✅ 好: 对象类型使用接口
interface User {
  name: string
  age: number
}

// ❌ 不推荐: 简单对象使用 type
type UserType = {
  name: string
  age: number
}
```

### 3. 利用接口合并

```typescript
// 为第三方库扩展类型
declare module 'express' {
  interface Request {
    user?: {
      id: string
      name: string
    }
  }
}
```

## 十、总结

### 接口的核心概念
- **对象类型**: 描述对象的形状
- **函数类型**: 定义函数签名
- **可索引类型**: 定义数组或字典
- **类类型**: 定义类的契约

### 接口的特性
- 可选属性 `?`
- 只读属性 `readonly`
- 接口继承 `extends`
- 接口合并
- 混合类型

### 使用场景
- 定义 API 响应格式
- 定义事件类型
- 定义类契约
- 定义函数类型
- 扩展第三方库类型
