# 类型系统理论基础

## 概述

类型系统是 TypeScript 的核心,本章将深入探讨类型系统的理论基础,包括类型论基础、类型检查策略、类型推断原理等。

## 一、类型论基础

### 1.1 什么是类型

**类型**是一组值的集合,以及可以对这组值执行的操作。

```typescript
// 类型定义了值的范围和可用的操作
type Number = 0 | 1 | 2 | ... // 数字的集合

// 不同类型有不同的操作
const a: number = 10
const b: number = 20
const sum = a + b // ✅ 数字加法

const c: string = 'hello'
const d: string = 'world'
const concat = c + d // ✅ 字符串拼接
```

### 1.2 类型系统的作用

**1. 检测错误**

```typescript
function add(a: number, b: number): number {
  return a + b
}

add(1, '2') // ❌ 编译时错误,避免运行时问题
```

**2. 抽象化**

```typescript
// 抽象用户概念
interface User {
  id: number
  name: string
}

// 无需关心具体实现细节
function greet(user: User) {
  console.log(`Hello, ${user.name}`)
}
```

**3. 文档化**

```typescript
// 类型即文档
function fetchData(
  url: string,
  options?: {
    method?: 'GET' | 'POST'
    headers?: Record<string, string>
    body?: any
  }
): Promise<Response> {
  // 函数签名清晰说明了用法
  return fetch(url, options)
}
```

**4. 优化**

```typescript
// 类型信息可用于编译优化
const numbers: number[] = [1, 2, 3]
numbers.forEach(n => console.log(n))
// 编译器知道 n 是 number,可以进行优化
```

### 1.3 类型系统的分类

#### 静态类型 vs 动态类型

| 特性 | 静态类型 | 动态类型 |
|------|---------|---------|
| 类型检查时机 | 编译时 | 运行时 |
| 类型注解 | 需要显式或推断 | 不需要 |
| 错误发现 | 早 | 晚 |
| 灵活性 | 较低 | 较高 |
| 示例语言 | TypeScript, Java, C++ | JavaScript, Python, Ruby |

```typescript
// 静态类型 (TypeScript)
function add(a: number, b: number): number {
  return a + b
}

add(1, '2') // ❌ 编译错误

// 动态类型 (JavaScript)
function add(a, b) {
  return a + b
}

add(1, '2') // '12' - 运行时不报错,但结果非预期
```

#### 强类型 vs 弱类型

| 特性 | 强类型 | 弱类型 |
|------|--------|--------|
| 类型转换 | 显式 | 隐式 |
| 类型安全 | 高 | 低 |
| 灵活性 | 较低 | 较高 |
| 示例语言 | Python, Haskell | JavaScript, C |

```typescript
// JavaScript (弱类型) - 隐式类型转换
'5' - 3  // 2 (string → number)
'5' + 3  // '53' (number → string)
[] == false // true

// TypeScript (相对强类型)
let a: number = 5
let b: string = '3'
// a + b  // ❌ 编译错误
```

### 1.4 名义类型 vs 结构类型

#### 名义类型系统

类型兼容性基于类型名称:

```typescript
// 名义类型系统 (如 Java, C#)
class Point {
  x: number
  y: number
}

class Point2D {
  x: number
  y: number
}

// Point 和 Point2D 是不同的类型,即使结构相同
// Point p = new Point2D() // ❌ 编译错误
```

#### 结构类型系统

类型兼容性基于结构:

```typescript
// TypeScript 使用结构类型系统 (鸭子类型)
interface Point {
  x: number
  y: number
}

interface Point2D {
  x: number
  y: number
}

const p1: Point = { x: 0, y: 0 }
const p2: Point2D = { x: 0, y: 0 }

// ✅ 结构相同,类型兼容
const p3: Point = p2
const p4: Point2D = p1
```

**结构类型的核心原则**:

```typescript
interface Named {
  name: string
}

class Person {
  name: string
  age: number

  constructor(name: string, age: number) {
    this.name = name
    this.age = age
  }
}

// ✅ Person 包含 Named 的所有属性
const p: Named = new Person('Alice', 25)
```

## 二、TypeScript 的类型系统

### 2.1 类型空间与值空间

TypeScript 有两个独立的空间:

```typescript
// 值空间 (编译后存在)
const value = 'hello'
function fn() { return 42 }

// 类型空间 (编译后移除)
type Name = string
interface User {
  name: string
}

// 区分方式
// - typeof 操作值空间
// - keyof 操作类型空间

const data = { x: 10, y: 20 }

// typeof 获取值的类型
type Point = typeof data // { x: number; y: number }

// keyof 获取类型的键
type Keys = keyof Point  // 'x' | 'y'
```

### 2.2 类型兼容性规则

#### 协变 (Covariance)

子类型可以赋值给父类型:

```typescript
interface Animal {
  name: string
}

interface Dog extends Animal {
  breed: string
}

// 协变: Dog → Animal
let animal: Animal = { name: 'Generic' }
let dog: Dog = { name: 'Buddy', breed: 'Labrador' }

animal = dog // ✅ Dog 是 Animal 的子类型
```

#### 逆变 (Contravariance)

函数参数类型是逆变的:

```typescript
type AnimalHandler = (animal: Animal) => void
type DogHandler = (dog: Dog) => void

let handleAnimal: AnimalHandler = (a: Animal) => console.log(a.name)
let handleDog: DogHandler = (d: Dog) => console.log(d.breed)

// 在严格模式下
// handleDog = handleAnimal // ✅ 安全 (参数逆变)
// handleAnimal = handleDog // ❌ 不安全

// 原因: handleDog 需要 breed 属性,但 Animal 可能没有
```

#### 不变 (Invariance)

必须类型完全匹配:

```typescript
interface Container<T> {
  value: T
  setValue(value: T): void
}

let numberContainer: Container<number>
let unknownContainer: Container<unknown>

// numberContainer = unknownContainer // ❌ 不兼容
// unknownContainer = numberContainer // ❌ 不兼容
```

### 2.3 类型宽度

类型的"宽度"指类型包含的值的范围:

```typescript
// 宽类型 (包含更多值)
type Wide = any        // 所有值
type AnyString = string // 所有字符串

// 窄类型 (包含更少值)
type Narrow = 'hello' | 'world' // 特定字符串
type SpecificNumber = 42        // 特定数字

// 赋值规则: 窄类型 → 宽类型
let wide: string
let narrow: 'hello'

wide = narrow // ✅ 窄类型赋值给宽类型
// narrow = wide // ❌ 宽类型不能赋值给窄类型
```

## 三、类型推断

### 3.1 类型推断的策略

TypeScript 使用多种策略推断类型:

#### 1. 最佳通用类型推断

```typescript
// 推断为 (string | number)[]
const mixed = [1, 'two', 3]

// 推断为 number[]
const numbers = [1, 2, 3]

// 推断为 never[] - 无法推断通用类型
const empty = []
```

#### 2. 上下文类型推断

```typescript
// 根据函数签名推断参数类型
type Handler = (event: { type: string; target: Element }) => void

const handler: Handler = event => {
  console.log(event.type) // event 类型被推断
}

// 根据对象属性推断
interface Config {
  callback: (value: string) => void
}

const config: Config = {
  callback: value => {
    console.log(value.toUpperCase()) // value: string
  }
}
```

#### 3. 控制流分析

```typescript
function process(value: string | number) {
  // value: string | number

  if (typeof value === 'string') {
    // value: string (类型缩小)
    return value.toUpperCase()
  } else {
    // value: number (类型缩小)
    return value * 2
  }
}
```

### 3.2 类型推断示例

```typescript
// 变量推断
let x = 10          // number
let y = 'hello'     // string
let z = [1, 2, 3]   // number[]
let obj = { a: 1, b: 'two' } // { a: number; b: string }

// 函数返回值推断
function add(a: number, b: number) {
  return a + b      // 返回值推断为 number
}

// 泛型推断
function identity<T>(arg: T): T {
  return arg
}

const result = identity(42)  // T 推断为 number, result: number

// 复杂类型推断
const users = [
  { id: 1, name: 'Alice' },
  { id: 2, name: 'Bob' }
]
// 推断为 { id: number; name: string }[]
```

## 四、类型安全

### 4.1 类型安全的层级

```
any (最不安全)
  ↓
unknown (类型安全)
  ↓
具体类型
  ↓
字面量类型 (最安全)
```

### 4.2 any vs unknown

```typescript
// any - 放弃类型检查
let anything: any = 'hello'
anything = 42
anything.nonExistentMethod() // ❌ 运行时错误,编译时不检查

// unknown - 类型安全
let unsure: unknown = 'hello'
unsure = 42

// ❌ 不能直接使用
// unsure.toUpperCase()

// ✅ 需要类型守卫或断言
if (typeof unsure === 'string') {
  console.log(unsure.toUpperCase())
}

// 或类型断言
console.log((unsure as string).toUpperCase())
```

### 4.3 类型断言

```typescript
// 类型断言告诉编译器"我知道这个类型"
const value: any = 'hello'

// 方式一: 尖括号语法
const length1: number = (<string>value).length

// 方式二: as 语法 (推荐)
const length2: number = (value as string).length

// 非空断言
interface User {
  name: string | null
}

const user: User = { name: 'Alice' }
const name = user.name! // 断言 name 不为 null

// 双重断言 (不推荐,不安全)
const something = value as any as number
```

## 五、高级类型概念

### 5.1 类型别名 vs 接口

```typescript
// 类型别名
type Point = {
  x: number
  y: number
}

// 接口
interface IPoint {
  x: number
  y: number
}

// 主要区别

// 1. 类型别名可以表示联合类型
type ID = string | number

// 2. 接口可以被类实现
class Point3D implements IPoint {
  x: number = 0
  y: number = 0
  z: number = 0
}

// 3. 接口可以声明合并
interface User {
  name: string
}

interface User {
  age: number
}

const user: User = {
  name: 'Alice',
  age: 25
}
```

### 5.2 类型运算

```typescript
// 联合类型 (OR)
type StringOrNumber = string | number

// 交叉类型 (AND)
type Name = { name: string }
type Age = { age: number }
type Person = Name & Age // { name: string; age: number }

// keyof 操作符
interface User {
  id: number
  name: string
  email: string
}

type UserKeys = keyof User // 'id' | 'name' | 'email'

// typeof 操作符
const config = {
  apiUrl: 'https://api.example.com',
  timeout: 5000
}

type Config = typeof config // { apiUrl: string; timeout: number }
```

### 5.3 条件类型

```typescript
// 基本语法
type IsString<T> = T extends string ? true : false

type A = IsString<string> // true
type B = IsString<number> // false

// 实用示例
type NonNullable<T> = T extends null | undefined ? never : T

type C = NonNullable<string | null> // string
type D = NonNullable<number | undefined> // number

// 分布式条件类型
type Exclude<T, U> = T extends U ? never : T

type E = Exclude<'a' | 'b' | 'c', 'a'> // 'b' | 'c'
```

## 六、类型系统的限制

### 6.1 运行时类型信息

TypeScript 类型在运行时不存在:

```typescript
interface User {
  name: string
}

const user: User = { name: 'Alice' }

// ❌ 运行时无法检查接口类型
// if (user instanceof User) { } // User 仅在类型空间存在

// ✅ 使用类型守卫函数
function isUser(obj: any): obj is User {
  return obj && typeof obj.name === 'string'
}

if (isUser(user)) {
  console.log(user.name)
}
```

### 6.2 类型擦除

```typescript
// 编译前
function greet(name: string): string {
  return `Hello, ${name}`
}

// 编译后
function greet(name) {
  return `Hello, ${name}`
}

// 类型信息丢失
```

### 6.3 泛型的限制

```typescript
// ❌ 不能使用 instanceof 检查泛型类型
function isString<T>(value: T): boolean {
  // return value instanceof string // ❌ string 是类型,不是值
  return typeof value === 'string'
}

// ✅ 正确的方式
function process<T>(value: T) {
  if (typeof value === 'string') {
    return value.toUpperCase()
  }
  return value
}
```

## 七、总结

### 核心概念

1. **类型系统** - 对值进行分类和约束的系统
2. **静态类型** - 编译时进行类型检查
3. **结构类型** - 基于结构而非名称的类型兼容
4. **类型推断** - 自动推断表达式的类型
5. **类型擦除** - 编译后类型信息被移除

### 类型系统设计原则

1. **渐进式采用** - 可以逐步添加类型
2. **类型安全** - 在编译时捕获错误
3. **表达力** - 可以表达复杂的类型关系
4. **工具友好** - 为 IDE 提供类型信息

### 学习建议

1. 理解类型空间与值空间的区别
2. 掌握类型兼容性规则
3. 学会使用类型推断
4. 了解类型系统的局限性
