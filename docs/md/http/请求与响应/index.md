# HTTP 请求与响应

> HTTP 通信的核心是请求-响应模型。客户端发送请求，服务器返回响应。理解请求和响应的结构对于开发 Web 应用和调试问题至关重要。

## 学习要点

- 📦 掌握 HTTP 请求的结构和各部分作用
- 📬 理解 HTTP 响应的组成和状态码含义
- 🔧 熟悉常用的请求头和响应头
- 🎯 了解不同请求方法的特性和使用场景

---

## 1. HTTP 请求结构

### 1.1 请求格式

```
请求行（Request Line）
请求头部（Request Headers）
空行
请求体（Request Body）[可选]
```

#### 完整请求示例

```http
POST /api/users HTTP/1.1
Host: example.com
Content-Type: application/json
Content-Length: 52
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9
Accept: application/json
User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64)
Connection: keep-alive

{"name": "张三", "email": "zhangsan@example.com"}
```

## 2. 请求行

### 2.1 请求行格式

```
请求行格式：方法 路径 HTTP版本

示例：GET /users/123 HTTP/1.1
     POST /api/login HTTP/1.1
     PUT /articles/456 HTTP/2
```

### 2.2 HTTP 方法详解

```
┌──────────┬────────────────────────────────────────────────────────────┐
│  方法      │  说明                                                        │
├──────────┼────────────────────────────────────────────────────────────┤
│  GET      │  获取资源。请求参数在 URL 中，请求体无意义，可缓存              │
│  POST     │  创建资源。请求体包含数据，不可缓存，非幂等                      │
│  PUT      │  更新资源（完整替换）。请求体包含完整数据，幂等                   │
│  PATCH    │  部分更新资源。请求体包含部分数据，非幂等                        │
│  DELETE   │  删除资源。幂等                                               │
│  HEAD     │  获取资源头部（不含响应体）。用于检查资源是否存在、获取元数据      │
│  OPTIONS  │  获取支持的 HTTP 方法。用于 CORS 预检请求                       │
│  CONNECT  │  建立隧道连接。用于 HTTPS 代理                                 │
│  TRACE    │  回显服务器收到的请求。用于诊断（有安全风险，通常禁用）           │
└──────────┴────────────────────────────────────────────────────────────┘
```

### 2.3 方法的幂等性与安全性

```
┌──────────┬──────────┬──────────┬─────────────────────────────────────────┐
│  方法      │  安全性    │  幂等性    │  说明                                      │
├──────────┼──────────┼──────────┼─────────────────────────────────────────┤
│  GET      │    ✓     │    ✓     │  只读，多次请求结果相同                      │
│  HEAD     │    ✓     │    ✓     │  只读，多次请求结果相同                      │
│  OPTIONS  │    ✓     │    ✓     │  只读，多次请求结果相同                      │
│  TRACE    │    ✓     │    ✓     │  只读，多次请求结果相同                      │
│  PUT      │    ✗     │    ✓     │  多次请求结果相同（覆盖）                    │
│  DELETE   │    ✗     │    ✓     │  多次请求结果相同（删除后仍是删除状态）        │
│  POST     │    ✗     │    ✗     │  每次请求可能创建新资源                      │
│  PATCH    │    ✗     │    ✗     │  每次请求可能产生不同结果                    │
└──────────┴──────────┴──────────┴─────────────────────────────────────────┘

安全性：不会修改服务器资源
幂等性：多次相同请求产生相同结果
```

## 3. 请求头部详解

### 3.1 请求头分类概览

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                           HTTP 请求头分类                                        │
├─────────────────────────────────────────────────────────────────────────────────┤
│  分类              │  功能描述                      │  典型头部                   │
├────────────────────┼────────────────────────────────┼────────────────────────────┤
│  通用头部          │  请求和响应都可使用             │  Cache-Control, Connection │
│  请求头部          │  客户端发送的请求信息           │  Host, User-Agent, Accept   │
│  实体头部          │  描述请求体内容                 │  Content-Type, Content-Length│
│  条件头部          │  条件请求                       │  If-Match, If-None-Match    │
│  安全头部          │  认证与授权                     │  Authorization, Cookie      │
│  缓存头部          │  缓存控制                       │  Cache-Control, Pragma      │
└────────────────────┴────────────────────────────────┴────────────────────────────┘
```

### 3.2 主机与连接相关头部

```
┌─────────────────────┬───────────────────────────────────────────────────────────────────────┐
│  头部名称             │  说明                                                                     │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Host               │  目标服务器的主机名和可选端口号                                            │
│                     │  • HTTP/1.1 必需字段，用于虚拟主机路由                                     │
│                     │  • 格式：Host: <host>:<port>                                             │
│                     │  • 示例：Host: www.example.com:8080                                      │
│                     │  • 默认端口可省略：Host: www.example.com                                  │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Connection         │  控制网络连接是否保持                                                     │
│                     │  • keep-alive：持久连接，可复用（HTTP/1.1 默认）                           │
│                     │  • close：请求完成后关闭连接                                              │
│                     │  • Upgrade：升级协议（如 WebSocket）                                      │
│                     │  • HTTP/2 和 HTTP/3 不使用此头部                                         │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Keep-Alive         │  持久连接参数（需配合 Connection: keep-alive）                             │
│                     │  • timeout：连接保持时间（秒）                                            │
│                     │  • max：连接最大请求数                                                    │
│                     │  • 示例：Keep-Alive: timeout=5, max=100                                  │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Upgrade            │  请求升级到其他协议                                                       │
│                     │  • 示例：Upgrade: websocket, h2c, TLS/1.0                                │
│                     │  • 需配合 Connection: Upgrade 使用                                        │
│                     │  • 常用于 WebSocket 握手和 HTTP/2 明文升级                                │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Via                │  代理服务器信息，用于追踪请求经过的代理                                    │
│                     │  • 格式：Via: <protocol> <host>:<port>                                   │
│                     │  • 示例：Via: 1.1 proxy.example.com:8080                                 │
│                     │  • 用于调试和避免请求循环                                                 │
└─────────────────────┴───────────────────────────────────────────────────────────────────────┘
```

### 3.3 内容协商相关头部

```
┌─────────────────────┬───────────────────────────────────────────────────────────────────────┐
│  头部名称             │  说明                                                                     │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Accept             │  客户端可接受的响应内容类型（MIME 类型）                                   │
│                     │  • 格式：Accept: <MIME_type>/<MIME_subtype> | */*                        │
│                     │  • 支持权重参数 q（0-1，默认1.0）                                         │
│                     │  • 示例：Accept: text/html,application/json;q=0.9,*/*;q=0.8             │
│                     │  • 常见类型：text/html, application/json, image/*, */*                   │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Accept-Charset     │  客户端可接受的字符集（已废弃，现使用 Content-Type 的 charset）            │
│                     │  • 示例：Accept-Charset: utf-8, iso-8859-1;q=0.5                         │
│                     │  • 现代浏览器通常不发送此头部                                              │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Accept-Encoding    │  客户端可接受的内容编码（压缩算法）                                        │
│                     │  • gzip：GNU zip 压缩（最常用）                                           │
│                     │  • deflate：zlib 压缩格式                                                │
│                     │  • br：Brotli 压缩（压缩率更高，HTTP/2 推荐使用）                          │
│                     │  • identity：不压缩（默认）                                               │
│                     │  • *：匹配任意编码                                                        │
│                     │  • 示例：Accept-Encoding: gzip, deflate, br                              │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Accept-Language    │  客户端可接受的自然语言                                                   │
│                     │  • 格式：<language>-<region> 或 <language>                               │
│                     │  • 支持权重参数 q                                                         │
│                     │  • 示例：Accept-Language: zh-CN,zh;q=0.9,en;q=0.8,en-US;q=0.7            │
│                     │  • 常见值：zh-CN, zh-TW, en-US, en-GB, ja, ko, fr, de                    │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  TE                 │  客户端可接受的传输编码                                                   │
│                     │  • trailers：支持 Trailer 头部                                           │
│                     │  • chunked：分块传输编码                                                 │
│                     │  • 示例：TE: trailers, gzip                                              │
└─────────────────────┴───────────────────────────────────────────────────────────────────────┘
```

```
│                     │  • Firefox 浏览器示例：                                                  │
│                     │    Mozilla/5.0 (Windows NT 10.0; Win64; x64; rv:121.0) Gecko/20100101   │
│                     │    Firefox/121.0                                                         │

│                     │    Mozilla/5.0 (Macintosh; Intel Mac OS X 14_2) AppleWebKit/605.1.15    │
│                     │    (KHTML, like Gecko) Version/17.2 Safari/605.1.15                      │
│                     │  • 移动端示例：                                                          │
│                     │    Mozilla/5.0 (iPhone; CPU iPhone OS 17_2 like Mac OS X)               │
│                     │    AppleWebKit/605.1.15 (KHTML, like Gecko) Version/17.2 Mobile         │
│                     │  • 爬虫示例：                                                            │
│                     │    Googlebot/2.1 (+http://www.google.com/bot.html)                      │
│                     │  • 用于内容适配、统计分析和爬虫识别                                       │
│                     │  • 注意：可被伪造，不应作为唯一的安全验证                                  │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  From               │  发送请求的用户邮箱地址（主要用于爬虫标识）                                 │
│                     │  • 示例：From: webmaster@example.org                                     │
│                     │  • 用于爬虫提供联系方式，便于网站管理员联系                                │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Referer [sic]      │  请求来源页面的 URL（注意拼写错误是历史遗留）                               │
│                     │  • 用于追踪用户来源、防盗链、统计分析                                      │
│                     │  • 包含完整 URL（含查询参数）                                             │
│                     │  • 示例：Referer: https://example.com/page.html?category=tech           │
│                     │  • 敏感场景可通过 Referrer-Policy 控制发送                                │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Referrer-Policy    │  控制 Referer 头部的发送行为                                              │
│                     │  • no-referrer：不发送 Referer                                           │
│                     │  • no-referrer-when-downgrade：降级时不发送（默认）                        │
│                     │  • origin：只发送源（协议+域名+端口）                                     │
│                     │  • origin-when-cross-origin：跨域时只发送源                               │
│                     │  • same-origin：同源时发送完整 URL                                        │
│                     │  • strict-origin：只发送源，降级时不发送                                  │
│                     │  • strict-origin-when-cross-origin：跨域时只发送源，降级时不发送           │
│                     │  • unsafe-url：始终发送完整 URL                                          │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Origin             │  请求发起的源（协议+域名+端口），用于 CORS                                 │
│                     │  • 与 Referer 的区别：只包含源，不含路径和查询参数                          │
│                     │  • 示例：Origin: https://www.example.com                                 │
│                     │  • 用于 CORS 跨域请求和 POST 请求的 CSRF 防护                             │
│                     │  • 由浏览器自动添加，JavaScript 不可修改                                   │
└─────────────────────┴───────────────────────────────────────────────────────────────────────┘
```

### 3.5 请求体相关头部

```
┌─────────────────────┬───────────────────────────────────────────────────────────────────────┐
│  头部名称             │  说明                                                                     │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Content-Type       │  请求体的媒体类型（MIME 类型）                                            │
│                     │  • application/json：JSON 数据                                           │
│                     │  • application/x-www-form-urlencoded：表单 URL 编码                       │
│                     │  • multipart/form-data：多部分表单数据（文件上传）                         │
│                     │  • text/plain：纯文本                                                    │
│                     │  • text/html：HTML 文档                                                  │
│                     │  • application/xml / text/xml：XML 数据                                  │
│                     │  • application/octet-stream：二进制数据                                   │
│                     │  • 可包含 charset 参数：text/plain; charset=utf-8                        │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Content-Length     │  请求体的字节长度                                                         │
│                     │  • 十进制整数，表示字节数                                                 │
│                     │  • 示例：Content-Length: 348                                             │
│                     │  • 与 Transfer-Encoding: chunked 互斥                                    │
│                     │  • 自动计算，手动设置可能导致错误                                         │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Content-Encoding   │  请求体的编码方式（压缩算法）                                              │
│                     │  • gzip、deflate、br、compress、identity                                 │
│                     │  • 示例：Content-Encoding: gzip                                          │
│                     │  • 请求中较少使用，主要用于服务器知道客户端能解压时                        │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Transfer-Encoding  │  传输编码方式                                                             │
│                     │  • chunked：分块传输编码                                                 │
│                     │  • 示例：Transfer-Encoding: chunked                                      │
│                     │  • 用于流式传输，无需预先知道 Content-Length                              │
│                     │  • HTTP/2 和 HTTP/3 不使用此头部（使用 DATA 帧）                          │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Content-Language   │  请求体的自然语言                                                         │
│                     │  • 示例：Content-Language: zh-CN                                         │
│                     │  • 请求中较少使用                                                         │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Content-Location   │  请求体资源的替代位置                                                     │
│                     │  • 示例：Content-Location: /documents/zh-CN/report.pdf                   │
│                     │  • 用于内容协商后指示资源位置                                             │
└─────────────────────┴───────────────────────────────────────────────────────────────────────┘
```

### 3.7 认证与授权相关头部

```
┌─────────────────────┬───────────────────────────────────────────────────────────────────────┐
│  头部名称             │  说明                                                                     │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Authorization       │  客户端认证信息                                                           │
│                     │  • 格式：Authorization: <auth-scheme> <credentials>                       │
│                     │  • Basic 认证：                                                          │
│                     │    Authorization: Basic dXNlcm5hbWU6cGFzc3dvcmQ=                        │
│                     │    （Base64 编码的 username:password）                                   │
│                     │  • Bearer Token（OAuth 2.0 / JWT）：                                     │
│                     │    Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...         │
│                     │  • Digest 认证：                                                         │
│                     │    Authorization: Digest username="admin", realm="test",               │
│                     │    nonce="...", uri="/", response="..."                                │
```

### 4.2 文件上传详解

```javascript
// 单文件上传
async function uploadFile(file) {
  const formData = new FormData();
  formData.append('file', file);
  
  const response = await fetch('/api/upload', {
    method: 'POST',
    body: formData
  });
  
  return response.json();
}

// 多文件上传
async function uploadFiles(files) {
  const formData = new FormData();
  
  // 方式1：每个文件一个字段
  for (const file of files) {
    formData.append('files', file);
  }
  
  // 方式2：带索引的字段名
  // files.forEach((file, index) => {
  //   formData.append(`file${index}`, file);
  // });
  
  const response = await fetch('/api/upload/multiple', {
    method: 'POST',
    body: formData
  });
  
  return response.json();
}

// 带进度的文件上传
function uploadWithProgress(file, onProgress) {
  return new Promise((resolve, reject) => {
    const xhr = new XMLHttpRequest();
    const formData = new FormData();
    formData.append('file', file);
    
    xhr.upload.addEventListener('progress', (e) => {
      if (e.lengthComputable) {
        const percent = Math.round((e.loaded / e.total) * 100);
        onProgress(percent);
      }
    });
    
    xhr.addEventListener('load', () => {
      if (xhr.status === 200) {
        resolve(JSON.parse(xhr.responseText));
      } else {
        reject(new Error(`Upload failed: ${xhr.status}`));
      }
    });
    
    xhr.addEventListener('error', () => reject(new Error('Upload error')));
    
    xhr.open('POST', '/api/upload');
    xhr.send(formData);
  });
}

// 使用示例
const file = document.querySelector('#fileInput').files[0];
await uploadWithProgress(file, (percent) => {
  console.log(`上传进度: ${percent}%`);
});
```

### 6.2 HTTP 状态码分类

### 6.3 常见状态码详解

#### 1xx 信息响应

```
┌──────────┬────────────────────────────────────────────────────────────┐
│  100     │  Continue - 继续发送请求体（Expect: 100-continue）          │
│  101     │  Switching Protocols - 协议切换（WebSocket 升级）            │
│  102     │  Processing - 请求已接收，正在处理（WebDAV）                  │
│  103     │  Early Hints - 预加载提示                                   │
└──────────┴────────────────────────────────────────────────────────────┘

101 状态码示例（WebSocket 握手）：
HTTP/1.1 101 Switching Protocols
Upgrade: websocket
Connection: Upgrade
Sec-WebSocket-Accept: s3pPLMBiTxaQ9kYGzzhZRbK+xOo=
```

#### 2xx 成功

```
┌──────────┬────────────────────────────────────────────────────────────┐
│  200     │  OK - 请求成功，响应体包含请求的数据                          │
│  201     │  Created - 资源创建成功，响应体包含新资源                     │
│  202     │  Accepted - 请求已接受，正在异步处理                          │
│  204     │  No Content - 请求成功，无响应体（DELETE 成功后常用）         │
│  205     │  Reset Content - 请求成功，重置表单                          │
│  206     │  Partial Content - 范围请求成功（断点续传、视频拖动）          │
└──────────┴────────────────────────────────────────────────────────────┘

使用场景：
• GET 成功 → 200 OK
• POST 创建成功 → 201 Created + Location 头部
• PUT 更新成功 → 200 OK 或 204 No Content
• DELETE 成功 → 204 No Content
• 异步任务 → 202 Accepted
```

#### 3xx 重定向

```
┌──────────┬────────────────────────────────────────────────────────────┐
│  300     │  Multiple Choices - 多个选项，可选择重定向                   │
│  301     │  Moved Permanently - 永久重定向，SEO 权重转移                │
│  302     │  Found / Moved Temporarily - 临时重定向                      │
│  303     │  See Other - 使用 GET 重定向到另一个 URL                     │
│  304     │  Not Modified - 资源未修改（缓存有效）                       │
│  307     │  Temporary Redirect - 临时重定向，保持请求方法                │
│  308     │  Permanent Redirect - 永久重定向，保持请求方法                │
└──────────┴────────────────────────────────────────────────────────────┘

重定向对比：
┌──────────┬──────────────┬───────────────────┬─────────────────────────┐
│  状态码   │  重定向类型   │  是否保持请求方法   │  典型场景                 │
├──────────┼──────────────┼───────────────────┼─────────────────────────┤
│  301     │  永久        │  可能转为 GET       │  网站迁移、URL 结构变更    │
│  302     │  临时        │  可能转为 GET       │  临时跳转                 │
│  307     │  临时        │  保持               │  表单提交后重定向          │
│  308     │  永久        │  保持               │  API 端点永久变更          │
└──────────┴──────────────┴───────────────────┴─────────────────────────┘

注意：301/302 可能将 POST 请求转为 GET，307/308 保持原始方法
```

#### 4xx 客户端错误

```
┌──────────┬────────────────────────────────────────────────────────────┐
│  400     │  Bad Request - 请求语法错误或参数无效                        │
│  401     │  Unauthorized - 未认证，需要登录                             │
│  403     │  Forbidden - 已认证但无权限访问                              │
│  404     │  Not Found - 资源不存在                                      │
│  405     │  Method Not Allowed - 不支持该 HTTP 方法                     │
│  406     │  Not Acceptable - 无法生成 Accept 指定的内容类型              │
│  408     │  Request Timeout - 请求超时                                  │
│  409     │  Conflict - 请求与服务器状态冲突（如重复创建）                 │
│  410     │  Gone - 资源已永久删除                                        │
│  412     │  Precondition Failed - 条件请求失败                          │
│  413     │  Payload Too Large - 请求体过大                               │
│  414     │  URI Too Long - URL 过长                                     │
│  415     │  Unsupported Media Type - 不支持的媒体类型                    │
│  422     │  Unprocessable Entity - 请求格式正确但语义错误                │
│  429     │  Too Many Requests - 请求过于频繁（限流）                      │
│  431     │  Request Header Fields Too Large - 请求头过大                 │
│  451     │  Unavailable For Legal Reasons - 因法律原因不可用             │
└──────────┴────────────────────────────────────────────────────────────┘

401 vs 403 区别：
• 401：未登录，需要认证
• 403：已登录，但权限不足
```

### 7.2 响应体相关头部

```
┌─────────────────────┬───────────────────────────────────────────────────────────────────────┐
│  头部名称             │  说明                                                                     │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Content-Type       │  响应体的媒体类型（MIME 类型）和字符集                                     │
│                     │  • 格式：Content-Type: <type>/<subtype>; charset=<charset>               │
│                     │  • 示例：Content-Type: text/html; charset=utf-8                          │
│                     │  • 示例：Content-Type: application/json; charset=utf-8                   │
│                     │  • 示例：Content-Type: image/png                                         │
│                     │  • 常见类型：text/html, application/json, image/jpeg, video/mp4          │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Content-Length     │  响应体的字节长度                                                         │
│                     │  • 十进制整数，表示字节数                                                 │
│                     │  • 示例：Content-Length: 348                                             │
│                     │  • 用于客户端判断响应是否完整                                             │
│                     │  • 与 Transfer-Encoding: chunked 互斥                                    │
│                     │  • HEAD 请求响应中表示资源大小                                            │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Content-Encoding   │  响应体的编码方式（压缩算法）                                              │
│                     │  • gzip：GNU zip 压缩（最常用）                                           │
│                     │  • deflate：zlib 压缩格式                                                │
│                     │  • br：Brotli 压缩（压缩率更高）                                          │
│                     │  • compress：Unix compress 格式（已废弃）                                 │
│                     │  • identity：不压缩                                                      │
│                     │  • 示例：Content-Encoding: gzip                                          │
│                     │  • 可同时使用多种编码：Content-Encoding: gzip, br                         │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Content-Language   │  响应体的自然语言                                                         │
│                     │  • 格式：<language>-<region> 或 <language>                               │
│                     │  • 示例：Content-Language: zh-CN                                         │
│                     │  • 示例：Content-Language: en-US                                         │
│                     │  • 用于内容协商后的语言标识                                               │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Content-Location   │  响应体资源的替代位置                                                     │
│                     │  • 示例：Content-Location: /documents/zh-CN/report.pdf                   │
│                     │  • 用于内容协商后指示实际返回的资源位置                                    │
│                     │  • 可用于区分同一资源的不同语言版本                                       │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Content-Range      │  范围响应的字节范围                                                       │
│                     │  • 格式：Content-Range: bytes <start>-<end>/<total>                      │
│                     │  • 示例：Content-Range: bytes 0-1023/2048                                │
│                     │  • 示例：Content-Range: bytes 1024-2047/2048                             │
│                     │  • total 为 * 表示未知大小：Content-Range: bytes 0-1023/*                │
│                     │  • 用于断点续传和分片下载                                                 │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Content-Disposition │  响应体的显示方式                                                        │
│                     │  • inline：浏览器内显示（默认）                                           │
│                     │  • attachment：下载文件                                                  │
│                     │  • 示例：Content-Disposition: inline                                     │
│                     │  • 示例：Content-Disposition: attachment; filename="report.pdf"          │
│                     │  • 示例：Content-Disposition: form-data; name="field1"                   │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Transfer-Encoding  │  传输编码方式                                                             │
│                     │  • chunked：分块传输编码                                                 │
│                     │  • 示例：Transfer-Encoding: chunked                                      │
│                     │  • 用于流式传输，无需预先知道 Content-Length                              │
│                     │  • HTTP/2 和 HTTP/3 不使用此头部                                         │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Trailer            │  分块传输结束后发送的尾部头部                                             │
│                     │  • 示例：Trailer: Expires, Content-MD5                                   │
│                     │  • 用于在分块传输结束后发送校验信息                                       │
└─────────────────────┴───────────────────────────────────────────────────────────────────────┘
```

### 7.3 缓存相关响应头部

```
┌─────────────────────┬───────────────────────────────────────────────────────────────────────┐
│  头部名称             │  说明                                                                     │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Cache-Control       │  缓存控制指令（响应端）                                                   │
│                     │  • public：可被任何缓存存储                                              │
│                     │  • private：只能被浏览器缓存，代理不可缓存                                 │
│                     │  • no-cache：可缓存但每次使用前必须验证                                   │
│                     │  • no-store：完全禁止缓存                                                │
│                     │  • max-age=<seconds>：缓存有效期（相对时间）                              │
│                     │  • s-maxage=<seconds>：共享缓存有效期（CDN/代理）                         │
│                     │  • must-revalidate：过期后必须验证                                       │
│                     │  • proxy-revalidate：共享缓存过期后必须验证                               │
│                     │  • no-transform：禁止代理转换响应内容                                    │
│                     │  • immutable：资源永不变化，无需验证                                      │
│                     │  • stale-while-revalidate=<seconds>：可接受过期时间                      │
│                     │  • stale-if-error=<seconds>：出错时可接受过期时间                        │
│                     │  • 示例：Cache-Control: public, max-age=3600, immutable                  │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  ETag               │  资源的版本标识符（实体标签）                                             │
│                     │  • 格式：ETag: "<etag-value>" 或 ETag: W/"<etag-value>"                  │
│                     │  • 强 ETag：ETag: "33a64df551425fcc55e4d42a148795d9f25f89d4"             │
│                     │  • 弱 ETag：ETag: W/"weak-etag"（语义等价，非字节相同）                    │
│                     │  • 用于协商缓存验证（配合 If-None-Match 使用）                            │
│                     │  • 通常为内容的哈希值或版本号                                             │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Last-Modified      │  资源最后修改时间                                                         │
│                     │  • 格式：Last-Modified: <HTTP-date>                                     │
│                     │  • 示例：Last-Modified: Wed, 21 Oct 2015 07:28:00 GMT                   │
│                     │  • 用于协商缓存验证（配合 If-Modified-Since 使用）                        │
│                     │  • 精度只有秒级，不如 ETag 精确                                          │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Expires            │  资源过期时间（绝对时间）                                                 │
│                     │  • 格式：Expires: <HTTP-date>                                           │
│                     │  • 示例：Expires: Thu, 01 Dec 2024 16:00:00 GMT                         │
│                     │  • HTTP/1.0 遗留字段，优先使用 Cache-Control: max-age                     │
│                     │  • 设置为 0 或过去时间表示已过期                                         │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Age                │  响应在缓存中存在的时间（秒）                                             │
│                     │  • 示例：Age: 3600                                                       │
│                     │  • 由缓存服务器（如 CDN）添加                                            │
│                     │  • 表示资源已在缓存中存储的时间                                           │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Vary               │  决定缓存键的请求头                                                       │
│                     │  • 告诉缓存服务器根据哪些请求头区分响应版本                               │
│                     │  • 示例：Vary: Accept-Encoding（根据压缩方式区分缓存）                     │
│                     │  • 示例：Vary: Accept-Language（根据语言区分缓存）                        │
│                     │  • 示例：Vary: User-Agent, Accept-Encoding                              │
│                     │  • Vary: * 表示每次请求都验证                                            │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Date               │  响应生成的日期时间                                                       │
│                     │  • 格式：Date: <HTTP-date>                                              │
│                     │  • 示例：Date: Wed, 21 Oct 2015 07:28:00 GMT                            │
│                     │  • 用于计算 Age 和缓存过期时间                                           │
└─────────────────────┴───────────────────────────────────────────────────────────────────────┘
```

### 7.5 Cookie 相关响应头部

```
┌─────────────────────┬───────────────────────────────────────────────────────────────────────┐
│  头部名称             │  说明                                                                     │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Set-Cookie         │  设置 Cookie（可多次出现）                                                │
│                     │  • 格式：Set-Cookie: <name>=<value>[; <属性>]*                           │
│                     │  • 示例：Set-Cookie: sessionId=abc123; Path=/; HttpOnly; Secure          │
│                     │  • 一个响应可包含多个 Set-Cookie 头部                                     │
│                     │  • 每个 Set-Cookie 只能设置一个 Cookie                                    │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Set-Cookie 属性     │                                                                          │
│  ─────────────────────────────────────────────────────────────────────────────────────── │
│  Expires            │  过期时间（绝对时间）                                                     │
│                     │  • 格式：Expires=<HTTP-date>                                            │
│                     │  • 示例：Expires=Wed, 21 Oct 2025 07:28:00 GMT                          │
│                     │  • 不设置则为 Session Cookie（关闭浏览器后失效）                          │
│                     │  • 设置为过去时间则删除 Cookie                                           │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Max-Age            │  有效期（秒）                                                             │
│                     │  • 示例：Max-Age=3600                                                    │
│                     │  • 优先于 Expires                                                        │
│                     │  • Max-Age=0 或负数立即删除 Cookie                                       │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Domain             │  生效域名                                                                 │
│                     │  • 示例：Domain=example.com                                              │
│                     │  • 默认为当前主机名（不含子域名）                                         │
│                     │  • 设置后包含所有子域名                                                   │
│                     │  • Domain=.example.com 可被所有子域名共享                                 │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Path               │  生效路径                                                                 │
│                     │  • 示例：Path=/admin                                                     │
│                     │  • 默认为当前路径                                                        │
│                     │  • 设置后对指定路径及其子路径生效                                         │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Secure             │  仅 HTTPS 传输                                                            │
│                     │  • 无值，仅设置属性名                                                     │
│                     │  • Cookie 只在 HTTPS 连接中发送                                          │
│                     │  • 强烈建议在 HTTPS 网站使用                                              │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  HttpOnly           │  禁止 JavaScript 访问                                                     │
│                     │  • 无值，仅设置属性名                                                     │
│                     │  • 防止 XSS 攻击窃取 Cookie                                               │
│                     │  • document.cookie 无法读取此 Cookie                                     │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  SameSite           │  跨站请求策略                                                             │
│                     │  • Strict：完全禁止跨站发送 Cookie                                       │
│                     │  • Lax：允许安全的跨站请求携带 Cookie（默认值）                            │
│                     │  • None：允许所有跨站请求携带 Cookie（需配合 Secure）                      │
│                     │  • 用于防止 CSRF 攻击                                                    │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  Priority           │  Cookie 优先级（Chrome 扩展）                                             │
│                     │  • 值：Low, Medium, High                                                 │
│                     │  • 用于 Cookie 数量超限时决定删除顺序                                     │
├─────────────────────┼───────────────────────────────────────────────────────────────────────┤
│  完整示例            │  Set-Cookie: sessionId=abc123;                                          │
│                     │    Domain=example.com;                                                   │
│                     │    Path=/;                                                               │
│                     │    Max-Age=86400;                                                        │
│                     │    HttpOnly;                                                             │
│                     │    Secure;                                                               │
│                     │    SameSite=Lax;                                                         │
│                     │    Priority=High                                                         │
└─────────────────────┴───────────────────────────────────────────────────────────────────────┘
```

### 7.7 安全相关响应头部

```
┌───────────────────────────┬─────────────────────────────────────────────────────────────────┐
│  头部名称                   │  说明                                                             │
├───────────────────────────┼─────────────────────────────────────────────────────────────────┤
│  Strict-Transport-Security │  强制使用 HTTPS（HSTS）                                          │
│  (HSTS)                    │  • max-age=<seconds>：有效期（必需）                              │
│                            │  • includeSubDomains：包含所有子域名                              │
│                            │  • preload：允许预加载到浏览器 HSTS 列表                          │
│                            │  • 示例：Strict-Transport-Security: max-age=31536000;            │
│                            │    includeSubDomains; preload                                     │
│                            │  • 首次访问后生效，有效期内强制 HTTPS                             │
├───────────────────────────┼─────────────────────────────────────────────────────────────────┤
│  Content-Security-Policy   │  内容安全策略（CSP）                                              │
│  (CSP)                     │  • 控制资源加载来源，防止 XSS 攻击                                 │
│                            │  • default-src 'self'：默认只允许同源                             │
│                            │  • script-src 'self' 'unsafe-inline'：脚本来源                    │
│                            │  • style-src 'self' 'unsafe-inline'：样式来源                     │

│  Content-Security-Policy   │  仅报告模式的 CSP                                                 │
│  -Report-Only              │  • 语法与 CSP 相同，但只报告不阻止                                 │

├───────────────────────────┼─────────────────────────────────────────────────────────────────┤

│                            │  • cross-origin：允许跨域加载                                      │
│                            │  • 示例：Cross-Origin-Resource-Policy: same-origin                │
│                            │  • 防止资源被其他网站加载                                          │
```

### 7.14 Set-Cookie 详解

```
Set-Cookie: <name>=<value>[; <属性名>=<属性值>]*

示例：
Set-Cookie: sessionId=abc123; Path=/; Domain=example.com; Max-Age=3600; HttpOnly; Secure; SameSite=Lax

┌─────────────────┬──────────────────────────────────────────────────────────┐
│  属性            │  说明                                                      │
├─────────────────┼──────────────────────────────────────────────────────────┤
│  Expires        │  过期时间（绝对时间）。不设置则为 Session Cookie            │
│  Max-Age        │  有效期（秒）。优先于 Expires                               │
│  Domain         │  生效域名。默认为当前域名，可设置子域名共享                  │
│  Path           │  生效路径。默认为当前路径                                   │
│  Secure         │  仅通过 HTTPS 传输                                         │
│  HttpOnly       │  禁止 JavaScript 访问（防止 XSS 攻击）                      │
│  SameSite       │  跨站请求策略：Strict | Lax | None                         │
└─────────────────┴──────────────────────────────────────────────────────────┘

SameSite 属性详解：
┌─────────────┬────────────────────────────────────────────────────────────┐
│  Strict     │  完全禁止跨站发送 Cookie，最严格                            │
│             │  从外部网站跳转过来也不会携带 Cookie                        │
├─────────────┼────────────────────────────────────────────────────────────┤
│  Lax        │  允许安全的跨站请求携带 Cookie（默认值）                     │
│             │  允许：GET 请求的链接、表单提交                             │
│             │  禁止：img、iframe、AJAX 请求                              │
├─────────────┼────────────────────────────────────────────────────────────┤
│  None       │  允许所有跨站请求携带 Cookie                                │
│             │  需要配合 Secure 属性使用                                   │
└─────────────┴────────────────────────────────────────────────────────────┘
```

### 7.3 Content-Disposition

```http
Content-Disposition: inline                          // 浏览器内显示
Content-Disposition: attachment                      // 下载文件
Content-Disposition: attachment; filename="report.pdf"  // 下载并指定文件名
```

### 8.1 RESTful API 设计

```javascript
// 用户资源 CRUD

GET /api/users
Response: 200 OK
{

POST /api/users
Request Body: { "name": "李四", "email": "lisi@example.com" }
Response: 201 Created
Location: /api/users/124
{
  "id": 124,
  "name": "李四",
  "email": "lisi@example.com"
}

// 完整更新用户
PUT /api/users/123
Request Body: { "name": "张三更新", "email": "zhangsan_new@example.com" }

// 部分更新用户
PATCH /api/users/123
Request Body: { "name": "张三更新" }
Response: 200 OK
{
  "id": 123,
  "name": "张三更新",
  "email": "zhangsan@example.com"
}

// 删除用户
DELETE /api/users/123
Response: 204 No Content
```

## 小结

### HTTP 方法特性速查

| 方法 | 安全性 | 幂等性 | 可缓存 | 请求体 |
|------|--------|--------|--------|--------|
| GET | ✓ | ✓ | ✓ | 无意义 |
| POST | ✗ | ✗ | 可选 | 有 |
| PUT | ✗ | ✓ | ✗ | 有 |
| PATCH | ✗ | ✗ | ✗ | 有 |
| DELETE | ✗ | ✓ | ✗ | 可选 |
| HEAD | ✓ | ✓ | ✓ | 无 |
| OPTIONS | ✓ | ✓ | ✗ | 无 |

### 状态码速查

| 类别 | 含义 | 常见状态码 |
|------|------|-----------|
| 1xx | 信息响应 | 100, 101, 103 |
| 2xx | 成功 | 200, 201, 204, 206 |
| 3xx | 重定向 | 301, 302, 304, 307, 308 |
| 4xx | 客户端错误 | 400, 401, 403, 404, 405, 409, 429 |
| 5xx | 服务器错误 | 500, 502, 503, 504 |
