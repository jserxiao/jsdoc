# HTTPS

> HTTPS（HTTP Secure）是 HTTP 的安全版本，通过 TLS/SSL 协议在客户端和服务器之间建立加密通道。理解 HTTPS 的工作原理对于前端开发者来说至关重要，它关系到用户数据安全和网站可信度。

## 学习要点

- 🔐 理解 HTTPS 的工作原理和安全模型
- 📜 掌握 TLS/SSL 握手过程
- 🏛️ 了解数字证书和 PKI 体系
- 🔑 熟悉对称加密与非对称加密的应用

## 2. 加密基础

### 2.1 对称加密

对称加密使用**相同的密钥**进行加密和解密。

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           对称加密流程                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   发送方                          传输通道                    接收方         │
│   ┌─────┐                         ┌─────┐                    ┌─────┐        │
│   │明文  │                         │     │                    │密文  │        │
│   └──┬──┘                         │     │                    └──┬──┘        │
│      │ 加密                        │     │                       │ 解密      │
│      ↓                            │     │                       ↓           │
│   ┌─────┐   密钥 K   ═════════════│═════│═══════  密钥 K    ┌─────┐        │
│   │密文  │ ═══════════════════════│═════│═════════════════→ │明文  │        │
│   └─────┘                         │     │                    └─────┘        │
│                                   │     │                                   │
│   常见算法：AES、ChaCha20、3DES                                             │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**优点**：加解密速度快，适合大量数据
**缺点**：密钥分发困难，需要安全地共享密钥

### 2.2 非对称加密

非对称加密使用**一对密钥**：公钥加密，私钥解密。

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          非对称加密流程                                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   发送方                          传输通道                    接收方         │
│   ┌─────┐                                                    ┌─────┐        │
│   │明文  │                                                    │私钥  │        │
│   └──┬──┘                                                    └──┬──┘        │
│      │ 用公钥加密                                                │           │
│      ↓                                                          ↓           │
│   ┌─────┐   公钥公开    ════════════════════════════════→  ┌─────┐        │
│   │密文  │ ══════════════════════════════════════════════→ │明文  │        │
│   └─────┘                                                    └─────┘        │
│                                                                             │
│   公钥：公开分发，用于加密                                                   │
│   私钥：严格保密，用于解密                                                   │
│                                                                             │
│   常见算法：RSA、ECC、DSA                                                   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**优点**：解决了密钥分发问题
**缺点**：计算速度慢，比对称加密慢 100-1000 倍

### 2.3 混合加密

HTTPS 采用**混合加密**方式：用非对称加密交换对称密钥，用对称加密传输数据。

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           混合加密方案                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   阶段一：密钥交换（非对称加密）                                              │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │  客户端 ←─────── 服务器公钥（明文）───────→ 服务器                     │   │
│   │  客户端 ──────── 对称密钥（公钥加密）───→ 服务器                        │   │
│   │                       ↓                                              │   │
│   │              双方拥有相同的对称密钥 K                                  │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│   阶段二：数据传输（对称加密）                                               │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │  客户端 ←─────── 加密数据（密钥 K）───────→ 服务器                     │   │
│   │              高效安全的数据通信                                       │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 3.2 TLS 1.3 握手（ECDHE 密钥交换）

TLS 1.3 简化了握手过程，只需 1-RTT（往返时间）：

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                        TLS 1.3 握手流程（ECDHE）                              │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   Client                                                Server              │
│     │                                                     │                 │
│     │  1. ClientHello                                     │                 │
│     │  ├─ 支持的 TLS 版本（仅 1.3）                         │                 │
│     │  ├─ 支持的加密套件                                   │                 │
│     │  ├─ 客户端随机数                                     │                 │
│     │  └─ 客户端 ECDHE 公钥（密钥共享）                      │                 │
│     │────────────────────────────────────────────────────→│                 │
│     │                                                     │                 │
│     │  2. ServerHello                                     │                 │
│     │  ├─ 服务器随机数                                     │                 │
│     │  └─ 服务器 ECDHE 公钥                                │                 │
│     │←────────────────────────────────────────────────────│                 │
│     │                                                     │                 │
│     │  3. EncryptedExtensions                             │                 │
│     │←────────────────────────────────────────────────────│                 │
│     │                                                     │                 │
│     │  4. Certificate                                     │                 │
│     │←────────────────────────────────────────────────────│                 │
│     │                                                     │                 │
│     │  5. CertificateVerify                               │                 │
│     │←────────────────────────────────────────────────────│                 │
│     │                                                     │                 │
│     │  6. Finished                                        │                 │
│     │←────────────────────────────────────────────────────│                
│     │                                                     │                 │
│     │  7. Finished（加密）                                 │                 │
│     │────────────────────────────────────────────────────→│                 │
│     │                                                     │                 │
│     │              加密通信开始（0-RTT 可选）                │                 │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 3.3 TLS 1.2 vs TLS 1.3 对比

| 特性 | TLS 1.2 | TLS 1.3 |
|------|---------|---------|
| **握手延迟** | 2-RTT | 1-RTT |
| **0-RTT** | 不支持 | 支持（会话恢复） |
| **加密套件** | 较多弱算法 | 仅保留强算法 |
| **密钥交换** | RSA/ECDHE | 仅 ECDHE |
| **前向保密** | 可选 | 强制 |
| **加密消息** | 部分明文 | 全程加密 |

## 5. 密钥派生

### 5.1 主密钥生成

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          密钥派生过程                                         │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   输入：                                                                     │
│   ├── 预主密钥（Pre-Master Secret）                                        │
│   │   └── 由客户端生成，用服务器公钥加密传输                                  │
│   ├── 客户端随机数（Client Random）                                        │
│   └── 服务器随机数（Server Random）                                        │
│                                                                             │
│                              ↓ PRF 函数                                      │
│                                                                             │
│   输出 - 主密钥（Master Secret）：                                          │
│   └── 48 字节，派生所有会话密钥                                             │
│                                                                             │
│                              ↓ 密钥扩展                                      │
│                                                                             │
│   最终密钥：                                                                 │
│   ├── 客户端写密钥（Client Write Key）                                     │
│   ├── 服务器写密钥（Server Write Key）                                     │
│   ├── 客户端写 MAC 密钥                                                    │
│   ├── 服务器写 MAC 密钥                                                    │
│   └── 客户端/服务器写 IV                                                   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 5.2 前向保密（Forward Secrecy）

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           前向保密原理                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   无前向保密（RSA 密钥交换）：                                               │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │  如果服务器私钥泄露 → 所有历史通信都可被解密                            │   │
│   │                                                                     │   │
│   │  攻击者记录了过去通信 ──→ 获得私钥 ──→ 解密所有历史数据                 │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│   有前向保密（ECDHE 密钥交换）：                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │  每个会话使用临时密钥对 → 即使私钥泄露，历史通信仍安全                   │   │
│   │                                                                     │   │
│   │  攻击者记录了过去通信 ──→ 获得私钥 ──→ 无法解密历史数据                 │   │
│   │                              ↑                                      │   │
│   │                   临时会话密钥已销毁                                   │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│   TLS 1.3 强制使用 ECDHE，确保前向保密                                       │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 7. 服务器配置

### 7.1 Nginx 配置示例

```nginx
server {
    listen 443 ssl http2;
    server_name example.com;

    # 证书配置
    ssl_certificate /etc/ssl/certs/example.com.crt;
    ssl_certificate_key /etc/ssl/private/example.com.key;

    # TLS 版本
    ssl_protocols TLSv1.2 TLSv1.3;

    # 加密套件
    ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256;
    ssl_prefer_server_ciphers on;

    # 会话缓存
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 1d;
    ssl_session_tickets off;

    # HSTS（强制 HTTPS）
    add_header Strict-Transport-Security "max-age=31536000; includeSubDomains" always;

    # OCSP Stapling
    ssl_stapling on;
    ssl_stapling_verify on;
    resolver 8.8.8.8 8.8.4.4 valid=300s;
}

# HTTP 重定向到 HTTPS
server {
    listen 80;
    server_name example.com;
    return 301 https://$server_name$request_uri;
}
```

### 7.2 Apache 配置示例

```apache
<VirtualHost *:443>
    ServerName example.com

    SSLEngine on
    SSLCertificateFile /etc/ssl/certs/example.com.crt
    SSLCertificateKeyFile /etc/ssl/private/example.com.key
    SSLCertificateChainFile /etc/ssl/certs/chain.crt

    SSLProtocol all -SSLv2 -SSLv3 -TLSv1 -TLSv1.1
    SSLCipherSuite ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256
    
    Header always set Strict-Transport-Security "max-age=31536000; includeSubDomains"
</VirtualHost>

<VirtualHost *:80>
    ServerName example.com
    Redirect permanent / https://example.com/
</VirtualHost>
```

### 9.2 混合内容处理

```html
<!-- 混合内容：HTTPS 页面加载 HTTP 资源 -->

<!-- ❌ 危险：被动混合内容（警告） -->
<img src="http://example.com/image.jpg">

<!-- ❌ 危险：主动混合内容（阻止） -->
<script src="http://example.com/script.js"></script>

<!-- ✅ 正确：使用 HTTPS -->
<img src="https://example.com/image.jpg">

<!-- ✅ 使用协议相对 URL -->
<script src="//example.com/script.js"></script>

<!-- ✅ CSP 策略阻止混合内容 -->
<meta http-equiv="Content-Security-Policy" content="upgrade-insecure-requests">
```
