# 闭包

> 闭包（Closure）是 JavaScript 最重要也最容易令人困惑的概念之一。理解闭包是理解 JavaScript 函数式编程、模块化、异步编程的基础。

## 学习要点

- 🔐 理解闭包的本质：函数 + 词法环境
- 🎯 掌握闭包的常见应用场景
- ⚠️ 避免闭包相关的陷阱和内存泄漏

---

## 1. 什么是闭包？

### 核心定义

**闭包 = 函数 + 函数声明时所在的词法环境**

简单来说，闭包让函数可以"记住"并访问它被创建时所在的作用域，即使该函数在其作用域之外执行。

### 最简单的闭包示例

```javascript
function outer() {
    const message = 'Hello';  // 外部函数的局部变量
    
    function inner() {
        console.log(message); // 内部函数访问外部变量
    }
    
    return inner;  // 返回内部函数
}

const fn = outer(); // outer 执行完毕，理论上 message 应该被销毁
fn(); // 'Hello' - 但 message 仍然存在！这就是闭包
```

### 为什么这很神奇？

```
正常情况：
  outer() 执行 → 创建 message → outer() 结束 → message 被垃圾回收

闭包情况：
  outer() 执行 → 创建 message → 返回 inner → inner 持有对 message 的引用
  → outer() 结束 → message 不会被回收（因为 inner 还引用着它）
```

## 2. 闭包的应用场景

### 应用 1：数据私有化

```javascript
function createCounter() {
    let count = 0; // 私有变量
    
    return {
        increment() { return ++count; },
        decrement() { return --count; },
        getCount() { return count; }
    };
}

const counter = createCounter();
console.log(counter.increment()); // 1
console.log(counter.getCount());  // 1
console.log(counter.count);       // undefined（无法直接访问）
```

### 应用 2：函数工厂

```javascript
function createMultiplier(multiplier) {
    return function(number) {
        return number * multiplier;
    };
}

const double = createMultiplier(2);
const triple = createMultiplier(3);

console.log(double(5));  // 10
console.log(triple(5));  // 15
```

### 应用 3：记忆化（缓存）

```javascript
function memoize(fn) {
    const cache = new Map();
    
    return function(...args) {
        const key = JSON.stringify(args);
        if (cache.has(key)) {
            return cache.get(key); // 从缓存返回
        }
        const result = fn.apply(this, args);
        cache.set(key, result);
        return result;
    };
}
```

### 应用 4：防抖与节流

```javascript
// 防抖：停止触发后一段时间才执行
function debounce(fn, delay) {
    let timer = null;
    return function(...args) {
        clearTimeout(timer);
        timer = setTimeout(() => fn.apply(this, args), delay);
    };
}

// 节流：固定间隔执行
function throttle(fn, limit) {
    let inThrottle = false;
    return function(...args) {
        if (!inThrottle) {
            fn.apply(this, args);
            inThrottle = true;
            setTimeout(() => inThrottle = false, limit);
        }
    };
}
```

## 3. 经典陷阱：循环中的闭包

### 问题代码

```javascript
for (var i = 1; i <= 3; i++) {
    setTimeout(function() {
        console.log(i);
    }, 100);
}
// 输出：4, 4, 4（而不是 1, 2, 3）
```

### 原因

`var` 是函数作用域，所有回调函数共享同一个 `i`。当回调执行时，循环已结束，`i` 已是 4。

### 解决方案

```javascript
// 方案 1：使用 let（推荐）
for (let i = 1; i <= 3; i++) {
    setTimeout(() => console.log(i), 100);
}
// 输出：1, 2, 3

// 方案 2：使用 IIFE
for (var i = 1; i <= 3; i++) {
    ((j) => {
        setTimeout(() => console.log(j), 100);
    })(i);
}

// 方案 3：使用 bind
for (var i = 1; i <= 3; i++) {
    setTimeout(console.log.bind(null, i), 100);
}
```

## 4. 内存泄漏注意事项

```javascript
// ⚠️ 可能的内存泄漏
function createHandler() {
    const largeData = new Array(1000000).fill('x');
    
    return function() {
        console.log(largeData.length); // 保留了大数组引用
    };
}
// largeData 无法被垃圾回收

// ✅ 解决方案：只保留需要的值
function createHandlerBetter() {
    const largeData = new Array(1000000).fill('x');
    const length = largeData.length; // 只保存需要的值
    
    return function() {
        console.log(length);
    };
    // largeData 可被垃圾回收
}
```

## 练习题

```javascript
// 1. 实现一个只能调用一次的函数
function once(fn) {
    let hasRun = false;
    let result;
    return function(...args) {
        if (hasRun) return result;
        hasRun = true;
        result = fn.apply(this, args);
        return result;
    };
}

// 2. 以下代码输出什么？
function createFunctions() {
    var funcs = [];
    for (var i = 0; i < 3; i++) {
        funcs.push(function() {
            console.log(i);
        });
    }
    return funcs;
}
var fns = createFunctions();
fns[0](); // ?
fns[1](); // ?
fns[2](); // ?
// 答案：都是 3

// 3. 如何修复上面的代码？
// 答案：将 var 改为 let，或使用 IIFE
```
