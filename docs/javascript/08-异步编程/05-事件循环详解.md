# 事件循环详解：微任务与宏任务

> 事件循环是 JavaScript 异步编程的核心机制。《你不知道的JavaScript》将此作为重点内容深入讲解，理解它对于编写正确的异步代码至关重要。

## 学习要点

- 🔍 理解事件循环的工作原理
- 📦 掌握宏任务和微任务的区别
- 🔄 理解任务队列的执行顺序
- 🎯 能够预测代码执行结果

---

## 1. 什么是事件循环？

JavaScript 是单线程语言，所有代码都在一个线程上执行。事件循环是 JavaScript 实现异步编程的核心机制。

### 单线程的挑战

```javascript
// 问题：同步代码会阻塞
console.log('开始');

// 假设这是一个耗时操作
for (let i = 0; i < 1000000000; i++) {}

console.log('结束'); // 必须等待循环结束
```

### 事件循环的基本结构

```
┌─────────────────────────────────────────────────────────────┐
│                        事件循环模型                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌───────────────┐                                        │
│   │   调用栈       │  ← 同步代码执行的地方                    │
│   │  (Call Stack) │                                        │
│   └───────┬───────┘                                        │
│           │                                                 │
│           │ 执行完毕                                         │
│           ▼                                                 │
│   ┌───────────────┐     ┌───────────────┐                  │
│   │   微任务队列   │ ←── │  Promise.then │                  │
│   │ (Microtask Q) │     │  queueMicrotask│                  │
│   └───────┬───────┘     └───────────────┘                  │
│           │                                                 │
│           │ 清空后                                           │
│           ▼                                                 │
│   ┌───────────────┐     ┌───────────────┐                  │
│   │   宏任务队列   │ ←── │  setTimeout   │                  │
│   │ (Macrotask Q) │     │  setInterval  │                  │
│   └───────┬───────┘     │  I/O 操作     │                  │
│           │              └───────────────┘                  │
│           │                                                 │
│           └──────────────────► 渲染                         │
│                    (UI Rendering)                           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 2. 宏任务（Macrotask）

宏任务是由宿主环境（浏览器或 Node.js）发起的任务。

### 常见宏任务

```javascript
// 1. setTimeout
setTimeout(() => {
    console.log('setTimeout');
}, 0);

// 2. setInterval
setInterval(() => {
    console.log('setInterval');
}, 1000);

// 3. setImmediate（Node.js）
setImmediate(() => {
    console.log('setImmediate');
});

// 4. I/O 操作
fs.readFile('file.txt', (err, data) => {
    console.log('文件读取完成');
});

// 5. UI 渲染
// 浏览器在两次宏任务之间进行渲染

// 6. requestAnimationFrame（特殊，在渲染前执行）
requestAnimationFrame(() => {
    console.log('动画帧');
});
```

### 宏任务队列特点

```javascript
// 宏任务队列是先进先出（FIFO）
setTimeout(() => console.log('A'), 0);
setTimeout(() => console.log('B'), 0);
setTimeout(() => console.log('C'), 0);

// 输出顺序：A, B, C
```

## 3. 微任务（Microtask）

微任务是由 JavaScript 引擎发起的任务，优先级高于宏任务。

### 常见微任务

```javascript
// 1. Promise.then / .catch / .finally
Promise.resolve()
    .then(() => console.log('Promise.then'));

// 2. queueMicrotask
queueMicrotask(() => {
    console.log('queueMicrotask');
});

// 3. MutationObserver
const observer = new MutationObserver(() => {
    console.log('DOM 变化');
});
observer.observe(document.body, { childList: true });

// 4. process.nextTick（Node.js，优先级最高）
process.nextTick(() => {
    console.log('nextTick');
});

// 5. V8 的 async/await（本质是 Promise）
async function test() {
    await Promise.resolve();
    console.log('await 后');
}
```

### 微任务队列特点

```javascript
// 微任务会在当前宏任务结束后立即执行
// 并且会清空所有微任务

console.log('1. 同步开始');

Promise.resolve()
    .then(() => console.log('3. 微任务 1'))
    .then(() => console.log('4. 微任务 2'));

Promise.resolve()
    .then(() => console.log('5. 微任务 3'));

console.log('2. 同步结束');

// 输出顺序：
// 1. 同步开始
// 2. 同步结束
// 3. 微任务 1
// 4. 微任务 2
// 5. 微任务 3
```

## 4. 事件循环执行顺序

### 完整执行流程

```
1. 执行同步代码（调用栈）
     ↓
2. 调用栈清空，检查微任务队列
     ↓
3. 执行所有微任务（直到队列清空）
     ↓
4. 浏览器渲染（如果需要）
     ↓
5. 执行一个宏任务
     ↓
6. 回到步骤 2
```

### 经典面试题

```javascript
console.log('1');

setTimeout(() => {
    console.log('2');
    Promise.resolve().then(() => {
        console.log('3');
    });
}, 0);

Promise.resolve()
    .then(() => {
        console.log('4');
    })
    .then(() => {
        console.log('5');
    });

setTimeout(() => {
    console.log('6');
}, 0);

console.log('7');

/*
执行流程：

第一轮事件循环：
1. 同步代码：
   - console.log('1') → 1
   - 注册 setTimeout 回调到宏任务队列
   - 注册 Promise.then 回调到微任务队列
   - 注册第二个 setTimeout 到宏任务队列
   - console.log('7') → 7
   - 调用栈清空

2. 微任务队列：
   - 执行 Promise.then → 4
   - 注册下一个 .then 到微任务队列
   - 执行下一个 .then → 5
   - 微任务队列清空

3. 宏任务队列：
   - 取出第一个 setTimeout 回调

第二轮事件循环：
1. 执行 setTimeout 回调：
   - console.log('2') → 2
   - 注册 Promise.then 到微任务队列

2. 微任务队列：
   - 执行 Promise.then → 3

第三轮事件循环：
1. 宏任务队列：
   - 取出第二个 setTimeout 回调
   - console.log('6') → 6

输出顺序：1, 7, 4, 5, 2, 3, 6
*/
```

### 图解执行过程

```
第一轮事件循环
┌─────────────────────────────────────────────────────────────┐
│ 调用栈                          │ 微任务队列    │ 宏任务队列  │
├─────────────────────────────────┼──────────────┼────────────┤
│ console.log('1')                │              │            │
│ setTimeout                      │              │ [callback1] │
│ Promise.resolve().then          │ [then1]      │            │
│ setTimeout                      │              │ [cb1, cb2]  │
│ console.log('7')                │              │            │
│ ────────────────────────────────│──────────────│────────────│
│ 调用栈清空                       │              │            │
│ 执行微任务 then1 → '4'           │ [then2]      │            │
│ 执行微任务 then2 → '5'           │ []           │            │
│ 微任务队列清空                    │              │            │
└─────────────────────────────────┴──────────────┴────────────┘
                                    ↓
第二轮事件循环
┌─────────────────────────────────────────────────────────────┐
│ 执行宏任务 cb1                   │ 微任务队列    │ 宏任务队列  │
├─────────────────────────────────┼──────────────┼────────────┤
│ console.log('2')                │              │ [cb2]      │
│ Promise.resolve().then          │ [then3]      │            │
│ ────────────────────────────────│──────────────│────────────│
│ 执行微任务 then3 → '3'           │ []           │            │
└─────────────────────────────────┴──────────────┴────────────┘
                                    ↓
第三轮事件循环
┌─────────────────────────────────────────────────────────────┐
│ 执行宏任务 cb2                   │ 微任务队列    │ 宏任务队列  │
├─────────────────────────────────┼──────────────┼────────────┤
│ console.log('6')                │              │ []         │
└─────────────────────────────────┴──────────────┴────────────┘
```

## 5. async/await 的事件循环

```javascript
async function async1() {
    console.log('async1 start');
    await async2();
    console.log('async1 end');
}

async function async2() {
    console.log('async2');
}

console.log('script start');

setTimeout(() => {
    console.log('setTimeout');
}, 0);

async1();

new Promise(resolve => {
    console.log('promise1');
    resolve();
}).then(() => {
    console.log('promise2');
});

console.log('script end');

/*
执行过程分析：

1. 同步代码：
   - console.log('script start') → 'script start'
   - setTimeout 注册到宏任务队列
   - 调用 async1()
     - console.log('async1 start') → 'async1 start'
     - 调用 async2()
       - console.log('async2') → 'async2'
     - await async2() 后面的代码相当于 Promise.then
     - 注册到微任务队列
   - new Promise
     - console.log('promise1') → 'promise1'
     - resolve() 注册 .then 到微任务队列
   - console.log('script end') → 'script end'

2. 微任务队列：
   - 执行 async1 的 await 后代码 → 'async1 end'
   - 执行 Promise.then → 'promise2'

3. 宏任务队列：
   - 执行 setTimeout → 'setTimeout'

输出顺序：
script start
async1 start
async2
promise1
script end
async1 end
promise2
setTimeout
*/
```

### async/await 转换为 Promise

```javascript
// async/await 本质上是 Promise 的语法糖
async function async1() {
    console.log('async1 start');
    await async2();
    console.log('async1 end');
}

// 等价于
function async1() {
    console.log('async1 start');
    return Promise.resolve(async2()).then(() => {
        console.log('async1 end');
    });
}
```

## 6. Node.js 的事件循环

Node.js 的事件循环与浏览器有所不同。

### Node.js 事件循环阶段

```
   ┌───────────────────────────┐
┌─>│           timers          │ setTimeout/setInterval
│  └─────────────┬─────────────┘
│  ┌─────────────┴─────────────┐
│  │     pending callbacks     │ 执行推迟到下一个循环的回调
│  └─────────────┬─────────────┘
│  ┌─────────────┴─────────────┐
│  │       idle, prepare       │ 内部使用
│  └─────────────┬─────────────┘
│  ┌─────────────┴─────────────┐
│  │           poll            │ I/O 回调
│  └─────────────┬─────────────┘
│  ┌─────────────┴─────────────┐
│  │           check           │ setImmediate
│  └─────────────┬─────────────┘
│  ┌─────────────┴─────────────┐
└──┤      close callbacks      │ close 事件
   └───────────────────────────┘
```

### Node.js 微任务优先级

```javascript
// Node.js 中：
// process.nextTick > Promise.then

process.nextTick(() => console.log('nextTick1'));
Promise.resolve().then(() => console.log('promise1'));
process.nextTick(() => console.log('nextTick2'));
Promise.resolve().then(() => console.log('promise2'));

// 输出：nextTick1, nextTick2, promise1, promise2
```

### setTimeout vs setImmediate

```javascript
// 在 I/O 回调中
const fs = require('fs');

fs.readFile(__filename, () => {
    setTimeout(() => console.log('timeout'), 0);
    setImmediate(() => console.log('immediate'));
});

// 输出：immediate, timeout（setImmediate 先执行）

// 在主模块中（顺序不确定）
setTimeout(() => console.log('timeout'), 0);
setImmediate(() => console.log('immediate'));
// 输出顺序不确定，取决于性能
```

## 7. 实战技巧

### 利用微任务延迟执行

```javascript
// 使用 Promise.resolve().then() 延迟执行
function defer(fn) {
    Promise.resolve().then(fn);
}

console.log('同步代码');
defer(() => console.log('延迟执行'));
console.log('更多同步代码');

// 输出：同步代码, 更多同步代码, 延迟执行
```

### 利用 queueMicrotask

```javascript
// 更明确地创建微任务
function logWhenIdle(message) {
    queueMicrotask(() => {
        console.log(message);
    });
}

logWhenIdle('空闲时执行');
```

### 确保代码在 DOM 更新后执行

```javascript
// 修改 DOM 后立即获取尺寸
const element = document.getElementById('box');
element.style.width = '200px';

// DOM 更新在微任务之后，宏任务之前
// 使用 requestAnimationFrame 确保在渲染后执行
requestAnimationFrame(() => {
    console.log(element.offsetWidth); // 新的宽度
});

// 或者使用 setTimeout
setTimeout(() => {
    console.log(element.offsetWidth); // 新的宽度
}, 0);
```

## 小结

| 任务类型 | 示例 | 执行时机 | 优先级 |
|---------|------|---------|--------|
| 同步代码 | 普通代码 | 立即执行 | 最高 |
| process.nextTick | Node.js | 微任务前 | 次高 |
| 微任务 | Promise.then | 宏任务前 | 高 |
| requestAnimationFrame | 浏览器 | 渲染前 | 中 |
| 宏任务 | setTimeout | 渲染后 | 低 |

**执行顺序口诀**：
1. 同步代码先执行
2. 微任务队列清空
3. 渲染（如果需要）
4. 执行一个宏任务
5. 回到步骤 2
